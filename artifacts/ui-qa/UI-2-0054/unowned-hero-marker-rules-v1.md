# Unowned Hero Marker Rules v1

## 目標

把 `未持有武將` 的顯示做成可跨 `日常人物頁`、`血脈命鏡 Loading`、`覺醒 / 命鏡預覽` 共用的視覺規則，讓玩家在看見尚未獲得的武將時，同時感受到：

- 還沒得到，但已可被看見
- 有收藏慾與抽卡誘惑感
- 不會誤認成已解鎖角色
- 不會因為遮罩太重而破壞角色商品感

## 核心原則

1. 未持有不是黑影剪影。
2. 未持有角色仍要保留 `80%` 以上的角色辨識與美感。
3. 規則重點是 `微遮罩 + badge + 提示文案`，不是整張灰掉。
4. 日常人物頁與命鏡頁都要能吃同一套語言，但濃度不同。

## 顯示分級

### A. 日常人物頁

用途：
- 圖鑑式角色預覽
- 未持有角色詳情試看

建議規則：
- 角色立繪保持清楚，不做整體去飽和。
- 左上或人物側邊掛一條 `未得 / 未持有 / 尚未結識` ribbon。
- 右側 meta 區可加一枚小 badge。
- 血脈 / 故事條仍可見，但可在 CTA 區補一句：
  - `完成招募後可解鎖完整血脈故事`

### B. 血脈命鏡 Loading

用途：
- 輪播展示未持有名將
- 過場誘惑圖

建議規則：
- 角色本體維持完整美感。
- 疊一層很薄的 `veil`，只用來區隔「未持有」狀態。
- 右上角保留一枚明確 badge。
- 可加一行小字：
  - `尚未結識`
  - `命鏡僅現其影`
  - `未納英靈`

### C. 覺醒 / 命鏡預覽

用途：
- 未持有角色的高階覺醒頁預覽

建議規則：
- 正常角色面與血脈面都可顯示。
- 不鎖畫面，不藏內容。
- 只在 CTA 或底部提示：
  - `取得武將後可啟動命鏡覺醒`

## UI 元件拆分

### 日常人物頁

- `gdv3.unowned.ribbon`
  - 人物左上角或側邊斜條
- `gdv3.label.unowned`
  - 人物旁小提示
- `gdv3.unowned.badge`
  - 右側資訊區小 badge

### 血脈命鏡

- `mirror.unowned.veil`
  - 全主視覺區輕遮罩
- `mirror.unowned.badge`
  - 右上 badge
- `mirror.label.unowned`
  - 右上小提示

## 視覺語言

### 不建議

- 純黑剪影
- 大面積問號遮擋
- 角色整張去彩
- 強鎖鏈 / 強封印 / 恐怖裂紋
- 像「尚未開放」功能的灰 disabled 狀態

### 建議

- 淡金、舊銀、灰青玉色 badge
- 半透紗霧 veil
- 柔和字樣：
  - `尚未結識`
  - `命鏡現影`
  - `英靈未納`
  - `待招募`

## 文案規則

推薦詞庫：

- `尚未結識`
- `待招募`
- `命鏡現影`
- `英靈未納`

避免詞：

- `未解鎖`
- `未開放`
- `敬請期待`
- `？？？`

原因：
- 這些詞比較像功能未開，不像角色尚未收入帳下。

## 與抽卡誘因的關係

未持有標記不能讓玩家覺得「這張不能看」。
相反地，它應該讓玩家覺得：

> 我已經看見這名武將的命格與氣質，只差把他招入麾下。

## 可行性評估

### 高可行

- ribbon / badge / hint
- loading 畫面 veil
- 文字提示切換

### 中可行

- 依稀有度切換不同未持有 badge family
- 依陣營切換不同未持有色系

### 目前不建議

- 角色進場時大面積封印動畫
- 常駐動態霧面特效
- 複雜 unlock 演出

原因：
- 你們目前特效仍是短板，這部分先靠靜態 UI 物件就能成立。
