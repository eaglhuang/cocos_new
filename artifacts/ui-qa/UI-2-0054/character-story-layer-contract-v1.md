# 角色雙故事層契約 v1

## 定位

本文件將角色頁與命鏡頁中的故事內容，正式拆成兩條並行敘事：

1. `歷史趣聞`
2. `血脈傳聞`

這兩條不是彼此替代，而是共同解釋：

- 他在史實中是誰
- 他為何會成為這樣的人

## 核心原則

### 1. 歷史趣聞講表象人生

- 講史實印象、成名事件、人生橋段、對外形象。
- 讓玩家認出這就是三國名將本人。

### 2. 血脈傳聞講內在命格

- 講血脈面、命紋靈獸、宿命牽引、古老基因影響。
- 用來合理化：
  - 趙雲為何七進七出
  - 諸葛亮為何鞠躬盡瘁
  - 劉備為何如此具魅力

### 3. 不是二選一，而是上下疊加

- 歷史趣聞讓角色被辨認
- 血脈傳聞讓角色變成《三國傳承》的角色

## 日常人物頁規則

### 版型

- 下方使用 `6 格 story strip`
- 左到右讀是一段完整人生與血脈故事

### 推薦節奏

1. 少年 / 初習
2. 入世 / 成名起點
3. 歷史名場面
4. 命運介入 / 轉折
5. 血脈真相 / 靈獸感應
6. 承接未來 / 英靈伏筆

### 內容分工

- 第 1~3 格偏歷史趣聞
- 第 4 格是轉折點
- 第 5~6 格偏血脈傳聞

## 命鏡 / 覺醒頁規則

### 版型

- 下方使用 `5 格命運故事帶`

### 推薦節奏

1. 凡身起點
2. 英雄崛起
3. 危機 / 犧牲
4. 命鏡接觸 / 血脈覺醒
5. 英靈延續 / 宿命承接

## Tab / Drawer 規則

若日後要擴成可點開閱讀，正式欄位固定為：

- `historicalAnecdote`
- `bloodlineRumor`
- `storyStripCells`

### Drawer 內容

- 標題
- 1 段短摘要
- 3~6 格分鏡縮圖
- 可選語音 / 旁白欄位

## 內容長度建議

### 格內標題

- 4~10 字

### 格內摘要

- 12~28 字

### Drawer 正文

- 60~140 字

## 與命紋靈獸的關係

- 血脈傳聞中必須出現對應的 `命紋靈獸`
- 但它不是可愛寵物，而是：
  - 預兆
  - 夜談對象
  - 靈魂引導者
  - 命格化身

## 與 UI 的關係

故事層應回流到以下 UI：

- 日常人物頁 story strip
- 血脈命鏡下方命運帶
- Loading tips 池
- 未持有武將預覽
