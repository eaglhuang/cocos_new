# UI-2-0040 Reward Container / Bundle Props Spec

## Family

- `A8 reward / bundle props family`

## 使用情境

- 商城 bundle
- 獎勵揭示
- 任務獎勵
- 活動禮包
- 材料包 / 補給包

## 子類型

### 1. 高價值寶箱

- 角色：大型獎勵、稀有掉落、首儲或高價值包裝
- 外觀：漆木或金屬主體、正面徽章、厚重蓋體、明顯鎖件
- 主色：暗紅、紫金、金棕、深藍金邊
- 視覺重點：中心徽章、金屬鎖片、邊角護件、開蓋結構

### 2. 文書 / 卷冊包

- 角色：書卷、情報、技能、知識型獎勵
- 外觀：竹簡、書匣、木盒、紙冊、繩結、封籤
- 主色：竹棕、紙米、深木、靛藍書衣、低飽和墨綠
- 視覺重點：綁繩、書脊、卷軸端頭、封條

### 3. 資源 / 材料容器

- 角色：補給、素材、工藝、一般資源包
- 外觀：木箱、工具盒、補給箱、簡化禮盒
- 主色：綠黑、木棕、鐵灰、暗紅
- 視覺重點：開口、把手、箱腳、材質分區

### 4. 禮盒 / 節慶包

- 角色：活動包裝、節慶登入獎勵、限時商品
- 外觀：亮色漆盒、金邊、綁帶、封印章
- 主色：紅金、紫金、藍金、暖木紅
- 視覺重點：綁帶、印章、正面圖章、蓋體反光

## 製作規則

1. 必須先做 family kit，再畫個別 proof。
2. 四個 proof 至少同批產出：
   - chest
   - gift box
   - scroll pack
   - material crate
3. 先定義 `value_signal`，再決定材質與色票。
4. 不可過度追求寫實細節，必須保住 `128 / 64` 下的識別。
5. 容器主體要有明確正面，不可畫成無方向的擺件。

## 尺寸組

- hero proof：`512 x 512`
- standard export：`256 x 256`
- cell proof：`128 x 128`
- small QA：`64 x 64`

## Screen-context QA

- 與 `reward-cell` 並排時，數量文字優先，容器本體不可搶掉數值
- 與商城 bundle price / CTA 疊放時，主輪廓不可被標價牌吃掉
- 放進活動 banner 時，需保住中心徽章與主要綁帶
