# UI-2-0047 Macro Style Rules

## 摘要

這份規則不是只回答「某張圖漂不漂亮」，而是回答「高品質 UI 為什麼看起來穩定」。從 29 張參考圖看下來，真正讓品質拉開差距的不是單張 icon 細節，而是同一個功能會共享同一套色彩語意、同一類 carrier、同一組字級階層、同一批狀態稿，以及同一個縮圖 QA 標準。

## 規則 1：同一功能的 icon 必須成 family 批次生產

適用條件
- 同一畫面的功能鍵、狀態指示、資源圖示、據點 pin、tab icon。

規範
- 不要單張逐個生圖。
- 至少以 `功能群組 × family × 狀態組` 批次建立。
- 每批至少包含 `normal`、`pressed/selected`、`disabled/locked`、`small / medium / large` 或 `32 / 64 / 128`。

製作技巧
- 先定 `carrier`，再定 `glyph`，最後才定小 badge / 光效。
- 同一批要共用邊框厚度、材質噪點、投影強度與發光顏色。

常見失敗點
- 每張 icon 單獨生圖，結果同功能卻像來自不同遊戲。
- 先畫 glyph、最後才想 carrier，導致按鈕與 icon 的質感不合。

## 規則 2：顏色要先綁功能語意，不能只綁美感

適用條件
- icon、CTA、badge、tab、progress、pin、card rarity。

規範
- 金色系：高價值、主 CTA、里程碑、SSR/高級獎勵。
- 紅色系：危險、敵方、警告、進攻壓力、不可逆操作。
- 綠色系：守護、補益、可佔領、友方增益、成功狀態。
- 藍/青色系：冷卻、資訊、次級稀有、科技感、可互動節點。
- 紫色系：稀有、神祕、法術、抽卡 SR 檔位。
- 灰黑系：鎖定、未啟用、背景功能、次要入口。

製作技巧
- 先建立語意色票，不要每張圖各自挑色。
- 同功能跨畫面仍應維持主色語意，只允許材質或亮度微調。

常見失敗點
- 同樣是「關閉」或「警示」，不同畫面卻用不同顏色。
- 抽卡稀有度、道具稀有度、按鈕重要度互相打架。

## 規則 3：畫面調性不是單色，而是深淺成對

適用條件
- 幾乎所有完整畫面。

規範
- 深色戰場 / 主城背景，必須搭配淺色資訊 plate 或高亮焦點。
- 淺色主面板，必須搭配深色側欄 / 深色分隔 / 深色操作節點。
- 一個畫面通常只保留一個主焦點高亮區，其餘區塊退到次層。

參數
- 深底常落在 `#1A1A1A ~ #2A2520`
- 淺面常落在 `#E8DFD0 ~ #F0E8D8`
- 金色焦點常落在 `#D4AF37 ~ #FFE088`

製作技巧
- 先定畫面主地板是深還是淺，再決定卡片與 CTA 反差。
- 不要整頁都亮，也不要整頁都髒黑。

常見失敗點
- 全畫面都用中灰，沒有層級。
- 背景、面板、按鈕、icon 都是同一種明度。

## 規則 4：同功能的文字階層必須固定

適用條件
- 按鈕、資訊條、資源數值、稀有度、卡牌標題、據點資訊卡。

規範
- 主標題：28~42 px 視覺級，通常是襯線 / 書法感標題字。
- 區塊標題：22~30 px。
- 按鈕字：20~28 px，主 CTA 不可小於次 CTA。
- 數值字：18~30 px，需比敘述字更有重量。
- 輔助字：14~18 px，仍需在縮圖可讀。

製作技巧
- 漢字主標與英數數值可以分字重，但要共用同層級基準。
- 金色 CTA 常用大字 + 深底；淺面板正文則用黑字 + 細描邊。

常見失敗點
- 主按鈕與次按鈕用字一樣大，失去階層。
- 稀有度、數值、說明字搶同一層權重。

## 規則 5：Carrier 與 Glyph 必須分層設計

適用條件
- icon、pin、badge、戰場按鈕、據點標籤。

規範
- `carrier`：決定材質、邊框、體積感、可點擊性。
- `glyph`：決定功能語意。
- `badge/overlay`：決定狀態或稀有度，不應搶主語意。

製作技巧
- 先產出空 carrier 套件，再批次替換 glyph。
- 同一 family 允許同 carrier 搭多 glyph，而不是每張都重畫容器。

常見失敗點
- glyph 與底板黏成單一圖，無法穩定量產。
- badge 做得比 glyph 還大，焦點被搶走。

## 規則 6：同一 family 必須同時做狀態稿

適用條件
- 所有可互動 icon、button、reward cell、tab。

規範
- 至少同時建立 `normal`、`hover/selected`、`pressed`、`disabled/locked`。
- 如果要自動生成，prompt / 規格也要一次描述完整狀態組。

製作技巧
- 用亮度、邊光、飽和度、陰影深度控制狀態，不要只改色相。
- disabled 不是單純降低透明度，還要降低材質價值感。

常見失敗點
- 只做 normal，後面再補 pressed，結果風格跑掉。
- disabled 只做灰階，卻保留高亮金邊，視覺仍像可點擊。

## 規則 7：卡片、頭像、容器、torso 不能混用 icon 的製作邏輯

適用條件
- portrait、card art、reward chest、building tile、torso。

規範
- portrait 看裁片與臉部可讀，不看 icon 邊框邏輯。
- card art 看角色/場景敘事與 rarity 框。
- 容器看輪廓辨識、材質價值、開啟感。
- torso 看 silhouette、胸肩辨識、配件節點。

製作技巧
- 每種 family 要有自己的 `crop_mode / silhouette_rule / value_signal`。
- 不要把所有東西都當成一張有邊框的小圖。

常見失敗點
- 用 icon 的高亮厚邊去包 portrait。
- 用卡牌的敘事構圖去做容器，造成縮圖讀不清。

## 規則 8：縮圖 QA 必須前移

適用條件
- 所有需要在 128 以下尺寸使用的素材。

規範
- 至少驗 `128 / 64 / 32`
- 若進戰場 HUD，必驗 `64 / 32`
- 若進微型 badge，必驗 `32 / 24`

製作技巧
- 先在灰底、深底、淺底各看一次。
- 同時看單張與實際畫面拼版。

常見失敗點
- 只看原尺寸很好看，縮小後全糊。
- 只在中性灰底驗圖，沒放回真實畫面。

## 規則 9：面板與按鈕的材質庫要少而穩

適用條件
- 所有 frame / fill / bleed / shadow / CTA / tab。

規範
- 主材質庫建議固定：
  - dark metal
  - parchment
  - gold CTA
  - red destructive
  - brush-strip / ink smear
- 同畫面最多 2~3 套主材質，不要每塊都換。

製作技巧
- 用材質切層級，不要用十種顏色切層級。
- family 切換時，先換材質，再決定需不需要換色。

常見失敗點
- 同一頁同時出現太多金屬、紙、木、玉石，焦點混亂。

## 規則 10：真正缺的不是單張美術，而是生產欄位

目前專案還缺
- `asset_family`
- `presentation_role`
- `color_role`
- `state_set`
- `size_set`
- `crop_mode`
- `value_signal`
- `background_tone`
- `qa_board_path`

這些欄位若不進規格，AI 之後還是會每張圖重新猜一次。

## 製程突破建議

1. 先做 family kit，再做單張資產。
2. 同功能圖示一次出整批，不接受逐張補洞。
3. 生圖 brief 必須帶：family、功能語意、顏色語意、狀態組、尺寸組、使用畫面。
4. QA 必須保留 compare board 與 reference board。
5. 先做跨畫面語意色票，再做個別 icon prompt。

## 這輪結論

高品質 UI 缺的通常不是更會畫，而是：
- 缺固定的功能語意色票
- 缺 family 批次生產
- 缺狀態組與尺寸組
- 缺 screen-context QA
- 缺規格欄位化

把這五件事補齊，量產穩定度會比單純多做幾張漂亮圖更有效。
