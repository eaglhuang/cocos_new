這套技術架構的核心目標是：**「在資源有限的地端環境（甚至是一般家用顯卡），同時支撐 200 個 NPC 的智慧運作。」**

為了達成這個目標，我們選擇的工具必須具備極致的效能、低記憶體占用，且全部為開源或免費。以下我為你梳理的這套「輕量級 AI NPC 技術棧」：

## ---

**1\. 推論引擎：llama.cpp (底層) \+ Ollama (介面)**

這是目前地端執行 LLM 的黃金標準。

* **為什麼選它：** \* **極低顯存占用**：支援 **GGUF (Quantization)** 技術，可以將原本 10GB 的模型壓縮到 2-3GB 運算，且智商損耗極小。  
  * **CPU/GPU 混合運算**：如果你的顯卡顯存滿了，它可以自動把剩下的運算丟給 CPU 跑，保證不會崩潰。  
* **如何與 Cocos 介接：**  
  * 你可以運行 **Ollama**，它會幫你把 llama.cpp 封裝成一個簡單的 **REST API**。你的 Cocos 腳本只需要用 fetch 發送 JSON 即可獲取對話。

## **2\. 向量資料庫：ChromaDB**

負責 NPC 的「長期記憶」與「世界知識」。

* **為什麼選它：**  
  * **內嵌式 (In-process)**：它不像 MySQL 需要安裝複雜的伺服器，它可以直接作為 Python 的一個 Library 運行，存儲在一個資料夾裡。  
  * **高效檢索**：當玩家問到「三國歷史」或「某個角色的往事」，系統只需毫秒級時間就能從萬筆資料中撈出正確的背景資訊。  
* **替代方案：** 如果你希望更輕量、甚至整合進遊戲存檔，可以使用 **SQLite-vss**，它讓你在傳統資料庫裡就能跑向量搜尋。

## **3\. Embedding 模型：FastEmbed / Sentence-Transformers**

這是將文字轉換成「數值向量（Vector）」的關鍵工具，就像是 NPC 的「翻譯官」。

* **推薦模型：** all-MiniLM-L6-v2 或 BGE-m3-tiny。  
* **特點：**  
  * **速度極快**：在 CPU 上就能每秒處理數百個句子。  
  * **免費開源**：完全由 Hugging Face 社群維護，無需任何 API 費用。

**4\. 推薦的小型模型（LLM）**

為了在 200 個 NPC 之間快速切換，你不能用大模型。建議鎖定 **「1.5B 參數以下」** 的模型：

| 模型名稱 | 開發者 | 優勢 | 適合場景 |
| :---- | :---- | :---- | :---- |
| **Qwen2.5-0.5B/1.5B** | 阿里 | **中文能力最強**，指令遵循極好。 | 三國題材、中文對話。 |
| **Phi-3.5-mini** | 微軟 | 邏輯推演強，體積稍大但智商極高。 | 複雜的任務邏輯、決策判斷。 |
| **Llama-3.2-1B** | Meta | 生態系最完整，微調資源多。 | 泛用型、多國語言支持。 |

## ---

**5\. 整合層：FastAPI (Python)**

你需要一個微型的「AI 中台」來連接 Cocos 與 AI 工具。

* **運作流程：**  
  1. **Cocos** 發送事件（例如：玩家送禮物給趙雲）。  
  2. **FastAPI** 接收請求，調用 **ChromaDB** 查詢趙雲對該玩家的好感度歷史。  
  3. **FastAPI** 根據\*\*效用 AI（Utility AI）\*\*公式計算出趙雲應該「高興」還是「懷疑」。  
  4. **FastAPI** 將這些資訊與意圖傳給 **Ollama (llama.cpp)** 生成台詞。  
  5. **Cocos** 接收台詞並顯示。

## ---

**為什麼這套架構最省資源？**

1. **GGUF 量化技術**：  
   它讓 1.5B 的模型在 1GB 顯存內就能跑，這意味著你的遊戲本體還有大量資源來跑渲染（Cocos）。  
2. **按需喚醒 (On-demand)**：  
   200 個 NPC 的「行為邏輯」在 CPU 跑（幾毫秒），只有當玩家「正在對話」的那一個 NPC 需要生成台詞時，才動用 GPU 推論。  
3. **快取機制 (KV Cache)**：  
   如果你有多個 NPC 的個性模板是重複的，llama.cpp 支援緩存相同的前綴引導詞（System Prompt），大幅減少重覆計算的開銷。

**目前實作上的小建議：**

如果你是在單機開發，可以先下載 **Ollama** 並執行 ollama run qwen2.5:1.5b，這會是你最快能測試「NPC 靈魂」的方式。你準備好先從哪一個組件開始試跑看看了嗎？
