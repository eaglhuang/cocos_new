這是一個非常吸引人的挑戰！為了符合 TA 玩家對「儀式感」與「懷舊經營」的偏好，我們不使用現代感太強的按鈕，而是將商城包裝成一個\*\*「家族祭壇」**或**「書房奏摺」\*\*的視覺體驗。

以下是我為這套「許願商城」設計的 UI 原型規劃與動態流程：

# ---

**🏛️ 「聚賢閣」：許願商城 UI 規劃書**

## **1\. 視覺風格與佈局 (Layout)**

整體介面應採古樸木質色調，背景為隱約可見的家祠祭壇或是一張展開的長幅絹布。

### **A. 頂部狀態欄 (Header)**

* **資源顯示**：左側顯示「糧草」、「金錢」，右側顯示「鑽石」與「通用加速卡數量」。  
* **家族名號**：中央顯示玩家的家族堂號（例如：穎川荀氏）。

### **B. 中央核心區：【三巡祭壇】**

這裡分為三個掛載「奏摺」的插槽：

* **許願插槽 (Slot 1-3)**：  
  * **空位狀態**：顯示一個虛擬的「祭拜」手勢，文字提示：「請主公誠心許願」。  
  * **進行狀態**：顯示一張捲起的奏摺，上方有沙漏倒數圖標。  
  * **完成狀態**：奏摺散發淡淡金光，文字提示：「顯靈！點擊領取」。

### **C. 底部切換區 (Tabs)**

* **【珍寶閣】**：即時課金區（現貨、禮包）。  
* **【誠心願】**：本系統核心（時間償還區）。  
* **【陳列室】**：已獲得的英靈虎符與傳家寶展示。

## ---

**2\. 交互邏輯與流程 (UX Flow)**

### **步驟一：誠心許願**

當玩家點擊空位時，彈出一個半透明的卷軸選單，列出今日「三巡」可供許願的課金商品。

**UI 细節**：商品下方會明確標註對比：

* 現貨：$4.99 (立即獲得 10 張)  
* 許願：⏳ 24:00:00 (獲得 1 張)

### **步驟二：時間償還**

點擊確認後，卷軸合攏，伴隨一聲清脆的木魚或鐘鳴聲。

* **動態表現**：背景有名將英靈的虛影一閃而過（若玩家擁有對應性格的名將，會增加「誠心值」）。  
* **加速操作**：點擊進行中的奏摺，可選擇「焚香（使用加速卡）」。

### **步驟三：顯靈領取**

時間結束，玩家點擊奏摺，畫面出現名將（如諸葛亮）的半身像說一段賀語：*「主公至誠感天，此物已為您備好。」*

## ---

**3\. 結合「人性化事件」的特殊反饋**

為了強化您說的生活感，許願過程中可能出現以下隨機 UI 事件：

* **【神速】**：張飛突然衝出來把沙漏踢碎，時間直接減半。  
* **【多得】**：小喬在奏摺旁邊放了一朵花，領取時數量機率變成 2 張。  
* **【驚喜】**：日報回報說今日許願的商品被敵將「遺失」，不需要等了，直接領取。

**4\. Vibe Coding 技術建議 (Cocos Creator)**

### **數據驅動的 UI 渲染**

您可以使用一個簡單的陣列來管理許願狀態，並透過 cc.Tween 實作奏摺的展開與合攏：

TypeScript

interface WishSlot {  
    id: number;  
    targetItem: string;  
    startTime: number;  
    duration: number; // 以秒為單位  
    isFinished: boolean;  
}

### **離線進度校驗**

在 update() 或是 onEnable() 時，讀取本地存儲的 startTime 與 duration。

TypeScript

let elapsed \= (Date.now() \- slot.startTime) / 1000;  
let progress \= elapsed / slot.duration;  
if (progress \>= 1) {  
    // 觸發顯靈狀態  
}

1. **具體化「三巡」內容**：要不要幫您定義早、中、晚三個時段分別會出現哪些「高 CP 值」的許願商品？
