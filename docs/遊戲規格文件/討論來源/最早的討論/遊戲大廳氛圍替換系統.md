這是一個非常務實且具備商業靈活性的設計。將大廳背景設計為\*\*「可替換的場景主題（Themed Backdrop System）」\*\*，不僅能透過晝夜變化提升代入感，還能在春節、週年慶等節日期間，以最低的開發成本營造強烈的慶祝氛圍。

以下是針對「動態氛圍替換」與「商業化佈置」的技術設計補充，已整合入規格書中：

### ---

**🌙 幕府大廳：晝夜變化與節慶氛圍系統 (Dynamic Atmosphere & Theme)**

#### **1\. 晝夜環境切換 (Day & Night Cycle)**

* **自動偵測**：系統抓取玩家設備的本地時間（Local Time）。  
  * **白晝 (06:00 \- 18:00)**：大廳門外透入明亮的日光，室內色調清亮，武將動作較為積極（如：練兵、翻閱文書）。  
  * **夜晚 (18:00 \- 06:00)**：門外為深藍色夜空，室內色調轉為暖橘色（燭光感），環境中加入微弱的蟲鳴與燈火搖曳效果。  
* **技術實現**：在大廳的最底層設有 BGSprite，根據時間變量切換對應的晝夜大圖。

#### **2\. 商業化主題佈置 (Seasonal Theme System)**

為了讓大廳能隨時進行「節慶變裝」，我們將大廳層級拆解為 **「背景、裝飾、前景」** 三層：

* **背景層 (Static Background)**：可替換的 2.5D 大圖（如：平時、春節紅幕、中秋月明）。  
* **裝飾層 (Decoration Slots)**：在大廳牆邊、柱子、天花板預設 4-6 個「節慶插槽」。  
  * **春節**：插槽掛上大紅燈籠、貼上「五虎豐登」春聯。  
  * **週年慶**：插槽放置金色的勳章旗幟或慶祝用的酒罈。  
* **前景層 (Foreground Overlay)**：動態的環境特效粒子（如：白天的浮塵、夜晚的螢火蟲、春節的飄落紅屑）。

#### **3\. 氛圍替換技術架構 (Technical Architecture)**

為了確保 Vibe Coding 能夠輕鬆實作與維護，我們採用 **「配置清單 (Manifest)」** 管理：

{  
  "themes": {

    "cny\_festival": { "bg": "bg\_cny\_hall", "decorations": \["red\_lantern", "couplet"\], "particle": "p\_fireworks" }  
  }

**📝 遊戲入口大廳文件補充：氛圍管理系統**

| 功能組件 | 設計說明 | 開發重點 |
| :---- | :---- | :---- |
| **環境光色調 (Color Grading)** | 透過簡單的 cc.Color 疊加（Overlay）在全畫面上，快速改變冷暖色調。 | 無需重新渲染 3D 模型，僅靠色調變化即可實現晝夜切換。 |
| **節慶插槽 (Event Slots)** | 大廳預設點位，平時隱藏，節慶時啟動顯示特定 Prefab（燈籠、錦旗）。 | 方便美術隨時擴充裝飾品，而不影響武將刷新邏輯。 |
| **動態切換邏輯 (Time Switch)** | 每次啟動或從戰鬥返回大廳時校驗時間，若跨越晝夜門檻則自動切換主題。 | 確保玩家即便玩到半夜，也能感受到大廳「亮燈」的變化。 |

**📈 總結：為什麼這對商業化很有利？**

1. **儀式感與新鮮感**：玩家每次重大節日登入，都會發現議事廳「變樣了」。這種與現實世界同步的感覺，能大幅提升玩家對「世家經營」的沉感。  
2. **低成本運營**：您只需要換一張背景大圖、掛上幾個小裝飾品 Prefab，就能讓整個遊戲看起來煥然一新，不需要改動核心代碼。  
3. **付費氛圍引導**：例如在「春節主題」期間，商城許願池可以變裝為「新春納福箱」，讓玩家更願意在節慶氣氛下進行消費。
