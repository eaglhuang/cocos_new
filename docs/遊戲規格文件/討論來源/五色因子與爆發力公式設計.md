主公聖明，這確實是開發最容易「翻車」的地方。如果 100 個武將乘上 14 個祖先，資料量確實會爆炸。

為了讓系統**好玩、好寫、不崩潰**，我們必須把這套「因子色譜」做成**類別化（Categorized）**，而不是 100 個武將寫 100 套獨立代碼。

### ---

**🎨 一、 五色因子定義：【邏輯分類與功能範圍】**

我們把功能分成「基礎」與「擴充」，MVP 版本（最小閉環）建議只做 **藍、粉、綠**，白與黃可以後續更新。

| 顏色 | 代表意義 | 功能範圍 (影響什麼) | 製作優先順序 |
| :---- | :---- | :---- | :---- |
| **🔵 藍色** | **基礎屬性** | 武/智/統/政/魅的「初始值」與「成長上限」。 | **1 (必做)** |
| **💗 粉色** | **兵種/地形** | 騎/步/弓適性；山/水/林適性。影響棋盤移動與傷害。 | **1 (必做)** |
| **🟢 綠色** | **戰法種子** | 學習特定大招的「習得權限」與「TP 點數折扣」。 | **2 (必做)** |
| **⚪ 白色** | **特殊天賦** | 抗性、性格（冷靜/暴躁）、特殊被動（如：名門）。 | 3 (選做) |
| **🔴 紅色** | **生命體魄** | 影響「最大壽命」與「精力回覆速度」。 | 3 (選做) |

**💡 為什麼不會爆炸？**

因為我們是用 **「標籤庫 (Tag Pool)」**。比如「騎兵適性★3」這份程式碼是共用的，只是掛在趙雲祖先身上還是馬超祖先身上的區別而已。100 個武將其實只是 100 行 JSON 資料。

### ---

**🧬 二、 爆發力 (Explosive Power) 的明確規格**

為了讓 AI 能算出數值，我們必須給出一套**公式**，而不是感覺。爆發力是決定子嗣「屬性跳升」的機率。

#### **1\. 爆發力計算公式：**

$$爆發力 \= 基礎分(50) \+ \[血脈共鳴\] \+ \[異脈加成\] \+ \[名將契合\]$$

#### **2\. 加分規格表：**

| 加分項目 | 觸發條件 | 爆發力加成 | 備註 |
| :---- | :---- | :---- | :---- |
| **血脈共鳴** | 雙方祖先 14 人中，擁有 3 個以上相同標籤（如：皆有 \[北境\]）。 | **\+15** | 強化優點，但也容易變近親。 |
| **異脈加成** | 雙方祖先 14 人完全沒有重複的 ID。 | **\+20** | 混血優勢，基礎數值穩。 |
| **名將契合** | 史實關係（如：周瑜+小喬）或 宿敵關係（如：諸葛亮+司馬懿）。 | **\+25** | 隱藏彩蛋，吸引玩家收集。 |
| **精力加成** | 父母當前精力皆為 100%。 | **\+10** | 鼓勵玩家讓武將休息。 |

**💡 四、 商業與遊戲性的核心：【洗因子】**

主公，這套系統最耐玩的地方在於：**「洗出純種」或「洗出變異」**。

1. **純種玩家**：會瘋狂收集帶有 \[騎兵★3\] 的武將互配，目標是養出一隻 14 個祖先全是 \[騎兵\] 的超級神騎。  
2. **變異玩家**：會拿 \[文官\] 配 \[武將\]，賭那個 10% 的爆發力跳升，看能不能生出「文武雙全」的怪胎。

### **👑 總結建議：**

1. **功能範圍**：第一版只做 **藍、粉、綠** 三色。  
2. **爆發力規格**：採用「標籤匹配機制」，AI 只要比對標籤重複幾次，就能算出分數。  
3. **防爆炸**：所有祖先共用同一套「因子字典（Dictionary）」。
