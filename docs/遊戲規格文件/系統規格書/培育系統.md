# 培育系統規格書

> **來源文件**：鏡像時空育才系統設計.md (doc_spec_0157)、養成系統與教官來源定義.md (doc_spec_0151)、子嗣系統規格書：三國傳承.md、三國傳承：賽馬娘養成融合提案.md

---

## A. 系統描述

培育系統（對標《賽馬娘》訓練模式）是 36 回合制的鏡像時空育才機制。玩家將 5 歲以上子嗣送入培育時空，用快照（鏡像分身）進行 36 回合培養。外界時間 = 1 季。培育中可指派教官，觸發因子覺醒、學習戰法、選擇分支。培育完成後數值寫回本體。

---

## B. 系統目的

1. **深度養成迴圈**：36 回合決策密度高，每回合都有選擇
2. **時間壓縮**：外界 1 季=培育 36 回合，允許大量培養
3. **教官價值**：退役武將→教官→培育中發揮價值（課金循環）
4. **因子覺醒**：培育是除戰場外的另一條覺醒路線
5. **少將養成包裝**：把子嗣的成長過程濃縮成可讀的 36 回合學年，讓玩家能明確感受從璞玉到畢業可戰的節奏

### 自動培育限制條件

> **設計理念**：培育是核心玩法迴圈，自動培育若無門檻開放會大幅降低遊戲耐玩度。因此採取「後期解鎖 + 品質懲罰 + 商業道具消耗」三重門檻。

| 限制項目 | 內容 |
|---|---|
| **前置條件 1** | 玩家累計手動培育畢業次數 ≥ 30 次 |
| **前置條件 2** | 已達成「培育大師」成就（至少 1 名 S 級畢業生） |
| **前置條件 3** | 玩家世代數 ≥ 5（遊戲後期） |
| **消耗道具** | 每次自動培育需消耗「自動培育券」×1（商城/活動獲取） |
| **品質上限** | 自動培育畢業評級最高僅達 A（無法獲得 S 級） |
| **因子覺醒** | 自動培育中不觸發因子覺醒事件 |
| **戰法學習** | 自動培育僅學習基礎戰法（不可學習進階/稀有戰法） |
| **事件機制** | 不觸發隨機事件（好感、受傷、特殊商店等） |
| **教官加成** | 教官 TP 加成僅套用 50%（自動模式教官效果減半） |

### 三階段學年包裝

1. 36 回合正式拆成 `Round 1–12` 基礎打底、`Round 13–24` 路線專精、`Round 25–36` 畢業衝刺三個區段，供 UI、劇本與事件權重共用。
2. 畢業結算除了寫回五維與戰法，也要輸出可供關卡與戰前 intel 使用的 `Graduation_Tags`，例如「槍將預備」、「夜襲適性」、「水戰候補」。
3. 此包裝只影響培育節奏與資訊呈現，不改變外界仍是 `1 季 = 1 次培育` 的正式時間權威。

## F. 培育數值模型與公式 (Nurture Logic & Formulas)

### F-1. 每回合訓練效果 (Per-Round Gain)

每次選擇訓練（兵營/書院/沙盤/巡視/清談）時，屬性獲得量計算如下：

$$Stat\_Gain = Training\_Base \times (1 + Mentor\_Bonus) \times (1 + Synergy\_Bonus) \times (1 + Event\_Modifier)$$

*   **Training_Base (基礎訓練值)**：
    *   Round 1–12 (啟蒙): 10 ~ 15
    *   Round 13–24 (發奮): 15 ~ 25
    *   Round 25–36 (衝刺): 25 ~ 40
*   **Mentor_Bonus (教官效果)**：指派的 6 名教官 (Deck) 提供的屬性獲得率加成（如：關羽提供武力獲得 +20%）。
*   **Synergy_Bonus (羈絆加成)**：當多名教官出現在同一個訓練格子時觸發，每多一人 +10%。
*   **Event_Modifier (事件修正)**：培育中突發事件（如「頓悟」）提供的瞬時倍率。
*   **TP 獲得量**：每次訓練固定獲得 5 TP + 教官溢出加成。

---

### F-2. 體力與精力管理 (Energy & Failure)

*   **精力 (Vigor)**：初始 100。
*   **消耗**：每次訓練消耗 15~25。
*   **恢復**：選擇「休息」恢復 40~60；或觸發醫療事件恢復。
*   **失敗率規則**：
    *   Vigor > 70：失敗率 0%
    *   Vigor 30-70：失敗率 1% - 15%
    *   Vigor < 30：失敗率 20% - 60% (紅區警告)
*   **訓練失敗**：不獲取屬性，且有機率獲得負面因子（如 [挫敗]、[厭學]）。

---

### F-3. 世家血脈保底機制 (Family Priority)

> 依 MCQ Q32 裁決，姓氏匹配可觸發強保底。

1.  **判定條件**：子嗣姓氏 == 支援卡教官家族名（例：關姓子弟使用 SSR 關羽卡）。
2.  **效果 A（事件優先）**：該教官的「核心靈感事件」觸發權重頂格，**36 回合內保證觸發完畢三段事件**。
3.  **效果 B（折扣加碼）**：購買對應家傳戰法時，TP 需求在 50% 折扣基礎上再減 20%（即原價 4 折）。

---

### F-4. 重修與畢業紀錄 (Retraining slots)

> 依 MCQ Q31/Q69 裁決，單一子嗣有 3 個記錄槽位。

1.  **重修流程**：畢業結算後，玩家可選擇「保留並重修」。
2.  **存檔上限**：系統最多保存該子嗣最近 3 次的畢業快照實例。
3.  **無代價畢業**：重修完全不附加任何副作用（無疲勞因子）。
4.  **定型 (Commit)**：玩家從 3 份成績單中選取一組進行「正式畢業」，寫入武將 Prowess 實力層，其餘 2 份記錄徹底銷毀且不得重來。

### F-5. 自動培育懲罰公式

```
AUTO_PENALTY = 0.7  // 訓練獲取量 * 0.7

```

### F-7. 戰法槽位（Loadout Capacity）

> **來源**：關卡設計與美術挑戰建議.md (doc_spec_0068)

武將可裝備的戰法數量由**智力（INT）**決定，形成「智力=戰術廣度」的策略深度。

```
Loadout_Capacity = max(1, floor(INT / 30))  // 智力每 30 點增加 1 個槽位
// 智力  1~29 → 1 槽
// 智力 30~59 → 2 槽
// 智力 60~89 → 3 槽
// 智力 90~99 → 4 槽（目前上限）
```

| INT 範圍 | 戰法槽位數 | 範例武將 |
|---|---|---|
| 1–29 | 1 槽 | 低智武將（純武力型） |
| 30–59 | 2 槽 | 均衡武將 |
| 60–89 | 3 槽 | 謀士型（如郭嘉） |
| 90–99 | 4 槽（上限） | 頂尖謀士（如諸葛亮） |

### F-8. 子嗣資訊分層揭露（Revelation_Level）

> **對應裁決**：Q4 採組合式方案，正式順序為「隱藏 → 傾向 → 區間 → 精確值」。

1. 子嗣未進入正式評估前，不直接公開五維精確值，避免培育前的資訊答案被一次看完。
2. 分層揭露目的是把子嗣養成做成長線判讀與事件推進。

| Revelation_Level | UI 顯示內容 | 正式邊界 |
|---|---|---|
| 0 (Hidden) | 完全隱藏 | 僅顯示剪影與家族來源 |
| 1 (Trend) | 顯示傾向 | 如「偏武」、「內政型」 |
| 2 (Bracket) | 顯示區間 | 如「武力 60–69」 |
| 3 (Exact) | 顯示精確值 | 完整公開，可進行戰術推薦 |

## N. 血脈保底機制補遺（2026-04-12）

> **正式共識**：依 MCQ Q32 裁決（選項 D），世家血脈連動機制為正式規則。

| 條件 | 效果 |

| 子嗣姓氏 == 支援卡家族（如 關家子弟裝備 SSR 關羽卡） | 連續事件觸發優先排程至最高、**36 回合前保證第 3 段事件觸發且 100% 成功**、金戰法 PT 需求 -20% |

## 雙層數值培育
培育主要提升的是實力層 (Prowess, 0~2000+)，並受制於先天的資質層 (Talent, 0-100)。
