## A. 系統描述

戰法系統管理所有部隊級的可施放技能。**戰法 (Tactic)** 指部隊級的變動技能，消耗 **TP**，會改變地圖格狀態或造成傷害。這與消耗精力且由模板固定的 **個人奧義 (General Ultimate)** 不同。

---

## F. 戰場戰法公式模型 (Battle Formulas)

### F-1. 傷害與結算公式

$$Final\_Damage = (Base\_ATK \times Skill\_Mult \times Counter\_Ratio) \times (1 + Terrain\_Bonus + Weather\_Bonus + Adaptation\_Bonus)$$

*   **Counter_Ratio (兵種相剋)**：相剋 1.3，被剋 0.8。
*   **Skill_Mult**：各戰法基礎倍率。

### G-0. 戰法發動權限與執行範圍 (Execution Scope)
根據戰法來源的不同，其發動與操作規則分為以下三種模式：

1. **(A) 武將個人習得的戰法（如教官靈感、血統傳承）**：
   * 所有裝備該武將的部隊皆有權限發動。
   * 會受到玩家手動指令或軍師 AI 控制，但**必須符合該戰法特有的兵種限制**（例如：習得「騎兵衝鋒」但帶領步兵時不可發動）。
2. **(B) 虎符精銳部隊的戰法**：
   * **綁定部隊，自動發動**：僅限裝備該虎符的「那支特定部隊」具備此能力。
   * 特徵是**條件觸發型自動技**，當戰場狀態吻合設定條件時，不需消耗額外操作或 TP（視具體設定）即可自動使用其專屬能力。
3. **全局場景戰法（如火燒赤壁、水淹七軍）**：
   * **場景自動生效**：由智將發動後，戰鬥開始時或指定回合，全場景自動執行大範圍變更（如改變整個地面的 Tile State）。
   * **我軍協同聯動**：在該場景作用期間，我方部隊若行動符合該環境戰術的相容條件（例如火燒場景中進行攻擊），也會觸發額外的場景戰法附屬特效或傷害。

## H. UI 強約束、環境判定與資源需求

1.  **UI 強約束 (Weather Gray-out)**：
    *   ⚠️ 細節請見 **[戰場適性系統.md](戰場適性系統.md)**。當前天氣若觸發禁用條件，包含「火」之戰法按鈕變灰，禁止點擊並顯示 Toast 提示。
2.  **美術特效配合**：
    *   **品質分級**：SSR/UR 戰法需有「半身英靈面」閃現演出。
    *   **環境回饋**：火計需在格子上持續顯示火焰 VFX；水計需有流動波紋，且藍色水格本體要保有右向沖刷的方向感。
    *   **資源目錄**：`assets/fx/tactics/` 下需包含對應 ID 的 Prefab。
3.  **戰場掠奪 (Loot)**：
    *   特定戰法（如峽谷落石、火燒敵營）被擊破或佔領後，觸發金色文字飄揚效果，顯示獲得的資源清單。

### G-5-2. 戰法評分算法（scoreTactic）

```
score = 0
// 適性規則命中加分
foreach rule in tactic.aptitudeRules:
  generalGrade = getAptitude(general, rule.scope, rule.key)
  if gradeToNum(generalGrade) >= gradeToNum(rule.minGrade):
    score += 10

// 偏好屬性加分
foreach stat in tactic.preferredStats:
  score += general.stats[stat] × 0.01

// 稀有度門檻
if score >= tier.minScore: 候選戰法
```

### G-5-4. Battle / General Detail 正式資料流（2026-04-15）

1. 武將 seed 資料以 `assets/resources/data/generals.json` 內的 `tacticSlots[]` 為唯一入口。
2. `GeneralDetail` Skills 分頁直接讀 `tacticSlots[]`，再查 `tactic-library.json` 取 `displayName / category / aptitudeRules / sourceGeneIds`。
3. `BattleScene` 不再維護平行戰法名單，而是由 `BattleSceneLoader` 讀同一份 `tacticSlots[]`，組出 HUD `戰法摘要` 與可執行的戰鬥技能 metadata。
4. 戰鬥演出層統一透過 `battleSkillId -> assets/resources/data/skills.json` 轉譯；`skills.json` 目前有 **13** 條 signature battle skills，供戰法/奧義共用時間軸與效果包裝。

### G-5-4-1. BattleScene 流程責任邊界（2026-04-20）

- `BattleScene` 只保留場景入口、生命週期、資源組裝與把事件交給 `BattleUIBridge.subscribeEvents()` 的接線，不直接承擔戰法流程細節。
- `BattleSceneLoader` 只負責資料載入與組裝，例如 `tacticSlots[]`、戰法摘要來源與初始技能 metadata。
- `BattleSkillTargetingFlow` 負責戰法 / 奧義的選目標、棋盤預覽、取消與點擊解析。
- `BattleSceneFlow` 負責回合 Banner、場景戰法提示、`replay`、`restart` 與回合收尾。
- 後續若要新增場景級流程，優先放進 `BattleSceneFlow`，不要再把流程邏輯堆回 `BattleScene`。
- `BattleUIBridge` 負責全域戰鬥事件訂閱、全域輸入轉接、戰鬥 VFX、combat visual queue 與 UI 呈現，不碰戰法目標解析或重開局流程。
- `TurnFlowManager` 負責 UI 事件轉接與回合推進，不碰戰法目標解析或重開局流程。

1. 每張卡只收同一種 runtime 家族，不跨 family 混收，避免驗收時無法定位失敗來源。
2. 同卡必須同時達成四個門檻才可關卡：`unit test`、`畫面表演`、`數值公式`、`整合演出流程`。
3. 32 類戰法不是 32 張卡，而是先壓成 5 個可驗收模組包；每個戰法只靠 `targetMode + module stack + tuning params` 掛上對應卡。
4. 若後續新增戰法，只能掛回上述既有 family；只有當 module stack 無法描述時，才允許新開卡。
