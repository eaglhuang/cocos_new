> **2026-04-26 覆寫裁定：畢業後成長與戰場容量**
>
> 1. 武將沒有傳統 `Level / EXP`。`Prowess` 是培育畢業後的戰場實力快照，除裝備、英靈卡、虎符與特定外部加成外，不用刷副本經驗線性提升。
> 2. 畢業後的戰場成長感由 `General_Fame -> General_Rank -> Command_Capacity` 承接，不直接增加基礎 Prowess；此處 `Command_Capacity` 是可見帶兵上限。
> 3. 帶兵上限公式建議：`Command_Capacity = Base_By_General_Rank + LEA * 50 + Fame_Bonus + Equipment_Command_Bonus + Event_Bonus`。
> 4. 出征軍勢公式：`Sortie_Troop_Pool_Max = min(Command_Capacity, City_Troops_Current, Stage_Sortie_Cap)`。
> 5. 部署限制公式：`CanDeploy = Sortie_Troop_Pool_Current >= Unit_Troop_Cost && Current_Food >= Food_Cost && Field_Unit_Count < Field_Unit_Limit`。
> 6. 戰損公式：`Lost_Troops = Unit_Troop_Cost - Returned_Troops`；戰後 `City_Troops_Current = City_Troops_Before - Total_Lost_Troops`，並由 `Recruit_Regen` 逐回合補回。
# 數值系統（Game Numerical System）

> 本文件定義遊戲內所有戰鬥數值、屬性計算、成長曲線與傷害結算的統一規範。
> 格式：A–J 標準規格

---

## A. 系統描述

數值系統是遊戲所有戰鬥與成長機制的底層數學框架，統一管理角色屬性、傷害計算、成長曲線、資源倍率與各子系統間的數值流轉。

**Unity 對照**：類似 Unity 中 `ScriptableObject` 驅動的 Stats System，但這裡以 JSON 配置 + 純 TypeScript 公式引擎（`FormulaSystem`）實現，不依賴 Component。

**核心職責**：
- 定義武將六色屬性（STR/INT/LEA/POL/CHA/LUK）的數值範圍與成長規則
- 統一戰鬥傷害、治療、剋制、適性修正的計算公式
- 管理 EP（爆發力）、精力（Vitality）、SP（技能能量）三大運作值
- 定義成長曲線（年齡修正、轉生上限、培育增益）
- 提供數值平衡調校的統一介面（所有常數集中管理）

## B. 系統目的

| 目的 | 說明 |
|---|---|
| 數值一致性 | 所有子系統的數值計算從此統一參照，避免各系統各自為政 |
| 平衡調校基底 | 集中管理倍率、常數、上限值，便於後期數值調整 |
| 可預測的戰力評估 | 玩家可透過公式理解「為什麼這一刀打這麼痛」 |
| 成長感可量化 | 屬性、等級、因子、轉生每一步都能在數值上看到差異 |
| AI 縮放基準 | 為 AI 武將強度系統提供可靠的數值基準線 |

### E-3. 傷害計算公式

```
// === 主傷害公式 ===
final_damage = max(1, floor(
    base_attack
    × counter_mod           // 兵種剋制（1.3 / 0.8 / 1.0 / 1.5機械被近戰）
    × terrain_atk_mod       // 地形攻擊修正
    × weather_atk_mod       // 天氣攻擊修正
    × troop_atk_mod         // 兵種適性攻擊修正
    × general_buff          // 武將加成
    - base_defense
    × terrain_def_mod       // 地形防禦修正
    × weather_def_mod       // 天氣防禦修正
    × troop_def_mod         // 兵種適性防禦修正
))

// === 最終攻擊力 ===
final_attack = base_attack × (1 + terrain_mod.atk) × (1 + weather_mod.atk) × (1 + troop_mod.atk)

// === 最終防禦力 ===
final_defense = base_defense × (1 + terrain_mod.def) × (1 + weather_mod.def) × (1 + troop_mod.def)

// === 最終移動力 ===
final_move = base_move + terrain_mod.move + troop_mod.move

// === 最終命中率 ===
final_accuracy = base_accuracy + terrain_mod.acc + weather_mod.acc

// === 治療公式 ===
heal_amount = max(fixed_value, target_max_HP × heal_ratio)

// === 暴擊判定 ===
crit_chance = base_crit + LUK / 200
crit_damage = final_damage × 1.5  // 暴擊倍率
```

### E-15. 初期 / 中期 / 後期錨點

| 階段 | 武將帶兵上限 | 城市兵源上限 | 單次普通兵投入 | 單次精銳投入 | 場上部隊上限 | 每回合補兵 | 玩家感受 |
|---|---|---|---|---|---|---|---|
| 初期 | 8,000 | 8,000~10,000 | 400~700 | 1,200~1,500 | 3 | 600~900 | 連續叫兵很明顯，但亂送會痛 |
| 中期 | 13,000~16,000 | 14,000~20,000 | 700~900 | 1,500~2,000 | 4 | 1,200~1,800 | 可同時鋪更多線，開始講節奏 |
| 後期 | 20,000~30,000 | 22,000~40,000 | 900~1,200 | 2,000~3,000 | 5 | 2,000~3,500 | 大軍壓境，官職與治理差距會被看見 |

**平衡原則**

1. 初期不靠超低兵力做假無限召喚，而是保留足夠大的帶兵池與低耗部隊。
2. 中期開始讓官職與治理明顯影響城市兵源與帶兵上限，玩家能感受到養成結果。
3. 後期的成長重點是更大的軍勢池與更高的單次投入，而不是把所有兵種縮成碎數字。

## F. 公式相關

### F-1. 傷害公式總覽（偽代碼）

```typescript
function calculateDamage(
    attacker: UnitStats,
    defender: UnitStats,
    terrain: TerrainType,
    weather: WeatherType
): number {
    // 1. 取得兵種剋制倍率
    const counterMod = getCounterMod(attacker.troopType, defender.troopType);
    // COUNTER_MOD_TABLE: { [key]: 1.3 | 0.8 | 1.0 | 1.5 }

    // 2. 取得地形修正
    const terrainAtk = getTerrainMod(attacker.terrainAptitude, terrain).atk;
    const terrainDef = getTerrainMod(defender.terrainAptitude, terrain).def;

    // 3. 取得天氣修正
    const weatherAtk = getWeatherMod(attacker.weatherAptitude, weather).atk;
    const weatherDef = getWeatherMod(defender.weatherAptitude, weather).def;

    // 4. 取得兵種適性修正
    const troopEfficiency = APTITUDE_EFFICIENCY[attacker.troopAptitude];
    // { S: 1.2, A: 1.1, B: 1.0, C: 0.9, D: 0.8 }

    // 5. 武將加成（統率/STR/INT 視武器類型）
    const generalBuff = attacker.generalStatBonus;

    // 6. 暴擊判定
    const isCrit = Math.random() < (attacker.baseCrit + attacker.LUK / 200);
    const critMod = isCrit ? 1.5 : 1.0;

    // 7. 最終傷害
    const rawDamage = attacker.attack * counterMod * (1 + terrainAtk)
        * (1 + weatherAtk) * troopEfficiency * generalBuff * critMod
        - defender.defense * (1 + terrainDef) * (1 + weatherDef);

    return Math.max(1, Math.floor(rawDamage));
}
```

function calculateEP(parents: GeneralPair, ancestors14: GeneralID[]): number {
    let ep = 50;

    // 血脈共鳴：14 人中 ≥3 同標籤
    if (countMaxSameTag(ancestors14) >= 3) ep += 15;

    // 異脈加成：14 人完全無重複 ID
    if (new Set(ancestors14).size === 14) ep += 20;

    // 名將契合：史實/宿敵配對
    if (isHistoricalPair(parents)) ep += 25;

    // 精力加成
    if (parents.father.vitality === 100 && parents.mother.vitality === 100) ep += 10;

    // 天命波動
    const avgLuk = (parents.father.LUK + parents.mother.LUK) / 2;
    if (avgLuk > 80) ep += randomRange(5, 15);
    else if (avgLuk < 40) ep -= randomRange(5, 10);

    return ep;
}

### F-4. 主將帶隊總戰力公式（Battle Power Rating）

> **Unity 對照**：類似 RPG 中的 Combat Power (CP) 計算，將所有影響戰鬥結果的因子歸一為一個可比較的數值，用於快速判斷勝敗、匹配平衡、UI 顯示。

```typescript
// === 單一主將帶隊的總戰力評估 ===
// 用途：快速預判一場戰役的勝敗、UI 戰力顯示、AI 決策依據
// 注意：英靈卡正式只有 1 張；參謀官 / 後勤官屬 sortie support，主要影響資源池，這裡只做戰力折算
interface SortieSupportContext {
    spiritCard: SpiritCard | null;    // 每名主將最多 1 張
    strategist: General | null;       // 出征前編制，非戰場額外單位
    deputy: General | null;           // 出征前編制，非戰場額外單位
    baseFoodCap: number;              // 關卡或主將提供的原始糧草上限
}

function calculateBattlePowerRating(
    general: General,              // 主將
    troopDef: TroopDef,            // 當前指揮兵種
    tally: TigerTally | null,      // 裝備的虎符（可無）
    support: SortieSupportContext, // 命槽 + 出征支援
    terrain: TerrainType,          // 地形
    weather: WeatherType           // 天氣
): number {

    // ═══════ 1. 主將基礎戰力 ═══════
    // 六維加權（武力系與智力系根據兵種類型側重不同）
    const isPhysical = ['CAVALRY', 'INFANTRY', 'SHIELD', 'PIKE'].includes(troopDef.Base_Type);
    const primaryStat = isPhysical
        ? general.STR * 0.5 + general.LEA * 0.3 + general.LUK * 0.05
        : general.INT * 0.5 + general.LEA * 0.3 + general.LUK * 0.05;
    const secondaryStat = general.CHA * 0.1 + general.POL * 0.05;
    const generalPower = (primaryStat + secondaryStat) * AGE_MODIFIER[general.agePhase];

    // ═══════ 2. 兵種基礎戰力 ═══════
    const troopBasePower = (troopDef.Stats.Attack + troopDef.Stats.Defense) * 0.5
        + troopDef.Stats.Move * 50 + troopDef.Stats.Range * 30;

    // ═══════ 3. 兵種適性修正 ═══════
    const aptitude = APTITUDE_EFFICIENCY[general.troopAptitude[troopDef.Base_Type]];
    // S=1.2, A=1.1, B=1.0, C=0.9, D=0.8

    // ═══════ 4. 虎符加成（含星級、適性與血脈共鳴） ═══════
    let tallyPowerMod = 1.0;
    if (tally && troopDef.Is_Special) {
        const QUALITY_MOD = { UR: 1.5, SSR: 1.3, SR: 1.15, R: 1.0 };
        const qualityMod = QUALITY_MOD[tally.Quality];
        const starMod = 1 + tally.Star * 0.08;          // 1★~5★ → +8%~+40%
        const resonanceMod = (tally.Resonance_Template
            && tally.Resonance_Template === general.Template_ID) ? 1.1 : 1.0;
        // 虎符適性加成（地形、家族、性別）
        const affinityMod = getTallyAffinityMod(tally, general, terrain);
        tallyPowerMod = qualityMod * starMod * resonanceMod * affinityMod;
    }

    // ═══════ 5. 英靈卡支援加成 ═══════
    // 英靈卡在戰力公式中只吃當前主卡，不再採多卡同時計算
    const spiritBonus = support.spiritCard
        ? Math.min(SPIRIT_CARD_POWER_CAP, support.spiritCard.power_rating)
        : 0;

    // ═══════ 6. 出征支援折算 ═══════
    // 參謀官 / 後勤官的正式影響是資源池，不直接乘算最終傷害；Battle Power 只做預估折算
    const strategistSupport = support.strategist
        ? Math.min(
            STRATEGIST_POWER_CAP,
            Math.floor((support.strategist.INT * 0.7 + support.strategist.POL * 0.5) * STRATEGIST_POWER_RATE)
        )
        : 0;
    const deputyRate = support.deputy
        ? Math.min(DEPUTY_FOOD_CAP_RATE_MAX, 0.03 + support.deputy.LEA * 0.001 + support.deputy.POL * 0.0006)
        : 0;
    const deputyFoodBonus = Math.floor(support.baseFoodCap * deputyRate);
    const deputySupport = Math.min(DEPUTY_POWER_CAP, Math.floor(deputyFoodBonus * DEPUTY_FOOD_POWER_WEIGHT));

    // ═══════ 7. 地形修正 ═══════
    const terrainMod = getTerrainMod(general.terrainAptitude, terrain);
    const terrainFactor = 1 + (terrainMod.atk + terrainMod.def) * 0.5;

    // ═══════ 8. 天氣修正 ═══════
    const weatherMod = getWeatherMod(general.weatherAptitude, weather);
    const weatherFactor = 1 + (weatherMod.atk + weatherMod.def) * 0.5;

    // ═══════ 9. 轉生倍率 ═══════
    const reincarnationMod = 1 + general.reincarnation * 0.15;  // 每轉 +15%

    // ═══════ 總戰力公式 ═══════
    const totalPower = Math.floor(
        (generalPower + troopBasePower + spiritBonus + strategistSupport + deputySupport)
        * aptitude
        * tallyPowerMod
        * terrainFactor
        * weatherFactor
        * reincarnationMod
    );

    return totalPower;
}

// === 戰役勝敗快速預判 ===
function predictBattleOutcome(
    attackerPower: number,
    defenderPower: number
): { winRate: number; result: string } {
    const ratio = attackerPower / Math.max(1, defenderPower);
    // ratio > 1.3 → 攻方大優，winRate ~85%+
    // ratio 0.9~1.1 → 勢均力敵
    // ratio < 0.7 → 攻方大劣
    const winRate = Math.min(0.95, Math.max(0.05,
        0.5 + (ratio - 1.0) * 0.6
    ));
    const result = winRate > 0.65 ? '攻方優勢' :
                   winRate > 0.45 ? '勢均力敵' : '攻方劣勢';
    return { winRate: Math.round(winRate * 100), result };
}
```

### F-4a. 戰場支援防火牆

1. 英靈卡正式只允許 1 張，不再採多卡陣列疊加。
2. 虎符是兵種權限與戰場定位來源；英靈卡是命紋 / 祖靈被動來源，兩者允許同裝，但責任必須分離。
3. 參謀官 / 後勤官只影響資源池，不直接加入最終攻擊倍率乘算；這兩個角色在 `Battle Power Rating` 中僅做折算，用於 UI 與 AI 估算。
4. 出征支援角色不參與虎符 Set 計算，也不得把自己的虎符或英靈卡效果直接投射到主將身上。
5. 同一武將不得在同一場戰鬥同時擔任主將、參謀官與後勤官兩個以上角色。
