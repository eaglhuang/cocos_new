# 結緣系統（配種）規格書

> **來源文件**：遊戲系統：結婚生子與傳承.md、子嗣系統規格書：三國傳承.md、精力系統優化建議.md (doc_spec_0148)、遊戲機制：精氣、大器、回流.md、戰棋遊戲的武將傳承與繁衍.md (doc_spec_0126)、三國傳承：賽馬娘養成融合提案.md

## A. 系統描述

結緣（配種）是玩家創造次世代武將的核心機制。透過選擇父母雙方武將進行結緣，系統根據雙方血統、因子、精力狀態產生子嗣。子嗣在 0~5 歲為神祕期，5 歲開獎時揭曉五維與因子，之後進入培育或直接轉教官。

---

## B. 系統目的

1. **世代循環引擎**：名將會老死，但因子永流傳——驅動玩家持續繁衍
2. **策略決策點**：父母選擇、精力管理、名士占卜形成深度決策鏈
3. **數值核心**：爆發力公式決定子嗣品質，連結血統系統
4. **情感連結**：5 歲開獎的開盲盒感與「大器晚成」的驚喜反轉
5. **和平血統線**：允許玩家在不把核心名將持續推上前線的情況下，仍能以保種為目的延續家系

## F. 公式相關

### 受孕率
```
Conception_Rate = Base(70%) × (1 + CHA_avg/200) × (1 + LUK_avg/200) × Vitality_Factor
Vitality_Factor: >50% → ×1.0; 30~50% → ×0.5; <30% → ×0.1

// 高齡與多次自然親傳只作 soft penalty，不另開新資源條：
// Age_Factor = Father_Age_Factor × Mother_Age_Factor
// Birth_Load_Factor = max(0.4, 1 - 0.15 × max(0, Mother_Previous_Births - 1))
// 最終受孕率上界限制（避免保底/特化組合出現不合理的 100% 成功率）：
// Final_Conception_Rate = min(Conception_Rate × Age_Factor × Birth_Load_Factor, 0.95)
```

### 子嗣因子繼承機率
| 來源 | 機率 |
|---|---|
| 父母（直系） | 100% 進入因子池 |
| 祖父母 | 50% 進入因子池 |
| 曾祖父母 | 25% 進入因子池 |

### 子嗣屬性決定
```
Base_Stat = (Father_Stat + Mother_Stat) / 2 × (EP / 100)
// EP = 爆發力，由血統共鳴+異脈+契合+天命波動決定
```

### 大器晚成（Is_Daqi）判定與效果（標準化說明）
```
// 判定時點：5 歲開獎時（Unveil）完成基礎判定並寫入 Is_Daqi
// 判定條件（需同時滿足）：
// 1) Unveil_Base_Stat <= 30（非常低的初始數值暗示）
// 2) 名士暗示或天象標記（Oracle_Hint == LATE_BLOOMER）
// 3) 隨機種子檢查通過（避免過度確定化，維持驚喜感）
Is_Daqi = (Unveil_Base_Stat <= 30 && Oracle_Hint == LATE_BLOOMER && Random() < 0.5)

// 效果：大器晚成不會在 0~5 歲立即提升屬性；其為一個長期成長 Modifier。
// 當角色年齡 >= 25 歲時，啟動成長加成：
if (Age >= 25 && Is_Daqi):
  Stat_Growth_Rate *= 1.5  // 長期成長速率×1.5（持續影響至角色壽命期）
```

// 備註：此處為標準化規範，其他文件（培育、轉職、名士預言）應以此邏輯為準。

### 近親劣化
```
if (Ancestor_14.contains(same Bloodline_ID)):
    Vitality_Max -= 30%
    Mutant_Rate(Negative) += 50%
```

## K. 近親繁殖（Inbreeding）範圍設定（新規）

// 目標：以可配置參數定義禁止配對的近親範圍，避免模糊或全域封鎖。

- 參數：`Inbreeding.BlockGenerations`（整數，預設 3）
  - 含義：若父母雙方在最近 `BlockGenerations` 代內存在共同祖先（例如有相同的祖父母），則禁止配對。
  - 範例：`BlockGenerations = 3` → 若在 祖父母（3 代）內有交集，則配對被系統阻擋。

- 實作建議：在結緣前進行檢查
```
if (Ancestors_Intersection_Generations(father, mother) <= Inbreeding.BlockGenerations):
    reject("近親範圍內不可配對：請選擇血緣更遠的配對。")
```

- UI：在配對介面新增顯示「近親警告」的 tooltip，並展示共同祖先的最小世代距離。

// 備註：此設定為系統層級配置（`config/gameplay.json`），可由設計在不同階段調整數值以達到平衡或倫理考量。

## L. 精力值（Vigor）與生育限制補充

> **來源**：養成循環與營收節奏探討.md (doc_spec_0066)

### 精力冷卻機制

精力值（Vigor）是獨立於壽命的武將體力資源，控制結緣行為的**時間節奏**。

### 和平血統線與退役保種邊界

1. 退役武將若仍以活體 `General` 身分留存，且尚未轉為教官卡，可參與低風險的保種型結緣。
2. `Peace_Lineage` 只是既有結緣的模式標記，仍完整沿用 `Vigor`、`Pregnancy_Lock`、`Breeding_Cap` 與近親檢查，不另開 `宗廟`、`生命精氣槽` 或其他獨立母系統。
3. 武將一旦被卡片化為 `Mentor / Support_Card`，後續便只保留教學職能，不再作為親傳父母來源；血統延續只屬活體武將線。
4. 傳家寶、家族藏物與裝備轉移仍由倉庫 / 道具 / 俘虜處置等既有系統承接，本規格不額外建立祖廟或宗廟收藏線。

```
// 精力自然回復
Vigor_Recovery = 1 unit / governance_month   // 每治理月 +1 點

// 結緣後精力凍結期
Post_Breeding_Cooldown = 3 governance_months  // 母方結緣後需 3 個治理月才能再次行動
// 注意：此期間母方精力不回復（Pregnancy_Lock 狀態）

// 父方精力消耗
Father_Vigor_Cost = 50  // 父方結緣消耗精力
Mother_Vigor_Cost = 100 // 母方結緣消耗精力（+待產凍結）
```

### 一生生育上限

每位武將的**親傳子嗣**上限為 **3~5 次**（具體由武將根基與精力上限決定）。超過上限後，武將生育力枯竭，無法再受孕（即便精力回滿也無效）。

```
Lifetime_Breeding_Cap = 3 + floor(General.Vitality / 30)  // 精力上限越高，生育次數越多（上限 5）
// 精力上限 <90 → 3 次; 90~119 → 4 次; ≥120 → 5 次
```

> **付費道具連動**：補元散可快速回復精力縮短冷卻；傳宗秘藥可提升 Lifetime_Breeding_Cap +1（每武將最多使用 1 次）。

## M. 血統模擬器與名士批語嵌入規格

> **整併來源**：名士預言：血統模擬器新玩法.md
> **衝突處理**：名士批語屬於「方向性提示」，不能覆蓋 `名士預言系統.md` (doc_spec_0006) 的準確率與偏見機制。

### 結緣介面的模擬器布局

- 父母放入後，介面中央顯示 14 人血統樹 / 矩陣摘要與預估爆發力（EP）。
- 介面左右可同時展示 2~3 位名士的批語卡片，形成「多視角解籤」體驗。
- 名士批語預設只給方向性暗示，例如「偏武」「偏智」「宜為人師」「大器晚成」，不直接洩漏最終五維數字。

### 矛盾批語原則

- 不同名士可以對同一組父母給出互相矛盾的結論，這是正式設計的一部分，不是錯誤。
- 玩家應把批語理解為培育方向建議，而不是百分之百準確的數值預測。
- 若玩家使用 `求籤令` 指定特定名士，則改以該名士的偏見領域輸出更聚焦的批語。

### 祖輩顯靈彩蛋

- 當血統矩陣中存在特定史實名將或宿命血脈時，可額外插入「祖輩顯靈」批語。
- 祖輩顯靈只強化情感與風格，不單獨改寫數值；數值仍回到血統與因子公式計算。
