# 官職系統規格書

> **來源文件**：幕府大廳 UI 設計與優化.md (doc_spec_0120)、遊戲玩法：幕府議事廳設計.md、策略養成遊戲平衡與營運探討.md (doc_spec_0081)、遊戲設計規格書：三國傳承.md
> **定位**：承接大廳、雙牆任務、世界沙盤解鎖與治理權限的長線進度母規格。

---

## A. 系統描述

官職系統是玩家在治理 / 大廳 / 戰場三線循環中的正式長線進度。它不只是 UI 標籤，而是統一管理「能看到哪些任務」、「能解鎖哪些大廳功能」、「世界沙盤能推進到哪一階段」、「能否參與高階外交 / 聯盟事務」的權威系統。

正式共識如下：

1. 官職已升格為獨立母規格，不再只作為大廳內部掛點。
2. 大廳、領地治理、他國 AI 與世界沙盤，都只能讀取官職系統輸出的權限快照。
3. 天下大亂或版圖重組時，官職等級與已解鎖的大廳 / 沙盤權限不倒退。

**Unity 對照**：可視為帳號級 `Progression Track + Unlock Table`。大廳只是顯示與互動入口，真正的權限真相由獨立的 Progression System 控制。

---

## B. 系統目的

1. **進度統一**：把任務上限、世界沙盤、外交層級與功能開放統一收束到單一真相來源。
2. **節奏分層**：避免玩家過早看到全部高難內容，改用官職逐層打開。
3. **治理回饋**：讓內政、戰功、聯盟參與都有長線帳號回報。
4. **賽季穩定**：天下大亂只重洗版圖，不重置玩家已獲得的中樞進度。

## F. 公式相關

### F-1. 官職經驗來源

```typescript
function calcOfficerExpGain(task: OfficerTaskResult): number {
    const difficultyBase = {
        EASY: 10,
        NORMAL: 18,
        ELITE: 30,
        HIGH: 48,
        HELL: 72,
    }[task.difficulty];

    const domainBonus = task.boardType === 'MILITARY' ? 1.1 : 1.0;
    const successBonus = task.success ? 1.0 : 0.4;
    return Math.floor(difficultyBase * domainBonus * successBonus);
}
```
