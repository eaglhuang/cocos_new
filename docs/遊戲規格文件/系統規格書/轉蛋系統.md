# 轉蛋系統規格書

> **來源文件**：轉蛋與戰場武將處置系統.md (doc_spec_0155)、賽馬娘式轉蛋系統平衡規劃.md (doc_spec_0127)

## A. 系統描述

採用「雙池分工轉蛋制」：`三國名將池（含女性）` 負責抽取可直接培養與上陣的名將本體，`支援卡（教官）池` 負責抽取培育用的教官卡。兩池用途不同，但都屬於正式商業化抽卡入口。

正式共識如下：

1. 轉蛋系統**不產出**英靈卡與虎符卡。
2. 英靈卡與虎符卡統一由名將死亡結算產出。
3. 名將生前退役只進教官 / 傳承路徑，不直接從轉蛋或退役流程拿卡。
3-b. **轉蛋預設無虎符**：玩家從轉蛋獲得的武將，基本出廠都不會自帶虎符軍隊。唯有透過長期培育且該武將死亡結算時，才會留下專屬的虎符遺產。
4. 名將池中的「三國名將」包含女性名將，不另立女性專屬主戰池。
5. 女性名將一律留在名將池內，以 `Female_Role_Type` 區分戰鬥型與輔助型，不拆到支援卡池。

## B. 系統目的

1. **角色獲取主管道**：提供三國名將本體與培育教官兩條正式抽取路線
2. **商業分工清楚**：名將池賣角色廣度，支援卡池賣培育深度
3. **公平性保障**：保底機制確保付費有回報
4. **與死亡傳承解耦**：抽卡只處理活體名將與教官，不混入英靈 / 虎符遺產
5. **雙池敘事清楚**：名將池提供血統種子，支援卡池提供 36 回合培育深度，避免玩家把抽卡誤解成唯一戰力來源

### B-1. 雙池分工的正式敘事

1. `三國名將池（含女性）` 的正式功能是提供可上陣、可繁衍、可進入家族與壽命流程的名將本體，因此它負責的是**血統種子 / 角色廣度**。
2. `支援卡（教官）池` 的正式功能是提供 36 回合培育中的事件、TP、爆發率與上限體驗，因此它負責的是**培育深度 / 成長天花板**。
3. 兩池都可以商業化，但兩者都不得販售 `英靈卡`、`虎符卡` 或舊稿中的 `DP` 部署點概念。
4. 抽卡所得只決定角色入口與培育工具，不直接覆蓋戰場上的 loadout 判讀、地形對策與操作風險。

### G-1. GachaMain 商業主視覺與人物立繪規格

1. `GachaMain` 的 Banner 人物圖屬於商城 / 轉蛋入口的商業主視覺，不等同武將人物介紹頁的資訊展示立繪；前者優先追求第一眼吸引力、池別情緒與完整構圖，後者優先追求可讀性與長時間瀏覽。
2. 短期 proof 可重用現有去背 PNG（例如 `assets/resources/sprites/generals/*_portrait.png`），但必須採用人物介紹頁已驗證的 soft-alpha 流程：建立獨立 artwork carrier，依 SpriteFrame `originalSize` / `offset` 做邏輯框 fit，並套用 `PortraitSoftMask` / `effects/portrait-soft-mask` 的 feather 材質，避免硬邊與白底外框。
3. 正式商業版優先規劃 pool-specific key visual：可包含完整人物構圖、背景、武器、旗幟、召喚光效與池別視覺符號。若 key visual 自身已完整承擔 Banner art area，可使用不透明完整圖；透明 PNG 不是硬性要求。
4. UI layout 不得保留 `opacity: 0` 的人物佔位或無用裝飾層。若該池人物圖尚未 ready，應直接省略人物節點，讓 Banner 背景、文字與 fade 維持乾淨結構。
5. 驗收標準：不得出現人物素材邊界矩形、白底鬼影、半透明色塊、硬切 alpha 邊；人物主視覺不得壓住池名條、池別說明 chip 或抽卡 CTA 的可讀性。

## 重複卡與傭兵二轉連動 (Q73)
若玩家此前已透過「傭兵系統結義」將某武將轉正為永久角色，隨後在轉蛋中抽到**同名史實武將**，將不會只視為一般覺醒材料，而是觸發特殊的**「二轉機制（2nd Awakening）」**，解鎖更深層的資質成長與實力上限。
