## A. 系統描述

大廳系統是玩家登入後最核心的長線中樞場景，負責把治理、出征、商城、英靈、人物互動與世界進度收攏到同一個 2.5D 議事廳空間中。它不是單純的首頁，而是長局經營的「司令部場景」：玩家在這裡接任務、看世界局勢、感受晝夜與節慶變化，也透過武將的在場感把靜態選單轉成有生活感的策略空間。

**Unity 對照**：概念上接近把 `MainMenu Scene`、`Campaign Hub`、`World Map Overlay` 與 `Shop Entry` 疊成一個持久 Hub Scene。大廳中的熱點相當於 Unity 中固定 Anchor 的互動節點；武將刷新池則類似輕量 NPC ambient system，但不走完整 NavMesh，只做預設點位輪播。

## B. 系統目的

| 目的 | 說明 |
|---|---|
| 長線中樞 | 將治理、戰場、商城、英靈、人物與世界進度收斂到單一入口 |
| 沉浸感 | 用議事廳、沙盤、武將在場與晝夜光色取代冰冷選單 |
| 任務分流 | 將軍事 / 外交與內政 / 人脈任務明確分牆，降低資訊混亂 |
| 世界感 | 用世界沙盤、遠征階段與外敵動向，讓玩家感受天下不只是一張戰場 |
| 商業掛點 | 許願祭壇與商城入口內嵌於場景，不需額外跳脫世界觀 |
| 低成本營運 | 背景、裝飾、粒子與色調可用配置切換，支援節慶與檔期變裝 |

## E. 製作功能清單

### E-1. 大廳空間總構

| 區塊 | 說明 | 互動類型 |
|---|---|---|
| 左牆 | 全國軍事 / 外交任務牆，偏深紅色語言 | Scroll + 任務接取 |
| 右牆 | 領地內政 / 人脈任務牆，偏土黃與卷軸語言 | Scroll + 任務接取 |
| 中央沙盤 | 顯示中原、邊境、全球遠征三階段進度 | 熱點 + 放大進入 |
| 武將點位區 | 刷新 3~5 位武將在固定點位待命 | 點擊對話 / 查看資訊 |
| 許願祭壇 | 早中晚三巡的時間償還商城入口 | 倒數 / 收取 / 加速 |
| 頂部狀態列 | 顯示玩家資源、官職、時段與提示 | 常駐資訊 |

### E-2. 雙牆任務架構

1. 左牆承接軍事、外交、圍堵、夜襲、遠征前置等高風險任務。
2. 右牆承接內政、人脈、尋訪、賑災、治安、商路與官職累積任務。
3. 已接取任務必須自動置頂並顯示高亮狀態。
4. 任務難度統一為五級：簡易、一般、菁英、高級、地獄。
5. 軍師或高智力武將提出的軍事任務，應能直接掛接場景戰法與關卡設計系統。

### E-3. 任務生成與武將自告奮勇

1. 任務池至少在進入大廳、每日早中晚切換、手動刷新時重算。
2. 官職越高，可見的任務難度上限與沙盤階段越高。
3. 武將與任務標籤高度契合時，允許觸發「自告奮勇」對話事件。
4. 自告奮勇屬於角色彩蛋事件，可提供臨場語氣、插話或一次性小演出，但不建立固定成功率公式。
5. 自告奮勇應作為角色互動機制，而不是額外的獨立玩法頁。

### E-4. 世界沙盤與遠征階段

| 階段 | 名稱 | 說明 |
|---|---|---|
| S1 | 中原統一 | 以本土割據與州郡爭奪為主 |
| S2 | 邊境征伐 | 解鎖邊疆勢力、朝貢與防線事件 |
| S3 | 全球遠征 | 開放異域強敵、世界遠征與最終霸業目標 |

1. 沙盤初期可為縮小桌面模型，隨官職或進度解鎖擴展為更完整世界圖。
2. 世界沙盤應展示遠征箭頭、異域動向與大勢力壓力，不必一開始就全部可操作。
3. 海外或邊境高階敵軍可採非對稱設計，讓高數值敵人同時保留戰術破口。
4. 大廳中的世界沙盤只負責世界態勢與入口，不重複定義戰場內規則。
5. 若觸發天下大亂或世界重組，已解鎖的世界沙盤階段與標記歷史必須保留，不因版圖洗牌而倒退。
6. `S3 全球遠征` 應以「遠征檔案（Expedition Dossier）」先揭露異域勢力輪廓，例如草原霸權、海上商盟與西方遠征軍，讓玩家先感知全球壓力，再逐步解鎖互動。
7. 全球遠征的外敵可採非對稱設計，不要求每一股外敵都等同本土三國勢力的完整治理迴圈；大廳只需承接其世界態勢、進度與入口。

### E-5. 大廳武將刷新池

1. 大廳採定點刷新，不做完整走位 AI。
2. 預設可配置 6~8 個點位，每次實際顯示 3~5 位武將。
3. 進入大廳與每 60 秒可觸發一次洗牌刷新。
4. 武將以原地 Idle 循環、點頭、揮手、看沙盤等動作營造存在感。
5. 頭頂應顯示姓名與官職，讓玩家可直覺點擊。
6. 高難任務被接取時，可讓在場武將做短時集體反應，強化臨戰氛圍。

### E-6. 晝夜與節慶氛圍替換

1. 白天與夜晚至少需要兩套基礎光色與背景狀態。
2. 節慶切換採三層分離：背景、裝飾、前景粒子。
3. 節慶插槽平時可空置或隱藏，活動時透過配置打開。
4. 主程式不得為每個節慶硬寫分支，應以主題配置驅動。
5. 許願祭壇與商城入口是最優先的節慶變裝熱點。

### E-7. 許願祭壇與商城入口

1. 許願祭壇維持經濟系統既有的雙軌商城原則：即買 100%，許願為時間償還下修產量。
2. 大廳只定義空間與互動語言，不覆寫經濟系統中的數值與價格權威。
3. 祭壇採早 / 中 / 晚三巡節奏，每巡可見多個待選商品插槽。
4. 插槽需支援空位、進行中、完成待領取三種視覺狀態。
5. 武將彩蛋事件可短暫影響許願時間或產量，但其數值上下限仍受經濟系統約束。

### E-8. 官職與功能解鎖

1. 官職是大廳功能解鎖與任務上限的重要進度線。
2. 官職提升可解鎖更高難度任務、更多雙牆欄位、世界沙盤階段與新熱點。
3. 官職的完整權威定義已移至 `官職系統.md` (doc_spec_0014)；大廳只負責讀取官職快照並呈現 UI 掛載點與進度反饋。

### E-9. 任務詳情彈窗與派遣流程

1. 軍事與內政任務皆必須可點擊，點擊後在畫面中央彈出 800×900px 的任務詳情視窗，採彈出動畫與阻斷式 Modal。
2. 任務視窗需完整顯示任務標題、任務內容、情報度、派遣武將、獎勵與耗費資訊。
3. 情報度預設可從 50% 起始；50% 時內容允許模糊、遮罩或省略，派探子再探後可升至 100% 並揭露完整任務內容。
4. 行動按鈕固定為橫排四項：`再探`、`執行`、`等待`、`委任`。再探會刷新情報度；執行開始任務；等待保留任務；委任則交由 AI 自動執行。
5. 行動人物必須提供下拉選單供玩家指派武將，並可在任務對應武將上顯示毛遂自薦 / 自告奮勇標記；若有自薦者，應在清單中優先提示。
6. AI 委任可透過勾選框或開關啟用，啟用後由 AI 自動選將、執行與回報，但任務結算規則仍由任務系統統一掌控。
7. 任務視窗需視覺化呈現糧草、兵力、銀兩與時間四種耗費，以及基礎獎勵與完美達成加成的分層資訊。
8. 關閉機制支援 X 按鈕、ESC 與背景點擊。

### E-10. 英靈世家總覽與家族入口

1. 大廳提供君主層 `世家總覽` 熱點，作為玩家管理麾下所有世家與英靈集合的主入口。
2. `世家總覽` 以家族分支為主分組，至少顯示家族名、主卡、家族深度、名聲值與可跳轉入口。
3. 若某武將已形成世家分支，則其人物頁命頁與大廳武將點位可顯示 `世家入口` 捷徑；未形成者不顯示。
4. `世家總覽` 不等於單一武將詳情，它跳的是該家族的陳列 / 管理頁；`英靈陳列室` 則負責單家族卡群瀏覽與主卡切換。
5. 若玩家尚未解鎖世家系統，入口可隱藏，並以大廳提示引導至相關主線。

## F. 公式相關

### F-1. 大廳任務可見性

```typescript
function getVisibleHallTasks(player: HallPlayerState, board: HallBoard): HallTask[] {
    return board.tasks.filter(task => {
        if (task.requiredRank > player.rankTier) return false;
        if (task.requiredStage > player.worldStage) return false;
        if (task.boardType === 'MILITARY' && player.intel < task.requiredIntel) return false;
        return true;
    });
}
```

### F-2. 自告奮勇彩蛋判定

```typescript
function shouldShowVolunteerEvent(task: HallTask, general: GeneralSnapshot): boolean {
  return task.tags.some(tag => general.roleTags.includes(tag))
    && general.mood === 'EAGER'
    && !task.flags.includes('VOLUNTEER_SHOWN');
}
```

1. 自告奮勇只定義「是否出現彩蛋事件」，不定義固定成功率加成公式。
2. 依據最終裁決（Q66=A），自告奮勇具有固定強度：觸發後該任務成功率固定 +50%，硬編進大廳母規格。

### F-3. 大廳武將刷新池

```typescript
function rollHallCharacters(pool: GeneralUID[], hallLevel: number): GeneralUID[] {
    const visibleCount = hallLevel >= 4 ? 5 : 3;
    return weightedSample(pool, visibleCount, general => {
        return general.isFavorite ? 2.0 : general.isAssigned ? 1.2 : 1.0;
    });
}
```

### F-4. 許願祭壇顯示層公式

1. 實際數值、等待時間與產量比率以經濟系統為權威。
2. 大廳僅負責將 `Wish_Pool.Active_Wishes` 映射為場景中的進行中插槽。
3. 加速後剩餘時間不得低於零，完成後應立即轉為可收取狀態。

### F-5. 任務情報度與委任狀態

```typescript
function getHallTaskIntelRevealPercent(task: HallTask): number {
    return Math.max(0, Math.min(task.intelRevealPercent ?? 50, 100));
}

function canRevealFullHallTask(task: HallTask): boolean {
    return getHallTaskIntelRevealPercent(task) >= 100;
}

function canUseReconAction(task: HallTask): boolean {
    return getHallTaskIntelRevealPercent(task) < 100;
}

function isAutoAssignEnabled(task: HallTask): boolean {
    return task.aiAssignEnabled === true;
}
```

1. `Required_Intel` 只負責軍事任務是否可見；`Intel_Reveal_Percent` 只負責任務內容是否完整揭露，兩者不得混用。
2. `Intel_Reveal_Percent` 預設為 50；`再探` 會把它刷新到 100。
3. 50% 情報只能顯示模糊版任務敘述與部分數值，100% 情報才允許完整內容、完整獎勵與完整耗費。
4. `委任` 狀態應讓任務由 AI 自動執行，但仍需保留玩家可回看任務詳情與最終結算紀錄。

## J. 名詞定義

| 名詞 | 代碼/Key | 定義 |
|---|---|---|
| 大廳 | Hall_Hub | 玩家登入後的長線中樞場景，整合治理、世界、商城與人物互動 |
| 幕府議事廳 | Main_Hall | 大廳主場景的世界觀名稱，偏策略與議政語言 |
| 軍事牆 | Military_Board | 左側任務牆，承接軍務、外交與高風險出征任務 |
| 內政牆 | Domestic_Board | 右側任務牆，承接內政、人脈與治理類任務 |
| 世界沙盤 | World_Sandtable | 用於呈現中原、邊境與全球遠征進度的中央世界圖 |
| 遠征檔案 | Expedition_Dossier | 世界沙盤在 S3 用來描述異域勢力、戰線壓力與解鎖節點的摘要資料 |
| 許願祭壇 | Wish_Altar | 將經濟系統的時間償還商城嵌入大廳的空間化入口 |
| 自告奮勇 | Volunteer_Event | 武將主動請纓某個任務的彩蛋互動事件，預設不綁固定成功率公式 |
| 任務詳情視窗 | Hall_Mission_Detail_Dialog | 軍事 / 內政任務點擊後開啟的中央任務彈窗 |
| 情報度 | Mission_Intel_Percent | 任務內容目前可見的完整程度；50% 時允許模糊顯示，100% 時完全揭露 |
| 毛遂自薦 | Volunteer_Badge | 任務清單或武將下拉中提示武將主動請纓的標記 |
| AI 委任 | Mission_AI_Assign | 勾選後由 AI 自動選將與執行任務的委任狀態 |
| 官職快照 | Officer_Snapshot | 大廳讀取官職系統輸出的當前階位與權限摘要 |
| 點位刷新 | Spot_Rotation | 武將在固定互動點位之間輪播顯示的輕量在場機制 |
| 節慶插槽 | Festival_Slot | 平時隱藏、活動期間由主題配置啟用的裝飾熱點 |
