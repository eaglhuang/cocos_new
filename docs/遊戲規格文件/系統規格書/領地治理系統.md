> **2026-04-26 覆寫裁定：人口、城市兵源與補兵 UI**
>
> 1. 治理頁的人口是生產引擎，不是戰中逐次扣除的人口池。人口透過動員率、民心、治安與政策推導 `城市兵源上限 City_Troop_Cap` 與 `補兵速度 Recruit_Regen`。
> 2. `城市兵源 / City_Troops` 是玩家看得到的軍事庫存；出征會從中撥出 `出征軍勢`，戰後戰損回扣城市兵源，並由治理逐回合補回。
> 3. 治理頁建議採 `人口核心儀表盤`：中央顯示人口與民心，四條流向分別連到金錢、糧草、城市兵源上限、補兵速度。
> 4. 每個產出卡需拆出 `基礎值 / 人口紅利 / 政策修正 / 總計`，城市兵源卡額外顯示 `目前兵源 / 上限`、`每回合補兵`、`距離回滿`。
> 5. 政策只調整人口貢獻傾向，例如農桑偏糧、軍備偏補兵、安民偏人口成長；不得引入逐場手動徵兵流程。
> 6. 初期治理至少要能支撐 `8,000` 左右的出征軍勢保底，避免玩家在沒有官職時失去召喚式戰鬥手感；中後期才讓人口、民心與官職拉開差距。
# 領地治理系統規格書

> **來源文件**：三國遊戲模式串接與循環.md (doc_spec_0055)、養成系統與教官來源定義.md (doc_spec_0151)

---

## A. 系統描述

治理模式是三大模式循環的第一環（治理→作戰→培育）。玩家作為太守管理領地，每季分配武將執行內政任務（農耕/商業/外交/軍備/偵查）。治理產出的資源供戰場與培育使用。太守系統支持「自動治理」——閒置武將按照其五維自動最優分配。

---

## B. 系統目的

1. **資源供給端**：治理是遊戲資源的主要產出源
2. **武將使用場景**：文官型武將在治理模式發揮最大價值
3. **策略分配**：有限的武將需要在治理/作戰/培育間分配
4. **放置收益**：自動治理讓輕度玩家也能穩定產出

## F. 公式相關

### 內政任務產出
| 任務 | 主屬性 | 產出 | 公式 |
|---|---|---|---|
| 農耕 | POL | 糧草 | Base(100) + General.POL × 2 |
| 商業 | CHR | 黃金 | Base(50) + General.CHR × 3 |
| 外交 | INT+CHR | 名聲 | Base(20) + (General.INT+General.CHR) × 1 |
| 軍備 | STR | 城市兵源 / 補兵速度 | Base(80) + General.STR × 2 |
| 偵查 | INT | 情報 | Base(30) + General.INT × 2 |

### 自動治理分配
```
for each General in Idle_Pool:
    Best_Task = argmax(Task_Affinity(General, task) for task in Available_Tasks)
    assign(General, Best_Task)

Task_Affinity(General, task) = General[task.Primary_Stat] × task.Weight
```

### 世襲主城保護

> **來源**：策略養成遊戲平衡機制探討.md (doc_spec_0060)

玩家的起始主城具有「世代不滅」的永久保護。即使所有外圍城池都被佔領，玩家仍能保有主城作為最後基地，防止完全淘汰。

```
// 主城 (Home_City) 保護規則
Home_City.Is_Protected = true  // 永久不可被攻陷
Home_City.Min_Guaranteed_Territory = 1  // 保證至少保有 1 個城池

// 當主城外的所有城池失守時，觸發「背水一戰」被動能力
if (Player_Territory_Count == 1):
    Home_City.Defense_Multiplier = 2.0  // 主城防禦力翻倍
    Home_City.Troop_Production += 50%   // 兵力生產加速
```

### 非培育子嗣內政職位

> **來源**：遊戲機制推演與玩家歷程.md (doc_spec_0064)

未進入培育鏡像的子嗣可指派擔任地方官職，提供持續性資源強化。這讓每個子嗣都有利用價值，無論是否進入核心培育流程。

| 職位 | 推薦子嗣類型 | 每季效果 |
|---|---|---|
| 郡城縣令 | 任何子嗣 | 黃金產出 +20% |
| 屯田官 | POL 值高的子嗣 | 培育糧草消耗 -20% |
| 工匠 | STR 值高的子嗣 | 建築完成時間 -30% |
| 說客 | CHR 值高的子嗣 | 傭兵結義成功率 +15% |

```
// 子嗣內政貢獻公式
Official_Output = Base_Output × (1 + Child_RelevantStat / 100)
```

### AI 橡皮筋系統（動態平衡）

> **來源**：策略養成遊戲平衡機制探討.md (doc_spec_0060)

防止 AI 勢力過度壟斷地圖，維持競爭活力。

```
// 橡皮筋觸發條件
if (any_ai.territory_ratio > 0.30):  // 任一 AI 佔領地圖超過 30%
    trigger_anti_coalition(strongest_ai)
  // 其他 AI 與非霸主玩家自動形成「反 X 同盟」
    // 效果：同盟 AI 對最強 AI 的攻擊力 +20%，外交關係必然惡化

// 30~50% 領土時：隱性外交壓力
// >50% 領土時：全圖圍攻，最強 AI 自動陷入多線作戰
```
