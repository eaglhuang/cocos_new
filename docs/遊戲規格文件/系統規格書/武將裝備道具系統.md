# 武將裝備道具系統規格書

> **定位**：本檔是整併母本，用來彙總武將可持有、可裝備、可消耗、可傳承的物件規則，並同時定義單一 `Inventory Service` 的資料層。細部戰場平衡、虎符兵種規則、商業化價格與 UI 版型，仍以各子系統母規格為準。

## A. 系統描述

武將裝備道具系統管理所有與武將個體、武將養成、戰場出征、俘虜收穫與傳承回收有關的物件。其核心範圍包含：

1. 一般道具：消耗品、材料、特殊道具、貨幣型道具。
2. 一般裝備：武器、防具、飾品、名馬、古書、秘寶、傳家寶物。
3. 特殊裝備：虎符與其他可解鎖特殊兵種的專屬裝備。
4. 命 / 祖靈裝備：英靈卡，屬於血脈層與人物頁 `命` 區域的正式插槽，不與一般裝備欄混用。
5. 共用資料層：單一 `Inventory Service` 依類別 / 槽位 / 綁定狀態管理主城倉庫、個人裝備、虎符投影、命槽綁定與鎖定槽；但資料統一不等於前端要做成一個混雜大背包。
6. 取得與回收：商城、任務、掉落、掛機、俘虜、死亡結算、退役回收。

> **Unity 對照**：可視為 `ItemDatabase + Inventory + Equipment Slots + Shared Stash` 的組合。一般道具偏向 `InventoryManager`，一般裝備偏向角色 `Equipment Slots`，虎符則是帶有戰場權限的特殊裝備資產。

## B. 系統邊界

1. `道具系統（付費免費道具）` 負責「可消耗、可持有、可轉化」的共通物件語意。
2. `武將背包（倉庫）系統` 負責「武將個體裝備欄」與「主城倉庫」之間的移動。
3. `兵種（虎符）系統` 負責「虎符作為特殊裝備」與「對應兵種指揮權」。
4. `經濟系統` 負責貨幣、商店、加速卡與時間償還。
5. `數值系統` 負責裝備數值、虎符星級、品質與適性加成。

本檔只整理上述規則的共同層，不重新定義各子系統已拍板的細節。

## C. 共同原則

1. **裝備與道具都必須可追溯來源**：商城、任務、掉落、掛機、俘虜、死亡結算與封賞都應在資料上標記來源。
2. **可消耗物件與可裝備物件分開看待**：道具以消耗為主，裝備以欄位與戰力為主。
3. **退役不等於消失**：可傳承或可回收的物件，應優先回倉庫或下一代，而不是直接刪除。
4. **虎符保留單裝唯一性**：每位武將僅能裝備 1 枚虎符，不可疊加。
5. **英靈卡保留命槽唯一性**：每位武將僅能插入 1 張英靈卡，且英靈卡不視為可隨意丟賣的普通背包裝備。
6. **資料統一、展示分域**：`Inventory Service` 是單一真相來源，但前端至少區分「共用倉庫」「武將裝備欄」「命 / 祖靈欄」三個使用語境。
7. **出征支援不等於裝備欄位**：參謀官 / 後勤官屬於 sortie loadout，不占用一般裝備欄，也不額外提供第二枚虎符或第二張英靈卡。
8. **商業化不直接販售勝負結果**：可販售的是效率、容量、加速與便利，不是直接把戰場結果賣掉。

## D. 物件分類

| 類別 | 說明 | 代表例 |
|---|---|---|
| 消耗品 | 使用後數量減少，立即觸發效果 | 補元散、美酒、安胎藥、送子符、求籤令、入學令、刷新券 |
| 材料 | 用於升級、突破、合成或教學的中介物 | 戰法筆記、虎符精華、神魂精華、一般強化材料 |
| 一般裝備 | 由武將持有並影響戰場數值 | 武器、防具、飾品、名馬、古書、秘寶 |
| 傳家寶物 | 俘虜、退役或事件產出的稀有裝備型物件 | 古劍、佩劍、傳家寶、戰利品 |
| 特殊裝備 | 直接解鎖特殊兵種或戰場權限 | 虎符、特種軍隊對應道具 |
| 命 / 祖靈卡 | 掛在武將 `命` 槽的血脈傳承物件 | 英靈卡、祖靈卡面、世家命紋載體 |
| 貨幣 | 系統性支付媒介 | 黃金、名聲、城市兵源、玉璧、精元、結義點數 |

> 傳家寶物在語意上可視為「可裝備的稀有寶物」，但在經濟流程上也可被當作可販售資產。

## E. 持有與槽位

### E-1. 武將個人裝備欄

| 欄位 | 性質 | 預設狀態 | 主要效果 |
|---|---|---|---|
| 武器 | 基礎欄 | 開啟 | 武力、智力加成 |
| 防具 | 基礎欄 | 開啟 | 統帥、生命加成 |
| 飾品 | 基礎欄 | 開啟 | 政治、魅力加成 |
| 名馬 | 進階欄 | 依條件解鎖 | 移動力、騎兵適性 |
| 古書 | 進階欄 | 依條件解鎖 | TP 折扣、戰法學習加速 |
| 秘寶 | 進階欄 | 依條件解鎖 | 特殊被動與額外效果 |
| 虎符 | 特殊欄 | 單裝 | 解鎖特殊兵種、改變戰場指揮權 |
| 命 / 英靈卡 | 命槽 | 單插 | 提供祖靈被動、血脈共鳴與人物頁 `命` 區域摘要 |

### E-2. 前端持有視圖分域

| 視圖 | 收納內容 | 正式說明 |
|---|---|---|
| 共用倉庫 | 一般道具、一般裝備、未裝備虎符 | 可視為玩家的主城物資池；支援篩選、鎖定、擴容與整理 |
| 特殊裝備分頁 | 虎符與特殊兵種相關查詢 | 虎符雖屬 `Inventory Service`，但建議以前端獨立 tab / list 呈現，避免和消耗品混在同一格子清單 |
| 命 / 祖靈欄 | 已裝配或可裝配的英靈卡 | 英靈卡不應塞進普通背包格位；資料語意屬 `命`，人物頁裝配入口集中在 `命 / Bloodline` 頁，`命` 頁同時承接共鳴 / 故事詳情 |
| 人物頁 `寶 / GEAR` 頁 | 一般裝備六槽、傳家寶、道具與裝備操作 | 正式一般裝配主入口；集中處理更換、卸下、使用與評價，不再把這些資訊塞到 overview |
| 人物頁 `兵 / APTITUDE` 頁 | 虎符槽、兵種適性、地形適性、天氣適性、戰鬥定位 | 正式虎符裝配主入口；集中處理虎符更換、卸下、詳情查詢與適性說明 |
| 出征編制欄 | 主將、參謀官、後勤官 | 這是 sortie loadout，不是物件存放格；只引用武將 UID 與支援效果 |

### E-3. 主城倉庫

1. 主城倉庫是全域共用的物品儲存空間。
2. 未裝配到特定武將的物品，應優先存放在倉庫。
3. 倉庫預設 50 格，擴容每次 +20 格，上限 200 格。
4. 重要物品可鎖定，避免誤賣、誤分解或誤移轉。
5. 英靈卡不占用一般倉庫格數；應由 `Display_Bucket = CLAN_RELIC / DESTINY_SLOT` 一類欄位區分顯示層。

### E-4. 裝備流轉

1. 裝備可以在倉庫與武將背包之間移動。
2. 武將退役時，已裝備物品應自動回倉庫。
3. 退役後可傳承的裝備，允許轉移到後代或重新配置給其他武將。
4. 虎符在一般規則下同樣是可轉交的特殊裝備，但仍維持「同一時間只可由一名武將持有」。
5. 英靈卡在資料層上可解除綁定並重新指定裝配者，但前端必須走 `命` 區域或世家陳列室操作，不得混在一般販售 / 分解流程中。

## F. 取得與消耗

### F-1. 取得管道

| 管道 | 適用物件 | 備註 |
|---|---|---|
| 商城購買 | 付費道具、材料、加速卡、容量類服務 | 以玉璧或現金相關貨幣結算 |
| 任務獎勵 | 消耗品、材料、一般裝備 | 支援日常循環與成就投放 |
| 戰場掉落 | 裝備、材料、補給 | 連結推圖與資源回收 |
| 掛機產出 | 免費道具、入學令、基礎資源 | 保障無課玩家節奏 |
| 俘虜處置 | 傳家寶物、教官轉化前獎勵 | 與戰場處置系統連動 |
| 死亡結算 | 虎符、英靈卡 | 名將死亡後正式產出 |
| 封賞 / 招降移交 | 虎符 | 成功招降或封賞時移交持有權 |

### F-2. 消耗原則

1. 消耗品使用後數量減少。
2. 強化材料只在升級、突破、合成、轉化時消耗。
3. 裝備變更應先檢查數量與鎖定狀態，再執行扣除或移轉。
4. 虎符變更會影響忠誠度，屬於高風險操作。

### F-3. 傳家寶與俘虜

1. 俘虜的無期徒刑流程可產出傳家寶物。
2. 傳家寶物可裝備、可販賣，也可作為收藏圖鑑素材。
3. 退役裝備回收與俘虜寶物產出應共用同一套來源追蹤欄位。

## G. 數值與戰場

### G-1. 一般裝備數值

```text
final_stat = base_stat + Σ(equipped_item[i].stat_bonus)
```

1. 一般裝備的加成以欄位相加為主。
2. 裝備效果以角色面板與戰場實戰共同顯示。
3. 古書與名馬一類進階裝備可附帶戰法或移動面修正。

### G-2. 虎符數值

1. 虎符品質由死亡戰績與傳承品質共同決定。
2. 虎符星級 1~3 星可透過戰鬥經驗升星，4~5 星需消耗虎符精華突破。
3. 虎符品質、星級、共鳴、適性與忠誠度都會進入戰場公式。
4. 不同虎符對地形、家族、性別可能有額外適性加成。

### G-3. 命槽與出征支援規則

1. 每位武將僅可插入 1 張英靈卡；英靈卡定位為血脈 / 祖靈被動，不作為第二套特殊兵種權限來源。
2. 英靈卡與虎符允許同角色同裝，但責任分工必須拆開：英靈卡負責人物被動、祖靈共鳴與命紋語意，虎符負責兵種解鎖與戰場調度。
3. 出征前可額外編制 1 名參謀官與 1 名後勤官。參謀官提供計謀點數支援，後勤官提供軍糧攜帶上限支援；兩者屬 sortie loadout，不是裝備欄。
4. 參謀官 / 後勤官不應把自己的虎符、英靈卡或 Set 羈絆直接投射到主將身上；其正式作用以支援資源池為主，避免乘算膨脹。

### G-4. 相容戰力規則

1. 武將裝備虎符後，才可指揮對應特殊兵種。
2. 同一枚虎符同時間只能由一名武將持有。
3. 拔除虎符會降低忠誠度，低忠誠會引發拒命或叛變風險。
4. 特種兵出戰仍需吃糧草，不是純展示型資產。

## H. 商業與節流

### H-1. 主要商業點

| 商業點 | 意義 |
|---|---|
| 倉庫擴容 | 用容量驅動付費，不直接賣戰力 |
| 欄位解鎖 | 讓高價值裝備與寶物有更多承載空間 |
| 自動整理 | 降低管理成本，作為高級便利功能 |
| 裝備強化 | 傳家寶與虎符精華等材料的主要消耗口 |
| 加速卡 | 把免費許願或免費等待轉成付費效率 |
| 禮包組合 | 把多項道具與加速包打包成商品 |

### H-2. 時間償還與許願商城

1. 許願商城可作為「免費等時間、付費換現貨」的雙軌介面。
2. 免費玩家透過等待取得商品，課金玩家透過現貨取得商品。
3. 加速卡可縮短等待時間，形成額外消耗口。
4. 高價值物件若走許願機制，應清楚標示產量衰減與不可重疊限制。

### H-3. 經濟連動

1. 特訓加速、軍備維護與補給包都可包裝成道具商品。
2. 補元散、美酒、入學令等道具是多系統共用的主要消耗口。
3. 商業化重點應放在效率、便利、容量與長線留存，而不是直接販售戰果。

## I. 介面與文案

### I-1. 通用操作

| 操作 | 文案意圖 |
|---|---|
| 裝備 | 把物件裝到武將身上 |
| 卸下 | 從武將身上移回倉庫 |
| 鎖定 | 防止誤操作 |
| 分解 | 回收材料 |
| 擴充倉庫 | 提升持有容量 |
| 前往商城 | 引導補貨或補足缺口 |

### I-2. 重要提示

1. 倉庫滿時應明確提示無法存入。
2. 道具不足時可提示前往商城或改用替代來源。
3. 虎符裝備、升星與突破應有獨立確認文案。
4. 傳家寶物入手、退役回倉、裝備傳承都應有故事化敘述。

### I-3. 人物頁與戰場摘要

1. 人物頁正式拆成 `命 / 寶 / 兵` 三種裝配語境：`命` 頁承接英靈卡命槽，`寶 / GEAR` 頁承接一般裝備 / 傳家寶 / 道具，`兵 / APTITUDE` 頁承接虎符槽與戰場適性。
2. 人物頁 overview 不再承接命槽英靈卡摘要或虎符摘要；overview 只保留人物識別、核心數值與故事層。
3. `寶 / GEAR` 頁至少要提供「一般裝備六槽」「傳家寶欄」「道具摘要」「裝備評價與使用操作」四類資訊；`兵 / APTITUDE` 頁至少要提供「虎符槽（含兵種 / 星級 / 糧耗 / 投入兵力）」「兵種適性」「地形適性」「天氣適性」「戰鬥定位」五類資訊。
3. 戰場 HUD 需能顯示來自參謀官的計謀點支援與來自後勤官的糧草上限支援，但不得誤導成額外一位可上場武將。
4. 戰場或摘要卡可顯示品質、星級、來源、適性與是否已裝備。
5. 裝備變更結果要能回扣到武將頁、戰場頁與倉庫頁。

## J. Data Schema 概要

### J-1. 現行可對應的資料真相

| 子域 | 主要資料概念 | 來源母規格 |
|---|---|---|
| 一般道具 | `ItemDef`、`PlayerInventory` | `道具系統（付費免費道具）` |
| 武將裝備 | `GeneralEquipment`、`Warehouse` | `武將背包（倉庫）系統` |
| 虎符持有 | `Talisman_Inventory` | `兵種（虎符）系統` / `Data Schema` |
| 英靈卡持有 | `Inventory_Service(Item_Type=SPIRIT_CARD)`、`SpiritCard_Collection` projection | `英靈世家系統` / `Data Schema` |
| 出征支援編制 | `Sortie_Loadout` | `戰場部署系統` / `Data Schema` |

### J-2. 現行欄位原則

1. 一般道具資料應包含 ID、品質、堆疊上限、效果型別與過期時間。
2. 武將裝備資料應能表達武將 UID、裝備槽位與倉庫格位。
3. 虎符資料應能表達品質、星級、來源、裝備者、對應兵種與是否顯示。
4. 英靈卡資料應能表達 `Spirit_ID`、`Spirit_Family_Key`、血脈等級、`Resonance_Band`、`Glow_State`、`cardRole`、`activeMainSpiritUid`、裝配中的武將 UID、顯示區桶位（陳列 / 命槽）與是否允許雙卡同裝。
5. `Sortie_Loadout` 應能表達 `Commander_UID / Strategist_UID / Deputy_UID` 與由兩個支援角色提供的摘要數值。
6. `PlayerInventory`、`GeneralEquipment`、`Talisman_Inventory`、`SpiritCard_Collection`、`Sortie_Loadout` 的欄位命名與投影若要調整，必須先同步回本檔與 `Data Schema`，不能再視為互相獨立的多套資料層。

## L. 維運規則

1. `Inventory Service` 仍是單一資料真相，但前端不得把「所有東西」強迫塞進同一個格子背包；至少要拆成倉庫 / 特殊裝備 / 人物頁寶頁 / 人物頁兵頁 / 命頁詳情 / 出征編制六個語境。
2. 英靈卡正式視為 `命 / 祖靈` 層的單插槽裝備，不占一般倉庫容量，也不走一般裝備販售 / 分解操作。
3. 每位武將正式限制為 `1 張英靈卡 + 1 枚虎符`；兩者可同裝，但不得各自再開第二槽。
4. 英靈卡的裝配 / 卸下 / 查詢入口集中在人物頁 `命 / Bloodline` 頁；虎符的裝配 / 卸下 / 查詢入口集中在人物頁 `兵 / Aptitude` 頁；`寶 / GEAR` 只負責一般裝備、傳家寶與道具。
5. 參謀官 / 後勤官屬於 sortie loadout，只提供資源支援摘要，不得透過文案或 UI 暗示成第二套可疊戰力裝備。
5. 若後續要調整 `PlayerInventory`、`GeneralEquipment`、`Talisman_Inventory`、`SpiritCard_Collection` 或 `Sortie_Loadout` 的欄位或命名，先同步更新本檔與 `Data Schema`，再回寫子規格。
