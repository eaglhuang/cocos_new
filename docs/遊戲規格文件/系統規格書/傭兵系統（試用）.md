# 傭兵系統（試用）規格書

> **來源文件**：傭兵系統規格書：人才經營進化.md、傭兵系統規格書：試用轉永久.md、傭兵養成與留存系統設計.md (doc_spec_0133)、傭兵職能分流與傳承機制.md (doc_spec_0134)

---

## A. 系統描述

傭兵系統讓玩家「試用」其他玩家（或系統）的武將。傭兵來自轉蛋池（非永久獲得），有 6 日結義期。期間玩家可將傭兵投入戰場或培育，且**能力以 100% 基準固定發揮**，不再存在 95% / 150% 之類的階段差異；差別只在於是否永久、是否可轉為教官。6 日結束後玩家選擇：❶ 結義（消耗結義點數轉為永久）、❷ 退場（失去傭兵）。退役武將（65 歲）可轉為教官卡繼續發揮價值。

---

## B. 系統目的

1. **降低首抽壓力**：「先試後買」比直接花錢更友善
2. **社交引擎**：傭兵自帶原主名牌→病毒行銷
3. **長線留存**：6 日倒數製造每日登入動機
4. **資源沉澱**：結義點數消耗綁定轉蛋池循環

## F. 公式相關

### 結義點數需求
| 傭兵稀有度 | 結義所需點數 |
|---|---|
| R | 100 |
| SR | 300 |
| SSR | 600 |
| UR | 1200 |

### 傭兵 6 日數值
```
Mercenary_Stats = Original_Stats × 1.0  // 固定 100%，不再存在試用期 95% 或 150% 階段差異
Daily_Bond += 10  // 每日好感度 +10
Bond_Max = 60  // 6日滿
```

### 退役轉教官
```
if (General.Age >= 65 && Status == "Active"):
    Retire() → Generate_Tiger_Tally → Create_Mentor_Card
    Mentor_TP_Base = General.Total_TP × 0.3
```

- **問題**：傭兵結義機制：點數 vs. 美酒

## 結義與傭兵特定機制更新 (Q72, Q73, Q74 裁決)
1. **D3/D4 互動純敘事**：六日結義歷程中的 D3（微醺）與 D4（信賴），均設定為純劇情對話事件，**無額外隱藏戰法或爆發機率加成**。
2. **30 天冷卻期**：同一個武將若結義成功或被玩家放棄，30天內絕對不會再次刷新出現在玩家的傭兵選單中。
3. **二轉機制**：玩家將傭兵結義轉正（成為永久武將）後，若後續透過「轉蛋」抽到該同名武將，系統將自動觸發**二轉（2nd Awakening）**，大幅解鎖屬性實力上限。
4. **俘虜豁免**：處於「試用/傭兵狀態」的武將，在戰場上**絕對不會被俘虜**，保障玩家的 .99 初期投資不被 AI 掠奪。
