## A. 系統描述

血統系統是《三國傳承》的靈魂骨架。每位武將擁有上三代（14 位長輩）的家族圖譜，因子從這 14 人中按機率繼承。初代名將透過「虛擬祖先生成器」自動補齊血統資料。系統包含爆發力計算、近親判定、血脈歸宗等機制。相較《賽馬娘》的 2 代 6 人制，本作採 3 代 14 人制以增加深度。

---

## B. 系統目的

1. **配種深度**：14 人血統矩陣讓配種成為一門學問
2. **長線留存**：百年世家經營，曾祖因子在玄孫身上覺醒
3. **差異化**：比《賽馬娘》多一代＋紫色運氣因子，形成策略差
4. **公平性**：初代名將與世家名門通過不同機制各有優勢
5. **世界觀主體**：把「祖靈命紋 / 命紋靈獸 / 血脈逸聞」做成產品辨識主軸，而不是只停留在數值面板

## K. 進階血統機制補充

> **來源**：名將轉身與子嗣盲盒雙軌制.md (doc_spec_0059)、遊戲機制推演與玩家歷程.md (doc_spec_0064)

### 覺醒名號（Awakening Title）

滿足天才條件的子嗣，系統自動授予特殊封號，永久附加於武將名稱之後。覺醒名號具有視覺標識與部分數值加成效果。

**觸發條件（以下任一）：**
- 培育畢業評級 S，且五維總和 ≥ 460
- 爆發力（EP）≥ 95
- 14 人血統矩陣存在「滿連鎖共鳴」（5 個以上相同標籤因子）

**名號範例：**

| 名號 | 條件說明 | 附加效果 |
|---|---|---|
| 常山真龍 | 14 人全騎兵因子 | 騎兵適性強制升為 S |
| 鬼才後裔 | 智力畢業值 ≥ 80 | 秘笈事件觸發率 +20% |
| 麒麟 | EP ≥ 98 | 培育中所有訓練效果 ×1.1 |
| 虎膽龍威 | STR 畢業值 ≥ 90 | 單挑爆發力 +10% |

### 混種戰法（Hybrid Skill）

子嗣獨有特性：同時繼承父方與母方的**不同領域戰法因子**，在培育中可觸發「跨域融合」事件，解鎖混種戰法。

**觸發條件：**
```
// 培育進行至第 15~25 回合時判定（每回合有機率觸發）
Hybrid_Trigger_Chance = 0.05 × (Has_Father_Skill_Gene ? 1 : 0) × (Has_Mother_Skill_Gene ? 1 : 0)
// 父母雙方都有不同領域的綠色戰法因子時才有觸發機會
```

**混種戰法範例：**

| 父方技能源 | 母方技能源 | 混種戰法 |
|---|---|---|
| 騎衝因子 | 火攻因子 | 火騎突刺（騎兵衝鋒×火焰附加） |
| 水路因子 | 弓兵因子 | 水波箭雨（水域強化+弓矢） |
| 大盾因子 | 計策因子 | 金剛詭計（防禦姿態+陷阱部署） |

### 因子媒介（Gene Catalyst）

退役武將可選擇「靈魂傳承」而非直接轉教官。此路徑將武將一生最強的 2 個因子提煉為「靈魂因子媒介」，在下一輪培育中賦予特定子嗣使用。

正式邊界補充：

1. `退役` 與 `死亡` 是兩條不同的生命週期出口。
2. 退役時可選擇教官或因子媒介路徑。
3. 死亡時則另外結算 `英靈卡` 與 `虎符卡`，該路徑詳見 [英靈世家系統.md](C:\Users\User\3KLife\docs\系統規格書\英靈世家系統.md (doc_spec_0022)) (doc_spec_0022) 與 [兵種（虎符）系統.md](C:\Users\User\3KLife\docs\系統規格書\兵種（虎符）系統.md (doc_spec_0012)) (doc_spec_0012)。

**機制流程：**
```
// 退役時選擇路徑：教官卡（標準）or 靈魂因子媒介（特殊）
if (Retirement_Choice == GENE_CATALYST):
    Catalyst = {
        General_ID: 退役武將 UID,
        Top_Genes: General.Genes.sortByLevel().take(2),
        Boost_Type: "Extra_15th_Ancestor"  // 作為「第15位祖先」提升繼承概率
    }
    // 指定一位 0~5 歲子嗣，注入因子媒介
    Target_Child.Extra_Ancestor = Catalyst
    // 效果：提升對應因子的繼承機率 +25%，但僅限 1 次使用
```

> **設計理念**：因子媒介讓課金玩家的 5 轉名將「退役不廢」，其精華以另一種形式延續至下一代，強化培育系統的長線價值。

## L. 血統色譜與祖先矩陣落地原則

> **整併來源**：五色因子與爆發力公式設計.md (doc_spec_0057)、賽馬娘機制三國化轉化.md (doc_spec_0093)
> **衝突處理**：若早期討論與現行正式規格不一致，一律以本檔與 `因子爆發系統.md` (doc_spec_0010) 的現行六色標準為準。

### 正式色譜標準

- 正式資料模型維持 **六色因子**：藍 / 粉 / 綠 / 紫 / 白 / 紅。
- 早期討論中的「五色因子」屬於 MVP 探索稿，現已整併為：
  - 紫色承接「天命 / 運氣波動」
  - 紅色承接「生命體魄 / 壽命 / 精力回復」
- 因子色譜的資料層不可再退回五色版本；若 UI 需要漸進顯示，可只先曝光部分色系，但底層資料與繼承規則仍以六色儲存。

### 祖先矩陣與 Tag Pool 原則

- 14 人祖先矩陣只保存「共享字典引用」，不為每名武將各自手寫一套獨立因子邏輯。
- `Ancestors_JSON` 的每個因子條目都應指向共用字典（如 Gene_Dictionary / Tag Pool），藉此避免 100 名武將 × 14 祖先的資料爆炸。
- 爆發力判定優先看「標籤組合」與「世代權重」，不是看武將姓名本身；武將姓名只在史實契合與名將彩蛋上產生額外加成。

### MVP 揭露節奏

- MVP 與新手期 UI 可優先揭露藍、粉、綠三色，讓玩家先理解屬性、適性與戰法種子。
- 紫、白、紅可以透過 Discovery / 覺醒機制在後期逐步亮相，但不能在資料層缺席。
- 祖先矩陣的可視化採「先父母兩代、後展開 14 人全圖」的漸進式資訊密度設計，避免新手在第一次接觸血統系統時資訊過載。

## M. 祖紋命篆與命紋靈獸落地原則

### 視覺主體原則

- 懷舊只負責導入，血統介面真正的主體必須是 `祖紋命篆 + 命紋靈獸 + 六色因子`
- 血統 UI 不走科幻基因實驗室風格，也不走重奇幻召喚獸風格
- 正式視覺語言應偏向：
  - 甲骨、金文、小篆、帛書殘筆、玉刻痕
  - 祠堂族譜、命盤、血脈樹卷
  - 暗金、玉白、月銀、墨青、暗赤的高級古老色調

### 命紋靈獸規則

- 每位名將都有一個獨特基因，並對應一個主要 `命紋靈獸`
- 命紋靈獸不是可愛寵物，而是祖靈圖騰、血脈伴靈、命格顯影
- 命紋靈獸應同時出現在：
  - 角色服裝裝飾與胸甲 / 腰封 / 髮飾 / 玉佩符號
  - 血統 UI badge
  - 血脈逸聞連環漫畫
  - 英靈虎符與特種軍隊圖騰

### 資料與 UI 契約原則

- 首波不新增 runtime public API，也不重做資料模型
- 先以既有欄位與概念組合出視覺語意：
  - `Ancestors_JSON`
  - `Bloodline_ID`
  - `Genes[]`
  - `Awakening_Title`
  - `Gene_Catalyst`
- 若後續進入正式美術落地，只新增獨立 `art-direction contract` 文件，固定：
  - 因子色系對應的紋樣與材質
  - 祖先連線樣式
  - 共鳴 / 警告 / 覺醒 / 媒介的視覺 badge
  - 命紋靈獸與角色符號 slot 對應表

### 血脈內容層原則

- 名將內容層分成兩種：
  - `歷史趣聞`：對應史實逸事與人格魅力
  - `血脈傳聞`：解釋其勇猛、謀略、魅力、執念等為何被古老血脈牽引而成
- 血脈傳聞建議以 `3~5 格靜態連環漫畫` 呈現，不依賴角色動畫產能
