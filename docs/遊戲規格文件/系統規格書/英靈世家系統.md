## A. 系統描述

英靈世家系統負責處理名將死亡後的兩條正式傳承產物，以及這些產物與家族積分、姓氏歸屬、供養與血脈延續的關係。

正式共識如下：

1. 名將在**死亡結算**時，同步留下 `英靈卡` 與 `虎符卡`。
2. 名將在**生前退役**時，只進入教官 / 傳承路徑，不直接生成英靈卡與虎符卡。
3. `英靈卡` 用於世家供養、祖靈加成、傳道與血脈陳列。
4. `虎符卡` 用於兵種軍勢化身、特殊兵種調度與戰場配置。
5. 轉蛋系統不再產出英靈卡或虎符卡；轉蛋只保留 `三國名將池（含女性）` 與 `支援卡（教官）池`。
6. 每一張死亡後生成的英靈卡，都必須同步生成一筆完整的英靈快照，保存當年的英勇事蹟、最終屬性、最終技能與戰報摘要，供家族展示列與詳情頁查詢。

> **Unity 對照**：英靈卡接近長線帳號級的 `Legacy Relic / Account Relic`，負責被動加成與家族展示；虎符卡接近可裝備的 `Equipment + Unit Unlock Token`，負責解鎖與部署特殊軍勢。兩者都來自角色生命週期結算，而不是一般抽卡掉落。

## B. 系統目的

1. **死亡不等於消失**：讓名將死亡後轉化為可持續經營的世家資產。
2. **傳承拆層**：把祖靈供養與戰場兵符拆成兩種不同用途的卡，避免單一系統過載。
3. **姓氏零和**：家族歸屬、絕學傳承與功勳結算統一走單一姓氏歸屬，不允許雙邊重複受益。
4. **長線世家經營**：透過人頭分、功勳分與名聲值，建立可持續但不失控的英靈成長曲線。
5. **回收付費價值**：高成本培養的名將死亡後，仍可透過英靈卡與虎符卡延續戰略價值與展示價值。

## D. 英靈卡生命週期

### D-1. 誕生條件

1. 史實名將死亡時必定生成 1 張對應英靈卡。
2. 女性名將若具正式名將模板，與史實名將同樣適用必出規則。
3. 非名將角色一律不生成英靈卡；若未來要開放例外，必須另立正式規格。
4. 名將若生前已進入退役 / 教官路徑，死亡時仍需依正式規則補發英靈卡與虎符卡。

### D-2. 唯一性原則

1. 同一存檔世界中，同一名將模板只維持 1 張正式英靈卡主體。
2. 同名模板再次死亡時，不新增第二張平行主卡，而是進行舊卡覆蓋與成長結算。

### D-3. 屬性覆蓋規則

```text
SpiritCard.FinalStat = max(ExistingSpiritCard.Stat, DeathSnapshot.Stat)
SpiritCard.LineageScore = ExistingLineageScore + CurrentDeathContribution
```

1. 舊卡與新死亡快照以「高者覆蓋」處理核心數值。
2. 家族貢獻與供養等級採累計制，不因新死亡結算而清空。
3. 已解鎖的祖靈階位不倒退，只允許維持或升階。

### D-4. 傳道權限

1. 英靈卡可以授與特定子嗣或血脈分支祖靈加護。
2. 傳道屬於英靈卡路徑，不屬於一般教官卡的抽卡突破路徑。
3. 英靈卡若要直接傳授專屬奧義，需消耗專屬資源與次數，不等於一般培育教官事件。

## E. 虎符卡來源與歸屬

### E-1. 虎符卡正式來源

1. 虎符卡只來自名將死亡結算。
2. 虎符卡與英靈卡在同一次死亡結算中同步產生，但用途不同。
3. 虎符卡不屬於轉蛋道具，也不是一般商店常駐掉落物。

### E-2. 虎符卡與英靈卡的差異

| 比較項 | 英靈卡 | 虎符卡 |
|---|---|---|
| 主用途 | 世家供養、祖靈加成 | 特殊兵種、軍勢化身 |
| 裝備位置 | 人物頁 `命` 頁的英靈卡槽（命系語意） / 世家 / 血脈 / 陳列室 / 傳道介面 | 武將虎符槽 / 人物頁 `兵` 頁虎符卡 |
| 成長來源 | 家族積分、供養、名聲值 | 虎符精華、戰績、適性與突破 |
| 視覺語言 | 祖紋命篆、命紋靈獸、宗族匾額 | 兵種旗幟、甲紋、軍勢圖騰 |

### E-3. 雙卡同裝正式規則

1. 同一名將死亡結算生成的英靈卡與虎符卡，允許由同一名角色同時裝備。
2. 英靈卡占用命系單插槽，人物頁的裝配入口位於 `命 / Bloodline` 頁；虎符卡占用虎符槽，人物頁裝配入口位於 `兵 / Aptitude` 頁，兩者不互斥。
3. 每名武將正式只能插 1 張英靈卡、裝 1 枚虎符；不得把雙卡同裝誤解成雙英靈或雙虎符。
4. 若同時滿足同姓或血脈共鳴條件，可分別吃到各自系統的加成；不因雙卡同裝而打折。

### E-4. 英靈卡命槽規則

1. 英靈卡的正式人物入口分兩層：`命 / Bloodline` 頁負責裝配、替換、卸下、設為主卡、共鳴、家族與故事說明；`世家 / 陳列室` 承接展示與回看。
2. 英靈卡在資料層仍可放入 `Inventory Service`，但前端應以 `人物頁命頁英靈卡槽 / 世家陳列室 / 命頁詳情 / 傳道介面` 顯示，不與消耗品或一般裝備共格。
3. 命槽為單插槽；替換英靈卡時必須提示原卡被動將解除，避免玩家誤以為能累積多張祖靈被動。
4. 英靈卡應優先提供血脈、地形、兵種傾向或戰法效率等「角色身份型加成」，避免與虎符的戰場兵種權限互搶定位。

## F. 家族積分與世家等級

### E-5. 家族光效與主卡標籤

1. 英靈卡光效分三階：`SELF_RESONANCE`、`FAMILY_RESONANCE`、`ECHO_RESONANCE`。
2. 若英靈卡插在自身對應模板的武將身上，顯示 `SELF_RESONANCE` 強光，並給出完整加成。
3. 若英靈卡插在同家族成員身上，顯示 `FAMILY_RESONANCE` 暖光，並給出高於一般卡片的加成，但必須封頂。
4. 同一家族若誕生多張英靈卡，UI 只要提供 `設為主卡` 切換；每一瞬間只允許 1 張卡成為 `主卡`，其餘都標記為 `副卡`。
5. 主卡才吃當期家族供養與培養公式；副卡不參與當期成長，只保留已累積進度與收藏 / 故事價值。
6. 切換主卡時必須顯示確認提示，並且只允許在人物頁 `寶` 頁 / 陳列室 / 出征前進行，戰鬥中鎖定，避免玩家誤切導致培養目標混淆。
7. `曹氏 / 孫氏` 這種名將密度高的家族，仍可擁有很多英靈卡，但系統只要回答一個問題：`你現在要養哪一張？`

### E-6. 家族供養與單一公式

1. 同家族成員的所有供養、祭奠、功勳、名望輸入仍可匯入家族層，但只會灌進當前 `主卡` 的成長公式。
2. 家族層的正式 read model 只需要知道 `Active_Main_Spirit_UID` 與 `Family_Contribution_Pool`；前者決定誰吃池，後者決定這一輪有多少資源可分配。
3. 單卡成長公式保持單一路徑即可：

```text
CardGrowth = min(CardCapRemaining, FamilyContributionPool)
```

4. 若當前主卡已接近封頂，剩餘家族池可保留到下次主卡切換再用，不需要拆成多卡分配。
5. 家族池溢出時再轉為 `Family_Fame` 與 `Family_Depth`，作為長線世家等級與視覺升格來源。
6. 對玩家的說法保持一句話就能懂：`你現在設誰當主卡，家族貢獻就先養誰。`

### F-1. 積分構成

家族積分由 `人頭分` 與 `功勳分` 組成。

正式範圍採 **完全個人存檔制**：只統計玩家自己存檔中的家族線，不採同伺服器共享，也不混入他人同姓積分。

```text
ClanScore = HeadScore + MeritScore
```

#### 人頭分

| 類型 | 權重 |
|---|---|
| 本家後代 | 100% |
| 俘虜轉化後納入本家者 | 80% |
| 外嫁且最終選擇隨母姓的後代 | 120% |

#### 功勳分

功勳分由下列行為合成：

1. 戰場總傷害
2. 總擊殺數
3. 攻城與據點貢獻
4. 特定名將挑戰 / 劇本節點達成

### F-2. 世家階位

| 階位 | 說明 |
|---|---|
| 傳家之寶 | 解鎖英靈卡基礎加成與陳列 |
| 名門望族 | 解鎖宗族加成與次級祖靈效果 |
| 萬世宗親 | 解鎖跨代供養強化與視覺升格 |
| 武聖神祇 | 解鎖高階祖靈顯化與頂階傳道權限 |

### F-3. 名聲值與供養消耗

1. 英靈卡的高階加成不能無成本常駐，必須消耗 `家族名聲值`。
2. 家族成員若出現叛逃、作惡、低評價結局，會削弱家族名聲值與供養倍率。
3. 名聲值不足時，英靈卡仍存在，但高階世家效果會降階或失效。

## G. 姓氏歸屬與絕學限制

### G-1. 姓氏零和原則

1. 子嗣必須在父姓與母姓中擇一歸屬。
2. 一旦選定姓氏，該子嗣的家族積分、世家供養、宗族關係與絕學權限皆綁定該姓氏。
3. 禁止同一子嗣同時為父母兩家累積正式世家積分。

### G-2. 家傳絕學限制

1. 家傳絕學需符合該姓氏的宗族權限才能發揮完整威力。
2. 非本姓但裝備同名英靈卡或虎符卡時，可獲得弱化版效果，不得享有完整宗族特權。
3. 同姓或具正式血脈共鳴者，裝備對應虎符卡時可觸發額外共鳴效果。

## H. 防火牆與反膨脹規則

### H-1. 近親衰減

1. 近親聯姻不視為高效率刷分手段。
2. 近親後代容易出現隱藏屬性劣化、負向因子增加與家族積分效率下降。

### H-2. 同宗不相殘

1. 同宗姓氏間互戰時，功勳積分應打折計算。
2. 該規則用於阻止玩家透過內戰刷英靈卡成長。

### H-3. 質重於量

1. 高品質後代應提供較高回向收益。
2. 低品質後代不應因數量優勢就把高品質世家培養線完全稀釋。

### I-3. 世家總覽、英靈陳列室與多卡管理

1. `世家總覽` 是君主層正式入口，負責彙總麾下所有世家分支、主卡狀態、家族深度與名聲值。它可從大廳 / 治理中樞進入，不依賴單一武將頁存在。
2. `英靈陳列室` 是家族層的集合瀏覽頁，應以家族 / 模板為主分組，讓玩家先看到整個家族的卡群，再決定要養哪一張。
3. `武將命頁世家入口捷徑` 是條件式入口，只有 `HasFamilyBranch = true` 或等價條件成立時才顯示；尚未形成世家分支的武將不顯示該按鈕。
4. `SpiritCard_Collection` 是英靈卡的集合投影，用來承接同一存檔世界內的所有英靈卡，不是普通倉庫格，也不是消耗品清單。列表至少要顯示 `Spirit_ID`、來源武將、`cardRole`、`resonanceState`、`lineageScore`、裝配者、鎖定狀態與是否可切換主卡，並支援依 `家族鍵`、`共鳴層級`、`家族貢獻分數`、`取得時間` 排序。
5. 當同家族存在多張英靈卡時，集合層只會有 1 張 `Active_Main_Spirit_UID`；其餘卡片都保留在集合中，狀態為 `BACKUP` 或 `ECHO`，不會被刪除。
6. `設為主卡` 必須經確認；若當前處於戰鬥中或出征鎖定中，集合視圖只顯示鎖定狀態，不提供切換。
7. `英靈卡詳情頁` 負責顯示單卡的來源、成長、加成與傳道能力；`英靈陳列室` 負責管理多卡與切換主卡，兩者不能混成同一個視圖。
8. 若卡片尚未裝入任一武將命槽，陳列室應顯示為 `未裝配`；若已裝配，則同步顯示對應武將 UID 與命槽狀態。
9. `英靈陳列室` 的家族展示列，必須額外顯示該名將的英勇事蹟、完整屬性快照與技能快照，讓玩家可直接回看「他當年強在哪裡」。

## J. 正式共識摘要

1. 轉蛋只保留 `三國名將池（含女性）` 與 `支援卡（教官）池`。

### I-5. 英靈陳列室 UI 頁面規格

1. `英靈陳列室` 是家族層的 archive 頁，採 `列表 + 詳情` 的雙區域閱讀模式，目的不是替代 `寶 / GEAR`，而是讓玩家回看已故名將的完整生命紀錄。
2. 頁面資料來源分成三層：`Data.Spirit_Archive_Snapshot` 負責展示列與詳情內容，`Data.Spirit_Card_Collection` 負責對應當前英靈卡歸屬與主卡焦點，`Data.Spirit_Family_State` 負責家族展示焦點與預設排序。
3. 展示列預設排序與篩選規則以 `武將人物介面規格書.md` 為準；當玩家從 `命` 頁或家族頁進入時，應保留原本的家族篩選條件。
4. 詳情頁只讀，不提供調整屬性、重置技能、拆卡或刪除快照等操作。若玩家要回到當前可操作的英靈卡，應跳回 `寶 / GEAR` 頁。
5. 若某張歷史名將同時也是目前仍可操作的主卡，展示列可顯示 `現行主卡` 與 `歷史封存` 雙標籤，但快照內容仍必須與現行戰力區分。
6. 若家族內有多張已故名將，展示列以展示權重排序；權重相近時，優先顯示最近死亡且英勇事蹟更完整的條目。
