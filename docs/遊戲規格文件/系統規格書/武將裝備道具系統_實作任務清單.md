# 武將裝備道具系統實作任務清單

> 目的：把武將裝備、英靈卡、虎符與人物頁 `命 / 寶 / 兵` 分頁拆成 `UI Spec / Layout Contract / Data Read Model` 三層實作任務。
> 依據：`武將裝備道具系統.md`、`英靈世家系統.md`、`戰場部署系統.md`、`武將人物介面規格書.md`、`主戰場UI規格書.md`、`數值系統.md`、`Data Schema文件（本機端與Server端）.md`
> 共識用語：`參謀官`、`後勤官`；英靈卡本人 = `SELF_RESONANCE`；同家族 = `FAMILY_RESONANCE`；多卡不允許線性疊加。

## 0. 前置共識

- [ ] 移除 `SMART` 兵種在正式出征與 UI 顯示中的所有入口。
- [ ] 統一支援職稱為 `參謀官` 與 `後勤官`，不再使用 `參謀 / 副將` 作為正式用語。
- [ ] 確認英靈卡光效分三階：`SELF_RESONANCE` / `FAMILY_RESONANCE` / `ECHO_RESONANCE`。
- [ ] 確認同家族多張英靈卡只能做家族深度累積，不可把戰鬥加成線性堆滿。

## 1. UI Spec 任務

- [ ] 人物頁正式 tab 收斂為 `將 / 屬 / 命 / 技 / 寶 / 兵`，其中 `命 / Bloodline` 承接英靈卡、`寶 / GEAR` 承接一般裝備 / 傳家寶 / 道具、`兵 / Aptitude` 承接虎符與戰場適性。
- [ ] `寶` 頁完成 `一般裝備 2x3 grid` 的 UI spec，涵蓋稀有度、裝備名、強化值與空槽文案。
- [ ] `命` 頁完成 `英靈卡槽` 的 UI spec，需顯示 `尚未插入英靈卡 / 主卡 / 副卡 / 設為主卡 / 戰鬥中不可切換` 等正式文案。
- [ ] `兵` 頁完成 `虎符槽` 的 UI spec，需顯示 `尚未裝備虎符 / 兵種 / 星級 / 糧耗 / 投入兵力 / 查看詳情` 等正式文案。
- [ ] `將` 頁完成 `名聲 / 官職 / 統領軍勢` 區塊，顯示 `武將名聲 / 官職 / 帶兵上限 / 可用城市兵源 / 預估出征軍勢`。
- [ ] `寶` 頁 action 不再固定放 `升星 / 突破 / 培育 / 評價` 四鍵；改為依目前選中裝備、傳家寶或道具顯示上下文操作。

## 2. Layout Contract 任務

- [ ] `GeneralDetailComposite` tab bar 與 route key 對齊 `Overview / Stats / Bloodline / Tactics / Equip / Aptitude`，本地化顯示 `將 / 屬 / 命 / 技 / 寶 / 兵`。
- [ ] `Equip` child 補上 `GearModules/EquipmentSection / UtilitySection / ActionSection` 三段 layout contract。
- [ ] `Aptitude` child 補上 `AptitudeModules/TroopSection / TerrainSection / WeatherSection / TallySection / RoleSection` 五段 layout contract。
- [ ] 一般裝備六槽採 `WEAPON / HEAD / BODY / BOOTS / ACCESSORY_A / ACCESSORY_B` 內部鍵值。
- [ ] 英靈卡替換、設為主卡與虎符更換共用既有 dialog shell，不直接把確認文案寫死在卡片 prefab。
- [ ] 自身 / 同家族 / 非家族 三種 glow state 的 sprite / effect contract。
- [ ] `命` 頁的英靈卡詳情與 `兵` 頁的虎符詳情都導向 `spirit-tally-detail-main.json`，但不得由 `寶` 頁代管。

## 3. Data Read Model 任務

- [ ] `Data.Spirit_Card_Collection` 補齊 `Spirit_Family_Key`、`Resonance_Band`、`Glow_State`、`Card_Role`、`Family_Primary_Flag`。
- [ ] `Data.Spirit_Family_State` 作為家族級 read model，提供 `Active_Main_Spirit_UID` 與家族供養池。
- [ ] `Data.Inventory_Service` 維持 `Display_Bucket` 與 `Slot_Type` 的語義分層，避免一般裝備六槽、命頁英靈卡槽、虎符槽與一般背包混流。
- [ ] `Data.Generals[UID].Profile_Presentation.Default_Tab` 僅允許 `Overview / Stats / Bloodline / Tactics / Equip / Aptitude`。
- [ ] Spirit card read model 支援 `SELF_RESONANCE / FAMILY_RESONANCE / ECHO_RESONANCE` 的狀態計算。
- [ ] 需要時提供家族主卡判定欄位，確保同家族多卡僅由單一主卡吸收成長。

## 4. 實作順序

1. 先回填 schema 與 read model 欄位。
2. 再補 layout contract，讓前後端有一致的欄位與狀態約定。
3. 最後落 UI Spec 與互動文案。
4. 完成後再跑一次平衡檢查，確認多家族卡不會破壞戰鬥上限。
