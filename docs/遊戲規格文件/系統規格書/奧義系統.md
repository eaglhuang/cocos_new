# 奧義系統規格書

> **來源文件**：因子血統論系統規格書.md (doc_spec_0073)、轉蛋與戰場武將處置系統.md (doc_spec_0155)、子嗣系統規格書：三國傳承.md

---

## A. 系統描述

奧義系統定義武將的終極必殺技——每位武將最多可擁有 5 個奧義，根據轉生次數（1~5 轉）逐步解鎖。奧義是比一般戰法更強大的技能，施放時消耗精力而非 TP，具有華麗的演出特效與強力效果。名將擁有史實專屬奧義；子嗣的奧義由遺傳因子與培育決定。

> **正式設計邊界（2026-04-15）**：奧義屬於「武將層能力」，設計上**不與格子系統綁死**。格子、地形、推退、火海、水域等只是戰場執行時可選的承載方式；奧義本體優先描述為武將屬性強化、全隊增益、敵軍全體減益、敵將傷害 / 斬殺 / 控制等高階效果。

> **Unity 對照**：類似 Ultimate Skill System，以「解鎖條件 + 精力消耗 + 華麗演出」為特徵。相當於 Unity 中 Animator Override Controller 驅動不同奧義動畫，SearchableObjectPool 管理解鎖狀態。

---

## B. 系統目的

1. **高光時刻**：奧義施放是戰場最具爽感的操作，製造「必殺瞬間」的情緒高峰
2. **轉生驅動**：5 個奧義分別在 1~5 轉解鎖，直接驅動重複抽卡的動力
3. **角色差異化**：每位名將的專屬奧義是其獨一無二的身份標識
4. **精力博弈**：奧義消耗精力，施放時機的抉擇（戰場 vs 結緣精力分配）增加策略深度
5. **傳承亮點**：子嗣可繼承父母的奧義因子，覺醒出家傳奧義

## C. 商業套路

| 商業點 | 說明 |
|---|---|
| 轉生解鎖 | 5 個奧義需 5 轉才能全部解鎖，驅動大量重複抽卡 |
| 補元散消耗 | 奧義消耗精力，頻繁使用需補元散回復 |
| 限定奧義 | 活動限定版武將擁有獨佔奧義，增加稀有度 |
| 奧義升級 | 重複解鎖同一奧義可升級，提升效果 |
| 奧義演出 | 華麗的奧義動畫是遊戲最佳宣傳素材 |

## F. 公式相關

### 轉生解鎖規則

| 轉生次數 | 解鎖奧義欄位 | 屬性上限提升 | 說明 |
|---|---|---|---|
| 1 轉（初見） | 奧義 1（基礎奧義） | 1000 | 首次取得即可使用 |
| 2 轉 | 奧義 2（進階奧義） | 1100 | 需重複抽到一次 |
| 3 轉 | 奧義 3（專精奧義） | 1200 | 角色特色定義 |
| 4 轉 | 奧義 4（覺醒奧義） | 1350 | 強力覺醒技 |
| 5 轉 | 奧義 5（究極奧義） | 1500+ | 角色完全體 |

### 精力消耗

```
vitality_cost = ult_base_cost × (1 - vitality_skill_reduction)
// 基礎消耗：奧義1=5%、奧義2=6%、奧義3=7%、奧義4=8%、奧義5=10%
Current_Vitality -= vitality_cost
if (Current_Vitality < 0): 
  // 強制退出戰場，並視為精力耗盡
```

### 奧義傷害公式

```
ult_damage = base_power × (1 + ult_level × 0.15) × stat_modifier
// stat_modifier 依武力/智力等主屬性決定
// base_power: 奧義1=200、奧義2=300、奧義3=400、奧義4=500、奧義5=700
```

### F-4. 奧義效果分層（與格子系統解耦）

1. **自身 / 主將屬性強化**：如爆發、免控、再行動、斬殺線強化。
2. **全隊增益**：如攻防速提升、護盾、士氣回復、精力回補、反擊率提升。
3. **敵方全體減益**：如沉默、崩解、魅惑、減速、驅散、易傷標記。
4. **敵將傷害 / 控制**：如單體重創、扇形斬擊、範圍智力傷害、倒戈、共享傷害。
5. **戰場表現層**：若奧義需要呈現火海、水域、位移、連鎖跳躍等效果，統一經由 `battleSkillId -> skills.json` 包裝進戰鬥時間軸，不把格子狀態反過來當成奧義 ownership 資料。

### F-5. Battle / General Detail 正式資料流（2026-04-15）

1. 武將 seed 資料以 `assets/resources/data/generals.json` 內的 `ultimateSlots[]` 為唯一入口。
2. `GeneralDetail` Skills 分頁直接讀 `ultimateSlots[]`，再依 `templateId + slot` 查 `ultimate-definitions.json` 取 `name / description / scalingStat / vitalityCostPct / inheritanceGeneId`。
3. `BattleScene` 只讀同一份 `ultimateSlots[]` 與已解鎖狀態，將其映射為奧義按鈕列表與戰鬥可施放技能，不另建 battle-only 奧義表。
4. `battleSkillId` 僅負責戰場執行與演出封裝；奧義歸屬、槽位、解鎖條件仍由武將資料與 `ultimate-definitions.json` 決定。

### F-7. 奧義具體實作步驟（可直接排 Sprint）

1. **Step 1: 先做 8 個效果家族，不先做 65 條特例**
  - `UBURST / UTEAMBUFF / UTEAMHEAL / UALLDEBUFF / USINGLE / UAREA / UCONTROL / USPECIAL`
  - 每個家族統一輸入：`caster / targets / scalingStat / coeff / duration / flags`
2. **Step 2: 奧義定義只負責參數，不負責流程**
  - `ultimate-definitions.json` 僅描述 `family / coeff / duration / targetMode / battleSkillId / cinematicProfile`
  - 禁止每條奧義在 code 內長出一個 `switch case` 專用流程
3. **Step 3: 先補戰場必要四大能力**
  - 自身爆發
  - 全隊增益
  - 敵軍全體減益
  - 敵將定點傷害 / 控制
  - 這四大類完成後，65 條奧義大多數即可先進入可玩狀態
4. **Step 4: 再補特殊規則能力**
  - 復活
  - 增益轉移
  - 行動條竊取 / 重置
  - 死亡觸發
  - 場勢切換 / 全場優勢態勢
5. **Step 5: GeneralDetail / BattleScene 一起接**
  - `GeneralDetail` 顯示：名稱、說明、解鎖轉數、主屬性、家傳因子、效果家族
  - `BattleScene` 顯示：可施放列表、SP 成本、冷卻 / 限用、命中目標摘要
6. **Step 6: AI 規則按家族寫，不按角色寫**
  - `UBURST`：看斬殺線與自身危險值
  - `UTEAMBUFF / UTEAMHEAL`：看我方血線、Buff 空窗、戰局落後
  - `UALLDEBUFF / UCONTROL`：看敵方密度、關鍵敵將、下一回合威脅
  - `USINGLE / UAREA`：看最優擊殺 / 最優期望傷害
7. **Step 7: 驗收順序**
  - family unit test
  - 每位名將至少 1 條 smoke 奧義 test
  - `GeneralDetail Skills` 與 `BattleScene UltimateBtn` 各做一輪 UI smoke capture

### F-8. 實作優先順序建議

1. **P0 首批可玩包**：`UBURST + UTEAMBUFF + UTEAMHEAL + USINGLE`
2. **P1 戰場策略包**：`UALLDEBUFF + UAREA + UCONTROL`
3. **P2 高階規則包**：`USPECIAL`
4. **P3 組合奧義包**：待獨立 `組合奧義` 子系統母規格成立後，再承接跨武將聯動

## J. 名詞定義

| 名詞 | 定義 |
|---|---|
| 奧義 | 武將的終極必殺技，消耗精力施放，每場限用 1 次，威力遠超一般戰法 |
| 轉生（轉） | 重複抽到同一名將時觸發的突破機制，1~5 轉依序解鎖奧義與屬性上限 |
| 精力消耗 | 奧義施放的代價，直接扣除武將的精力百分比（5%~10%） |
| 專屬奧義 | 與名將 Template_ID 綁定的固定奧義，代表該武將的特色能力 |
| 家傳奧義 | 子嗣透過繼承父母的綠色因子，在培育中覺醒的奧義 |
| 奧義演出 | 奧義施放時的全屏特效、鏡頭運動與動畫表現 |
| 究極奧義 | 5 轉解鎖的最終奧義，威力最強、演出最華麗 |
| 組合奧義 | 特定武將搭配時可施放的聯合奧義（如周瑜+諸葛亮=赤壁大火） |
| 奧義圖鑑 | 記錄玩家已解鎖的所有奧義的收集系統 |


## 傳承系特殊奧義：英靈附體
- 當兵力陷入劣勢時可發動。機制為「召喚培育期間所繼承的祖先虛影」，並同步施放該祖先的成名奧義，實現一回合內「本體 + 祖先」雙重發招的逆轉效果。
