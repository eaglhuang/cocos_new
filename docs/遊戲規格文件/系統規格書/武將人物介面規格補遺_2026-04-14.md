# 武將人物介面規格補遺_2026-04-14：人性化與生活感擴充

本補遺旨在將「112 份歷史討論文件」中關於 **「武將人性化行動」**、**「動態生活敘事」** 與 **「俘虜處置邏輯」** 的細節正式轉化為 UI 視覺規格，補足母規格書（doc_spec_0011）在生活感方面的缺口。

---

## 1. 性格標籤系統 (Personality Icon Family)

依據 doc_spec_0102 與 doc_spec_0128，武將具有驅動離線行為的四種核心性格。人物頁 Overview (v3) 必須顯式展示此標籤。

### 1.1 性格圖騰定義

| 性格類型 | 視覺意象 | 核心行為傾向 (離線/晨報) | UI 提示文案 |
| :--- | :--- | :--- | :--- |
| **義氣型** | 兩手相握 / 結義印 | 尋訪在野武將、縮短招降時間、增加他將好感。 | 「重情重義，人脈廣布」 |
| **豪爽型** | 酒碗 / 裂帛 | 找人喝酒提升屬性、市場消費觸發折扣、武力切磋。 | 「豪氣干雲，廣結善緣」 |
| **睿智型** | 羽扇 / 奏摺 | 內政優化、協助子嗣課程、自動完成每日任務。 | 「深謀慮，治世良才」 |
| **貪玩型** | 骰子 / 飛鳥 | 隨機遊歷撿回道具、觸發搞笑命運事件、意外發現。 | 「放浪不羈，奇遇不斷」 |

### 1.2 視覺位置
- **優先位置**：人物 Header 區，緊鄰「職能標籤（Combat/Mentor）」。
- **表現形式**：小型圓形徽章，帶有專屬顏色（義-綠、豪-紅、睿-藍、貪-紫）。

## 2. 忠誠與信任體系 (Loyalty & Trust Gauge)

依據 doc_spec_0155，區分轉蛋、招降、傭兵三種狀態，並影響虎符沒收邏輯。

### 2.1 忠誠度條 (Loyalty Bar)
- **顯示條件**：非轉蛋來源之武將（如戰場招降、地下城救美、傭兵）。
- **視覺規範**：
  - **數值範圍**：0 ~ 100。
  - **鎖定態**：轉蛋武將顯示 `100 (鎖定圖標)`。
  - **動態條**：低於 60 顯示橙色警示；低於 30 顯示紅色（代表隨時可能下野或被勸降）。
- **連動操作**：進行「沒收虎符」操作時，UI 需彈出提示：「沒收此虎符將導致 **[武將名]** 忠誠度下降 30 點。」

---

## 3. 武將動態日誌氣泡 (General Narrative Bubble)

依據 doc_spec_0112，將「主公晨報」的數據實例化到個別人物頁中。

### 3.1 視覺結構
- **名稱**：`GeneralJournalPeek` (掛點於母規格 738)。
- **位置**：人物立繪旁（對話氣泡態）或 Summary Strip 下方（橫向滾動文字態）。
- **內容組合**：`[時間戳] + [人性化動作描述]`。
  - *範例*：*「昨晚：張飛因酒後與華雄切磋，助其武力提升了 1 點。」*
  - *互動*：點擊該氣泡可進入「完整生平實錄」彈窗。

---

## 4. 任務適合度提示 (Mission Suitability Tag)

依據 doc_spec_0074 與 doc_spec_0120，加強大廳任務牆與人物詳情頁的對應感。

### 4.1 推薦標籤 (Suitability Badge)
- **邏輯**：系統實時比對目前大廳「已開啟」但「未派遣」的任務。
- **UI 表現**：若該武將對某任務有「自告奮勇」加成（成功率 +50%），在人物頁顯示：
  - 🔥 **自薦推薦**：[任務名稱]
  - *點擊跳轉*：直接引導至大廳該任務牆位置。

## 6. 結論
本補遺完成後，武將人物頁面將不再只是靜態數字，而是能反映 **「生活軌跡」**、**「情感忠誠」** 與 **「性格特色」** 的人性化窗口。本件作為 `UI-2-0059` 與 `UI-2-0061` 的上位設計準則。
