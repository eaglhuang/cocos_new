## 2. 工具鏈詳細說明

### 2.1 ingest-generals-wiki.js（Wiki 爬取）

**用途**: 從維基百科取得武將歷史摘要，生成 `temp_workspace/raw-wiki.json`

**執行方式**:
```bash
node tools_node/ingest-generals-wiki.js [--limit 50] [--dry-run]
```

**輸出格式**: `temp_workspace/raw-wiki.json`
```json
{
  "version": "1.0.0",
  "source": "wikipedia",
  "generatedAt": "<ISO timestamp>",
  "count": 50,
  "data": [ /* GeneralBase 陣列 */ ]
}
```

**速率限制**: 每次請求間隔 500ms，含 User-Agent 設定

**快取機制**: `temp_workspace/wiki-cache.json` 避免重複請求

---

### 2.2 ingest-generals-ai.js（AI 故事生成）

**用途**: 從 `master/generals-base.json` 讀取武將資訊，生成 lore 欄位，輸出 `temp_workspace/raw-ai.json`

**執行方式**:
```bash
node tools_node/ingest-generals-ai.js [--limit <n>] [--dry-run] [--force]
```

**輸出欄位**:
- `historicalAnecdote` — 歷史趣聞（200 字內）
- `bloodlineRumor`     — 血脈傳聞（坊間說法）
- `storyStripCells`    — 6 格故事條（slot 1-6，各含 text）

**容錯**: 單筆生成失敗時回報 WARN 並跳過，不整批失敗

---

### 2.3 merge-generals-master.js（合併去重）

**用途**: 合併 raw-wiki.json 與 raw-ai.json 至 master/generals-base.json + master/generals-lore.json

**執行方式**:
```bash
node tools_node/merge-generals-master.js [--dry-run] [--wiki <path>] [--ai <path>]
```

**合併策略**: 以 `id` 為 key，**已存在資料優先，不覆蓋**（保護手動修正）

**自動後置**: 合併完成後自動執行 `validate-generals-data.js`

---

### 2.4 validate-generals-data.js（品質驗證）

**用途**: 驗證 master/ 目錄與 generals.json 中所有資料的品質

**執行方式**:
```bash
node tools_node/validate-generals-data.js [--file <path>] [--dir <path>]
```

**驗證規則**:

| 規則 | 欄位 | 標準 |
|------|------|------|
| R1 必填欄位 | id, name, faction | 不得為空 |
| R2 屬性範圍 | str/int/lea/pol/cha/luk | 整數 0-100 |
| R3 EP 值 | ep | >= 0 |
| R4 稀有度 | rarityTier | common/rare/epic/legendary/mythic |
| R5 武將類別 | characterCategory | civilian/general/famed/mythical/titled |
| R6 基因格式 | genes[].id | 不得為空 |
| R7 故事格式 | storyStripCells[].slot, .text | 不得為空 |

## 4. 執行順序（標準流程）

```
步驟 1: 爬取 Wiki 資料
  node tools_node/ingest-generals-wiki.js --limit 50

步驟 2: 生成 AI 故事（可選，首次或 --force）
  node tools_node/ingest-generals-ai.js

步驟 3: 合併至 master
  node tools_node/merge-generals-master.js

步驟 4: 自動驗證（merge 工具已內建）
  → 若失敗，進入步驟 5

步驟 5: 人工審核修正（見第 5 節）
  → 修正後重新執行步驟 3
```

---

## 5. 人工審核 Checklist

合併後執行 `validate-generals-data.js` 通過後，人工審核以下項目：

### 5.1 數值合理性
- [ ] str 最高值武將是否合理（呂布 99、關羽 97 等）
- [ ] int 最高值武將是否合理（諸葛亮 99、郭嘉 97 等）
- [ ] rarityTier 分布是否合理（legendary 應 < 20%）
- [ ] ep 排名前 10 是否符合遊戲設計預期

### 5.2 內容品質
- [ ] historicalAnecdote 是否有明顯亂碼或截斷
- [ ] bloodlineRumor 是否通順
- [ ] storyStripCells 是否具有敘事連貫性（6 格流暢）

### 5.3 完整性
- [ ] 所有武將均有 rarityTier 與 characterCategory
- [ ] 關鍵 25 位（五虎上將等）均有完整 lore 欄位
- [ ] 沒有重複 id

### 5.4 修正方式
1. 直接修改 `assets/resources/data/master/generals-base.json` 或 `generals-lore.json`
2. 執行 `node tools_node/validate-generals-data.js` 確認 0 errors
3. 不要修改 `temp_workspace/raw-wiki.json`（視為原始快照，可重新爬取覆蓋）
