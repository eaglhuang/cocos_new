# 戰場部署系統規格書

> **來源文件**：遊戲系統規格書.md — 第 3 節「5x8 棋盤與戰場平衡 (Battle System)」

---

## A. 系統描述

戰場部署系統定義玩家在 5×8 棋盤戰鬥中的出兵機制。廢除傳統「部署點數（DP）」系統，改為**糧草即時扣除**與**兵種冷卻**雙重約束。核心特性：(1) 糧草不足或精銳兵種冷卻中時，點擊仍可產出「降級普通小兵」保持操作連貫；(2) 英靈代打機制允許戰鬥中呼叫英靈附體或代替子孫進行單挑；(3) 委任 AI 根據武將智力決定部署邏輯，高智力 AI 會策略性保留英靈虎符。

> **Unity 對照**：DP 系統等同 RTS 中的「供給值（Supply）」——我們改為「糧草即時消耗」；兵種冷卻等同「訓練佇列 Cooldown」；降級產出類似 StarCraft 中供給到頂改產 Marine 的設計思路；英靈代打等同格鬥遊戲的「觸發 Ultimate / Assist」。

---

## B. 系統目的

1. **操作爽度**：無 DP 限制 + 降級產出確保「每次點擊都有反饋」
2. **戰術深度**：糧草管理 + 兵種冷卻創造資源壓力與優先序選擇
3. **英靈價值化**：英靈代打讓退役名將在戰場上繼續發揮，強化情感傳承
4. **自動化體驗**：委任 AI 為非核心戰鬥提供省心選擇（根據武將智力品質導向不同 AI 策略）
5. **經濟連動**：糧草作為戰鬥核心消耗品，連動治理系統的資源產出

## F. 公式相關

### 糧草即時扣除

```typescript
// === 糧草消耗（取代 DP）===
// 每部署一組兵消耗糧草
const DEPLOY_FOOD_COST: Record<string, number> = {
    INFANTRY:  10,    // 步兵：最便宜
    ARCHER:    12,
    PIKE:      12,
    SHIELD:    14,
    CAVALRY:   18,    // 騎兵：較貴
    NAVY:      20,    // 水軍：需舟船
    SIEGE:     30,    // 機械：最貴
    SMART:     15,    // 智將
    SPECIAL:   25,    // 特殊兵種（虎符兵）：高消耗
};

// 戰場總糧草預算 = 玩家當前糧草儲備（上限 10,000）
// 戰鬥結束後消耗結算，從玩家資源扣除
```

### 兵種冷卻

```typescript
// === 兵種冷卻（回合數）===
const DEPLOY_COOLDOWN: Record<string, number> = {
    INFANTRY:  1,   // 基礎兵：1 回合即可再產
    ARCHER:    1,
    PIKE:      1,
    SHIELD:    1,
    CAVALRY:   2,   // 騎兵：需 2 回合
    NAVY:      2,
    SIEGE:     3,   // 機械：需 3 回合
    SMART:     2,
    SPECIAL:   4,   // 特殊兵種：最長冷卻
};

// 急行令：消耗 1 張 → 目標兵種冷卻 -1 回合（最低 0）
```

### 精銳部署上限與保留槽

```typescript
// === 精銳部署上限 ===
// 只計入「場上存活中的精銳 / 特殊兵 / 攻城器械」；普通小兵不佔上限
const ELITE_DEPLOY_LIMIT = {
    base_active_cap: 4,
    siege_scene_bonus: 1,        // 攻城戰預留 1 個器械槽
    officer_tier_bonus_cap: 1,   // 官職 / 任務加成最多再 +1
};

function getEliteDeployCap(state: BattleState): number {
    const siegeBonus = state.scenePresetId === 'SCENE_SIEGE' ? ELITE_DEPLOY_LIMIT.siege_scene_bonus : 0;
    const officerBonus = Math.min(state.officerTierDeployBonus ?? 0, ELITE_DEPLOY_LIMIT.officer_tier_bonus_cap);
    return ELITE_DEPLOY_LIMIT.base_active_cap + siegeBonus + officerBonus;
}

function canDeployElite(state: BattleState): boolean {
    return state.activeEliteCount < getEliteDeployCap(state);
}
```

- 精銳部署上限的目的，是避免高品質虎符與高糧草存量把戰場直接推成「滿版精銳」。
- `SPECIAL / CAVALRY / NAVY / SIEGE / SMART` 預設都算精銳出場名額；`MILITIA` 與基礎步弓盾槍不計入。
- 若超出上限，仍沿用本系統的核心原則：**優先降級產出普通小兵，而不是讓點擊失效**。

### 降級產出

```typescript
// === 降級機制 ===
// 當玩家點擊部署，但目標兵種處於冷卻或糧草不足時
// → 自動降級產出「普通小兵」（militia），確保操作連貫
function deployUnit(player: Player, targetTroop: string): DeployResult {
    const cooldownReady = player.cooldowns[targetTroop] <= 0;
    const foodEnough = player.food >= DEPLOY_FOOD_COST[targetTroop];

    if (cooldownReady && foodEnough) {
        // 正常部署精銳
        player.food -= DEPLOY_FOOD_COST[targetTroop];
        player.cooldowns[targetTroop] = DEPLOY_COOLDOWN[targetTroop];
        return { unit: targetTroop, quality: 'ELITE' };
    } else {
        // 降級產出普通小兵（固定消耗 5 糧草，無冷卻）
        if (player.food >= MILITIA_FOOD_COST) {
            player.food -= MILITIA_FOOD_COST;
            return { unit: 'MILITIA', quality: 'BASIC' };
        }
        // 糧草完全耗盡 → 無法部署
        return { unit: null, quality: null };
    }
}

const MILITIA_FOOD_COST = 5;   // 普通小兵消耗

// === 普通小兵 vs 精銳 ===
// 普通小兵攻防 = 基礎兵種的 40%
// 沒有特殊能力、沒有兵種剋制加成
// 存在意義：保持操作爽度、填充戰線、拖時間等精銳冷卻
const MILITIA_STAT_RATIO = 0.4;
```

### 英靈代打

```typescript
// === 英靈附體 ===
// 戰鬥中呼叫英靈附體到在場武將身上
// 持續 2~3 回合，能力大幅提升
interface SpiritPossession {
    spirit_id: string;           // 英靈 ID（退役名將）
    target_general_uid: string;  // 附體目標武將
    duration: number;            // 持續回合（2~3）
    stat_boost: number;          // 攻防提升百分比
    ultimate_unlock: boolean;    // 是否解鎖英靈奧義
}

const POSSESSION_CONFIG = {
    duration_base: 2,              // 基礎持續 2 回合
    duration_resonance_bonus: 1,   // 血脈共鳴額外 +1 回合
    stat_boost_rate: 0.50,         // 攻防 +50%
    cooldown_per_battle: 1,        // 每場戰鬥只能使用 1 次
    daily_free_uses: 1,            // 每日免費 1 次
    extra_use_cost: 20,            // 額外使用消耗精元
};

// === 英靈代打（單挑）===
// 英靈完全代替子孫進行單挑
// 使用英靈的歷史最高屬性（退役時記錄）
// 勝負結果由英靈承擔，不影響子孫 HP
function spiritDuel(spirit: SpiritCard, enemyGeneral: General): DuelResult {
    // 英靈使用退役時的 maxHp / stats 進行單挑
    // 若英靈與對手有「宿敵」關係 → 攻擊 +20%
    const nemesisBonus = hasNemesis(spirit, enemyGeneral) ? 1.2 : 1.0;
    // ... 套用標準單挑公式
    return executeDuel(spirit.retireStats, enemyGeneral.currentStats, nemesisBonus);
}
```

### 委任 AI

```typescript
// === 委任 AI 邏輯 ===
// 根據武將 INT（智力）決定 AI 部署品質
// 高智力 → 策略性保留英靈虎符直到遇敵方主將
// 低智力 → 隨機部署、不考慮兵種剋制

type AIDeployQuality = 'LOW' | 'MEDIUM' | 'HIGH' | 'EXPERT';

function getAIDeployQuality(generalINT: number): AIDeployQuality {
    if (generalINT >= 90) return 'EXPERT';   // 諸葛亮級
    if (generalINT >= 70) return 'HIGH';      // 周瑜級
    if (generalINT >= 50) return 'MEDIUM';    // 一般武將
    return 'LOW';                              // 張飛級（勇猛但不擅策略）
}

// AI 部署特性
const AI_DEPLOY_BEHAVIOR: Record<AIDeployQuality, string> = {
    LOW:    '隨機部署，不考慮剋制，精銳立即使用',
    MEDIUM: '基本剋制意識，精銳在前 3 回合使用',
    HIGH:   '根據敵方兵種剋制部署，保留虎符兵到關鍵時刻',
    EXPERT: '完美剋制+保留英靈虎符直到遇到敵方主將，糧草管理最優',
};

const AI_RESERVE_POLICY = {
    LOW:    { keepFoodFloorRate: 0.00, reserveSpecialForBoss: false, reserveSiegeSlots: 0 },
    MEDIUM: { keepFoodFloorRate: 0.10, reserveSpecialForBoss: false, reserveSiegeSlots: 0 },
    HIGH:   { keepFoodFloorRate: 0.18, reserveSpecialForBoss: true,  reserveSiegeSlots: 1 },
    EXPERT: { keepFoodFloorRate: 0.25, reserveSpecialForBoss: true,  reserveSiegeSlots: 1 },
};
```

| AI 品質 | 保留策略 | 正式定義 |
|---|---|---|
| LOW | 幾乎不保留資源 | 看到可用精銳就直接下，不區分主將戰或場景目標 |
| MEDIUM | 保留少量糧草 | 會避免在前兩回合把糧草耗空，但仍可能提早交掉虎符兵 |
| HIGH | 保留 1 個特殊兵 / 器械槽 | 會等到敵方主將、營帳、城門等關鍵目標出現再下高價值單位 |
| EXPERT | 嚴格保留特殊兵與器械節奏 | 會依 `scenePresetId` 與敵方陣容，把高 `TigerTallyScore` 單位留給決勝窗口 |

## K. 特殊戰場環境與模組化格子補充

> **整併來源**：戰場互動矩陣與養成動力.md (doc_spec_0091)、夜襲戰場視覺規格提案.md (doc_spec_0078)、模組化戰場系統開發策略.md (doc_spec_0089)
> **衝突處理**：本章是在現行 5×8 棋盤、糧草與冷卻體系之上疊加環境變體，不改動主戰場的基本回合框架。
> **完整規格**：場景戰法（Scene Gambit）與格子戰法（Tile Tactic）的完整定義、觸發流程、交互規則，請見 **戰法場景規格書.md (doc_spec_0039)**。

### 戰場環境的正式定位

- 戰場特殊玩法不應靠另做一張完全獨立的棋盤系統，而應透過「格子狀態 + 任務變數 + 視覺主題」在同一個 5×8 基礎上切換。
- **場景戰法**（Scene Gambit）由高 INT 軍師在戰場外發動成功後，進入戰鬥時載入預繪靜態背景 + 預設 Tile-State 佈局。場景背景是畫死的，不可動態切換。
- **格子戰法**（Tile Tactic）是武將在戰場上消耗 TP 施放的即時技能，僅在指定格子上疊加特效（如火攻幾格、落石幾格、暴風幾格），不改變背景。
- 夜襲、火攻、圍堵等只是場景變體，不可覆蓋主規則中的糧草、冷卻、降級產出與英靈代打。

### 已規劃的 6 種場景戰法

| 場景 ID | 場景名稱 | INT 門檻 | 預繪背景風格 | 核心玩法 |
|---|---|---|---|---|
| SCENE_FOREST_AMBUSH | 森林埋伏戰 | ≥ 75 | 冷色夜戰密林 | 陷阱抓捕、隱身伏擊 |
| SCENE_NIGHT_RAID | 夜襲作戰 | ≥ 70 | 月光營帳群 | 佔領營帳、夜間暴擊 |
| SCENE_CANYON_ROCK | 峽谷落石戰 | ≥ 65 | 險峻峽谷 | 落石傷害、路線封鎖 |
| SCENE_FLOOD | 水淹七軍戰 | ≥ 80 | 深藍水體渡河 | 水流推移、適性削弱 |
| SCENE_FIRE_RAID | 火燒偷襲戰 | ≥ 75 | 暗暖橘色火場 | 火焰蔓延、物資掠奪 |

### Tile-State 模組分類

| 模組 | 代表功能 | 典型戰場 |
|---|---|---|
| Normal Tile | 一般移動與接戰 | 標準遭遇戰 |
| Hazard Tile | 進入或停留時受傷害 | 火燒 / 落石 |
| Force-Move Tile | 強制推移、退後、滑移 | 水淹 / 潮汐 |
| Stealth Tile | 隱匿、伏擊、陷阱 | 森林埋伏 / 夜襲死角 |

### Tile-State preset 權威表

| Preset_ID | 對應場景 | 組成重點 | 對部署的直接影響 |
|---|---|---|---|
| `SCENE_FOREST_AMBUSH` | 森林埋伏戰 | 高比例 `STEALTH` + 陷阱 `HAZARD` | 優先保留機動與伏擊單位，低視野不鼓勵早交遠程精銳 |
| `SCENE_NIGHT_RAID` | 夜襲作戰 | `STEALTH`、營帳目標、局部火源 `HAZARD` | AI 應保留高爆發虎符兵給營帳與主將擊殺窗口 |
| `SCENE_CANYON_ROCK` | 峽谷落石戰 | 固定落石 `HAZARD` + 路線封鎖 | 精銳部署以耐打前排與擊退單位優先 |
| `SCENE_FLOOD` | 水淹七軍戰 | 大量 `FORCE_MOVE` + 深水 `HAZARD` | 優先水戰 / 位移抗性單位，減少低適性騎兵投入 |
| `SCENE_FIRE_RAID` | 火燒偷襲戰 | 可蔓延 `HAZARD` + 局部 `STEALTH` | 高火抗與高機動單位優先，避免把低韌性精銳一次壓上 |

- 模組化棋盤的正式原則是「**同一張 5×8 棋盤 + 不同 preset**」，而不是為每種場景再做一套新棋盤系統。
- `scenePresetId` 只控制格子狀態、部署權重與目標物件，不改動本系統的糧草、冷卻、降級產出與英靈代打主規則。
