# 名將挑戰賽系統規格書

> **來源文件**：遊戲系統規格書.md — 第 4 節「名將挑戰賽 (The Tournament)」、賽馬娘機制轉化三國戰場.md (doc_spec_0067)

---

## A. 系統描述

名將挑戰賽（Tournament）是以「兩週為一季」的非同步離線 PvP 賽事系統，仿照星座杯賽制設計。賽季分為兩階段：第一週為「培育期」（公告場地屬性，玩家針對性培育克制武將）；第二週為「競賽期」（五戰三勝制，非同步離線對戰，決賽一場定勝負）。每季新發行的虎符具備克制本季 BOSS 的特定標籤，驅動新卡需求與商業連動。

> **Unity 對照**：類似 Guild War + Season Pass 混合系統。培育期等同 Unity 中的「賽前 Lobby 準備階段」；競賽期的非同步離線對戰等同「AI 鏡像對戰」——雙方各上傳 JSON 快照（含五圍、因子、虎符配置），由伺服器模擬戰鬥。

### 賽季命名規則（杯賽制）

賽季以知名三國武將或史實地點命名，強化三國沉浸感：

| 賽季類型 | 命名規則 | 範例 |
|---|---|---|
| 武將盃 | 以名將命名 | 張飛盃、呂布盃、趙雲盃 |
| 古戰場盃 | 以史實地點命名 | 赤壁盃、官渡盃、長坂盃 |
| 九州盃（特別賽） | 全服大型賽事，每季末舉行 | 九州天下盃（積分制，月度結算） |

### 月度場景賽（特別賽制）

> **來源**：賽馬娘機制轉化三國戰場.md (doc_spec_0067)

每個月額外舉辦一場「特定戰法場景賽」，指定本月主推的**特殊地形組合或天氣條件**，強迫玩家使用不同的策略組合。場景賽不計入主賽積分，但提供限定獎勵與稱號。

| 月份 | 場景賽類型 | 特殊限制 |
|---|---|---|
| 1 月 | 冰天雪地 | 禁用騎兵類戰法 |
| 2 月 | 赤壁水戰 | 只允許水域/弓兵適性武將 |
| 3 月 | 春耕奇謀 | 智力戰法威力 ×1.5 |
| 4 月 | 山地奇兵 | 步兵適性 S 級加成翻倍 |
| … | … | … |

## B. 系統目的

1. **長線目標**：給玩家提供每兩週一次的培育方向與競爭動力
2. **培育驅動**：公告場地屬性（如：張飛盃/雨天/森林）引導玩家針對性培育
3. **虎符商業連動**：每季新虎符具備克制 BOSS 的特定標籤，創造新卡需求
4. **公平競技**：非同步離線對戰避免操作差距，回歸策略深度
5. **社交與排名**：排名系統提供炫耀與社交比較的動力

### 賽季鏡像快照與鎖定規則

```typescript

interface TournamentTeamSnapshot {
  snapshot_id: string;
  season_id: string;
  registered_at: string;
  ruleset_id: string;
  generals: string[];
  tiger_tally_scores: number[];
  scene_preset_tags: string[];
}

const TOURNAMENT_SNAPSHOT_RULES = {
  lock_on_first_entry: true,

  competition_phase_replace_limit: 1,

```

- 名將挑戰賽的權威資料不是即時角色狀態，而是**報名當下凍結的鏡像快照**。
- 培育期允許每日重登錄 1 次；進入競賽期後，僅允許 1 次緊急替換，避免玩家無限追版本與即時洗配置。
- 快照必須連同 `TigerTallyScore`、主要戰法、場景標籤一起鎖定，伺服器模擬只讀取快照，不回頭抓玩家當下背包狀態。

### 排名積分

```typescript
// === ELO 簡化排名 ===
// 勝 +25~40 分（視雙方實力差距）
// 敗 -15~30 分
function calculateEloChange(myRank: number, opponentRank: number, isWin: boolean): number {
    const expected = 1 / (1 + Math.pow(10, (opponentRank - myRank) / 400));
    const k = 32;  // K-factor
    const actual = isWin ? 1 : 0;
    return Math.round(k * (actual - expected));
}

// === 段位對照 ===

    DIAMOND:  { min: 2500, max: 2999 },

  const SEASON_ROLLOVER_RULE = {
```

  ### 賽季 rollover 與歷史留檔

  - 每季結束時，排名分數不直接歸零，而是依段位做 soft reset，保留長線追逐感。

## K. 九州盃與月度場景賽補充設計

> **來源**：賽馬娘機制轉化三國戰場.md (doc_spec_0067)

### 九州盃（全服大型限定賽）

九州盃為每季末（第 14 天）舉辦的全服積分制大型賽事，以**極端場景限制**（單一地形 + 特定天氣 + 指定兵種克制）為核心，驅動玩家針對該場景專門培育特定方向的武將。

| 九州盃類型 | 地形 | 天氣 | 特殊規則 | 培育驅動方向 |
|---|---|---|---|---|
| 赤壁盃 | 水域 | 大風 | 火攻傷害 ×2 | 水域 S 適性 + 火攻戰法 |
| 官渡盃 | 平原 | 晴天 | 騎兵衝鋒傷害 ×1.8 | 平原 S 適性 + 高武力騎兵 |
| 虎牢盃 | 山地 | 夜戰 | 單挑武力直接結算 | 山地 S 適性 + 高武力 |
| 夷陵盃 | 森林 | 大雨 | 弓兵射程減半；谷地陷阱觸發率 ×2 | 步兵/機關適性 + 謀略型 |

### 月度場景賽積分公式

```
Monthly_Score = Σ[(屬性總和 × 場景地形倍率) + (戰法點數 × 場景克制係數)] × 存活係數
// 公式與 戰場適性系統.md (doc_spec_0041) F 節「競技場積分公式」相同
// 但地形/天氣限制更極端，「針對性培育」的加成效果因此更顯著
```

地形倍率沿用 `戰場適性系統.md` (doc_spec_0041) 定義（S=1.5, A=1.2, B=1.0, C=0.85, D=0.70）；
`場景克制係數`為本月特殊加成，由後台每月設定（預設值 = 1.0）。

## 🗳 MCQ 決策記錄（Q61）

- **問題**：虎符特種兵部署冷卻回合機制
