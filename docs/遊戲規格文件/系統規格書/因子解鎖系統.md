# 因子解鎖系統規格書

> **來源文件**：逐步開鎖系統的設計.md (doc_spec_0097)、遊戲節奏：基因解鎖與視覺遮罩.md

---

## A. 系統描述

因子解鎖分三階段，對應玩家等級與武將成長的節奏。初始所有因子為灰色鎖頭（「???」），透過培育窺探、戰場覺醒、名士占卜三種途徑逐步解鎖。三階段定義：Lv1~20 只見藍粉兩色；Lv21~45 開放綠白；Lv46+ 開放紫紅。視覺遮罩層層剝離，營造「真相逐步浮現」的懸疑感。

---

## B. 系統目的

1. **新手保護**：初期只看 2 色因子，避免資訊過載
2. **漸進式懸疑**：「原來他還有這個因子！」的驚喜
3. **長線動機**：高等級解鎖紫紅因子=長期目標
4. **名士系統連動**：占卜可提前窺探未解鎖因子

## F. 公式相關

### 三階段解鎖表
| 階段 | 玩家等級 | 可見因子 | Discovery_Level |
|---|---|---|---|
| Phase 1 | Lv 1~20 | 🔵藍色、💗粉色 | 1 |
| Phase 2 | Lv 21~45 | + 🟢綠色、⚪白色 | 20 |
| Phase 3 | Lv 46+ | + 🟣紫色、🔴紅色 | 45 |

### 培育窺探機率
```
Per_Round_Peek_Chance = 0.08 × (1 + Mentor_Rank × 0.1)
// 教官等級越高，窺探機率越高
```

### 戰場覺醒觸發
```
Per_Battle_Awakening_Chance = 0.05
// 特定動作（如騎兵衝鋒→粉色騎兵因子覺醒）提升到 0.15
```

## I. Data Schema 需求

### 本機端 (Client)
```json
{
  "Gene_Visibility": {
    "Player_Level": 30,
    "Current_Phase": 2,
    "Visible_Types": ["Blue","Pink","Green","White"],
    "Locked_Types": ["Purple","Red"]
  },
  "GeneObject_Example": {
    "Type": "Green",
    "ID": "GREEN_DRAGON_04",
    "Level": 4,
    "Is_Locked": false,
    "Discovery_Level": 20
  }
}
```

// 可配置項：`Discovery_Level` 閾值由 `config/gameplay.json` 管理，示例：
// {
//   "DiscoveryThresholds": { "Phase1": 20, "Phase2": 45 }
// }
// 開發者/設計師可透過修改該檔調整可見門檻，而文件僅描述行為語義；實作需載入該 config 以同步閾值。

// 註：`Discovery_Level` 與 `Is_Locked` 的語義：
// - `Discovery_Level`：決定玩家 UI 是否能看見該因子（提示或摘要），是可見性門檻。
// - `Is_Locked`：運行時的鎖定旗標，若為 true 因子不會生效，需要覺醒動作才能解鎖。
// 兩者分離可讓設計同時支持「先看到暗示」與「稍後解鎖生效」的體驗。

### Server 端
| 欄位 | 型別 | 說明 |
|---|---|---|
| Player_ID | VARCHAR(32) PK | 玩家 |
| Player_Level | INT | 玩家等級 |
| Unlock_Phase | INT | 當前解鎖階段 1/2/3 |

## J. 名詞定義

| 名詞 | 代碼/Key | 定義 |
|---|---|---|
| Discovery_Level | Discovery_Level | 因子在 UI 上可見的等級門檻 |
| 視覺遮罩 | Visual_Mask | 未達等級時因子顯示為「???」 |

| 三階段 | Three_Phase | Lv1~20 / 21~45 / 46+ 的解鎖分層 |
| 先知陷阱 | Oracle_Trap | 名士預覽不完全準確的設計 |
