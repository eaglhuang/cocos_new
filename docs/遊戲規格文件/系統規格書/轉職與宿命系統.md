# 轉職與宿命系統規格書

> **來源文件**：轉職與宿命系統設計.md (doc_spec_0156)、傭兵職能分流與傳承機制.md (doc_spec_0134)

---

## A. 系統描述

武將在 5 歲開獎時進行第一次「宿命分流」——系統根據因子偏好建議武官或文官路線。玩家可接受或抗命。20 歲時可進行「轉職」，從單一職能轉為雙職能（如武官追加軍師副職）。65 歲退役時，退役路線由職能決定：武官→虎符→戰技教官；文官→竹簡→智識教官。宿命系統在 5 歲定下大方向，轉職系統在 20 歲提供軌道修正。

---

## B. 系統目的

1. **5 歲開獎的決策感**：天命分流=開盲盒後的首個重大選擇
2. **20 歲轉職=軌道修正**：給玩家「培養方向出錯」的補救空間
3. **退役分流=資源化**：不同路徑退役的教官有不同加成方向
4. **因子視覺化**：職能分流結合因子色彩，UI 一目了然

---

## C. 商業套路

| 商業點 | 說明 |
|---|---|
| 轉職令 | 20 歲轉職需消耗轉職令 |
| 抗命風險 | 5 歲抗命→成長率下降（但可能大器晚成） |
| 雙職能強化 | 雙職能武將稀有度更高→驅動精心培育 |

## F. 公式相關

### 5 歲宿命推薦
```
if (Blue_STR + Blue_POW > Blue_INT + Blue_POL):
    Recommend = "武官"
else:
    Recommend = "文官"
```

### 抗命懲罰
```
if (Player_Choice != Recommend):
    Growth_Rate × 0.8  // 成長率降 20%
    // 但若觸發「大器晚成」，25 歲後 Growth_Rate × 1.5
```

### 轉職條件
```
if (General.Age >= 20 && has(轉職令)):
    Add_Sub_Class(target_class)
    // 副職能力 = 主職能力 × 0.6
```

### 退役分流
| 主職能 | 退役產物 | 教官類型 |
|---|---|---|
| 武官 | 虎符 | 戰技教官（藍粉因子加成） |
| 文官 | 竹簡 | 智識教官（白綠因子加成） |
| 雙職能 | 虎符+竹簡 | 全能教官（稀有） |

## 🗳 MCQ 決策記錄（Q61）

- **問題**：虎符特種兵部署冷卻回合機制
- **衝突說明**：doc_spec_0109（虎符系統設計與商業化探討）明確提出：英靈精銳每10回合才能出一波，普通精銳每3回合可出；召喚消耗糧草，此冷卻是核心平衡閥。doc_spec_0107（虎符系統平衡與養成策略）也強調英靈虎符冷卻極長、轉蛋精銳虎符冷卻短，以此區隔兩類定位。但現行 doc_spec_0012（兵種虎符系統）完全沒有特種兵部署冷卻機制，只有星級/數值系統。若不加冷卻，英靈部隊可能被連續召喚，導致平衡崩盤；若加冷卻，需新增 TigerTally.Cooldown_Turns 欄位並更新 BattleController 邏輯。
- **裁決**：**選項 C** — 維持現有星級+數值系統，不設特種兵部署冷卻

## 🗳 MCQ 決策記錄（Q62）

- **問題**：特種軍隊召喚是否消耗糧草
- **衝突說明**：doc_spec_0109（虎符系統設計與商業化探討）提出：召喚英靈部隊瞬間扣除大量糧草，強迫玩家在「維持戰線（普通兵）」與「瞬間爆發（英靈兵）」之間取捨，並且可課金購買「點將令」刷新冷卻。doc_spec_0107 也暗示「轉蛋精銳糧草消耗中等、英靈虎符糧草消耗高」是差異化手段之一。但現行 doc_spec_0012 完全沒有糧草消耗機制，也沒有「點將令」道具定義。若採納，需在 TigerTally schema 新增 GrainCost 欄位，並在 BattleController 加入召喚扣糧邏輯。

## 🗳 MCQ 決策記錄（Q63）

- **問題**：英靈虎符vs精銳虎符商業差異化是否正式化
- **衝突說明**：doc_spec_0107 與 doc_spec_0109 均建議將虎符分為「英靈虎符（Heritage Seal）= 名將死亡轉化，有招牌技能但冷卻長/糧耗高」與「精銳虎符（Gacha Seal）= 常態數值穩定，低冷卻低糧耗」。此二元差異化是兩份文件的核心商業主張。但現行 doc_spec_0012 只有「死亡傳承虎符」和「名將專屬虎符」兩張表，兩者都走相同的星級/品質公式，沒有「英靈主動技能爆發 vs 精銳持續輸出」的明確規格。若採 A，需在 TigerTally 新增 TallyCategory（SPIRIT/ELITE）欄位並配合冷卻/糧耗欄位；若採 B，維持現有 schema 不動，但失去討論文件的核心商業亮點。
- **裁決**：**選項 B** — 維持現有統一品質體系（R/SR/SSR/UR），不做英靈/精銳概念二分
- **回寫時間**：2026-04-13 00:34

- **問題**：同陣容英靈虎符集齊羈絆加成是否正式化
- **衝突說明**：doc_spec_0107 明確提出：若同時裝備多個同陣營英靈，冷卻時間集體縮短 10%，以刺激玩家去收集完整名將陣容。doc_spec_0109 也提到「集齊五虎將或江東十二虎臣的英靈虎符」作為核心收集動力。現行 doc_spec_0012 在 F 節已定義家族共鳴（+8%）與史實羈絆（+5%），但沒有「集齊特定組合觸發 Set buff」的概念。若採 A，需要新增 SetBonusDef 資料表並在 BattleController 判定玩家當前陣容是否滿足 Set 條件；若採 B 則維持現有 schema 不變。
