## A. 系統描述與核心分層

### 兩層戰法體系

本專案的戰法系統分為 **兩個完全不同的層級**，必須嚴格區分：

| | 場景戰法（Scene Gambit） | 格子戰法（Tile Tactic） |
|---|---|---|
| **觸發時機** | 戰場外，進入戰鬥前 | 戰場內，回合中施放 |
| **發動者** | 高智力軍師（INT 門檻判定） | 任何武將（消耗 TP） |
| **影響範圍** | 整個戰場背景 + 全局 Tile-State 預設 | 指定格子（1~9 格不等） |
| **背景圖** | 專屬預繪場景（AI 繪製靜態背景） | 不改變背景，僅疊加格子特效 |
| **可動態切換？** | ❌ 不可。場景背景為靜態畫面，進場即固定 | ✅ 可。格子特效逐回合變化 |
| **設計理由** | 動態場景渲染成本過高，改用預繪+格子效果模擬 | 即時特效疊加在固定背景上，技術可行 |

> **Unity 對照**：Scene Gambit ≈ 載入不同 Scene 或切換 Skybox + TileMap preset；Tile Tactic ≈ Runtime 在 Grid 上 Instantiate 特效 Prefab。但我們的 Scene 背景是靜態圖不做動態切換，所以只用預繪方式。

## B. 場景戰法（Scene Gambit）— 戰場外觸發的特殊戰鬥

### B-1. 觸發流程

> **唯一觸發來源**：接受治理大廳的「軍事任務」。玩家接取後，進入下一場戰鬥時自動套用。

### B-2. 發動條件與判定（收斂規則）

1. **軍師 INT 門檻**：發動者 INT ≥ 門檻值。
2. **情報門檻**：對目標國家達到固定 `requiredIntelValue`。
3. **成功率公式**：$P = \min(95\%, 50\% + (INT - threshold) \times 2\%)$。
4. **環境修正**：兵種剋制為主要乘區；其餘地利採百分點加法彈性修正。

### B-3. 六種已規劃場景戰法

#### 🌲 森林埋伏戰（Forest Ambush Gambit）
| 項目 | 規格 |
|---|---|
| **場景 ID** | SCENE_FOREST_AMBUSH |
| **INT 門檻** | ≥ 75 |
| **情報門檻** | `requiredIntelValue = 45` |
| **核心玩法** | 我方開場隱身、敵方視野受限、陷阱格抓捕敵軍 |
| **Tile 預設** | 60% Stealth Tile + 20% Hazard Tile（陷阱）+ 20% Normal |
| **特殊規則** | 我方前 2 回合全員隱身；踩陷阱格：定身 1 回合 + 傷害 |

#### 🌙 夜襲作戰（Night Raid Gambit）
| 項目 | 規格 |
|---|---|
| **場景 ID** | SCENE_NIGHT_RAID |
| **INT 門檻** | ≥ 70 |
| **情報門檻** | `requiredIntelValue = 40` |
| **核心玩法** | 佔領營帳目標、夜間命中降低但暴擊提升 |
| **特殊規則** | 全局命中 -25%、暴擊 +20%；營帳為可攻擊目標物（佔領=勝利條件） |

#### 🪨 峽谷落石戰（Canyon Rockfall Gambit）
| 項目 | 規格 |
|---|---|
| **場景 ID** | SCENE_CANYON_ROCK |
| **INT 門檻** | ≥ 65 |
| **核心玩法** | 限制敵軍行進路線、落石造成高傷害 + 移動封鎖；**擊破落石可獲資源** |
| **Tile 預設** | 40% Hazard Tile (落石) + 30% Normal + 30% Force-Move |
| **擊破掉落** | 石材/木材 +100~300、隨機防具材料 +5、武將經驗 +50（金色浮動文字） |

#### 💧 水淹七軍戰（Flood Gambit）
| 項目 | 規格 |
|---|---|
| **場景 ID** | SCENE_FLOOD |
| **INT 門檻** | ≥ 80 |
| **核心玩法** | 水域擴散、敵軍被水流強制推移、非水戰適性敵軍嚴重削弱 |
| **Tile 預設** | 50% Force-Move Tile (水流) + 30% Hazard (深水) + 20% Normal |

#### 🔥 火燒偷襲戰（Fire Raid Gambit）
| 項目 | 規格 |
|---|---|
| **場景 ID** | SCENE_FIRE_RAID |
| **INT 門檻** | ≥ 75 |
| **核心玩法** | 火攻、蔓延、**掠奪敵營物資**、敵軍群聚受罰 |
| **掠奪掉落** | 糧草 +200~500、隨機武器材料 +10、教官經驗(TP) +20（金色浮動文字） |
| **特殊規則** | 我軍佔領/攻擊火焰格 = 掠奪成功；我方火抗 +50% |

#### 🏯 攻城戰（Siege Gambit）
| 項目 | 規格 |
|---|---|
| **場景 ID** | SCENE_SIEGE |
| **INT 門檻** | ≥ 60 |
| **核心玩法** | 攻破城門、器械操作、城牆地利加成 |
| **攻城器械** | 衝車（破門）、雲梯（攀爬）、投石機（AOE） |

## C. 格子戰法（Tile Tactic）— 戰場內即時施放

### C-1. 戰法分類
- **突擊類**：如騎兵衝鋒（擊退 1 格）。
- **控制類**：如離間（敵軍互鬥）、空城計（後退 2 格）。
- **環境類**：如火計、落石（即時在格子上生成 Hazard 特效）。

## E. 軍師智謀能量 HUD

- **能量積累**：每回合 $INT \times 0.5$。
- **發動特效**：能量滿 100 時，可觸發場景全局增幅（如火燒蔓延劇增）。
- **視覺**：垂直能量條由下往上填充，滿時噴發金色粒子。

## [Implementation Note | 2026-04-17] 水淹場勢視覺掛載規則（已落地）

- Flood 河道地板材質採「條件掛載」：
  - 只有 `battleTactic === BattleTactic.FloodAttack` 時，才會掛載 `water-ripple.effect`。
  - 非 Flood 戰法維持原本地板材質，不做多餘 shader 載入。
- Fallback 規則：
  - 若 ripple 載入失敗，回退至 `floodBaseFillMaterial`，避免畫面中斷。
  - 載入失敗後允許重試，避免一次失敗永久失效。
- 目的：
  - 避免所有戰法都承受水流 shader 成本。
  - 確保「發動水淹戰法」與「河道流動視覺」具一對一對應。
