# 武將戰績系統規格書

> **來源文件**：MVP 遊戲驗證規格書.md (doc_spec_0158)、三國遊戲模式串接與循環.md (doc_spec_0055)、轉蛋與戰場武將處置系統.md (doc_spec_0155)

---

## A. 系統描述

武將戰績系統負責記錄每一位武將在所有戰場中的表現數據：勝場、敗場、擊殺數、傷害點總數、守護力（被承受傷害＋治療量）。這些數據驅動武將評價、退役虎符品質、英靈牆等級、以及部分隱藏因子覺醒條件。

> **Unity 對照**：類似 Unity 的 Stats Tracker / Achievement System，透過 ScriptableObject 記錄累計戰績，並由 Observer Pattern 在每場戰鬥結束時更新數據。

---

## B. 系統目的

1. **角色價值量化**：讓玩家看到每位武將的「戰場履歷」，增強情感連結
2. **退役品質依據**：退役時虎符品質、教官加成等級依據戰績數據計算
3. **因子覺醒條件**：部分隱藏因子（如「百人斬」、「不敗神話」）需達成特定戰績門檻
4. **社交炫耀**：PvP 排行榜與好友比較的數據基礎
5. **營運數據**：後端可分析各武將的戰場表現，調整數值平衡

## F. 公式相關

### 核心戰績欄位定義

| 欄位 | 公式 | 說明 |
|---|---|---|
| 勝場 (Win) | `win_count += 1` (勝利時) | 累計勝利場數 |
| 敗場 (Lose) | `lose_count += 1` (失敗時) | 累計失敗場數 |
| 擊殺數 (Kill) | `kill_count += killed_enemies_this_battle` | 累計擊殺單位數 |
| 傷害點總數 (Damage) | `total_damage += damage_dealt_this_battle` | 累計造成的總傷害 |
| 守護力 (Guardian) | `guardian_power += damage_taken + healing_done` | 被承受傷害＋施放治療量 |

### 勝率

$$WinRate = \frac{Win}{Win + Lose} \times 100\%$$

### 退役戰績評分

```
retire_score = (win_count × 3) + (kill_count × 2) + (total_damage / 1000) + (guardian_power / 1000)

if (retire_score >= 10000): tiger_tally_grade = "SSR"
elif (retire_score >= 5000): tiger_tally_grade = "SR"
elif (retire_score >= 1000): tiger_tally_grade = "R"
else: tiger_tally_grade = "N"
```

### 里程碑門檻

| 成就 | 條件 | 獎勵 |
|---|---|---|
| 百人斬 | kill_count >= 100 | 戰法筆記 ×1 |
| 千人斬 | kill_count >= 1000 | 戰法筆記 ×3 + 專屬稱號 |
| 萬人斬 | kill_count >= 10000 | 虎符品質提升 + 英靈牆特殊動畫 |
| 不敗神話 | win_count >= 50 且 lose_count == 0 | 隱藏白色因子「不敗」 |
| 守護之壁 | guardian_power >= 50000 | 隱藏白色因子「鐵壁」 |
| 修羅 | total_damage >= 100000 | 隱藏紅色因子「殺意」 |

## I. Data Schema 需求

### 本機端 (Client)

```json
{
  "BattleRecord": {
    "UID": "string (武將 UID)",
    "Win_Count": "number",
    "Lose_Count": "number",
    "Kill_Count": "number",
    "Total_Damage": "number",
    "Guardian_Power": "number",
    "Milestones": ["string (已達成成就 ID)"],
    "Season_Stats": {
      "Season_ID": "string",
      "Season_Win": "number",
      "Season_Lose": "number",
      "Season_Kill": "number",
      "Season_Damage": "number",
      "Season_Guardian": "number"
    }
  }
}
```

```json
{
  "BattleSnapshot": {
    "Battle_ID": "string",
    "General_UID": "string",
    "Damage_Dealt": "number",
    "Damage_Taken": "number",
    "Healing_Done": "number",
    "Kills": "number",
    "Is_Winner": "boolean",
    "Battle_Date": "string (遊戲內季度)"
  }
}
```

### Server 端

| 欄位 | 型別 | 說明 |
|---|---|---|
| record_id | VARCHAR(32) PK | 戰績記錄 ID |
| player_id | VARCHAR(32) FK | 玩家 |
| general_uid | VARCHAR(32) FK | 武將 UID |
| win_count | INT | 累計勝場 |
| lose_count | INT | 累計敗場 |
| kill_count | INT | 累計擊殺 |
| total_damage | BIGINT | 累計傷害 |
| guardian_power | BIGINT | 累計守護力 |
| season_id | VARCHAR(16) | 當前賽季 ID |
| updated_at | DATETIME | 最後更新時間 |

## J. 名詞定義

| 名詞 | 定義 |
|---|---|
| 勝場 | 武將參與的戰鬥中，我方獲勝的場次累計 |
| 敗場 | 武將參與的戰鬥中，我方失敗的場次累計 |
| 擊殺數 | 武將在所有戰鬥中消滅的敵方單位數量累計 |
| 傷害點總數 | 武將在所有戰鬥中造成的傷害數值累計 |
| 守護力 | 武將承受的傷害總量＋施放的治療總量，衡量坦克/治療型角色的貢獻 |
| 戰績評分 | 綜合勝場、擊殺、傷害、守護力的加權分數，影響退役虎符品質 |
| 里程碑 | 戰績達到特定門檻時解鎖的成就，可觸發獎勵與隱藏因子覺醒 |
| 戰報快照 | 單場戰鬥的詳細數據記錄 |
| 賽季戰績 | 按賽季週期獨立統計的戰績，賽季結束時結算排名獎勵 |
