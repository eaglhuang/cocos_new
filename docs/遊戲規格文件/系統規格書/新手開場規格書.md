# 新手開場規格書

## 1. 文件目的

本文件整合 `docs/新手開場/` 目錄下所有內容，將分散的構想整理成單一、可執行、可交付的開場規格。

目標不是只寫劇情，而是把以下項目一次定義清楚：

- 開場要做什麼
- 每一步的分鏡與節奏
- 對話內容與台詞語氣
- 必須準備的功能與資料結構
- 角色規則、因子規則、血統規則
- 商業化流程與付費套路

## 2. 開場核心概念

這個新手開場的核心不是單純教學，而是用「情懷、血統、名將、紅顏、商業化」五條線一起拉住玩家。

本輪世界觀重定位後，正式主張調整為：

- 懷舊只是入口，不是產品主體
- 祖靈命紋、命紋靈獸、家族血脈才是玩家真正要記住的核心
- 開場的任務不是單純喚醒老三國情懷，而是讓玩家在 15 分鐘內明白「這不是一般三國抽卡，而是會留下血脈、英靈與家族傳承的三國世界」

### 2.1 英靈夢回開場原則

> **整併來源**：遊戲開場設計：傳承與養成.md
> **衝突處理**：本章是對現行新手開場的導入方式補強，不把主軸改回「直接扮演歷史君主」。

- 開場先讓玩家短暫操作一位傳說級英靈，完成一場高完成度教學戰，目的在於先展示完全體武將、棋盤戰術與高級特效的「終點樣貌」。
- 教學戰後，英靈不直接變成永久主角，而是轉化為玩家持有的「初始虎符 / 初始英靈記憶」，帶著少量基因與傳承期待進入本篇。
- 本篇仍維持玩家自立門戶、資源拮据、靠培育與傳承變強的正式方向，避免歷史君主開局過強而沖淡養成價值。
- 若玩家前期勢力受挫，不採強迫重開，而以時代更迭、主城保底與後續培育逆轉作為正式挽救路徑。

### 核心主軸

- 先用老玩家熟悉的像素《三國志》記憶當錨點
- 再用復古漫畫與 Win95 信件打破第四面牆
- 接著讓玩家抽到初始名將，建立「宿命感」
- 之後用血統與因子系統讓玩家理解養成深度
- 最後以紅顏、趕路倒數、加價購，推進第一次與第二次付費

### 2.2 三層世界語言

- `懷舊入口層`
  - 像素回憶、CRT、Win95、黑場古文
  - 目的：讓 30~55 歲三國情懷玩家快速入戲
- `祖靈命紋主體層`
  - 14 人祖先矩陣、六色因子、祖紋命篆、命紋靈獸、血脈共鳴
  - 目的：建立《三國傳承》真正的差異化記憶點
- `高級商品層`
  - 名將立繪、紅顏立繪、宿命轉蛋、稀有度包裝、卡面演出
  - 目的：承接收藏感、付費動機與視覺獎勵

### 2.3 開場視覺母語原則

- 不走重科技基因實驗室風格，避免把世界觀帶成科幻基因工程
- 不走重奇幻神魔風格，避免把三國辨識沖淡成 generic 東方幻想
- 正式採用 `祖紋命篆` 作為血脈視覺語言：
  - 來源感應接近甲骨、金文、小篆、帛書殘筆、玉刻痕
  - 感覺要高級、古老、神祕，但不可恐怖
- 命紋靈獸作為每位名將古老血脈的象徵，不是賣萌寵物，而是祖靈圖騰
- UI 不整套推翻重做，只允許在既有骨架上微調色調、紋樣、提示層與 badge 元素

### 風格關鍵字

- 復古漫畫
- Win95 視窗
- 8-bit 像素回憶
- 高畫質轉場
- 宿命轉蛋
- 血統因子
- 現代高級時尚紅顏
- 倒數計時與留存誘因

## 3. 完整流程總覽

### 0 到 15 分鐘總節奏

1. 0 到 3 分鐘：情懷定錨與信件開場
2. 3 到 5 分鐘：初始名將無限抽與第一次破冰付費
3. 5 到 12 分鐘：首戰教學與因子解鎖
4. 12 到 15 分鐘：紅顏系統、血統預覽與第二次破冰付費

### 玩家情緒曲線

- 先共鳴
- 再驚喜
- 再爽感
- 再理解深度
- 再產生擁有欲
- 最後進入留存與付費循環

## 4. 分鏡腳本

### 第一幕：情懷定錨

#### 分鏡 1：黑場與古文

- 畫面：黑底，暗金文字慢慢浮現
- 氣氛：古老、莊嚴、帶有宿命感
- 音效：低沉古琴或環境音
- 功能：建立情懷與時代感
- 補充：首批浮現的不只是古文旁白，還要出現片段 `祖紋命篆`、殘缺血脈印記與祖靈名號，讓玩家從第一秒就知道這個世界與「傳承」有關

#### 分鏡 2：現代書房與 Win95 信件

- 畫面：現代書房、CRT 螢幕、老電腦桌面
- 畫面元素：信封圖示、藍色視窗、灰色按鈕
- 互動：玩家點擊信件內容或按鈕
- 功能：讓玩家明確知道這不是普通開場，而是被召喚進入「時空計畫」
- 補充：信件附件內可夾帶簡化版 `家族血統觀測圖`、`因子終端快照` 或 `祖先矩陣片段`，讓 Win95 不是單純復古梗，而是血脈觀測終端

#### 分鏡 3：懷舊像素回憶

- 畫面：老玩家熟悉的像素地圖與六大諸侯頭像
- 作用：作為記憶錨點
- 情緒：讓老玩家瞬間認出來，產生「這不是我當年玩過的那個嗎」的共鳴
- 補充：像素畫面要逐步被六色因子光痕與祖紋命篆侵入、撕裂、接管，表示「舊三國記憶正在被重鑄為傳承世界」

#### 分鏡 4：時空震盪與旋渦

- 畫面：像素畫面中心出現白點，逐漸擴散成漩渦
- 演出：舊畫面被吸入，轉蛋盤由遠至近飛出
- 作用：把「舊靈魂」與「新軀體」連起來
- 補充：漩渦不是 generic portal，而是由祖紋命篆、血脈連線與六色因子重組成的「命盤裂口」

### 第二幕：宿命名將抽取

#### 分鏡 5：轉蛋機啟動

- 畫面：青銅與科技藍光混合的抽卡介面
- 功能：支援無限重置十連抽
- 核心：玩家可以反覆洗牌，直到抽到想要的那位名將

#### 分鏡 6：名將逐一登場

- 畫面：六位初始名將的高辨識立繪或 2D 骨架動畫
- 重點：每位武將都要有一句能擊中玩家記憶的台詞
- 功能：讓玩家對選定角色產生擁有欲

### 第三幕：首戰教學與因子解鎖

#### 分鏡 7：進入戰場

- 畫面：5 路 x 8 格深度的小棋盤
- 功能：讓玩家理解部署、推進、對推規則
- 教學：先以最簡單、最直接的方式引導戰鬥

#### 分鏡 8：因子解鎖特效

- 畫面：武將觸發因子時，出現文字爆發、圖紋亮起、色彩升級
- 功能：把抽象的「血統」具體化
- 作用：玩家知道因子不是隱藏參數，而是會直接改變戰鬥表現的核心資源
- 補充：亮起的圖紋應優先使用 `祖紋命篆` 與命紋靈獸對應紋樣，不用 generic 科技六角形或 sci-fi HUD 線條

### 第四幕：紅顏與血統定錨

#### 分鏡 9：紅顏登場

- 畫面：與名將轉蛋區隔的另一個池子
- 風格：柔和紫色、粉色、金色數據流
- 功能：讓玩家明白這一池不是戰力主體，而是血統與後代價值的入口

#### 分鏡 10：血統預覽與倒數

- 畫面：14 人祖先矩陣、因子雷達圖、預測子嗣結果
- 功能：展示配種與傳承的深度
- 留存：若角色還在趕路，主介面顯示倒數圖示
- 補充：14 人祖先圖與因子圖的版型可參考 `血統UI示意圖.png`、`14人血統示意圖.jpg` 的資訊分區與展開層級，但不沿用其科幻材質與特效

## 5. 劇情對話

### 5.1 信件正文

建議信件採用「現代與古代碰撞」的語氣，像是來自時空終端的召喚。

#### 信件內容主軸

- 問玩家是不是又在回味三國舊作
- 告訴玩家遺憾可以被改寫
- 提供一個夢境觀測位與全息頭盔
- 召喚幾位脾氣很硬、但願意跟隨玩家的宿命之將
- 最後用一句正式按鈕結尾：開始宿命轉蛋

### 5.2 初始名將登場台詞

#### 張飛

- 「主公！俺老張等這封信等得嗓子都乾了！」
- 語氣要粗獷、直接、帶豪氣

#### 孫策

- 「江東的風吹到了現代，還是這麼燙人。」
- 語氣要年輕、鋒利、帶王者感

#### 甘寧

- 「這長江水，往後就歸主公管，我負責開路！」
- 語氣要有游俠與衝鋒感

#### 夏侯惇

- 「雙眼尚在，戰魂未冷。」
- 語氣要穩、硬、忠誠

#### 魏延

- 「他們都說我有反骨，只有你願意給我這封信。」
- 語氣要孤傲、危險、但有被接納感

#### 張郃

- 「在這渾沌的亂世中，唯有主公的選擇，才配得上我這華麗的謝幕。」
- 語氣要優雅、精準、帶名將自尊

### 5.3 教學期台詞

- 水鏡先生或左慈負責導入系統規則
- 文字重點是：這些名將的因子會被封印，必須透過戰鬥逐步解鎖
- 需要讓玩家理解：血統不是裝飾，是戰力成長的本體

### 5.4 紅顏趕路與加價購台詞

#### 0.99 小額破冰

- 「妾身若搭馬車需三天，若有 0.99 盤纏，即刻便到。」
- 重點是把小額付費寫成「加速與便利」

#### 29.99 大包裝

- 「只需旅費 29.99，美人與文官可一同加入，文武齊備護主公。」
- 重點是把高價商品寫成完整陣營與完整體驗

#### 倒數留存

- 「主公，路途遙遠，妾身正在趕來。」
- 重點是讓玩家在主畫面看見倒數，形成持續期待

- 必須能作為教學期主戰角色使用
- 必須能直接連結後續的血統與因子系統

## 7. 因子與血統規則

### 7.1 因子規則

- 每位名將都要有一個最強主因子
- 主因子必須是固定且可視覺化的
- 其餘因子可由系統逐步解鎖或繼承

### 7.2 14 人祖先規則

- 初始名將會延伸出 14 位虛擬祖先作為血統矩陣
- 祖先必須遵守地域限制，不能亂生
- 例如：
  - 張飛的祖先地域應偏北境與幽平
  - 孫策的祖先地域應偏吳越與江淮
- 祖先因子可以由系統在地域人才庫中抽選

### 7.3 血統解鎖順序

1. 先顯示角色主因子
2. 再顯示次級因子
3. 之後開啟祖先矩陣
4. 最後讓玩家看到子嗣預測結果

### 7.4 視覺化需求

- 需要一個六色六角圖或同等可視化面板
- 需要祖先樹或矩陣視圖
- 需要因子變異預測圖
- 需要血統強弱與成長傾向的明顯視覺提示
- 六角圖不應只是通用雷達圖，應優先往 `命盤 / 家譜印輪` 感靠攏
- 祖先矩陣不應只是表格，應有 `祠堂族譜 / 血脈樹卷 / 命紋拓印` 的閱讀感
- 子嗣預測不只顯示數值，還要能看出父母系譜、因子流向、混種可能與覺醒名號預兆

## 8. 紅顏與支援卡規格

### 8.1 角色池分層

- 史實女將：馬雲騄、祝融夫人、呂玲綺、王異
- 史實紅顏：貂蟬、甄姬、小喬、二喬、蔡文姬、步練師
- 幻境紅顏：嫦娥、洛神、西王母、女媧等延伸型角色

### 8.2 三位核心紅顏草圖

#### 貂蟬

- 風格：黑天鵝與漢服元素融合
- 定位：運氣與智謀定錨
- 核心感受：高級、神秘、不可褻瀆

#### 甄姬

- 風格：冰雪奇緣與魏晉風骨融合
- 定位：體魄與才情定錨
- 核心感受：高冷、純淨、才情感強

#### 小喬

- 風格：龐克與吳越水鄉融合
- 定位：爆發與暴擊定錨
- 核心感受：靈動、叛逆、帶衝擊力

### 8.3 紅顏功能定位

- 紅顏不是單純美術角色，而是血統與後代品質的定錨器
- 玩家看到她們時，要直接感受到稀有度與基因價值
- 紅顏同時承擔抽卡、留存與第二次付費的入口

## 10. 商業化套路

### 10.1 第一道商業化：無限重置十連抽

- 先讓玩家免費洗牌，直到抽到想要的初始名將
- 這會把選角變成擁有欲建立過程
- 玩家不是在抽卡，是在挑命運

### 10.2 第二道商業化：29.99 全明星包

- 玩家選定名將後，立刻推薦整套同陣營文武包
- 目標是課長與重度玩家
- 語氣要大、完整、誇張，但不能太硬推

### 10.3 第三道商業化：0.99 破冰包

- 若玩家拒絕高價包，立即退而求其次
- 用極低門檻的價格打破第一次付費防線
- 核心是讓玩家感覺「不買可惜」

### 10.4 第四道商業化：倒數計時

- 支援紅顏或文官不是立刻到，而是顯示趕路中
- 主介面長期掛倒數
- 讓玩家產生等待與回流

### 10.5 第五道商業化：第二次情感付費

- 當玩家理解血統與後代後，再推出配種與預測功能的擴充商品
- 這時付費理由不是強度，而是完整性與未來性

## 11. 介面與演出規則

### 11.1 介面規則

- 信件要像 Win95，但背景可以帶水墨或懷舊煙雲
- 轉蛋區與紅顏區要明確區隔
- 因子與血統畫面要避免太花，資訊要一眼看懂
- 倒數要放在玩家每天都能看到的位置

### 11.2 演出規則

- 每一次轉場都要有明顯節奏變化
- 像素到高畫質的反差要大
- 名將登場台詞要短、準、狠
- 紅顏台詞要有高級感與誘惑感
- 因子解鎖要像角色真的成長了

- 用情懷吸引

- 用因子建立深度理解

4. 最後補紅顏趕路與商業化彈窗

## 16. UI 規格書

本節只定義畫面、演出、互動與狀態，不包含資料欄位與程式模組實作。

### 16.1 UI 設計目標

- 建立強烈的情懷錨點
- 讓玩家明確感受到「進入夢境」的過場
- 讓名將、紅顏、血統、商業化入口在視覺上完全分層
- 讓新手在前 15 分鐘內看懂自己正在做什麼

### 16.2 主要畫面模組

#### 開場序列

- 黑場古文畫面
- Win95 信件畫面
- 像素三國回憶畫面
- 漩渦吸入轉場畫面
- 宿命轉蛋主畫面

#### 角色序列

- 初始名將展示卡
- 名將登場對話框
- 紅顏角色卡
- 趕路中的紅顏提示圖示
- 文官或支援角色提示卡

#### 系統序列

- 因子六角圖面板
- 祖先矩陣面板
- 子嗣預測面板
- 血統成長提示面板
- 因子解鎖彈出視窗

#### 商業序列

- 0.99 小額破冰彈窗
- 29.99 全明星包彈窗
- 趕路倒數提醒視窗
- 主畫面常駐倒數圖示
- 重新抽取與確認彈窗

#### 戰鬥序列

- 5 路 x 8 格戰場主畫面
- 單位部署提示
- 因子觸發字卡
- 勝敗結算畫面
- 教學提示浮窗

### 16.3 畫面互動規則

- 信件畫面必須先引導點擊，再進入下一幕
- 像素畫面與高畫質畫面之間必須有明顯轉場
- 名將與紅顏不得在同一個主介面混雜顯示
- 因子與血統畫面需要讓數值資訊一眼可讀
- 倒數圖示必須常駐在玩家可見區域

### 16.4 UI 狀態規則

- 初始：黑場與信件
- 召喚：轉蛋與名將展示
- 教學：戰鬥與因子提示
- 深化：血統與祖先矩陣
- 留存：紅顏倒數與商業入口

### 16.5 UI 驗收重點

- 每個畫面是否有明確的進場與退場
- 文字是否能在手機上清楚閱讀
- 轉場是否足夠有節奏感
- 商業化視窗是否不會遮蔽核心教學
- 主畫面常駐資訊是否足夠簡潔

## 17. 程式規格書

本節只定義資料結構、控制器責任、流程銜接與實作邏輯，不重複描述畫面細節。

### 17.1 系統模組責任

#### OpeningFlowController

- 控制開場各階段順序
- 管理信件、漫畫、轉場、轉蛋、教學的串接
- 控制每一幕的進出條件

#### GachaManager

- 控制無限重置十連抽
- 控制初始名將與紅顏抽取池
- 控制保底與確認邏輯

#### CountdownManager

- 管理紅顏與支援角色的趕路倒數
- 管理主畫面常駐提示
- 管理加速與完成狀態

#### UI_BloodlineTree

- 呈現祖先矩陣
- 顯示因子繼承與血統關聯
- 支援子嗣預測顯示

#### AttributePanel

- 呈現名將屬性與因子
- 顯示主因子、次級因子與解鎖狀態
- 支援六角圖或等效圖表

### 17.2 靜態資料結構

- ancestorCount
- allowedRegions
- factorSlots
- mutationRules

### 17.3 程式流程規則

- 開場流程應採事件驅動，但每個事件都要可回溯
- 抽卡流程應支援重置與鎖定
- 因子與血統流程應以資料驅動，不硬編成 UI 文本
- 倒數流程應支援離線時間累積
- 所有商業入口都應從同一套配置表讀取

### 17.4 程式驗證重點

- 新手流程是否能在未登入資料情況下先顯示預設內容
- 抽卡重置後是否會正確回到初始狀態
- 因子預覽是否能正確對應角色資料
- 倒數完成後是否能正確切換為已到達狀態
- 商業化入口是否與流程狀態一致
