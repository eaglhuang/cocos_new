# 育成與戰鬥公式基準手冊

> **文檔定位**：本手冊萃取自《賽馬娘》的頂級數值公式設計，並將其本土化為《3KLife》專案的三國語境。此手冊作為「培育系統」、「戰鬥系統」與「資料中心教官卡設計」的**底層數學與定義參考基準**。

## 第二章：培育系統 (36回合) 的底層數學模型

單次訓練的屬性增長是由精密的乘區構成，這是確保平民玩家與課長玩家之間能夠透過運氣和配卡策略抗衡的關鍵。

### 2.1 屬性增長總公式

$$ \text{獲得屬性值} = \left( Base + \sum Bonus_{Card} \right) \times \left( 1 + Gr_{General} \right) \times M_{Morale} \times M_{Training} \times M_{Friendship} \times M_{Crowd} $$

*   **基礎值 (Base)**：設施等級固定給予的數值。例如 Lv5 騎術營固定給 15 STR。
*   **教官加值 ($Bonus_{Card}$)**：教官卡自帶的固定加值。**（加法區）**
*   **武將資質 ($Gr_{General}$)**：武將本體的成長率（如：趙雲武力成長率 +20%）。
*   **運勢乘區 ($M_{Morale}$)**：
    *   $M_{Morale} = 1 + (\text{目前基礎運勢係數} \times (1 + \sum \text{教官運勢效果提升}))$
    *   *註：預設最佳運勢為 0.2 (20%加成)。*
*   **陪伴效果乘區 ($M_{Training}$)**：
    *   $M_{Training} = 1 + \sum \text{教官陪伴效果提升}$
    *   *註：只要教官出現即可發動，全教官疊加（**加法後相乘**）。*
*   **傾囊相授乘區 ($M_{Friendship}$ - 核心爆發區)**：
    *   $M_{Friendship} = \prod (1 + \text{觸發傾囊相授的教官專屬加成})$
    *   *註：好感度滿 80 且出現在專精設施時觸發。**這是全公式唯一的「連乘」區**！是創造神仙面板的極限來源。*
*   **人數補正 ($M_{Crowd}$)**：每多一位未觸發傾囊的閒雜角色在同一格，額外 +5%。

### 2.2 疲勞度與失敗率曲線

體力降低會指數型增加失敗率，迫使玩家進行「休息」的決策。
*   **安全區**：體力 ≥ 70% 時，失敗率 0%。
*   **風險區**：體力介於 30% ~ 70% 時，失敗率由 5% 逐漸爬升至 35%。
*   **危險區**：體力 < 30% 時，失敗率 40% ~ 99%。

## 第三章：教官卡 (Mentor Card) 詞條定義與 Schema

在 `Data Schema` 中，一張教官卡必須具備以下屬性詞條來支撐上述公式。

### 3.1 教官卡核心詞條 JSON 範例

```json
{
  "Mentor_ID": "MENTOR_GUANYU_SSR",
  "Specialty_Facility": "CAVALRY_CAMP", // 專精兵營 (對應得意率)
  "Base_Stats": {
    "Teaching_Rate": 65,            // 教學頻率 (原: 得意率)
    "Starting_Bond": 30,            // 初始好感度 (加速進入傾囊相授)
    "Mentor_Burst_Bonus": 0.25,     // 傾囊相授加成 (25%，進入連乘區)
    "Presence_Bonus": 0.10,         // 陪伴效益 (原: 訓練效果提升，全局+10%)
    "Morale_Bonus": 0.30,           // 運勢效果提升 (放大好心情加成)
    "Stat_Bonus": {                 // 固定點數屬性加成
      "STR": 1,
      "LEA": 1
    },
    "Stamina_Discount": 0.15,       // 體力消耗減免 15%
    "Hint_Rate": 40,                // 戰法啟發發生率 (+40%)
    "Hint_Discount_Level": 3        // 戰法啟發折扣等級 (大幅減少 TP 消耗)
  }
}
```

### 3.2 詞條強度的真實意義
*   **得意率的陷阱與必須性**：數值為 65 並不代表 65% 出現，而是 `(基礎出現率 * 1.65) / 總權重`。低得意率的卡片即使爆發高，也難以與其他卡碰在一起形成「連乘」。
*   **初期羈絆的節奏感**：高起始羈絆等於變相節省了前 10 回合無效的「追著教官跑」的動作，是評價卡片節奏的頂級指標。

## 第四章：戰鬥底層計算與技能發電機制

將賽制對抗的概念帶入戰鬥邏輯中，賦予五維屬性更深層次的戰場意義。

### 4.1 智力 (INT) 檢核與戰法發動率

戰法（技能）不應 100% 發動，必須由智力決定其穩定性。
*   **發動率公式**：$$\text{戰法發動率} = 1 - \left( \frac{9000}{\text{INT (智力)}} \right) \%$$
*   *閾值參考*：
    *   INT 300 $\approx$ 70% 發動率
    *   INT 600 $\approx$ 85% 發動率
    *   INT 900 $\approx$ 90% 發動率
    *   INT 1200 $\approx$ 92.5% 發動率 (軟上限)

### 4.2 殘局爆發 (Last Spurt / 破釜沉舟)

取代賽馬娘的「終盤衝刺」，我們引入「破釜沉舟」機制。
*   **機制**：當戰鬥進入特定回合（如第 5 回合），系統解除常規限速。
*   **物理極速運算**：武將的**「目標爆發力」將轉換為：$基礎傷害 \times \sqrt{\frac{STR}{500}} \times 兵種終盤係數$**。
*   **加速度 (Acceleration)**：極速再高，若無法快速達到也無用。**加速度 $\propto$ 統率(LEA) 或 勇武(STR) 的平方根**。戰鬥中切換狀態或受到減速技能後，需要依靠此數值重新爬升到最高火力。
*   *屬性衰減設計 (Soft-Cap)*：借鏡超過 1200 屬性後收益折半的設定，當 $STR > 1200$ 時，溢出部分的平方根係數應額外乘上 $0.5$ 進行衰減，避免數值無限制崩壞。

### 4.3 體力 (HP/Stamina) 與根性 (Guts) 的燃燒模型

戰場上的續航不應該只看防禦力，而是導入「體力水池與燃燒率」的動態模型。
*   **體力上限公式**：$\text{總體力庫} = \text{基礎體能} + (\text{體力屬性(VIT)} \times 0.8 \times \text{兵種消耗率})$
*   **體力燃燒率**：當武將處於「破釜沉舟」的極速狀態時，體力會急遽燃燒。公式參考：$\text{每秒消耗} \propto (\text{當前速度} - \text{基準速度} + C)^2$。出力越狠，掉血越快。
*   **意志/根性 (Guts) 的作用**：在終盤衝刺時，高意志力能作為燃燒減免的護盾。
    *   *減免公式參考*：$\text{損耗減免係數} = 1 + \frac{200}{\sqrt{600 \times \text{WILL(意志)}}}$
    *   *意義*：在殘局雙方彈盡糧絕時，高意志力的武將能用比別人更少的體力，維持更久的極速狀態（即賽馬娘中的「根性壓勝」）。

## 第五章：血脈傳承 (因子繼承) 的機率模型

因子的優異程度與出現機率，直接與鏡像培育畢業時的「面板評價」掛鉤。

### 5.1 藍色因子（基礎屬性）機率
*   **數值 < 600 (B級以下)**：獲得 3 星藍色因子機率為 **0%**。
*   **600 ≤ 數值 < 1100 (B ~ S+)**：獲得 3 星藍色因子機率為 **6%**。
*   **數值 ≥ 1100 (SS 級以上)**：獲得 3 星藍色因子機率為 **10%**。
*   *設計意圖*：強制玩家必須將該項屬性拉到極限，才有資格生產出頂級種馬，創造了明確的重複遊玩循環（Loop）。

### 5.2 世代相性 (Compatibility)
*   **相性計算**：歷史淵源 (史實契合度) + 共同出戰次數 + 家族關聯。
*   **相性作用**：相性越高（如 ◎ 級），在結緣生育或培育中途傳承時，**繼承到白色戰法因子與紅色適性因子**的機率就越高。藍色因子固定給，但白/紅因子是靠相性拚機率。

## 第六章：教官卡 (支援卡) 種類分類與生產矩陣

為了應對長線的卡池營收營運，我們借鑑賽馬娘將教官卡做深度的原型（Archetype）分類。每一種分類都有其在培育公式中的獨特干預點，以產生不可取代的化學反應。

### 6.1 教官卡四大核心設計原型 (Archetypes)

| 類型分類 | 設計邏輯與公式干預點 | 3KLife 情境範例 | 營運與課金價值 |
| :--- | :--- | :--- | :--- |
| **數值炮台型**<br>(Stat Sticks) | **高教官教學頻率 (得意率) + 極高傾囊相授加成 (爆發%)**。<br>不依賴戰法啟發，唯一目的就是堆出破千的單項極限屬性。 | **武力教官・呂布**：只要他在騎術營，爆發就能帶來 +40 STR。 | 開啟新時代數值上限的剛需卡。 |
| **戰法大核型**<br>(Skill Banks) | **高戰法啟發率 + 戰法 TP 折扣滿等**。<br>本身不提供高屬性，但自帶版本強勢的「金戰法」，且能大幅降低學習成本。 | **策略教官・郭嘉**：提供【短兵相接・極】絕版戰法，且折扣達到 60%。 | 提供戰鬥解題能力、豐富戰術流派構建。 |
| **資源調度型**<br>(友人/幕僚卡) | **無專精設施，大幅降低體力損耗 + 定期回復運勢**。<br>這類卡無法觸發傾囊相授，而是保證你在 36 回合中「幾乎不用浪費回合休息」，保持高效訓練。 | **後勤教官・荀彧**：任何訓練只要有他同行，體力消耗 -25%，且固定回復士氣。 | 高端培育中不可或缺的節奏潤滑劑。 |
| **集團體系型**<br>(團體卡) | **多角色共綁、提供特有的「連動機制」**。<br>必須在特定條件/任務解鎖後，提供全局常駐光環，是特定育成劇本的最佳解。 | **桃園結義卡**：提供「情義計量表」，集滿後下 3 回合全訓練大成功率 100%。 | 針對特定劇本 (Scenario) 推出的特化人權卡。 |

### 6.2 技能／戰法組合的生產分類 (Skill Synergy Matrix)

教官卡除了提供數值，更重要的是「餵給武將哪種技能」。我們將戰鬥技能拆解為以下生產類別，供企劃後續填表：

1.  **條件觸發類 (限制型強力技)**
    *   *機制*：數值收益極大，但發動條件極度嚴苛。
    *   *範例*：【夜戰特化】(僅限夜晚天氣發動)、【先發制人】(僅限我軍先手時發動)。
2.  **屬性閾值類 (滾雪球型)**
    *   *機制*：進入戰鬥時，若某項屬性碾壓對方，則額外觸發增益。
    *   *範例*：【智力壓制】(若 INT 差距 > 200，敵人戰法發動率強制減半)。
3.  **動力學修正類 (物理速度系)**
    *   *機制*：這類技能**絕對不能在錯誤階段發動**（如在慢速區開上限技）。分為「加強初速（突圍）」、「加強加速度（變道）」、「加強極速上限（收割）」。
    *   *範例*：【長驅直入】(進入第5回合的當下，瞬間賦予加速度 buff，形成完美 Connection)。
    *   *範例*：【釜底抽薪】(強制扣除對方全體 200 點隱藏體力庫，讓高速度武將因缺血而提前失速)。

## 第七章：5x8 戰場格子策略映射與生產矩陣

為了讓賽馬娘的「數值力學」在 5x8 格子玩法中產生策略深度，我們必須將這些抽象的公式轉化為空間、位移與格子狀態的具體聯動。

### 7.1 四大技能原型之格子策略映射 (Grid Strategy Mapping)

| 技能原型 | 對應 5x8 格子玩法 (Mechanics) | 3KLife 定向設計範例 |
| :--- | :--- | :--- |
| **條件引爆類**<br>(Environment Trigger) | **地利與天時的絕對佔領**。<br>技能發動不僅看機率，更看武將站位的「Tile State」。 | **【陷陣之志】**：當武將位於「林地」或「山地」格子且敵軍在「平原」時，無視防禦發動突擊。 |
| **屬性碾壓類**<br>(Stat Dominance) | **區域控制 (ZOC) 與智力反制**。<br>利用屬性差距改變格子的「進入成本 (Move Cost)」。 | **【空城計】**：若發動者 INT > 敵方 300，周圍 2 格範圍內所有武將該回合無法執行「移動」指令。 |
| **動力學/極速系**<br>(Mobility Matrix) | **行動力 (AP) 爆發與空間跳躍**。<br>將「加速度」轉化為跨越格子的瞬間位移能力。 | **【神駒突圍】**：第 5 回合進入殘局時，AP 消耗歸零且移動距離 +3。當前路徑若為直線，傷害額外加成。 |
| **體力掠奪/破壞類**<br>(Attrition War) | **補給切斷與格簇腐蝕**。<br>利用格子範圍效果 (AOE) 進行體力扣除。 | **【斷其糧草】**：選定 3x3 區域，區域內敵軍每移動 1 格額外損耗 20% 體力，直至其進入「疲勞減速」狀態。 |

### 7.2 5x8 戰場生產功能矩陣 (Production Item List)

為了支撐上述玩法，程式與企劃需產出以下基礎功能組件：

#### 1. 格子狀態矩陣 (Tile State Matrix)
*   **[P0] 持續性狀態 (Buff/Debuff Grid)**：支援在特定網格上掛載「火海」、「陷阱」、「聖域」等狀態，用於配合「條件引爆」類技能。
*   **[P0] 動態阻擋 (ZOC)**：依據武將屬性（STR 壓制）動態產生的「不可通行」或「高移動成本」區域。

#### 2. 分段戰鬥控制器 (Phase Controller)
*   **[P0] 殘局判斷器**：負責在第 5 回合（或場上存活人數 < 50%）時切換為 `Last_Spurt_Mode`，更改屬性計算權重與 AP 消耗規則。
*   **[P1] 連接 (Connection) 監視器**：偵測特定技能是否在「進入殘局前 1 回合」發動，若吻合則給予數值累積加成。

#### 3. 體力與疲勞反饋系統 (Stamina UI)
*   **[P1] 隱藏體力計量表**：武將血條下方應有隱藏的「精力/體力條」，用於顯示「破釜沉舟」後的燃燒進度。
*   **[P2] 疲勞視覺效果**：當體力歸零後，武將在格子上的移動動畫應切換為「喘氣/緩慢」狀態，並強制變灰不可執行大招。

#### 4. 屬性對抗引擎 (Stat Matcher)
*   **[P0] 二次冪平方根解算器**：實現 $Base \times \sqrt{Value}$ 的戰鬥解算函數，並內建 1200 點後的 Soft-Cap 衰減邏輯。
*   **[P0] 智力檢核器**：所有戰法發動時必須 call 此函式驗證發動成功率，並支援由教官卡提供的 `Success_Bonus`。
