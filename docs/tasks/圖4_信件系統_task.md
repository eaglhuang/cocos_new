# 任務卡 — 圖4：信件系統（草案）

摘要：信件系統需以淺色底作為閱讀主體，左側列表與右側內文需在互動與閱讀性上分工明確。

改善項目：
- 信件列表（左側）使用羊皮紙框，列表 row 選中態底色加深 `#DDD4C2`
- 分類 Tab（左側垂直）選中時加左側金色指示條；idle 為深色
- 信件內文（右側 body）保留淺底與合理行距（line-height >= 1.4），文字色深褐
- 刪除 / 全部刪除按鈕需明確二次確認與錯誤狀態回報

驗收條件：
- 長信件在淺底上段落間距、對齊正常，無字體被裁切或行高異常
- 刪除操作有 confirm 流程，且 revoked / undone 不會造成資料一致性問題（工程驗證）

交付物：
- 列表 row sprite / skin JSON
- 內文版面樣式範例（長文測試）
- 測試截圖（選中 / 非選中 / 刪除確認）

預估工時：UI 美術 0.5~1 日；工程整合 0.5~1 日

負責人（建議）：UI 美術 / UI 工程 / QA
