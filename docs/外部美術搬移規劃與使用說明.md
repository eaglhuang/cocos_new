## 0. 匯入管理原則

本文件所有從 Cyberpunk 專案搬入的素材，**必須遵循本專案已建立的「VFX Core 特效積木庫」管理規範**：

1. **拆解為最小可共用單位（積木）** — 不搬入完整 Prefab，而是將貼圖、Shader、Mesh 拆開歸檔
2. **依視覺功能分類** — 放入 `vfx_core/textures/{分類}/`（glow / fire / smoke / impact / trails 等）
3. **統一命名** — `tex_{分類}_{描述}.png`（如 `tex_smoke_cyberpunk_vent.png`）
4. **登錄到 Registry** — 在 `vfx-block-registry.ts` 的 `VFX_BLOCK_REGISTRY` 中新增 `VfxBlockDef`
5. **宣告使用關係** — 在 `vfx-usage-table.ts` 的 `VFX_EFFECT_TABLE` 中登錄積木組合
6. **壓縮規格遵循 `@SIZE`** — 新建子資料夾使用 `@256` / `@128` / `@no-compress` 命名

> **擴充 SOP 完整步驟** 請見 `assets/bundles/vfx_core/README.md` 第「擴充 SOP」章節。

## 1. 專案概述

`cocos-example-cyberpunk-master` 是 Cocos 官方提供的高品質 3D 第三人稱射擊遊戲範例。它展示了 Cocos Creator 3.x 引擎在 **3D 渲染、物理碰撞、AI 尋路、IK 動畫、動態音效混合** 等方面的最佳實踐。專案採用 **Core / Logic 雙層架構**，核心框架（Core）可複用，業務邏輯（Logic）為遊戲特化。

### 技術亮點

| 特性 | 說明 |
|------|------|
| **Actor-Component 框架** | ActorBase → Actor → 各 Component（Move、Equipment、Sensor），分層清晰 |
| **Data-Driven Action 系統** | JSON 定義行為序列，運行時映射到 class 靜態方法 |
| **A* 導航系統** | 基於 Point-Node 的尋路，含隨機巡邏與最短路徑 |
| **Steering Behaviors** | Seek/Velocity 轉向行為，數學化的 AI 移動方案 |
| **IK 瞄準系統** | Aim IK + Bone Weight，運行時程序化骨骼旋轉 |
| **動態音軌混合** | MusicComposeState / MusicComposeRandom，多音軌 lerp 漸變 |
| **Ghost 殘影特效** | Mesh 拷貝 + Vertex Shader 位移，打擊感視覺增強 |
| **全域事件匯流排** | Msg 類：靜態 on/off/emit，全域解耦 |
| **物件池** | ResPool：自增長陣列池，pop/push + recycle 批次回收 |

## 2. 專案目錄結構

```
cocos-example-cyberpunk-master/
├── assets/
│   ├── animations/          # 動畫資源（AnimationGraph、AnimationClip）
│   ├── LightFX/             # 光影與後處理設定
│   ├── res/
│   │   ├── effect/          # 粒子特效 Prefab（爆炸、火花、煙霧等）
│   │   ├── model/           # 3D 模型（角色、場景、武器）
│   │   ├── shader/          # 自定義 Shader Effect
│   │   │   ├── fx-ghost.effect      # 殘影 Shader
│   │   │   ├── fx-ghost-skin.effect # 蒙皮殘影 Shader
│   │   │   ├── trail.effect         # 拖尾 Shader
│   │   │   └── ...
│   │   ├── sound/           # 音效（SFX + BGM）
│   │   └── texture/         # 貼圖資源
│   └── scripts/
│       ├── core/            # 可複用引擎框架層
│       │   ├── action/      # Data-Driven Action 系統
│       │   ├── actor/       # ActorBase 基類
│       │   ├── ai/          # Steering Behaviors
│       │   ├── audio/       # Sound、MusicCompose
│       │   ├── effect/      # fx 特效管理器
│       │   ├── ik/          # AimIK、IKSolverAim
│       │   ├── input/       # IActorInput 介面
│       │   ├── msg/         # 全域事件匯流排 Msg
│       │   ├── res/         # 資源載入 / ResPool 物件池
│       │   ├── sensor/      # SensorGround（地面偵測）
│       │   └── ui/          # UI 管理
│       └── logic/           # 遊戲特化業務層
│           ├── actor/       # Actor 子類（武器、裝備、移動、Brain）
│           ├── effect/      # FxGhost、FxGhostVertex、FxBulletTracer
│           ├── level/       # 關卡管理
│           └── navigation/  # NavSystem A* 尋路
```
