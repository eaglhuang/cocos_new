## 2. 🔴 P0 — Steering Behaviors 單位移動

### 2.1 為什麼需要

**現況問題**：三國 Demo 的 `UnitRenderer` 使用 `tween` 做線性位移，單位移動看起來像「滑行」而非「行軍」。突然加速/突然停止缺少物理感。

**Cyberpunk 作法**：使用 `SteeringBehaviors.Seek()` 計算**轉向力（steering force）**，讓移動有加速→巡航→減速的自然曲線。

```typescript
// Cyberpunk 原始碼（steering-behaviors.ts）
Seek(targetPos: Vec3) {
    UtilVec3.copy(this.desiredVelocity, targetPos);
    this.desiredVelocity.subtract(this.node.worldPosition); // 期望方向
    return this.desiredVelocity.subtract(this.moveVelocity); // 轉向力 = 期望 - 當前
}
```

### 2.2 如何以最佳方式實作

**方案**：在 `ServiceLoader` 中新增 `SteeringService`，提供 `seek()`、`arrive()`、`flee()` 三種行為。`UnitRenderer` 的移動 tween 改為每幀呼叫 `SteeringService`。

```typescript
// ===== 新建 assets/scripts/core/systems/SteeringService.ts =====
import { Vec3, v3 } from 'cc';

export class SteeringService {
    private static _tempDesired = v3();
    private static _tempSteering = v3();

    /**
     * Seek — 追蹤目標，回傳轉向力
     * @param currentPos 當前位置
     * @param targetPos  目標位置
     * @param currentVel 當前速度向量
     * @param maxSpeed   最大速度
     * @returns steering force（轉向力）
     */
    public seek(currentPos: Vec3, targetPos: Vec3, currentVel: Vec3, maxSpeed: number): Vec3 {
        Vec3.subtract(SteeringService._tempDesired, targetPos, currentPos);
        SteeringService._tempDesired.normalize().multiplyScalar(maxSpeed);
        Vec3.subtract(SteeringService._tempSteering, SteeringService._tempDesired, currentVel);
        return SteeringService._tempSteering.clone();
    }

    /**
     * Arrive — 靠近目標時減速
     * @param slowRadius 開始減速的距離
     */
    public arrive(currentPos: Vec3, targetPos: Vec3, currentVel: Vec3, maxSpeed: number, slowRadius: number): Vec3 {
        Vec3.subtract(SteeringService._tempDesired, targetPos, currentPos);
        const distance = SteeringService._tempDesired.length();
        let speed = maxSpeed;
        if (distance < slowRadius) {
            speed = maxSpeed * (distance / slowRadius); // 線性減速
        }
        SteeringService._tempDesired.normalize().multiplyScalar(speed);
        Vec3.subtract(SteeringService._tempSteering, SteeringService._tempDesired, currentVel);
        return SteeringService._tempSteering.clone();
    }

    /**
     * Flee — 逃離目標
     */
    public flee(currentPos: Vec3, threatPos: Vec3, currentVel: Vec3, maxSpeed: number): Vec3 {
        Vec3.subtract(SteeringService._tempDesired, currentPos, threatPos); // 反方向
        SteeringService._tempDesired.normalize().multiplyScalar(maxSpeed);
        Vec3.subtract(SteeringService._tempSteering, SteeringService._tempDesired, currentVel);
        return SteeringService._tempSteering.clone();
    }
}
```

**在 `UnitRenderer` 中的使用方式**：

```typescript
// UnitRenderer 改造（虛擬碼）
private _velocity = v3(0, 0, 0);
private _maxSpeed = 3.0;

updateMove(dt: number) {
    const steering = services().steering.seek(
        this.unitView.worldNode.position,
        this.targetPosition,
        this._velocity,
        this._maxSpeed
    );
    // 施加轉向力（截斷最大力量）
    Vec3.add(this._velocity, this._velocity, steering.multiplyScalar(dt));
    // 截斷速度
    if (this._velocity.length() > this._maxSpeed) {
        this._velocity.normalize().multiplyScalar(this._maxSpeed);
    }
    // 更新位置
    this.unitView.worldNode.position = this.unitView.worldNode.position.add(
        this._velocity.clone().multiplyScalar(dt)
    );
}
```

**ServiceLoader 註冊**：

```typescript
// ServiceLoader.ts 新增

```

## 3. 🟡 P1 — 動態音軌混合

### 3.1 為什麼需要

**現況問題**：三國 Demo 的 `AudioSystem` 只有單軌 BGM 播放 (`playBGM`)，場景切換時硬切音樂，缺少氣氛漸變。

**Cyberpunk 作法**：`MusicComposeState` 同時播放多條 AudioSource，透過 `math.lerp` 每幀漸變各軌音量，實現「探索→緊張→戰鬥」的無縫音樂過渡。

```typescript
// Cyberpunk 原始碼（music-compose-state.ts）
update(dt) {
    for (let i = 0; i < this._audios.length; i++) {
        this._curs[i] = math.lerp(this._curs[i], this.selects[i], dt * this.smooth);
        this._audios[i].volume = this._curs[i] * Sound.volume;
    }
}
```

### 3.2 如何以最佳方式實作

**方案**：擴充現有 `AudioSystem`，新增 `MusicBlender` 內部類別。

```typescript
// ===== AudioSystem 擴充 =====
interface MusicTrack {
    source: AudioSource;
    targetVolume: number;
    currentVolume: number;
}

// 在 AudioSystem 中新增：
private _musicTracks: MusicTrack[] = [];
private _blendSmooth = 0.1;

/**
 * 設定多軌混合 BGM
 * @param clips  多條音軌
 * @param smooth 混合速度（0.05 = 慢，0.2 = 快）
 */
public setupMusicBlend(clips: AudioClip[], smooth = 0.1): void {
    this._blendSmooth = smooth;
    this._musicTracks = clips.map(clip => {
        const source = this._hostNode!.addComponent(AudioSource);
        source.clip = clip;
        source.volume = 0;
        source.loop = true;
        source.play();
        return { source, targetVolume: 0, currentVolume: 0 };
    });
}

/**
 * 設定各軌目標音量（0~1），系統會自動 lerp 漸變
 * 例：setMusicBlend([0, 0, 1]) → 只播第三軌
 */
public setMusicBlend(volumes: number[]): void {
    volumes.forEach((v, i) => {
        if (this._musicTracks[i]) {
            this._musicTracks[i].targetVolume = v;
        }
    });
}

/**
 * 每幀更新 —— 需在遊戲主循環中呼叫
 */
public updateMusicBlend(dt: number): void {
    for (const track of this._musicTracks) {
        track.currentVolume = math.lerp(track.currentVolume, track.targetVolume, dt * this._blendSmooth);
        track.source.volume = track.currentVolume * this._masterVolume;
    }
}
```

**使用場景（三國 Demo）**：

```typescript
// BattleScene.start() 中
const bgmClips = [explorationClip, tensionClip, battleClip];
services().audio.setupMusicBlend(bgmClips, 0.1);

// 戰鬥回合開始時
services().audio.setMusicBlend([0, 0.3, 1]); // 主戰鬥音樂

// 探索/部署階段
services().audio.setMusicBlend([1, 0, 0]); // 平靜探索音樂

// 敵方回合（緊張）
services().audio.setMusicBlend([0, 1, 0.5]); // 緊張 + 輕戰鬥
```

|------|-------------|----------|

## 4. 🟡 P1 — Data-Driven Action 系統

### 4.1 為什麼需要

**現況問題**：三國 Demo 的武將技能效果散落在 `FormulaSystem`、`EffectSystem`、`BuffSystem` 的各處硬編碼中，新增技能需要改動多個檔案。

**Cyberpunk 作法**：`Action` 類別從 JSON 讀取行為序列，`UtilAction.do(name, data)` 透過靜態方法註冊表將字串映射到對應函數，實現 **「一個 JSON 描述一整個動作」**。

```typescript
// Cyberpunk 原始碼（action.ts 核心邏輯）
update(dt) {
    if (this._act.time >= cur.time) {
        UtilAction.do(cur.name, cur.data);  // 字串 → 靜態方法
        this._act.idx += 1;
    }
}

// UtilAction 靜態方法映射
static on_sfx(key: string)  { Sound.on(key); }
static on_fx(data: string, actor: ActorBase)  { actor.onFx(data); }
static on_anig(data: any, actor: ActorBase)  { actor._animationGraph.play(data.key, data.value); }
```

### 4.2 如何以最佳方式實作

**方案**：新建 `ActionSystem`，註冊到 `ServiceLoader`。

```typescript
// ===== 新建 assets/scripts/core/systems/ActionSystem.ts =====
import { services } from '../managers/ServiceLoader';

type ActionHandler = (data: any, context?: any) => void;

interface ActionStep {
    time: number;       // 觸發時間（秒）
    delay: number;      // 延遲（秒）
    name: string;       // 動作名稱
    data: any;          // 動作參數
}

export class ActionSystem {
    private _handlers: Map<string, ActionHandler> = new Map();

    constructor() {
        this.registerDefaults();
    }

    /** 註冊預設動作處理器 */
    private registerDefaults(): void {
        this.register('play_sfx', (key) => services().audio.playSFX(key));
        this.register('play_effect', (data, ctx) => {
            services().effect.playEffect(data.effectId, ctx?.position);
        });
        this.register('apply_buff', (data, ctx) => {
            services().buff.addBuff(ctx?.unitId, data.buffId, data.duration);
        });
        this.register('emit_event', (data) => {
            services().event.emit(data.key, data.value);
        });
    }

    /** 動態註冊新的動作處理器 */
    public register(name: string, handler: ActionHandler): void {
        this._handlers.set(name, handler);
    }

    /** 執行單一動作 */
    public execute(name: string, data: any, context?: any): void {
        const handler = this._handlers.get(name);
        if (handler) {
            handler(data, context);
        } else {
            console.warn(`[ActionSystem] 未註冊的動作: ${name}`);
        }
    }

    /** 執行一組動作序列（時間軸驅動） */
    public createTimeline(steps: ActionStep[], context?: any): ActionTimeline {
        return new ActionTimeline(steps, this, context);
    }
}

export class ActionTimeline {
    private _steps: ActionStep[];
    private _system: ActionSystem;
    private _context: any;
    private _time = 0;
    private _index = 0;
    private _done = false;

    constructor(steps: ActionStep[], system: ActionSystem, context?: any) {
        this._steps = steps;
        this._system = system;
        this._context = context;
    }

    get isDone() { return this._done; }

    update(dt: number): void {
        if (this._done) return;
        this._time += dt;
        while (this._index < this._steps.length) {
            const step = this._steps[this._index];
            if (this._time < step.time) break;
            this._system.execute(step.name, step.data, this._context);
            this._index++;
        }
        if (this._index >= this._steps.length) this._done = true;
    }
}
```

## 5. 🟢 P2 — Ghost 殘影特效

### 5.1 為什麼需要

**現況問題**：三國 Demo 的武將衝鋒和大招施放缺少視覺衝擊力，移動時只是平移 sprite。

**Cyberpunk 作法**：`FxGhostVertex` 每幀計算移動方向向量，傳入 Shader 的 `direction` uniform，在 GPU 端沿移動反方向偏移頂點，產生速度感殘影。

```typescript
// Cyberpunk 原始碼（fx-ghost-vertex.ts）
update(dt) {
    Vec3.lerp(this.lastPosition!, this.lastPosition!, this.currentPosition!, this.t);
    Vec3.subtract(this.direction, this.lastPosition!, this.currentPosition!).multiplyScalar(this.intensity);
    material.passes[0].setUniform(handle, this.directionUniform); // GPU 端頂點偏移
}
```

### 5.2 如何以最佳方式實作

**方案**：在 `EffectSystem` 中新增 `applyGhostEffect()` 方法，搭配搬入的 `fx-ghost.effect` Shader。

```typescript
// ===== EffectSystem 擴充 =====

/**
 * 對目標節點施加殘影效果
 * @param targetNode 要產生殘影的節點
 * @param intensity  殘影強度（預設 4）
 * @param duration   持續時間（秒）
 */
public applyGhostEffect(targetNode: Node, intensity = 4, duration = 1.0): void {
    const ghost = targetNode.addComponent(GhostEffect);
    ghost.intensity = intensity;
    ghost.duration = duration;
}

// GhostEffect 元件（簡化版 FxGhostVertex）
@ccclass('GhostEffect')
class GhostEffect extends Component {
    intensity = 4;
    duration = 1.0;
    private _elapsed = 0;
    private _lastPos = v3();
    private _direction = v4();

    start() { Vec3.copy(this._lastPos, this.node.worldPosition); }

    update(dt: number) {
        this._elapsed += dt;
        if (this._elapsed >= this.duration) {
            this.destroy();
            return;
        }
        const dir = v3();
        Vec3.subtract(dir, this._lastPos, this.node.worldPosition).multiplyScalar(this.intensity);
        Vec3.copy(this._lastPos, this.node.worldPosition);

        // 更新 Shader uniform
        const renderers = this.node.getComponentsInChildren(Sprite);
        renderers.forEach(r => {
            const mat = r.customMaterial;
            if (mat) {
                const handle = mat.passes[0].getHandle('direction');
                if (handle) {
                    this._direction.set(dir.x, dir.y, dir.z, 0);
                    mat.passes[0].setUniform(handle, this._direction);
                }
            }
        });
    }
}
```

## 7. 🟢 P2 — 全域事件匯流排強化

### 7.1 為什麼需要

Cyberpunk 的 `Msg` 類別支援 `bind(key, fn, target)` 自動綁定 `this`，三國 Demo 的 `EventSystem` 目前需要手動 `bind(this)`，容易遺漏導致 `this` 指向錯誤。

### 7.2 如何以最佳方式實作

```typescript
// EventSystem 擴充
public onBind(event: string, callback: Function, target: any): void {
    this.on(event, callback.bind(target));
}
```

小改動，但能防止常見的 `this` 丟失 bug。

## 9. 風險評估與注意事項

| 風險 | 影響 | 緩解措施 |
|------|------|----------|
| **Steering 與 tween 衝突** | 現有 tween 移動邏輯需要全面替換 | 先在新單位上驗證，確認穩定後逐步替換 |
| **多軌 BGM 記憶體** | 3~4 軌 AudioClip 同時載入增加記憶體 | 控制每軌壓縮格式（OGG）和取樣率（22kHz） |
| **ActionSystem 循環依賴** | `ActionSystem` 引用 `services()` 可能造成循環 | 延遲初始化（在 `init()` 而非 `constructor` 中註冊 handlers） |
| **Ghost Shader 相容性** | 2.5D Sprite 與 3D Shader 的 pass 結構不同 | 需為 Sprite 建立專用的 2D ghost effect |
| **A* 效能** | 大棋盤（20x20+）尋路可能耗時 | 限制最大搜索步數 + 使用 Jump Point Search 優化 |
| **JSON 設定錯誤** | 錯誤的 action name 不會編譯報錯 | `ActionSystem.execute()` 已有 console.warn 提示 |

## 12. 🟡 P1 — 兩專案素材管理方式比較分析與可借鑒優化

### 12.1 管理方式深度對比

#### A. 目錄結構管理

| 比較項目 | Cyberpunk | 三國 Demo (VFX Core) | 評價 |
|----------|-----------|---------------------|------|
| **組織原則** | 按「特效用途/Prefab」分資料夾（`fx_axe/`、`fx_grenade_0/`） | 按「視覺功能/積木類型」分資料夾（`glow/`、`fire/`、`smoke/`） | ⭐ 三國 Demo 更佳 |
| **貼圖與 Prefab** | 同一資料夾混放 `.png` + `.prefab` + `.mtl` | 貼圖、Shader、Mesh、Prefab **各自獨立資料夾** | ⭐ 三國 Demo 更佳 |
| **重複資源** | `flash11.png`、`spark31.png` 等在 muzzle01~04 + muzzle_enemy 中各存一份（**5 份重複**） | 每張貼圖只存一份在 `vfx_core/textures/`，Prefab 透過路徑引用 | ⭐ 三國 Demo 更佳 |
| **擴充方式** | 新建整個 `fx_新特效/` 資料夾，內含所有依賴 | 新貼圖放入已有分類 + Registry 登錄 | ⭐ 三國 Demo 更佳 |
| **Core / Logic 分層** | `scripts/core/` vs `scripts/logic/` 清晰分離 | `scripts/core/` + `bundles/` 分離，但規範文件更完善 | ≈ 各有優勢 |

**分析**：Cyberpunk 的「每個特效一個資料夾」方式直覺，但導致大量貼圖重複。例如 `flash11.png`（閃光貼圖）在 5 個 muzzle 資料夾中各複製一份，浪費磁碟空間且增加 Bundle 體積。三國 Demo 的「積木分類 + 統一引用」方式更優，**已無需借鑒此方面**。

#### B. 命名管理

| 比較項目 | Cyberpunk | 三國 Demo (VFX Core) | 評價 |
|----------|-----------|---------------------|------|
| **貼圖命名** | 無統一前綴（`smoke001.png`、`flash11.png`、`CubeParticle.png`） | 統一前綴 `tex_{分類}_{描述}.png` | ⭐ 三國 Demo 更佳 |
| **大小寫一致性** | 混用（`CubeParticle` vs `smoke001` vs `AxeHoles`） | 全小寫 + 底線分隔 | ⭐ 三國 Demo 更佳 |
| **版本號** | 數字後綴（`smoke001`、`smoke002`、`spark31`）含義不明 | 語義後綴（`_soft`、`_bright`、`_sharp`）含義清晰 | ⭐ 三國 Demo 更佳 |
| **壓縮規格** | 無標示 | `@SIZE` 命名 or `_TEX_COMPRESSION.md` 文件補充 | ⭐ 三國 Demo 更佳 |

**分析**：Cyberpunk 的命名缺乏規範（大小寫混用、數字後綴含義不明），是典型的「快速原型」命名。三國 Demo 已建立完善的命名體系，**已無需借鑒此方面**。

#### C. 素材治理流程

| 比較項目 | Cyberpunk | 三國 Demo (VFX Core) | 評價 |
|----------|-----------|---------------------|------|
| **使用追蹤** | 無機制（需手動在程式碼中搜尋） | `vfx-usage-table.ts` 程式化宣告 + `findUnusedBlocks()` 自動稽核 | ⭐ 三國 Demo 更佳 |
| **死資源偵測** | 無機制 | `findUnusedBlocks()` API 可在開發期間偵測 | ⭐ 三國 Demo 更佳 |
| **清理流程** | 無文件規範 | `_MANIFEST.md` 定義 Active/WIP/Archive 生命週期 | ⭐ 三國 Demo 更佳 |
| **新增流程** | 無文件規範 | `vfx_core/README.md` 擴充 SOP 7 步驟 | ⭐ 三國 Demo 更佳 |
| **視覺預覽** | 需在編輯器場景中手動放置 Prefab | `VfxComposerTool` 即時預覽積木組合 | ⭐ 三國 Demo 更佳 |

**分析**：Cyberpunk 完全缺乏素材治理機制，依賴開發者自律。三國 Demo 已建立完善的治理體系，**已無需借鑒此方面**。

### 12.2 ⭐ Cyberpunk 值得借鑒的管理模式

雖然在目錄/命名/治理方面三國 Demo 已優於 Cyberpunk，但 Cyberpunk 仍有以下管理模式值得學習：

#### 💡 借鑒 1：JSON Data-Driven 特效配置

**Cyberpunk 作法**：武器/技能的特效播放全部定義在 JSON 中（`data-equips.json`），包含 `fx`（特效 key）、`sfx`（音效 key）、`notify`（提示文字），不在程式碼中硬編碼。

```json
// Cyberpunk: data-equips.json
"damage": {
    "hit_body": { "fx": "fx_hit_body", "sfx": "sfx_axe_hit_body", "notify": "Hit body" },
    "hit_wall": { "fx": "fx_hit_wall", "sfx": "sfx_axe_hit_wall", "notify": "Hit_wall" }
}
```

**三國 Demo 現況**：技能特效的積木組合定義在 `vfx-usage-table.ts` 的 TypeScript 常量表中，修改需要重新編譯。

**優化建議**：將 `VFX_EFFECT_TABLE` 改為從 JSON 載入，實現「設計師可編輯、不需重新編譯」：

```typescript
// 現有方式（需編譯）
const VFX_EFFECT_TABLE: Record<string, string[]> = {
    "skill_fireball": ["fire_burst", "smoke_aura", "impact_ring"],
};

// 優化方式（JSON 驅動，免編譯）
// resources/data/vfx-effect-table.json
{
    "skill_fireball": ["fire_burst", "smoke_aura", "impact_ring"],
    "skill_charge":   ["glow_bright", "smoke_stretched", "trail_slash_w"]
}
```

**預估工時**：0.5 天
**影響範圍**：`vfx-usage-table.ts`（改為 JSON loader）

#### 💡 借鑒 2：特效-音效-通知三位一體綁定

**Cyberpunk 作法**：每個命中事件同時綁定 `fx`（視覺）+ `sfx`（聽覺）+ `notify`（邏輯通知），三者不可分離，避免「有特效沒音效」或「有音效沒回饋」的不一致問題。

**三國 Demo 現況**：`VfxBlockDef` 有 `audio?` 欄位，但這是積木級別的「建議音效」，並非事件級別的「強制綁定」。

**優化建議**：在 `VFX_EFFECT_TABLE` 中增加事件級別的音效與回調綁定：

```typescript
// 增強版 VFX_EFFECT_TABLE
interface VfxEffectEntry {
    blocks: string[];          // 積木 ID 列表
    sfx?: string;             // 事件級音效（覆蓋積木預設音效）
    notify?: string;          // 事件通知 key（觸發 EventSystem）
}
```

**預估工時**：1 天
**影響範圍**：`vfx-usage-table.ts`、`EffectSystem`

#### 💡 借鑒 3：Core / Logic 分層的命名空間隔離

**Cyberpunk 作法**：`scripts/core/` 與 `scripts/logic/` 嚴格分離。Core 層不引用 Logic 層，Logic 層可引用 Core 層。這保證了 Core 的可移植性。

**三國 Demo 現況**：`scripts/core/` 存在，但部分 Service（如 `EffectSystem`）同時包含通用邏輯和三國特化邏輯，分層不夠嚴格。

**優化建議**：檢視 `scripts/core/systems/` 下的所有 Service，將三國特化邏輯（如武將名稱映射、兵種屬性計算）提取到 `scripts/battle/` 或 `scripts/logic/`，保持 Core 層的純淨可複用性。

**預估工時**：2 天
**影響範圍**：`scripts/core/systems/` 下多個文件的重構
