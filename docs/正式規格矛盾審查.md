## 2. 仍待處理

### 2.1 舊 UI 流程文件中的 legacy key

- `docs/ui/UI-vibe-pipeline.md` 仍保留 `hud.icon.dp` 類型的舊命名，屬流程範例遺留，尚未改寫成現行 `grain` 命名。

### 2.2 歷史紀錄段落中的舊詞殘留

- `docs/keep.md` 的後半部日期式歷程紀錄仍可能出現 `DP` 或更早期措辭，但只要不與前段正式準則混用，暫視為歷史脈絡，不列為阻斷性矛盾。

### 2.3 待整併討論檔尚未回寫的正式承接

- `docs/討論來源整併狀態.md` 所列 `仍待整併` 檔案，尤其是 UI / 視覺 / 鏡頭相關提案，仍缺正式承接文件，因此不算已完成整平。
- 其中 `更舊的討論/三國傳承 UI 布局說明書.md`、`更舊的討論/遊戲視覺構圖與攝影機規格.md`、`更舊的討論/夜襲戰場視覺規格提案.md` 優先級最高。

### 2.4 Constants.ts DP 命名殘留

- `assets/scripts/core/config/Constants.ts` 仍使用 `INITIAL_DP`、`DP_PER_TURN` 常數名稱。規格書已統一為「糧草」，代碼常數名稱需跟進重命名為 `INITIAL_GRAIN`、`GRAIN_PER_TURN`。

### 2.5 terrain.json 未從 Constants.ts 抽取

- `demo_技術架構.md` § 8 已標記地形修正值應抽成 `terrain.json`，但目前仍 hardcoded 在 `Constants.ts` 的 `TerrainType` 相關常數中。此為 P0 優先事項。

### 2.6 教官系統 VIT_Max 縮寫不一致

- `docs/系統規格書/教官系統（支援卡）.md` 使用 `VIT_Max`，而 `武將系統.md` / `Data Schema文件.md` 使用 `Vitality_Max`。需統一為 `Vitality_Max`。

### 2.7 宿敵配對 EP 懲罰跨文件衝突

- `家族關係（史實相性）系統.md` 與 `血統理論系統.md` 對宿敵配對的 EP 懲罰描述可能不一致，需確認具體數值是否對齊。

### 2.8 組合奧義規則尚未定義

- `戰法場景規格書.md` J 節已保留「組合奧義」區段，但赤壁大火聯合、桃園結義等組合觸發條件與效果尚未正式定義。在新架構下，需額外釉清組合奧義是屬於場景戰法層還是格子戰法層。

### 2.9 混種戰法定義分散

- `戰法系統.md` 未定義混種戰法（Hybrid Skills），實際定義在 `血統理論系統.md` K 節。應在 `戰法系統.md` 加入交叉引用。

### 2.10 UIPreviewBuilder / UIValidationRunner 未實作

- `UI技術規格書.md` 規定了 UIPreviewBuilder 與 UIValidationRunner 兩個自動化工具，但目前尚無對應代碼檔。
