# [UI-2-0101] BattleScene 數值 Micro Glyph Suite

## 任務目標

為 `TigerTallyPanel` 與 `UnitInfoPanel` 建立同一套 battle status micro glyph 規格，首批只處理：

- `atk`
- `hp`
- `cost`

這批 glyph 的定位是「內容層的微型數值符號」，用來搭配 runtime number 顯示，不是 badge、不是 button icon、也不是收集貨幣圖示。

## 範圍

- 建立 `battle-status-icon-v1` 的正式 member spec
- 明確定義 `TigerTallyPanel` / `UnitInfoPanel` 的 docking target
- 回寫 prompt card / prompt txt / runtime reference / readiness
- 保留 `def / spd` 為第二批成員，不在這一輪先畫

## 不在本輪

- 不生圖
- 不改 `BattleHUD`
- 不把 `DEF / SPD` 提前塞進首批
- 不先決定 `TigerTally` 的兵種縮寫文字是否退場

## 美術語言

遵守 [美術風格規格書.md](/C:/Users/User/3KLife/docs/美術風格規格書.md) 與 BattleScene 非 HUD 語境：

- 背景層：水墨戰場氛圍
- 蒙版層：半透明暗色壓低背景噪訊
- 框體層：由 runtime frame / chip 承擔，不烘進 glyph
- 內容層：本單輸出的 micro glyph

## 品質要求

- `16px ~ 24px` 仍可辨識
- `cost` 必須是軍糧 / 補給語意，不可畫成金幣、商城幣、寶石
- 只允許 `glyph-only`
- 不可內建文字、數字、字母
- 不可把 ornate frame / medal / coin plate 烘進 icon
