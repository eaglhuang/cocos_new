# UI-2-0097 GeneralDetail Rarity Badge Formal Production

## 目標

統一生產 `GeneralDetailOverview` 的 rarity badge 家族，讓 common/rare/epic/legendary/mythic 都有正式可量產版本。

## Art Direction

- badge 是 header 的輔助資訊，不可比人物名與主匾額更搶眼
- 金色只能是節點提示，不可整塊俗亮
- mythic 不可再使用 placeholder 或 duplicate
- 五階層必須家族一致，只做材質、色階、紋理差異

## 交付幾何規格

- 五階 badge 都是**固定尺寸單張 sprite**，不是 9-slice，不可拉伸。
- `common / rare / epic / legendary / mythic` 必須各自輸出成五個獨立檔案，一檔只對應一個 badge。
- 禁止輸出「五枚 badge 一排」或「一張候選板再裁切」當成正式交付；若需要批次生成，也必須在導入前切成單張並逐張驗收。
- 每張 badge 需保留一致外輪廓與對齊基準，只允許材質、色階、紋理變化，不可每階尺寸與輪廓都漂移。

## AI 生成禁則

- AI prompt 不可寫成 `a set of 5 badges in a row`、`badge collection`、`badge sheet` 這類會誘發 strip/sprite-sheet 的描述。
- AI prompt 必須一次只生一個 badge，並明講 `single isolated badge asset`。
- 禁止把展示用排版圖、比較板、五連 badge strip 直接導入 runtime。
- mythic 與 legendary 必須分開生成或分開驗收，不可只靠色偏硬分。


## Reject If

- mythic 仍是 placeholder / duplicate
- epic 與 legendary 色語義不清
- badge 亮度過高，破壞 header 層次
- 任一 `formal/rarity_badge_*` 仍帶著其他等階 badge 的殘影、拼版或整條 strip
