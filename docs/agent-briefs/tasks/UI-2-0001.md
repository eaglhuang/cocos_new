## 開單原因
`lobby-main-default` 的主選單按鈕群目前使用純色 slot 或舊有 sprite，缺乏 `nav_ink` family 的墨黑質感，與設計規範不符。待 `UI-1-0001` 完成後須接線更新。

## 完整描述
1. 確認 `UI-1-0001` 已完成（目標路徑存在）。
2. 在 `assets/resources/ui-spec/skins/lobby-main-default.json` 中找到對應 `button-skin` slot。
3. 將 SpriteFrame 路徑更新為：
   - normal：`assets/resources/sprites/ui_families/common/nav_ink/btn_primary_normal`
   - pressed：`assets/resources/sprites/ui_families/common/nav_ink/btn_primary_pressed`
   - disabled：`assets/resources/sprites/ui_families/common/nav_ink/btn_primary_disabled`
4. 確認 `border` 為 `[20, 20, 20, 20]`（按鈕九宮格規範）。
5. 執行 JSON schema 驗證腳本（`npm run check:acceptance` 或 `tools_node/run-tests.js`）。
6. 更新本卡 frontmatter `status` 與 `notes`，並同步 manifest。
