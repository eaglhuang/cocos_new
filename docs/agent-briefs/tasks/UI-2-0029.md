# [UI-2-0029] 戰場 UI Portrait 資產清單

## 範圍

- BattleHUD 左右兩側武將頭像
- 戰場相關武將 portrait 命名規則
- 現有頭像資產覆蓋率與缺漏

## 任務目標

1. 盤點 BattleScene 實際需要的 portrait key 與尺寸。
2. 確認現有 `sprites/generals/*_portrait` 是否已覆蓋目前 roster。
3. 明確區分「資產缺件」和「runtime 尚未套用」兩種問題。

## 盤點結果

### 1. 畫面入口

- `assets/resources/ui-spec/layouts/battle-hud-main.json`
  - `playerPortrait`：`64x64`
  - `enemyPortrait`：`64x64`
- `assets/resources/ui-spec/skins/battle-hud-default.json`
  - `hud.portrait.player` 目前指向 `sprites/battle/portrait_player_placeholder/spriteFrame`
  - `hud.portrait.enemy` 目前指向 `sprites/battle/portrait_enemy_placeholder/spriteFrame`

### 2. 現有武將頭像資產

`assets/resources/sprites/generals/` 目前已存在：

- `zhang_fei_portrait.png`
- `guan_yu_portrait.png`
- `lu_bu_portrait.png`
- `cao_cao_portrait.png`
- `zhao_yun_portrait.png`

### 3. roster 覆蓋率

`assets/resources/data/generals.json` 目前 roster 共 5 名：

| 武將 id | 預期 portrait key | 現況 |
|---|---|---|
| `zhang-fei` | `sprites/generals/zhang_fei_portrait` | 已存在 |
| `guan-yu` | `sprites/generals/guan_yu_portrait` | 已存在 |

## 結論

- 就目前 roster 而言，BattleScene portrait 並不是「缺圖」問題，而是「HUD 還沒接到現有頭像資產」。
- 現有資產覆蓋率是 `5/5`，足夠支撐目前戰場雙方武將頭像顯示。
- 真正缺口在 runtime：BattleHUD 仍固定使用 placeholder slot，尚未按武將 id 動態載入 `sprites/generals/*_portrait`。

## 後續需求

- Agent1 需要接續 `UI-2-0033`，把 BattleHUD portrait 改成動態載入。
- 動態載入建議規則：
  - 以武將 id 轉成 snake_case，組成 `sprites/generals/{id}_portrait`
  - 載入失敗時才退回 `portrait_player_placeholder` / `portrait_enemy_placeholder`

  - `A2 HUD 頭像裁片 family`

  - `A5 主將半身 panel family`
