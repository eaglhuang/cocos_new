# UI-2-0094 GeneralDetailBloodlineV3 Canonical Finish Asset Pack

## 目標

接手 `UI-2-0093` 已經接近 ceiling 的狀態，專門補齊真正決定 canonical finish 的資產，不再讓 `layout / skin / shell tint` 去硬扛本來就該由 final art asset 解決的問題。

本卡聚焦三塊：

1. `final medallion`
2. `header ornament`
3. `portrait frame`

## 美術總監期待

- 北極星固定為 `Warm Parchment Base + Jade Header Chrome + Controlled Dark Contrast + Readable Atlas Hierarchy`，不要把 finish 理解成加深、加黑、加金屬重量。
- 右側 overview 仍以 parchment 承載面為主，新資產只能補結構與 finish，不能把閱讀主面板推回深綠重金。
- `jade` 只負責 header chrome、命紋 socket 與小面積 accent；它不是整塊主底，也不是用來把牌匾壓成厚重綠條。
- 這張卡要補的是「乾淨且高級」的 finish，不是用更多 ornament 製造熱鬧感。

## Canonical References

- 正式參考圖：`docs/UI品質參考圖/UI-2-0054_人物頁與血脈命鏡/人物日常介紹v3_正確參考.png`
- 目前 runtime 基線：`artifacts/ui-qa/UI-2-0093/r25-recheck-after-codec-fix/GeneralDetailOverview.png`
- 美術規格基準：`docs/美術風格規格書.md (doc_art_0002)` (doc_art_0002)

## 為什麼要從 UI-2-0093 拆出

`UI-2-0093` 的任務本質是 canonical layout / content polish。當 header 比例、右側 cards、crest socket 結構、portrait scale/background 都已經收過一輪後，若畫面還是欠缺參考圖那種「乾淨且高級」的 finish，問題就不再是 layout，而是 asset finish。

若繼續把這類問題留在 `UI-2-0093` 裡，只會出現：

- header 牌匾一直靠 tint 與 padding 假裝有 ornament
- crest 一直靠 proof bridge 撐 medallion 完成度
- portrait stage 一直靠暗底與色偏硬撐 frame 質感

所以這張卡的任務，是把 finish 問題抽離成獨立生產線。

## 資產範圍

### 1. Final Medallion

- 補 `crest-medallion-final` 的正式 `face / ring / socket` 結構
- 方向是 jade-parchment family 的命紋章，不是厚重金屬徽章
- 要能嵌入 `BloodlineCrestCard`，看起來像卡面內建 socket，而不是外掛貼片
- 外圈可有光暈與刻紋，但重心必須留在命紋章面本身，不可變成厚黑圈 + 金屬盾牌語言

### 2. Header Ornament

- 補 `header-ornament-final` 的左右端飾、主牌匾角節、中心重心
- 目標是把 header 從「有 jade band」推進到真正的「牌匾」
- 必須與 `jade-parchment-panel-final` 同 family，不可跳成另一套武將框語言
- header ornament 的工作是補牌匾節奏與中心錨點，不可因 ornament 讓 title 區被壓扁、縮字或變成厚重深綠橫條

### 3. Portrait Frame

- 補 `portrait-frame-final` 的正式舞台框，而不是回退到裝備框或廉價稀有度框
- 任務是建立左側人物舞台的 finish 與邊界，而不是增加額外裝飾噪音
- 要支援目前 `GeneralDetailOverview` 的縱向構圖，不能只在單一角色裁切上成立
- 左側舞台框可以有細柱、角節、內陰影與輕量金屬邊，但人物仍必須是主角，不能讓 frame 比角色更搶眼

## Final Pass Asset Direction（2026-04-08 補充）

以下清單是依目前最新 runtime capture 與正式參考圖的殘差整理，給後續 Agent 或美術直接對照。

### A. Header Ornament / Plaque

- 目標：右上 header 必須讀起來像完整 jade-parchment 牌匾，而不是一條深綠色帶上面放標題。
- 必須補足：
  - 中段主牌匾的結構感
  - 左右端飾與角節的視覺錨點
  - 標題區內側的暖 parchment 承載面
- 允許：薄 jade band、細金邊、低密度 ornament。
- 不允許：
  - 厚重深綠整條壓滿
  - 黑金裝備框語言
  - 直接燒字進 ornament 圖
  - ornament 比標題本身更搶眼

### B. Final Medallion / Crest Socket

- 目標：右下命紋區要像卡面內建的 jade socket，不是外掛 badge。
- 必須補足：
  - socket 深度
  - 內外圈層級
  - 中央命紋面可讀性
  - 與右側卡體的一致 family 語言
- 允許：淡 jade、pale parchment、細金框、低強度內光。
- 不允許：
  - 徽章像硬幣、盾牌、商城 token
  - 厚黑金屬外圈
  - 高飽和 neon jade
  - AI 生成燒字、印章文字、符號過多導致可讀性破壞

### C. Portrait Frame

- 目標：左側人物舞台要有正式邊界，但人物仍是主體。
- 必須補足：
  - 舞台邊界與角色分離
  - 輕量結構節點（角節 / 細柱 / 內側陰影）
  - 與右側 parchment 主卡相容的 finish
- 不允許：
  - 裝備格、稀有度框、商城邊框語言
  - frame 比人物更亮、更厚重
  - 過重暗底把角色整體壓髒

### D. Reject List（給 QA / Agent2 快速退件）

- 任一資產含燒入英文/中文標題、印章字樣、裝飾字樣，一律退件。
- header 若套上後仍主要依靠 runtime tint 才像牌匾，一律退件。
- medallion 若第一眼讀成 badge、coin、shield、equipment icon，而不是血脈 socket，一律退件。
- portrait frame 若把人物舞台壓成深色框景或搶走角色主體焦點，一律退件。
- 套入後若右側主卡整體比目前版本更深、更綠、更重，一律退件。
- 若任何資產只在單一角色裁切成立，換人就破版，一律退件。

### E. 本輪已知殘差

- header 已有較正確的牌匾節奏，但 final ornament 仍不足以形成參考圖那種中心錨點。
- 右下命紋區目前靠 runtime 色階與比例已接近 reference，但 final medallion 的 finish 與 socket 深度仍不足。
- 左側人物舞台仍主要靠構圖與底托成立，portrait frame family 尚未真正提供 canonical finish。

## Reject If

- 資產一套上去，右半邊閱讀主面板明顯比目前基線更深、更綠、更重。
- `header ornament` 需要靠額外 tint、padding 或 glow 補救，才看起來像完整牌匾。
- `medallion` 看起來像外掛 badge 貼片、裝備徽章或厚黑金屬盾牌，而不是卡面內建命紋 socket。
- `portrait frame` 讀感像裝備格、稀有度框或商城外框，而不是人物舞台邊界。
- 資產只在當前張飛裁切成立，換角色或換縱向構圖就失衡。
