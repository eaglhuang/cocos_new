## 開單原因
`UI-2-0019` 已確認 D-2 的正式驗收方向應是 `common-parchment / light-surface carrier consistency`，而不是 `paper.utility`。但目前：

1. `shop-main-default.json` 仍主要使用 dark color block 與 `commerce` family。
2. `gacha-default.json` 雖掛了 `common-parchment-default` fragment，卻還沒有明確、可供 QA 對照的 shared light-surface target。

若不補這一步，Agent2 即使有 `LoadingScene` preview hub，也沒有真正可驗的共同承載面。

## 完整描述
1. 盤點 `shop-main-main.json` / `shop-main-default.json`，挑出最適合承載淺底框體的主容器或次級容器。
2. 盤點 `gacha-main.json` / `gacha-default.json`，確認哪個區塊作為 D-2 的對照 light-surface target。
3. 將兩邊都接上可命名、可截圖、可在 QA notes 明確描述的 `common-parchment` / light-surface slots。
4. 回填 D-2 文件中的 target slots / target nodes，讓 Agent2 截圖時知道該看哪裡。
