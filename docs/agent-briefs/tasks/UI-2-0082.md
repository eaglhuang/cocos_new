# UI-2-0082 — Figma Proof Mapping Sync Pipeline

## 背景

Figma `09_Proof Mapping` frame 已有正式欄位定義（`familyId/template/tabs/railItems/titleKey/bodyKey`），且 keep.md §10 和 §12 定義了 Proof Mapping Contract。但目前的流程仍是：

1. 設計者在 Figma 填好 proof mapping
2. **人工抄錄**到 `artifacts/ui-qa/{family}/proof-mapping-*.json`
3. 再手動餵給 `scaffold-ui-spec-family.js`

這個「設計→開發」的人工搬運步驟，是 AI Agent 無法完全自動化的入場瓶頸之一。

## 目標

讓 Agent 能直接從 Figma 讀取設計交付物：
```bash
node tools_node/sync-figma-proof-mapping.js --frame-id lf8ByZq8VVBtBTBJ0IpahX
```

## 實作清單

### 1. sync-figma-proof-mapping.js（新建）
位置：`tools_node/sync-figma-proof-mapping.js`

功能：
- 透過 MCP Figma 工具讀取指定 frame（Figma 檔案：`lf8ByZq8VVBtBTBJ0IpahX`）
- 解析 frame 中的 text node（familyId, template, tabs, titleKey 等欄位）
- 解析 image node（proof screenshot, wireframe, slot-map）
- 驗證欄位完整性（依 keep.md §12 的 Proof Mapping Contract）
- 輸出 `artifacts/ui-qa/{family}/proof-mapping-{date}.json`

選項：
- `--frame-id <id>`：Figma frame 的 node-id（必填）
- `--family <id>`：限制只處理某個 family（可選）
- `--dry-run`：只輸出 JSON 預覽，不寫檔
- `--out <path>`：指定輸出路徑

### 2. scaffold-ui-spec-family.js 擴充
- 新增 `--figma-frame-id <id>` 選項
- 呼叫 sync-figma-proof-mapping 取得 proof mapping JSON
- 用此 JSON 作為 scaffold 的 config 輸入

### 3. 欄位一致性驗證
- 自動比對 Figma 欄位 vs 現有 repo proof mapping JSON
- 差異存在時輸出 diff report

## 依賴

- MCP Figma 工具已可使用（keep.md §14 確認 2026-04-04 可用）
- UI-2-0080 完成（ContentContractRef schema 定義完整後，才能用 Figma 欄位對接 requiredFields）
