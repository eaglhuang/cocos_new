# UI-2-0085 — Layered Skin Theme Stack

## 分派理由

這張卡適合 Agent1，因為需要改 `UISpecLoader`、`UISkinResolver` 與相容策略，屬於 runtime 架構工作。

## 背景

現況 skin 雖然已可載 fragment，但仍偏單張 manifest 合併。這不足以支撐 `base theme -> family -> state override -> screen override` 的批次換皮。

Unity 對照：目前比較像單一 Prefab；目標是接近 Prefab Variant + Theme ScriptableObject 疊層。

## 目標

建立 layered skin/theme 疊層能力，支援共用高品質 recipe 與局部覆寫，同時保持既有 screen 向後相容。

## 驗收條件

- [x] UISpecLoader 支援多層 merge chain（A1-5 完成 `_buildThemeStackSlots`）
- [x] UISkinResolver 可解析 layered override（themeStack 4-layer 已實作，含 circular ref guard）
- [x] 既有單 manifest skin 無需改檔也能跑（backward compat: 無 themeStack → 走原 flat merge）
- [ ] 至少 2 張既有 screen 完成 migration 驗證（需要 smoke test，待 UI-2-0091 資產就位後）
- [x] 優先順序與覆寫邏輯已文件化（見下方疊層策略）

## Phase G 第一版欄位草案（UI-2-0084 完成後可立即開始）

### 疊層合併策略

四層優先序（由低到高）：
```
1. global-base   — 全域設計 token / 預設 slot（eg. skin-base-v1）
2. family-theme  — family 級共用 slot（eg. skin-family-dark-metal）
3. stateOverride — 互動狀態覆寫 slot（eg. skin-state-pressed）
4. screen        — 畫面層局部覆寫（recipeRef.slotOverrides）
```

後面的 key 覆蓋前面，未重疊的 key 全部保留。

Unity 對照：相當於 Material Inheritance Chain：
- global-base ≈ Base Material
- family-theme ≈ Prefab Variant（共用骨架）
- stateOverride ≈ Animation 覆蓋特定屬性
- screen ≈ per-Renderer MaterialPropertyBlock

### UISpecTypes.ts 已預留的欄位（UI-2-0084 已加）

```typescript
// UISkinManifest 新增
themeStack?: {
    base?: string;              // 全域 base skin id
    family?: string;            // family-level skin id
    stateOverrides?: string[];  // state override skin id 清單（低到高）
};
recipeRef?: RecipeRef;          // 框體 recipe 引用
```

### UISpecLoader 改動計畫（Batch A1-5）

```typescript
// 現況（flat merge）
Object.assign(manifest.slots, fragment.slots);

// 目標（layered merge chain）
async function buildMergeChain(manifest: UISkinManifest): Promise<Record<string, SkinSlot>> {
    const chain: Record<string, SkinSlot>[] = [];
    // 1. base
    if (manifest.themeStack?.base) {
        const base = await loadSkinManifest(manifest.themeStack.base);
        chain.push(base.slots);
    }
    // 2. family
    if (manifest.themeStack?.family) {
        const family = await loadSkinManifest(manifest.themeStack.family);
        chain.push(family.slots);
    }
    // 3. stateOverrides（由低到高）
    for (const soId of manifest.themeStack?.stateOverrides ?? []) {
        const so = await loadSkinManifest(soId);
        chain.push(so.slots);
    }
    // 4. 本 manifest 自身槽（最高優先）
    chain.push(manifest.slots);
    // merge（後覆前，不刪未衝突 key）
    return Object.assign({}, ...chain);
}
```
