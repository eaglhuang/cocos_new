# [UI-2-0038] 建立 BattleHUD portrait 裁片量產規格

## 目標

把 `UI-2-0029` 的「roster 與 portrait 資產已存在」往前推到可量產階段，定義 BattleHUD / GeneralQuickView 可共用的 `A2 HUD 頭像裁片 family`。這張卡處理的是：

- 小尺寸 HUD 頭像的裁切語言
- 頭像旁 badge 的疊加規則
- 對自動生成流程有用的 prompt DNA

## 研究結論

### 1. 目前 BattleHUD 的真實約束

- `playerPortrait` / `enemyPortrait` 都是 `64x64`
- 這兩個 slot 在 topbar 內，旁邊直接接角色名與城血資訊
- 頭像不能走完整立繪思路，而是要優先服務：
  - `24~32px` 縮圖辨識
  - 左右對稱配置
  - 可掛 1 個小型 badge

### 2. A2 HUD 頭像裁片 family 的核心特徵

- 主視覺是「頭部 + 肩線 + 一個清楚的識別配件」
- 臉部需佔最終裁片寬度 `40%~52%`
- 視線與武器不得切斷臉部辨識
- 背景必須壓成深色或透明，不可保留完整場景
- 裁片外緣需預留 badge 落點，不可把關鍵五官放在左下角或右下角

### 3. 不合格來源的典型症狀

- 直接塞完整立繪，導致縮成 `64x64` 時只剩胸甲與武器
- 把頭像當卡面，保留太多身體、披風或底座
- 臉部過暗、眼睛被頭巾或頭盔陰影吃掉
- 左下角被 badge 壓住後，輪廓辨識直接崩掉
