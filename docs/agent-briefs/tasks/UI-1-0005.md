## 開單原因

`docs/UI參考圖品質分析.md (doc_ui_0051)` (doc_ui_0051) §2.6 定義「水平 Tab 框體家族」，但目前 `assets/resources/sprites/ui_families/common/tab/` 路徑完全缺失，導致：
- 畫面無法呈現正確的 Tab 選中 / 未選中視覺切換效果
- 框體分類完整性評分停在 3/10，Tab 是第一個要補全的缺失家族

本卡為 §7.3 優先改善行動中「補全框體分類」的直接措施。

## 完整描述

- **目標路徑**：`assets/resources/sprites/ui_families/common/tab/`
- `tab_active.png`（96×48）
  - 金色漸層填充：`#FFE064`（頂） → `#B48014`（底）
  - 僅上兩角圓角 R=8，下緣平直（無縫銜接面板頂邊）
  - 金色描邊 2px（`#C9A028`）
  - 9-slice border `[8, 12, 0, 12]`
- `tab_inactive.png`（96×48）
  - 暗灰色填充：`#222222`，alpha=216（約 85%）
  - dim 金色描邊 `#4D4635`，1px
  - 同 tab_active 上兩角圓角 R=8，下緣平直
  - 9-slice border `[8, 12, 0, 12]`
