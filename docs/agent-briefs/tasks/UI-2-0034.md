# [UI-2-0034] TigerTallyPanel 綁定 card art / rarity / type badge

## 背景

`UI-2-0030` 已確認 TigerTally 的 layout 與 skin 已留出以下 slot：

- `tally.card.art`
- `tally.card.rarity.normal / rare / epic`
- `tally.badge.type`

但 `TigerTallyPanel.ts` 目前只綁定名稱、攻擊、血量、cost 與 disabled mask，尚未真正依資料切換：

- card art
- rarity border
- unit type badge

## 目標

1. 依 `TallyCardData` 的資料動態套用 `ArtBg*`。
2. 依 `rarity` 切換 `RarityBorder*`。
3. 依 `unitType` 切換 `UnitTypeBadge*`。
4. 全部缺件時需沿用 `UI-2-0031` 的 fallback 契約。

## 參考依據

- `assets/resources/ui-spec/layouts/tiger-tally-main.json`
- `assets/resources/ui-spec/skins/tiger-tally-default.json`
- `assets/scripts/ui/components/TigerTallyPanel.ts`
- `docs/agent-briefs/tasks/UI-2-0030.md (doc_task_0053)` (doc_task_0053)
- `docs/agent-briefs/tasks/UI-2-0031.md (doc_task_0054)` (doc_task_0054)

## 驗收條件

- TigerTally 四張卡都能依資料切換 rarity 框。
- unit type badge 能依 `unitType` 顯示對應圖，缺圖時安全 fallback。
- card art 若未提供正式圖，至少能穩定退回 `tally_card_art_placeholder`，不出現 missing asset error。
