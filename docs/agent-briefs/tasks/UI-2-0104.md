# UI-2-0104 Portrait Stage & Illustration Finish

## 目標

把 `GeneralDetailOverview` 左側英雄立繪舞台（hero stage）的視覺規則**正式凍結**，使其成為 `detail-split family` 的可複用 baseline，供後續新投 / 換角色時直接套用，不重複調整。

## 現況

| Slot | 現值（v10） | 說明 |
|------|-----------|------|
| `gdv3.portrait.carrier` | `portrait_frame_final` | 立繪 frame 框，已確認為 final |
| `gdv3.portrait.backdrop` | opacity 0.28 | 背景疊層，輕量存在感 |
| `gdv3.portrait.shadow` | opacity 0.0 | 已歸零，不使用 |
| `gdv3.portrait.glow` | opacity 0.0 | 已歸零，不使用 |
| `gdv3.portrait.image` | 透過 `_loadPortrait()` 動態載入 | `sprites/generals/{id}_portrait` |

## Hero Stage 凍結規範

### 立繪載入 Seam（`GeneralDetailOverviewShell._loadPortrait()`）
- portraitResource 路徑格式：`sprites/generals/{id.replace(/-/g, '_')}_portrait`
- 立繪資源不存在時：顯示 fallback，不 crash，不 console.error
- 載入後 anchor 對齊：以 `(0.5, 0)` 底部居中為基準（如有偏移，須更新 skin slot 或 script）

### 立繪裁切 & Fit 規則
- **Fit 模式**：人物主體（上半身到腰部以上）必須在可視框內完整呈現
- **構圖優先區**：臉部 + 武器在中上 1/2 區域；底部允許 trim 至腰部
- **安全邊距**：頭頂距 frame 頂邊保留至少 8px，兩側保留至少 12px
- **人物主體不可被 frame / backdrop 遮擋**：`carrier` 始終在 `image` 後層（z-order）
- **換角色不破版**：不同身形（瘦高 / 魁梧 / 女性）均適用；不允許因角色切換而調整 carrier 尺寸

### 視覺層次與權重規則
- carrier（frame）：裝飾輔助，不可比人物更搶眼
- backdrop：微存在感（opacity ≈ 0.2–0.35），絕對不可覆蓋人物輪廓
- shadow / glow：目前 freeze 為關閉（opacity 0.0），如未來需要激活，須另開卡
- 與右側資訊卡的主次關係：人物為視覺主體，資訊卡為閱讀輔助；兩者不可互搶焦點

### Backdrop 色調規則（`gdv3.portrait.backdrop`）
- 色調偏中性至暗棕，避免大片高飽和填色
- opacity 範圍：0.2–0.35，正式值為 0.28（當前設定，已 freeze）
- 背景變化應跟隨 rarityTier 色彩系：legendary → 暖棕金 / epic → 深紫焦棕 / rare → 銀灰藍 / common → 淺石灰

### `portrait_frame_final` 使用規範
- 此資產為 9-slice 不可整框拉伸（`portrait_frame_final` 的 9-slice 邊距依資產真實值設定）
- 不允許在未評估 9-slice 合規性的情況下更換 frame 資產
- 若需要新的 frame 形狀，須另行設計並與 `portrait_frame_final` 家族一致

## 多角色相容性驗收

換入以下類型角色時，hero stage 不應破版：
- 典型男性武將（zhang-fei，魁梧）
- 典型男性武Bản謀士（zhuge-liang，修長）
- 女性角色（預期 diao-chan，纖細）
- 低 tier 角色（gongsun-zan，common）
