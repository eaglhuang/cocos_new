## 開單原因
`UI-2-0007` 涵蓋 19 個 skin，直接整包改動不利於 review 與回歸。首波先處理 nav / popup / duel 類畫面，建立 `*.shadow` slot 的命名與引用模板，讓後續波次可以直接複用。

## 完整描述
1. 更新下列 skin：`lobby-main-default.json`、`duel-challenge-default.json`、`result-popup-default.json`、`network-status-default.json`、`toast-message-default.json`。
2. 為主要容器或卡片新增對應的 `*.shadow` slot，統一引用 `sprites/ui_families/common/shadow/shadow_01`。
3. `spriteType` 一律使用 `sliced`，`border` 統一為 `[20,20,20,20]`，必要時補 `allowAutoAtlas: true`。
4. 命名優先對齊現有主視覺 slot，例如 `popup.card.shadow`、`duel.root.shadow`、`toast.shadow`。
5. 執行 JSON 契約驗證與 manifest 狀態同步，並在 notes 記錄實際完成的 slot 清單。
