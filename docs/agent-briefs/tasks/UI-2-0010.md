## 開單原因
即使 shadow 素材與 slot 都存在，若 `UIPreviewBuilder` 不支援自動渲染陰影層，設計資產仍然無法真正出現在預覽與 runtime 中。

## 完整描述
1. 在 `UIPreviewBuilder` 中定義 shadow layer 的建立與掛載順序。
2. 支援從 skin slot 讀取 shadow sprite、offset、opacity 等設定。
3. 確保 shadow 位於 frame 下方，不影響互動節點。
4. 執行 acceptance 與必要的 UI 預覽確認。

## 目前進度備註
1. 已在 `UIPreviewBuilder` 補上 `container + skinSlot` 背景處理、shadow slot 自動推導，以及非 `Layout` parent 的 shadow sibling 建立。
2. 本次再補上 detached shadow host，讓 parent 帶 `Layout` 的節點也能 layout-safe 渲染 shadow，而不被排版系統納入幾何計算。
3. `node tools_node/run-acceptance.js` 全綠；剩餘工作集中在 legacy `support-card` schema 與人工 preview QA，已由 [UI-2-0016](UI-2-0016.md) (doc_task_0040) 接續。

2. 無 shadow slot 的畫面不應報錯。
3. acceptance 驗證通過。
