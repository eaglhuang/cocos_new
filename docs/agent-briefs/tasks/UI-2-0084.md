# UI-2-0084 — FrameRecipe / MaterialRecipe 契約層

## 分派理由

這張卡適合 Agent1，因為它本質是 UI 契約與 runtime/tooling 底座，不是單純美術產出。需要同時碰 schema、TS 型別、驗證器與載入流程。

## 背景

目前專案已經有 template、content contract、strict validator，但參考圖的高品質語法仍主要停留在分析文件：`frame + bleed + shadow + fill + ornament` 還不是正式契約。

沒有 recipe 層，後續的 layered theme、AI 資產治理與 compare board 都只能各自猜測視覺語意。

## 目標

建立正式的 `FrameRecipe` / `MaterialRecipe`，讓高品質 UI 視覺語法可被：

1. skin / screen 引用
2. validator 檢查
3. scaffolder 預填
4. Agent 自動推理與回寫

## 推薦執行順序

### Wave 1：基礎語法層

1. `UI-2-0084` — FrameRecipe / MaterialRecipe 契約層
2. `UI-2-0085` — Layered Skin Theme Stack

這一波先把「高品質 UI 的語言」與「runtime 疊層載入方式」定出來。沒有這兩層，後面的 QA、AI 治理與資產 pilot 都會失去共同語意。

### Wave 2：中介契約 + 品質門檻

3. `UI-2-0088` — Proof Contract / Design AST 中間格式
4. `UI-2-0086` — Visual Asset QA Rule Engine

這一波把設計來源轉成 machine-readable 契約，並補上資產級品質 gate。

### Wave 3：驗收與治理流程

5. `UI-2-0089` — AI 生圖 ArtRecipe 治理
6. `UI-2-0087` — Golden Preview Baseline + Compare Board
7. `UI-2-0090` — UI Skill Orchestration Pack

這一波建立 AI 資產可追溯性、固定 compare board 驗收，以及 skill 編排邊界。

### Wave 4：試點證明

8. `UI-2-0091` — 高品質框體家族 Pilot 資產包

最後用 pilot family 驗證整套新架構不是只停在文件，而是能真的產出高品質、可量產、可驗收的資產。

## 可並行批次

### Agent1 熱路徑批次

#### Batch A1-1：Recipe Vocabulary 與 Schema
- 定義 `FrameRecipe` / `MaterialRecipe` 的欄位詞彙表
- 決定 `frame / bleed / shadow / fill / ornament / stateVariants` 的最小結構
- 明確規範 token / slot / material 的責任邊界

#### Batch A1-2：Canonical Family Matrix
- 把 dark-metal / parchment / gold-cta / destructive / tab 五大 family 收斂成 canonical matrix
- 補 family 差異：圓角、邊厚、bleed 強度、shadow 語意、典型用途

#### Batch A1-3：Runtime 接點設計
- 定義 recipe 要掛在哪一層：skin、screen、shared recipe registry
- 預設向後相容策略，避免既有 manifest 全面重寫

#### Batch A1-4：Validator Hook Design
- 定義哪些 recipe 屬性能被 `validate-ui-specs` / asset QA 使用
- 先切出 rule id 與可配置 threshold，為 `UI-2-0086` 預留接點

### Agent2 可提前平行準備的批次

#### Batch A2-1：Reference Family Mapping
- 重新整理參考圖中的 family 對照表
- 為 5 個 canonical family 各找 2~3 個代表畫面與反例
- 產出之後 compare board 與 pilot 要用的 baseline 候選 screen 清單

#### Batch A2-2：AI 資產治理前置盤點
- 先列出「可 AI 生」與「不可直接 AI 生」的資產類型草案
- 優先盤點：paper noise、metal grain、bleed overlay、corner ornament、背景水墨層
- 這批成果後續直接餵給 `UI-2-0089`

#### Batch A2-3：Pilot 驗收樣板前置
- 先定義 pilot compare board 要看的維度：框邊自然度、bleed 柔和度、字色對比、陰影層、背景融合
- 決定 dark-metal / parchment / gold-cta 各要在哪張試點畫面驗證
