# UI-2-0096 GeneralDetail Header Chrome Formal Production

## 目標

為人物頁 `GeneralDetailOverview` 生產正式的 header chrome 家族，取代目前過渡使用的 `panel_header_cap_*` 與舊 ornament 選件。

## Art Direction

- 核心用途是 `Jade-Parchment Header`
- 必須維持 `bleed + band + frame` 三層
- jade 面積低於 header 總量 30%
- 不能變成厚重深綠橫條
- 不能用大型 AI ornament 生硬壓在標題上
- 若來源圖是四角完整花角 + 四邊連續 ornament 的整框，禁止直接當 9-slice header frame 使用；必須先拆成 `cap / band / frame` 分件，或重畫 stretch-safe 中段

## 生產範圍

1. `header_cap_left_formal`
2. `header_cap_right_formal`
3. `header_center_accent_formal`（若需要）

## 交付幾何規格

- Header chrome 預設採 **`corner/cap + edge/band + center/fill` 分件法**，不是整張 ornate frame 直接切九宮。
- `header_cap_left_formal` / `header_cap_right_formal` 是固定尺寸角件或端帽，不可拉伸。
- `header_center_accent_formal` 若存在，必須只承擔中央紋章或小型 accent，不可把整條邊紋畫進去。
- 可拉伸的部分只能是 `band / fill / edge` 的 stretch-safe 中段；中段必須低紋理、低節奏、可水平延展。
- 若來源圖是完整花角整框，必須先人工拆成 `corner + edge + center/fill`，或另外重畫 9-slice 專用版本。
- 交付命名要反映分件用途，不接受 `header_full_frame_final.png` 這種整框檔名直接進 runtime。

## AI 生成禁則

- 禁止 prompt 要求 AI 生成「完整 header ornate frame」後再交給工程硬切。
- 禁止生成一整條含左右角、中央徽章、長邊連續紋樣的單張圖，因為這類圖不是 stretch-safe。
- AI prompt 必須明講產物類型：`left cap only`、`right cap only`、`stretch-safe center band only`、`isolated asset`。
- 若邊紋節奏需要保留，應改成獨立 edge tile 或固定寬度 band，不可讓 AI 在一張圖上同時解決全部幾何問題。

## Reject If

- 名字被 ornament 壓住
- 看起來像商城橫幅而不是人物頁匾額
- 深色佔比過高，破壞 parchment-first 閱讀秩序
- 把整圈 ornate frame 直接九宮拉伸，導致邊紋、角花、回字紋被拉扁或節奏斷裂
- 交付物仍是整張 ornate frame，而不是 `cap / band / frame` 可接線分件
