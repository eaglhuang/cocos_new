## 開單原因
`UI-2-0010` 的第一版 runtime 已讓非 `Layout` parent 的 shadow slot 能渲染，但實際盤點後，仍有多個已補 slot 的畫面因 `popup` 類型、parent 帶 `Layout`，或 legacy layout / itemTemplate 流程而無法顯示 shadow，不能留在同一卡的「完成」狀態裡模糊帶過。

## 完整描述
1. 補齊 `UIPreviewBuilder` 的 layout-safe shadow host，處理 parent 帶 `Layout` 的節點，例如 battle-log / tiger-tally 類節點。
2. 確認 popup 類 case 是否仍受 runtime 阻擋；若未阻擋，將其自 blocker 清單移除並記錄原因。
3. 盤點並補齊 legacy 畫面或非標準 schema 的 shadow 覆蓋，目前焦點為 `support-card`。
4. 維持 acceptance 全綠，並補人工 preview 確認。

## 如何驗證
1. parent 含 `Layout` 的節點在預覽時能正確生成陰影層，且不破壞原本的 layout 幾何。
2. `support-card` 若維持現行 legacy schema，需明確記錄支援策略；若接入，預覽需看到對應 shadow。
3. acceptance 驗證通過，且互動與顯示順序不回歸。

## Unit Test 建議
不必補純邏輯 unit test；至少維持 acceptance 全綠，並補一個小型 preview regression 或人工對照截圖。
