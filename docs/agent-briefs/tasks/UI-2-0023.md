# [UI-2-0023] Provide Reproducible Headless Preview Capture For D-Stage QA

## 摘要

這張卡的目標是讓 Agent2 不需要手動進 Inspector，就能對 `LoadingScene.previewTarget` 進行可重現的 screenshot capture。

現在這個目標已經達成：

1. `tools_node/capture-ui-screens.js` 可直接用 `puppeteer-core` 啟動 headless browser。
2. 工具會自動注入 `previewMode=true`、`previewTarget`、`LoadingScene.scene` uuid。
3. 工具會在 capture 前觸發 `asset-db/refresh`，並對可重試的 preview 問題做一次 retry。
4. 工具已改用 `domcontentloaded` + `window.__UI_CAPTURE_STATE__` 當成功條件，避開 Cocos Editor 預覽頁常駐長連線造成的 `networkidle2` timeout。
5. 成功與失敗都會留下 artifact，便於 Agent2 與 Agent1 分流處理。

## 驗證結果

已成功輸出：

- `artifacts/ui-qa/UI-2-0023/LobbyMain.png`
- `artifacts/ui-qa/UI-2-0023/ShopMain.png`
- `artifacts/ui-qa/UI-2-0023/DuelChallenge.png`

失敗時可留下：

- `*-debug.png`
- `*-debug-state.json`

## 後續

`Gacha` 目前仍無法成功 capture，但已確認不是 `UI-2-0023` 的工具缺陷。現況是：

1. `gacha-screen.json` 的 compound screen contract 不相容 `loadFullScreen()`。
2. 即使改由 preview-only `gacha-main-screen.json` 直指 `gacha-main`，`gacha-main.json` 仍是 legacy layout schema，與 `UIPreviewBuilder` 不相容。

該問題已拆到 [UI-2-0025](UI-2-0025.md) (doc_task_0049)。
