## 背景與動機

### §9.6 驗證腳本之缺口

`UI-1-0012` 建立的 `validate-skin-contracts.js` 涵蓋 §9.6 中 5 條 JSON 可驗證規則（R1~R5），但 **§3.3「金色邊框永遠存在」** 需要像素層級分析——只有查看 PNG 檔案的 border 區域是否有暖金/銅色像素才能確認。

原先 UI-1-0012 對此條規則的處理是「目視規則，腳本做路徑存在確認」，屬於降級實作。本任務將其升級為**真正的像素掃描**。

### §3.3 規格要求

> **§3.3 金色邊框永遠存在**：不論深淺框體，至少有 1px 金色/銅色邊界線。這是保持全局一致性的關鍵。

## 實作內容

### 腳本位置

`tools/validate-frame-sprites.ps1`

### 掃描目標

僅針對「預期含金色邊框」的 frame/panel sprite（不包含刻意使用稀有度色或紅色的 badge/item_cell/warning 族群）：

| Sprite 路徑 | 說明 |
|---|---|
| `dark_metal/frame.png` | §2.1 主框體 |
| `parchment/frame.png` | §2.2 羊皮紙框 |
| `gold_cta/frame.png` | §2.4 金色 CTA 框 |
| `progress_bar/track.png` | §4 軌道框 |
| `tab/tab_active.png` | §2.6 啟用 Tab |
| `diamond_tab/diamond_active.png` | §2.6 菱形 Tab |
| `circle_icon/circle_normal.png` | §6.1/6.2/6.4 圓形按鈕 |
| `nav_ink/btn_primary_normal.png` | §9.1 nav 家族 |
| `equipment/btn_primary_normal.png` | §9.1 equipment 家族 |
| `commerce/btn_primary_normal.png` | §9.1 commerce 家族 |

### 金色/暖銅色像素判斷準則

- R > 130, G > 80, B < 180
- R > B + 40（確保暖色，排除灰藍調）
- R ≥ G - 40（排除純綠）
- Pixel Alpha > 50（排除透明邊緣）

掃描區域：sprite 外邊緣 4px 環形區域

通過閾值：邊緣區域找到 **≥ 5 個暖金像素** 即 PASS
