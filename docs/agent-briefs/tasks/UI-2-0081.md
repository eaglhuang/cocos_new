# UI-2-0081 — Screen → Component Auto-Scaffolder 實作

## 背景

`scaffold-ui-spec-family.js` 解決了 JSON 骨架生成，但從 `screen.json` 到可運行的 TypeScript Panel 仍是手動步驟：

1. 人工建立 `SomePanel.ts`（extends UIPreviewBuilder）
2. 人工寫 `onReady(binder)` 的查找骨架
3. 人工在 UIConfig.ts 新增 UIID enum 成員

這三步對人類開發者是 30 分鐘，對 AI Agent 是每次都要重新推斷，缺乏確定性。

Unity 對照：等同於有了 ScriptableObject 設定，但沒有自動產出對應 MonoBehaviour 骨架的工具。

## 目標

一行指令，從 screen config 走到可執行的 Panel TypeScript 骨架：
```bash
node tools_node/scaffold-ui-component.js --screen lobby-main-screen
```

## 實作清單

### 1. scaffold-ui-component.js（新建）
位置：`tools_node/scaffold-ui-component.js`

功能：
- 讀取 `assets/resources/ui-spec/screens/{screenId}.json`
- 從 `familyId`（或 `contentRequirements.familyId`）選擇對應 template
- 用 `tools_node/templates/{family}-panel.template.ts` 產出 Panel TS
- 在 `UIConfig.ts` 的 UIID enum 中自動新增成員（標 TODO）

選項：
- `--screen <id>`：screen spec id（必填）
- `--family <id>`：強制指定 family（可選；預設從 screen spec 讀）
- `--out <dir>`：輸出目錄（可選；預設 `assets/scripts/ui/components/`）
- `--dry-run`：只預覽，不寫檔

### 2. Panel 樣板（新建）
位置：`tools_node/templates/`

| Family | 樣板檔 |
|--------|--------|
| `detail-split` | `detail-split-panel.template.ts` |
| `dialog-card` | `dialog-card-panel.template.ts` |
| `rail-list` | `rail-list-panel.template.ts` |
| `fullscreen-result` | `fullscreen-result-panel.template.ts` |

每個樣板含：
- `onReady(binder)` 骨架，依 content schema 的 `requiredFields` 預填 binder 查找
- `loadAndBuild()` 標準入口
- 必要 import
- JSDoc 說明 + Unity 對照

### 3. UIConfig.ts 自動新增 UIID entry
- 不覆寫現有 enum 內容
- 在 enum 末尾新增 `// TODO: 補 prefab 路徑` 標記的成員

## 驗收條件

1. `tools_node/scaffold-ui-component.js` 完成
2. 對 `lobby-main-screen` 執行後，能正確產出 `LobbyMainPanel.ts` 骨架
3. 產出的 TS 通過 `tsc --noEmit`（無型別錯誤）
4. 4 個 family-specific `.template.ts` 樣板存在於 `tools_node/templates/`
5. `--dry-run` 模式正常工作
6. encoding check 通過
