# [UI-2-0048] 建立功能語意色票規格

## 目標

把參考圖中的跨畫面用色規律整理成正式功能語意色票，讓之後的 icon、badge、CTA、progress、rarity 與 placeholder 都能依角色選色，而不是逐張自由發揮。

## 完成內容

- 建立正式規格：
  - [functional-color-palette-spec.md](C:\Users\User\3KLife\artifacts\ui-qa\UI-2-0048\functional-color-palette-spec.md)
- 統一定義的色票角色：
  - `cta-gold`
  - `danger-red`
  - `support-green`
  - `info-cyan`
  - `rare-purple`
  - `neutral-ink`
  - `paper-black`
  - `battle-light`
- 交代各系統使用條件：
  - Lobby
  - 武將介紹 / QuickView
  - Battle
  - Reward / Rarity

## 主要規則

- 高價值、主 CTA、稀有高亮優先使用 `cta-gold`
- 危險、敵方、刪除、破壞類使用 `danger-red`
- 守護、成功、補益、友軍傾向使用 `support-green`
- 資訊、連線、冷卻、狀態 pin 使用 `info-cyan`
- 稀有度、法術感、收藏高階可用 `rare-purple`
- disabled / placeholder / 次要資訊回到 `neutral-ink` 或 `paper-black`
