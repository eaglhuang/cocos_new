# [UI-2-0031] 建立戰場 UI Fallback 規範

## 目標

定義 BattleScene 缺圖時的統一 fallback 規則，避免不同面板各自退回不同 placeholder、純色或空白狀態，讓畫面至少保持一致、可讀、不中斷。

## 規範原則

1. 缺圖時優先退回同類型 placeholder，不直接顯示空白。
2. 若該類型沒有專用 placeholder，才退回 skin 內的 `_fallback` 顏色。
3. 缺圖不得中斷頁面建立；只允許 warning，不允許 throw。
4. 同一類資產要有固定命名規則，避免每個面板各自拼路徑。

## BattleScene Fallback 規則

### 1. 武將 portrait

- 正式路徑：
  - `sprites/generals/{snake_case_id}_portrait`
- player 缺圖 fallback：
  - `sprites/battle/portrait_player_placeholder`
- enemy 缺圖 fallback：
  - `sprites/battle/portrait_enemy_placeholder`

### 2. unit info type icon

- 正式路徑：
  - 後續可依兵種切到 `unitinfo.type.{troopType}` 或等價 sprite family
- 當前缺圖 fallback：
  - `sprites/battle/unitinfo_type_icon`
- 若 sprite 仍不存在：
  - 再退到 `unit-info-panel-default.json` 內 `_fallback` 色塊邏輯

### 3. tiger tally card art / badge

- card art 正式路徑：
  - 依卡片 template 或部隊類型對應正式卡圖
- card art 缺圖 fallback：
  - `sprites/battle/tally_card_art_placeholder`
- rarity badge 缺圖 fallback：
  - `tally_rarity_normal`
- type badge 缺圖 fallback：
  - `sprites/battle/tally_badge_type`

### 4. action / utility icon

- 正式路徑：
  - `action_util_endturn`
  - `action_util_tactics`
  - `action_util_duel`
  - `action_skill_circle*`
  - `action_sp_ring*`
  - `action_ult_bg*`
- 缺圖 fallback：
  - 優先保留 skin 內既有 `_fallback` 顏色
  - 不額外建立空白 sprite

## 實作要求

- runtime 套用 fallback 時，需記錄一次 warning，內容至少包含：
  - slot 名稱
  - 嘗試載入的正式路徑
  - 實際退回的 fallback 路徑或顏色
- 同一 slot 不要重複洗警告。

## 後續接手

- `UI-2-0033`：BattleHUD portrait 動態載入與 fallback 契約
- 其他 battle runtime 若新增動態 card art / badge，也必須沿用這份規則，不可另外自訂

## Family 保留原則

- `F7` 類缺圖時，應退到中性厚邊 badge 或既有 battle micro placeholder，不應退成獎勵格物件圖。
- `F8` 類缺圖時，應保留按鈕或工具 glyph 的 carrier 感，不應退成收藏卡圖。
- `F6` 類缺圖時，應退到 card art / rarity placeholder，不應退成單色工具 icon。
- `F5` 類缺圖時，應退到 commodity silhouette 或 reward-cell placeholder，確保數量仍是第一閱讀點。
