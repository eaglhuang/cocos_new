## 背景

`assets/scripts/ui/core/UIPreviewBuilder.ts` 目前屬於高風險中文密集檔，已經出現過編碼災難疑慮。檔案同時承擔 orchestration、中文字串、diagnostics、layout、style 組裝等多重責任，導致：

1. 非 ASCII 字元量過高，任何整檔覆寫都容易造成大面積損害。
2. 多個 Agent 同時修改時，merge 衝突面積過大。
3. review 與回救都過於困難。

## 目標

在不改變既有 runtime 行為的前提下，把 `UIPreviewBuilder.ts` 逐步拆成較小的 helper 模組，至少朝以下方向收斂：

1. `UIPreviewBuilder.ts`
2. `UIPreviewTextCatalog.ts`
3. `UIPreviewDiagnostics.ts`
4. `UIPreviewLayoutBuilder.ts`
5. `UIPreviewStyleBuilder.ts`

## 執行原則

1. 目前若仍有人在直接修 `UIPreviewBuilder.ts`，本卡不得同時開始。
2. 每次只抽一到兩個責任，不做大爆改。
3. 優先抽離中文字串與 diagnostics，再處理 layout/style。
4. 新檔一律納入 UTF-8 防災規則與後續 baseline。

## 驗收標準

1. `UIPreviewBuilder.ts` 行數、責任範圍與非 ASCII 密度明顯下降。
2. 行為不變，`npm run check:acceptance` 通過。
3. `npm run check:encoding` 通過。
4. 新拆出的 helper 檔命名清楚，能讓多人協作時降低衝突面積。
