## 開單原因
Agent2 在執行 D-1~D-3 截圖前盤點實際預覽入口，確認 `lobby-main-screen`、`shop-main-screen`、`gacha-screen`、`duel-challenge-screen` 雖然已有 screen/layout/skin JSON，但專案內沒有對應的 `UISpecLoader.loadFullScreen(...)` 掛載點。這代表目前無法在 Cocos Editor 內開出真正的 screen-driven 畫面，D 階段 QA 會被迫驗 legacy 手刻畫面，結果不可信。

Unity 對照：這就像只有 prefab 規格與 ScriptableObject 都好了，但沒有任何一個測試 scene / preview bootstrap 真正把 prefab instantiate 出來；此時做 Game View 截圖，驗的不是最終版本。

## 完整描述
1. 為 `lobby-main-screen` 建立可在 Editor 內直接預覽與截圖的 screen-driven host。
2. 同步補齊 `shop-main-screen`、`gacha-screen`、`duel-challenge-screen` 的實際掛載點，避免 D-2 / D-3 重複卡住。
3. host 必須走 `UISpecLoader.loadFullScreen(...)` 與既有 `UIPreviewBuilder` 流程，不能再落回舊版手刻 UI。
4. 補完後要把可用入口回填到 Agent2 文件，讓 `agent2-visual-qa-playbook` 與各張 D 卡都能指向正確路徑。
