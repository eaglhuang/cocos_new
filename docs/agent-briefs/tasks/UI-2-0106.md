# UI-2-0106 UI System Freeze + Runtime QA + Docs Backwrite

## 目標

把 `GeneralDetailOverview` 的所有視覺子系統**統一驗收**、做出 tier/icon policy 的最終決策、執行 runtime smoke matrix，並完成所有 docs backwrite，讓後續 detail-split 畫面可直接引用本案 baseline。

## 待做決策：Tier 導視系統

本卡開始前，必須明確回答以下決策：

> **GeneralDetailOverview 的稀有度導視，是：**
> - **Option A**: `tier-mark dock` 為主、`badge` 為輔（tier-mark 是圖文 glyph，badge 是小型 accent）
> - **Option B**: `badge underlay + label` 雙層（badge 是主視覺，label 加強可讀性）

**建議**：以 `docs/UI-icon-factory-workflow.md (doc_ui_0033)` (doc_ui_0033) 的 `icon family` 框架為依據，依 `structureMode` 選擇。若 badge 已成為獨立 family 資產（UI-2-0097 的輸出），則 Option B 更自然。若特別強調 tier 識別的 dock 系統，則選 Option A。

無論何種決策，必須填寫以下 `iconFamilyId` 欄位定義：

```json
{
  "iconFamilyId": "rarity-badge",
  "structureMode": "underlay|glyph",
  "runtimeTextPolicy": "none|label|label+badge",
  "underlayPolicy": "fixed-size|nine-slice-safe"
}
```

## QA Rubric（統一通過準則）

### Pass 條件（必須全滿足）
- [ ] Runtime 維持 clean pass（0 residuals，0 console errors）
- [ ] 所有 showcase samples 可正常載入（portrait fallback 允許，crash 不允許）
- [ ] rarityTier 視覺 accent 色正確（5 tier × 3 色點驗收：header / badge / crest glow）
- [ ] crestState 三態（rumored / revealed / awakened）在至少各一個 sample 下正確顯示
- [ ] header 無燒字，jade < 30%
- [ ] crest 無 badge 語義（無 star / chip / rating 形式）
- [ ] portrait stage：人物是視覺主體，換角不破版

### Fail 條件（任一即 fail）
- 出現 console error 或 TypeError
- 有任何欄位顯示 "???" 或空白
- portrait/crest/badge 任何資產 load 失敗導致 crash
- header jade 超過 30%
- crest 顯示為 badge 樣式
