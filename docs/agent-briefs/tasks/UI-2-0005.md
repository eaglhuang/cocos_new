## 開單原因
Tab 與 segmented control 類 UI 若要沿用 `button-skin`，必須能表示 `selected` 態，否則只能額外做 screen-specific hack。

## 完整描述
1. 更新 UI spec type 定義，讓 `button-skin` 可宣告 `selected`。
2. 在 `UISkinResolver` 與 `UIPreviewBuilder` 加入 selected state 解析與快取。
3. 若 skin 未提供 `selected`，自動 fallback 到 `normal`。

## 如何驗證
1. `button-skin.selected` 欄位可被 acceptance 流程正確解析。
2. runtime 可切換 selected state 且未提供時不報錯。

## Unit Test 建議
不需要新增獨立 unit test；沿用既有 acceptance 驗證與預覽檢查即可。
