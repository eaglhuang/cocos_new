# [UI-2-0033] BattleHUD portrait 改為動態載入 generals 資產

## 背景

`UI-2-0029` 已確認目前 roster 的 5 張武將頭像都存在於 `assets/resources/sprites/generals/`，但 BattleHUD 的 skin slot 仍固定指向：

- `sprites/battle/portrait_player_placeholder/spriteFrame`
- `sprites/battle/portrait_enemy_placeholder/spriteFrame`

也就是說，現在畫面上沒吃到真正 portrait 的主因不是缺圖，而是 runtime 還沒把武將 id 對應到現有頭像資產。

## 目標

1. BattleHUD 依當前 player / enemy general id 動態載入 portrait。
2. 路徑規則與其他 UI 保持一致，優先沿用 `sprites/generals/{snake_case_id}_portrait`。
3. 若找不到對應資產，再退回 player / enemy placeholder。

## 參考依據

- `assets/resources/ui-spec/layouts/battle-hud-main.json`
- `assets/resources/ui-spec/skins/battle-hud-default.json`
- `assets/resources/data/generals.json`
- `assets/resources/sprites/generals/*.png`
- `docs/agent-briefs/tasks/UI-2-0029.md`

## 驗收條件

- BattleHUD 左右頭像不再固定顯示 placeholder。
- `zhang-fei / guan-yu / lu-bu / cao-cao / zhao-yun` 至少能正確載入現有 portrait。
- 找不到圖時仍能安全回退到 placeholder，不出現 missing asset error。
