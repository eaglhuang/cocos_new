# UI-2-0105 UI Chrome & Crest Family Freeze

## 目標

整合 `UI-2-0094`（finish assets）、`UI-2-0095`（crest formal）、`UI-2-0096`（header chrome）、`UI-2-0097`（badge formal）四張卡的成果，正式凍結：

1. **Header Rarity Plaque** 三層結構規則
2. **Crest Medallion** 家族規則（face / ring / socket）
3. **Alias Policy** 與 provisional asset 的退場 / 留場條件
4. **Reject List**（燒字、整框拉伸、jade 過重、badge 語義誤入 crest）

## Header Rarity Plaque 凍結規範

### 三層結構：base / fill / band / cap / frame

| 層次 | Slot | 說明 |
|------|------|------|
| base | `gdv3.rarity.band` | parchment 底材，提供 header 基礎質感 |
| fill | `gdv3.rarity.fill` | 色彩層，依 rarityTier 調整飽和度 |
| band | `gdv3.rarity.band`（overlay）| 橫向帶狀 accent，連結兩側 cap |
| cap.left | `gdv3.rarity.cap.left` | 左端蓋件，`panel_header_cap_l_v4j_r5` |
| cap.right | `gdv3.rarity.cap.right` | 右端蓋件，`panel_header_cap_r_v4j_r5` |
| frame | （整框） | **禁止整框 9-slice 拉伸**，使用拆件架構 |

### Jade < 30% 規則
- header 可見面積中，jade 色（#5C7A3E 系）佔比**不超過 30%**
- 超過 30% 必須降低飽和度或縮小 jade 區域
- parchment-first 原則：header 主材為米棕/羊皮紙調，jade 僅為 accent

### Center Accent 權重規則
- header 中央標題區（headerTitle + headerName）必須是視覺首焦點
- 裝飾件（cap / band）的視覺重量不可超過標題文字
- band 色調應比標題背景深一階，形成文字對比基礎

## Crest Medallion 家族凍結規範

### 組件分層：face / ring / socket

| 組件 | Slot | 正式資產 | 說明 |
|------|------|---------|------|
| face | `gdv3.bloodline.crest.face` | `crest_face_final` | 中央圖騰面，因角色非固定 |
| ring | `gdv3.bloodline.crest.frame` | `crest_ring_final` | 外圈環形紋，9-slice 不可用，固定尺寸 |
| socket | `gdv3.bloodline.crest.fill` | `crest_fill_v3` | 底層填充，提供嵌入感 |
| glow | `gdv3.bloodline.crest.glow` | 依 crestState 動態 | awakened 才啟用 |
| inner | `gdv3.bloodline.crest.inner` | 依 crestState 動態 | revealed / awakened 才啟用 |

### Crest Face 動態映射規則
- `crestFaceResource` 由 content state 提供，允許角色個別化
- **Proof bridge**（`proof/dragon_medallion_face_v1`）：允許暫用，但 v3_final 正式版本完成後必須退場
- 退場條件：`crest_face_formal` 家族完成量產後，一次性替換所有使用 proof bridge 的 state

### Crest 禁則
- **禁止 badge 語義誤入 crest**：crest 表達「命紋 / 血脈特質」，不是「稀有度等級 badge」
- **禁止把 crest 設計成 coin / medal / achievement badge** 形狀
- **禁止在 crest 上燒入固定文字**（AI 生成時若出現文字 artifacts，必須 reject）
- **禁止 crest ring 使用整框 9-slice 拉伸**

### Non-9-slice Ornate Frame 禁則
- 凡是四角有完整花角 + 四邊連續 ornament 的整框，不可直接九宮拉伸
- 處理方式：固定尺寸、拆角邊件，或改畫 stretch-safe 中段版本
- `portrait_frame_final` 的 9-slice 邊距已知，若其他 frame 不符此條件，禁止直接套用

### Reject List（不得再引入的類型）
- AI-generated 資產含可辨識英文或漢字文字 artifacts
- badge 語言的 crest（star rating / level chip / achievement ribbon）
- 整框 ornate frame 的 9-slice 拉伸應用
- jade 佔比 > 30% 的 header 方案
- 新增未經 proof review 的 `provisional` 前綴資產至正式 skin
