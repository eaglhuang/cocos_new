# [UI-2-0099] BattleScene 兵種符號 Icon Suite

## 目標

為 `TigerTallyPanel` 與 `UnitInfoPanel` 建立可重用的兵種符號 suite，將兵種語意從暫用 icon / fallback 收斂成正式 factory icon family。

## 範圍

- `cavalry / infantry / shield / archer / pikeman / engineer / medic / navy` 正式 glyph 與 underlay
- `TigerTally` 的 type badge
- `UnitInfoPanel` 的 type icon
- icon family / suite / member key 命名與 runtime 路徑
- `siege / smart` 記錄為規格保留成員，不在這輪假裝完成

## 不包含

- TigerTally 卡面主圖
- ATK / HP / Cost 小型數值 glyph
- BattleHUD 既有圖示回頭重做

## 尺寸契約

- `troopTypeGlyph`、`unitInfoTypeIcon`、`tigerTallyTypeBadge` 的 source master 預設一律為 `256x256` 方形。
- 生圖 prompt 必須明寫 `square icon composition for 256x256 UI use`，避免模型輸出成寬幅插圖或帶場景構圖。
- 本卡的 `80%` 一律指 `glyph logical box` 佔 `troopTypeBadgeUnderlay` 有效承載區的 80%，不是取金色筆畫的 `alpha bounds` 去直接縮放。
- `logical box` 是兵種符號在 `256x256` 畫布上的語意外接矩形；弓弦中空、盾牌內凹、細槍桿周圍留白，都屬於同一顆符號的邏輯框，不可因為透明區多就把符號做小。
- 正規化報表若要保留 `alpha bounds` 只能作技術觀測，最終成品的 family 尺度應以 `logical box` 為準。
- `troopTypeBadgeUnderlay` 若需要 AI 生成，也應與 glyph 同步採 `256x256` 方形 master，維持組裝對位。
- 若 BattleScene / TigerTally / UnitInfo 在實裝時需要不同顯示大小，應縮放整體 badge 成品或 prefab 節點，不回頭改壞 glyph family 的 `logical box` 基準。
- 除非任務卡後續明確增列大 badge / 宣傳用途，否則本卡不得要求 `512x512` 以上的大圖輸出。

## 風格要求

- 遵守 [美術風格規格書](/C:/Users/User/3KLife/docs/美術風格規格書.md (doc_art_0002)) (doc_art_0002) 的戰場語言：`deep-ink battlefield + cold tactical HUD + gold CTA`
- 遵守虎符系統規格與特種軍隊補遺，不得把真實兵種集合裁成只剩四種常規兵
- icon 只屬於內容層，underlay 屬於框體語言，不得把整塊 frame 烘進 glyph PNG
- 兵種 glyph 必須走「交通符號等級」的極簡筆畫語言：單一武器或單一主體、低細節、遠讀優先，不做插畫式輪廓或材質刻畫
- 視覺分層要維持：
  - 背景層（水墨山水渲染）
  - 半透明暗色蒙版
  - 框體層（金屬/羊皮紙 badge underlay）
  - 內容層（兵種符號）

## 驗收

- `UI-2-0098` 的 `TigerTally` / `UnitInfoPanel` 皆可切換到正式 troop-type suite
- 首批 member spec 明確覆蓋 `cavalry / infantry / shield / archer / pikeman / engineer / medic / navy`
- `siege / smart` 已被記錄為系統規格保留成員，而不是遺漏
- suite 有明確 `family id / structure mode / suite id / member key`
- runtime capture 可驗證 badge 與 type icon 的可讀性
- 不把整張 badge frame 烘進 glyph，保留 runtime underlay + glyph 組裝能力

## 完成紀錄（2026-04-09）

### Pipeline 說明

1. 來源切片：`slice-ui-image-components.js --merge-gap` 從 `Ancient_warfare_1.jpeg` 切出 10 個乾淨符號（proximity-merge 解決 medic 被分成兩塊的問題）
2. 去背透明化：對切出的裁片以角落取色距離閾值去背，存為 256×256 透明 PNG
3. 正規化：`normalize-troop-icons.ps1` 偵測 alpha bounds、取中心點置中、uniform scale 至 targetBox 80%
4. 合板：符號疊至 `troop_dark_plate_256.png` underlay，輸出 composited PNG
