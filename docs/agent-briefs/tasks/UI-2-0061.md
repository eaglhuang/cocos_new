# [UI-2-0061] Story / Bloodline / Crest 內容契約

## 目標

將 `日常人物頁 v3` overview shell 裡原本靠 fallback 字串拼出的內容，收斂成可量產、可重用、可回寫到資料源的正式 content contract。

## 契約欄位

1. `historicalAnecdote`
2. `bloodlineRumor`
3. `storyStripCells`
4. `crestHint`
5. `crestState`

## 已完成

1. `GeneralConfig` 已新增上述欄位與對應型別。
2. `storyStripCells` 已固定為六格語意槽位：`origin / faction / role / awakening / bloodline / future`。
3. `crestState` 已固定為 `placeholder / rumored / revealed / awakened`。
4. `GeneralDetailOverviewMapper.ts` 已把 bloodline body、crest title / hint、story cells 收斂成正式 overview data。
5. `GeneralDetailOverviewShell.ts` 已改為直接吃上述 contract，不再從 `title / source / bloodlineId` 臨時拼成 generic 內容。
6. `generals.json` 已補首批示範欄位，供 preview 與後續 proof 使用。

## 完成定義

1. 後續新角色不需要再改 overview shell 節點，只需填入正式 contract。
2. `UI-2-0060` 與後續其他人物 family 可直接沿用相同欄位，不必再重發明 story / crest schema。
3. 文件、manifest 與交叉索引已同步回寫。
