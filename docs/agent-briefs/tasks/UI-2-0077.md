# [UI-2-0077] i18n 語系 hardcoding 移除

## 開單原因

14 個元件直接硬編碼語系字串：

```typescript
const i18n = await this._specLoader.loadI18n('zh-TW');
```

受影響元件：
- GeneralDetailPanel, UnitInfoPanel, UltimateSelectPopup, DuelChallengePanel
- ResultPopup, BattleLogPanel, ToastMessage, BattleHUD, TigerTallyPanel
- GeneralQuickViewPanel, ActionCommandPanel, StyleCheckPanel
- GeneralPortraitPanel, GeneralListPanel

`I18nSystem` 已作為服務存在於 `ServiceLoader`，但其 `currentLocale` 沒有被任何元件查詢。若未來需要支援繁中/簡中/英文切換，需要逐一改 14 個檔案——這是典型的 shotgun surgery 反模式。

對照 Unity：等同把語言直接 hardcode 在每個 `Text` Component 的 `GetComponent<TextMesh>().text = zhTWString`，而非通過 `LocalizationManager.GetString(key)`。

## 完整描述

1. 在 `I18nSystem.ts` 加入 `currentLocale` 屬性（預設 `'zh-TW'`）：
   ```typescript
   export class I18nSystem {
       private _locale = 'zh-TW';
       get currentLocale(): string { return this._locale; }
       setLocale(locale: string): void { this._locale = locale; /* 廣播 */ }
   }
   ```
2. 各元件將：
   ```typescript
   // 改前
   const i18n = await this._specLoader.loadI18n('zh-TW');
   // 改後
   const locale = services().i18n.currentLocale;
   const i18n = await services().specLoader.loadI18n(locale);
   ```
3. （選做/Phase 2）`I18nSystem.setLocale()` 廣播 EventTarget 訊號，讓已開啟 UI 可重新載入文字

3. 各元件開啟時 i18n 文字顯示正確，與遷移前等同
