# [UI-2-0054] 人物與命鏡主視覺方向收斂

## 任務目標

把人物日常介紹、血脈逸聞、命鏡 Loading / 覺醒等高辨識度畫面，收斂成可以量產的美術與內容方向。這張卡不是只做單張圖，而是要定義可重複使用的視覺語氣、內容欄位與 family 邊界。

本輪已聚焦在命鏡 `fullscreen-result` family，先完成 shared skeleton 之後，再把四種 state 的內容契約補齊：

1. `loading`
2. `awakening`
3. `ascension`
4. `unowned-preview`

## 本輪收斂

- 命鏡過場共用 `bloodline-mirror-loading-main` / `bloodline-mirror-loading-default`
- 命鏡四種 state 已集中到 `assets/resources/ui-spec/content/bloodline-mirror-states-v1.json`
- 既有 `BloodlineMirrorLoading` / `BloodlineMirrorAwakening` screen 已補上 `content.source` 與 `content.state`
- 後續 `ascension` / `unowned-preview` 應沿用同一份 contract，不另開第二套 layout

## 驗收重點

1. 畫面骨架固定使用同一份 `fullscreen-result` template family
2. 狀態差異優先落在 `content contract`，不是再複製 layout
3. 命鏡文案能直接對齊 `StateBadge / Header / Tip / Progress / StoryStrip`
4. 正式規格與交叉索引已同步回寫
