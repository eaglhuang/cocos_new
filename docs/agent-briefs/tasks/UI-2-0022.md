## 開單原因
目前武將列表的實際畫面仍停留在 placeholder 階段：

1. `general-list-default.json` 的 fullscreen 背景雖已掛 `detail.bg.fullscreen = ui-spec/placeholders/bg_ink_main`，但視覺上仍缺少與 `general-detail` 同級的框體層次。
2. `general.header.bg`、`general.list.bg`、`general.row.bg` 仍是 `color-rect`，導致列表區塊看起來像白模表格，而非水墨 + 金屬 / 紙質融合的正式 UI。
3. 截圖中 `HeaderBar`、資料列與底部返回區沒有形成明確九宮格承載面，與 `keep.md` (doc_index_0011) 已定義的框體標準不一致。

現有證據顯示，這比較像是 skin 與既有 panel family 的接線缺口，而不是素材檔不存在。

## 完整描述
1. 盤點 `general-list-main.json` / `general-list-default.json`，確認哪些節點應改接既有 panel family，而不是維持 `color-rect`：
   - `detail.bg.fullscreen`
   - `general.header.bg`
   - `general.list.bg`
   - `general.row.bg`
   - `BtnBack`
2. 優先重用既有可用資產或 family：
   - `ui-spec/placeholders/bg_ink_main`
   - `sprites/common/popup_card_bg`
   - 既有 lobby/common header strip / section card 類資產（若存在）
3. 必要時補充 skin slot 命名，讓 `HeaderBar`、列表容器與列背景有可驗證、可維護的框體語言，而不是單純改色。
4. 以 `LobbyScene` 或對應 preview 流程做 smoke check，確認武將列表不再呈現純色表格感。
5. 若實作過程確認現有 common/lobby 資產不足以支撐 row/header/frame 語言，再補開 Agent2 的素材卡，不要在本卡內混入素材產製。
