# [UI-2-0059] 日常人物頁 v3 UI Spec Skeleton

## 目標

在 `UI-2-0058` 完成 slot family / skin contract 之後，將 `GeneralDetailBloodlineV3` 的 overview shell 繼續收斂成可量產、可模板化、可回掛 `GeneralDetail` host 的正式 skeleton：

1. layout JSON
2. skin JSON
3. screen JSON
4. slot-map
5. node contract

## 執行原則

1. 先沿用 `UI-2-0058` 已收斂的 `general_detail` family slot。
2. runtime 只做填值與狀態切換，不再臨時建立最終字框。
3. node naming 要往 template / fragment 可複用方向收斂，避免只為單一 screen 命名。

## 已完成收斂

1. 先把 `Unowned*` 暫名節點收斂為 `PortraitModeRibbon / PortraitModeHint / OverviewModeBadge`。
2. `OverviewModeBadge` 正式內含 `OverviewModeBadgeLabel`，保持 badge 自帶文案節點。
3. 覺醒條已由舊的 progress 表意改為 `AwakeningBarTrack + AwakeningBarFill`，並對應 `gdv3.progress.track / fill` skin slot。
4. `GeneralDetailOverviewShell.ts` 已移除臨時建構最終字框的做法，改為只針對正式 layout 節點做填值。
5. 故事帶六格已收斂為 `StoryCellOrigin / StoryCellFaction / StoryCellRole / StoryCellAwakening / StoryCellBloodline / StoryCellFuture`。
6. `general-detail-bloodline-v3-screen.json` 已收緊為 `allowMissingSkin: false`。
7. skeleton 已補出 `OverviewStateChrome` 與 `AwakeningProgressGroup` 兩個可複用群組。
8. skeleton 已再往前收成 `OverviewSummaryModules` 與 `BloodlineOverviewModules`，讓上方摘要區與血脈區都具備可模板化的群組邊界。
9. `GeneralDetailOverviewShell.ts` 已改用欄位表批次填值，減少 summary / bloodline 模組 path 散落。

## 預期成果

1. `GeneralDetailBloodlineV3` 成為 layout / skin / screen 三層一致的正式 skeleton。
2. overview shell 能被 `GeneralDetail` host 穩定掛入，並保留分頁式擴充空間。
3. 後續 `UI-2-0061` 可以直接接在這份 skeleton 上補齊 story / bloodline / crest 的內容契約。
