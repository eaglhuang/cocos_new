# [UI-2-0027] 戰場 UI 圖像資產需求總表

## 目標

建立 BattleScene 所需圖像資產的單一需求表，避免後續 icon、頭像、部隊卡圖與狀態圖示漏件。

## 範圍

- 功能 icon
- 屬性 icon
- 武將頭像
- 兵種頭像
- 虎符部隊卡圖
- 狀態圖示
- 按鈕狀態圖 `normal / pressed / disabled`
- 缺圖 fallback 規則

## 預期產出

- 一份可追蹤的資產需求 manifest
- 每個 UI 節點對應唯一資產 key
- 每項資產標示用途、尺寸、來源、命名規範、bundle 位置與交付狀態

## Family 分流原則

- `F7 戰場微型屬性 badge`
  - 貼在頭像、卡角、資訊面板邊緣的小型即時辨識 icon。
- `F8 戰鬥功能按鈕`
  - `AUTO / x2 / 單挑 / 計謀 / 奧義 / 設定` 這類可點擊控制 icon。
- `F6 裝備 / 收藏 cell`
  - 虎符卡圖、稀有度框、收藏型 card art。
- `F5 資源掉落 cell`
  - 戰場掉落、材料、附數量的 commodity 類小實物。
- `F1 墨刷導覽 glyph`
  - 僅保留給 back / close / help / info 這類低負載導覽。

> 這張總表的共識是：BattleScene 不能再用單一「萬用 icon family」量產。

## 驗證

1. 資產需求清單可被後續任務直接引用。
2. 每個需求項都有 owner 或來源說明。
3. 任何缺件都能回查到對應卡號或備註。
4. 最終畫面不再依賴臨時 placeholder。
