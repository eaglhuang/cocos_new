# [UI-2-0046] 建立 BattleScene 真場景 headless capture 驗證鏈

## 目標

讓 `tools_node/capture-ui-screens.js` 能自動截取真正的 `BattleScene.scene`，而不是只截 screen-driven UI composite。這張卡主要解決：

- BattleScene scene UUID 載入
- BattleScene ready / error capture state 回報
- BattleScene 截圖輸出與 debug artifact
- 後續 Top HUD / 左側虎符卡 / 右側日誌可走反覆截圖校正

## 背景

目前的 capture pipeline：

- `LoadingScene.ts` 只支援 `LobbyMain / ShopMain / Gacha / DuelChallenge`
- `UIScreenPreviewHost.ts` 只負責掛載 screen JSON
- `battle-scene-main.json` 只是 UI composite，不含真實棋盤 / 單位 / 場景背景

因此就算截到 `battle-scene-main`，也不等於玩家在 BattleScene 中真正看到的畫面。

## 建議實作方向

1. `capture-ui-screens.js` 新增 `BattleScene` target。
2. 以 `BattleScene.scene.meta` UUID 直接開啟場景，而非走 `LoadingScene` 的 `UIScreenPreviewHost.showScreen()`。
3. 在 `BattleScene.ts` 啟動完成後，若偵測到 capture 模式，寫入 `window.__UI_CAPTURE_STATE__ = { status: 'ready', screenId: 'battle-scene' }`。
4. 若 `BattleScene.start()` 失敗，寫入 `status: 'error'` 與錯誤訊息。
5. capture 腳本等待上述 state 後再截圖，並保留 `debug.png / debug-state.json`。

## 驗收條件

1. `node tools_node/capture-ui-screens.js --target BattleScene --outDir artifacts/ui-qa/UI-2-0046` 可成功產出 `BattleScene.png`。
2. BattleScene ready 前不會提早截圖。
3. BattleScene 若初始化失敗，會留下可讀的 debug state。
4. `UI-2-0026` 可直接沿用這條驗證鏈做 Top HUD 校正。
