## 開單原因
目前 PreviewInEditor 的實際 log 顯示，多個本來應該已落地的 SpriteFrame 無法從 `resources` bundle 取回：

1. `toast.shadow` / `general.info.shadow` → `sprites/ui_families/common/shadow/shadow_01`
2. `common.button.primary` → `sprites/ui_families/common/equipment/btn_primary_normal`
3. `detail.header.bg`、`detail.summary.bg`、`detail.tabbar.rail.*`、`detail.field.*` → `sprites/ui_families/general_detail/*`
4. `detail.tabbar.rail.noise` → `sprites/ui_families/common/noise/metal_noise_256`

但盤點實體檔案後，上述 PNG 與 `.meta` 都已存在於 `assets/resources/sprites/ui_families/...`。因此這不是單純「素材缺圖」，而是 PreviewInEditor 的資源索引、bundle catalog、路徑正規化，或 editor refresh 契約沒有對齊。

## 完整描述
1. 盤點 `UISkinResolver` / `ResourceManager` / PreviewInEditor 的 `resources.load` 路徑組裝方式，確認是否有 path normalization 或 `spriteFrame` 子資源名不一致問題。
2. 驗證 `assets/resources/sprites/ui_families/...` 下既有資產，在 Editor refresh 後是否真的被 `resources` bundle 收錄。
3. 若問題出在 preview harness 啟動序或 asset-db refresh，補齊可重現且可自動驗證的流程。
4. 至少覆蓋以下代表性路徑：
   - `sprites/ui_families/common/shadow/shadow_01`
   - `sprites/ui_families/common/noise/metal_noise_256`
   - `sprites/ui_families/common/equipment/btn_primary_normal`
   - `sprites/ui_families/general_detail/detail_header_bg`
5. 修正後，重新以 PreviewInEditor 載入 `general-detail` 與相關畫面，確認上述錯誤不再出現。

## 完成摘要
1. `assets/scripts/core/systems/ResourceManager.ts` 新增資源路徑正規化，統一處理 `db://assets/resources/`、`assets/resources/`、副檔名與 `/spriteFrame` 後綴差異。
2. `loadSpriteFrame()` 改為依序嘗試 SpriteFrame 候選路徑，失敗後再退回 Texture2D 包裝，避免 PreviewInEditor 因單一路徑組裝失敗直接斷鏈。
3. `releaseAsset()` 現在會同步釋放單張 SpriteFrame 快取，避免後續 refresh / smoke 切換時殘留舊快取鍵。
4. 新增 `tools_node/validate-representative-spriteframe-assets.js`，並接入 `tools_node/run-acceptance.js`；代表性路徑 4/4 PASS，整體 acceptance 全綠。
5. 已成功執行 `curl.exe http://localhost:7456/asset-db/refresh`，把 editor asset-db refresh 流程納入本輪收斂。
