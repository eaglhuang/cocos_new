# [UI-2-0040] 建立 reward container / bundle props 量產規格

## 目標

把寶箱、禮盒、卷冊包、材料箱、補給箱這些會出現在商城 bundle、獎勵揭示、任務獎勵與活動包裝中的「價值容器」整理成同一套 `A8 reward / bundle props family`。這張卡不是單張插畫，而是建立一條可量產的容器語言。

## 本輪結論

- `A8` 不能只定義成「一個 chest」，而是要拆成四種 presentation role：
  - 高價值寶箱
  - 文書 / 卷冊包
  - 資源 / 材料容器
  - 禮盒 / 節慶包
- 這些資產都必須共享：
  - 明確 value signal
  - 可縮小的主輪廓
  - 正面徽章 / 鎖件 / 綁帶等焦點元素
  - 與 `F5 reward-cell` / `F6 collectible-card` 可共存的材質語言

## 驗收重點

- 容器本體不可只是 props 拼貼，要有明確的正面、蓋體、鎖件與開啟邏輯
- `128 / 64` 下要能保住輪廓、鎖件與價值信號
- 高價值 / 中價值 / 補給型的色票與材質需可一眼區分
- 同一批產出要包含 chest / gift box / scroll pack / material crate 四類 proof
