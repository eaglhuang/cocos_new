# [UI-2-0098] Agent3 接手 BattleScene 非 HUD Slice Factory Onboarding

## 任務目標

在不動到已於 2026-04-08 clean pass 的 `BattleHUD` 前提下，接手 `BattleScene` 內 HUD 以外的內容，先把 scene-level 邊界切乾淨，再判斷哪些區塊可直接 reuse 既有 screen、哪些需要補 onboarding、哪些先落成 runtime residual triage。

## 明確範圍

### 納入本卡

- `BattleLogPanel`
- `ActionCommandPanel`
- `TigerTallyPanel`
- `UnitInfoPanel`
- `BattleScenePanel` 對上述子面板的 orchestration
- 戰場 world-space / 非 HUD 互動與視覺 residual

### 明確排除

- `BattleHUD` 既有 warning close-out
- `BattleHUD` layout / skin / style 回頭重調
- 任何會改寫 `artifacts/ui-source/battle-hud/` clean-pass 結論的工作

### 邊界備註

- `DeployPanel` 目前有自己的 `deploy-panel-screen`，但在 runtime 是由 `BattleScene.ts` 直接管理，沒有被列進 `battle-scene-main.json` 的 composite panels。
- 因此本卡先把 `DeployPanel` 視為 BattleScene 非 HUD slice 的外部依賴與邊界觀察點，不把它混進 BattleHUD，也不在第一輪就強行併成新 screen。

## 現況結論

### 1. BattleScene 已經有可 reuse 的子 screen 邊界

`assets/resources/ui-spec/screens/battle-scene-main.json` 已把 BattleScene 拆成：

- `battleHUD`
- `tigerTally`
- `battleLog`
- `actionCommand`
- `unitInfoPanel`

這個拆法本身是對的，等同 Unity 裡先把一個大 Canvas 拆成多個職責明確的 Prefab/Panel 區，而不是把所有 UI 混在同一個根節點。

### 2. 非 HUD 四大子面板大多已可 reuse，但還沒形成 scene-level factory package

- `TigerTallyPanel` 已有 `tiger-tally-screen`，且 `UI-2-0034` 已補齊 card art / rarity / type badge 的 runtime 綁定與 fallback。
- `BattleLogPanel`、`ActionCommandPanel`、`UnitInfoPanel` 都已有各自的 `screen/layout/skin`，代表 screen 層級不是從零開始。
- 但目前只有 `battle-hud` 已完成 `ui-source/<screen-id>/intake -> family-map -> review -> runtime verdict` 的完整 package；BattleScene 非 HUD 還沒有自己的 scene-level package。

### 3. 現有 clean capture 證明 runtime 健康，但還沒證明非 HUD 已被獨立驗證

- `artifacts/ui-source/battle-hud/review/runtime-verdict.json` 已記錄 2026-04-08 `runtime-triage-r5` 為 clean `pass`。
- 該 verdict 使用的畫面其實是整張 `BattleScene`。
- 所以這張 capture 可以當 BattleScene 非 HUD 的基線證據，但不能直接等同「非 HUD slice 已完成獨立 factory 驗證」。

### 4. 目前最缺的是 composite slice 的 onboarding 與 dedicated residual triage

依 `UI-2-0053` style audit：

- `BattleLog`、`TigerTally`、`ActionCommand` 各自成立，但還缺 scene-level tone pair 與同一套 profile。
- `World-space` 與 screen-space UI 仍未納入同一條 capture QA。
- `BattleScenePanel` 雖已在 code 中負責串接四個非 HUD 子面板，但 artifact 層還沒有對應的 onboarding package。

## Agent3 本輪交付

1. 建立 `UI-2-0098` 任務卡，鎖定「BattleScene 非 HUD slice」而非 BattleHUD。
2. 建立 `artifacts/ui-source/battle-scene-non-hud/` 的 intake、family-map、generated-review、runtime-verdict 基線。
3. 把可 reuse、需 param-tune、需新 family / dedicated residual QA 的邊界先寫清楚，讓下一輪可以直接延續。

## 完成定義

- 不影響 `BattleHUD` clean pass。
- BattleScene 非 HUD 區段有清楚邊界、現況結論與下一步。
- 至少落到 intake / family-map / review / runtime verdict 基線。
