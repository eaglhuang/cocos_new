## 開單原因
此卡開立當下，`UI-1-0015` 與 `docs/UI參考圖品質分析.md` 的 D-2 任務文字仍把 `paper.utility` 當成 `shop-main` / `gacha` 的共同驗收目標，但目前實作狀態不一致：

1. [shop-main-default.json](/c:/Users/User/3KLife/assets/resources/ui-spec/skins/shop-main-default.json) 目前主要是 `commerce` / color slot。
2. [gacha-default.json](/c:/Users/User/3KLife/assets/resources/ui-spec/skins/gacha-default.json) 則已掛上 `common-parchment-default` fragment。

若不先對齊這件事，之後就算 `UI-2-0018` 補好 preview harness，Agent2 也無法確定 D-2 到底是在驗「paper.utility 按鈕家族一致性」，還是「淺色承載面板 / parchment 家族一致性」。

## 完整描述
1. 盤點 `shop-main` 與 `gacha` 現況，確認哪一個 light-surface family 才是 D-2 的正式目標。
2. 二選一收斂：
   - 若 D-2 仍要驗 `paper.utility`，則補上對應 skin 接線。
   - 若 D-2 已轉向 `common-parchment` / light-surface consistency，則同步修正文案、任務卡與品質文件。
3. 對齊完成後，回填可供 Agent2 視覺 QA 使用的明確 target slots / target nodes。
