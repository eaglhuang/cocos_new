## 開單原因
目前 `shadow_01.png` 已存在，但 skin 層仍缺乏標準化的 `*.shadow` slot，導致 runtime 無法一致地掛上陰影層。

## 完整描述
1. 盤點需要補 shadow 的 skin 範圍與 slot 命名規則。
2. 依 rollout 波次拆成子卡：`UI-2-0013`（lobby / popup / duel）、`UI-2-0014`（general / support / shop / gacha）、`UI-2-0015`（battle / system）。
3. 在各 skin 中新增 `*.shadow` slot，並統一引用 `sprites/ui_families/common/shadow/shadow_01`。
4. 定義 offset / opacity / 尺寸策略，避免不同畫面各自發明。
5. 執行 UI spec 驗證並更新 manifest。

## 如何驗證
1. 目標 skin 新增 shadow slot 後仍可通過 acceptance。
2. 首波已完成 `lobby.navbar.shadow`、`duel.root.shadow`、`popup.card.shadow`、`netstat.icon.shadow`、`toast.shadow`；第二波已完成 `detail.root.shadow`、`general.list.shadow`、`general.info.shadow`、`quickview.shadow`、`card.shadow`、`shop.tabs.shadow`、`gacha.banner.shadow`；第三波已完成 `hud.topbar.shadow`、`log.root.shadow`、`action.ultimate.shadow`、`deploy.root.shadow`、`unitinfo.root.shadow`、`tally.card.shadow`、`sandbox.main.shadow`。
3. slot 命名與路徑可被後續 runtime 渲染邏輯共用。
4. manifest `UI-2-0007` 的 `notes` 已完整包含三波 rollout 清單。
