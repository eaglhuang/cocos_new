# UI-2-0083 — Agent Strict Layout Validator 擴充

## 背景

現有的 `validate-ui-specs.js` 檢查 JSON 格式、引用完整性、缺失欄位，但有幾類問題無法阻擋：

1. **數值合理性問題**：spacing=2000（超出螢幕）、label fontSize=8（不可讀）
2. **Family 違規**：`dialog-card` family 出現 6 個 CTA 按鈕（超過 family 慣例的 2 個）
3. **Content Contract 缺口**：`requiredFields` 未宣告但在 layout 中用了 bind path
4. **節點層級結構問題**：嵌套過深（超過 10 層）、空白 container 過多

這些問題在 AI 量產中頻繁出現：AI 產出的 JSON 「文法正確」但「語義失控」。

Unity 對照：相當於 Unity 有語法正確的 Prefab，但 Canvas 層級或 RectTransform 數值邏輯錯誤。

## 目標

加入 `--strict` 模式，讓 `validate-ui-specs.js` 能擔任「AI 量產的品質守門員」：
```bash
node tools_node/validate-ui-specs.js --strict
node tools_node/validate-ui-specs.js --check-content-contract
```

## 規則清單（至少 15 條）

### 結構規則
1. `max-node-depth`：節點樹嵌套不超過 12 層
2. `max-children-per-container`：單一 container 子節點不超過 20 個
3. `no-empty-container`：不允許沒有任何子節點 + 沒有 skinSlot 的 container
4. `scroll-list-needs-itemTemplate`：`scroll-list` 類型必須有 `itemTemplate`

### 數值合理性規則
5. `spacing-range`：spacing 不超過 200px（相對於 1920 設計寬）
6. `font-size-range`：fontSize 在 10~96px 之間
7. `widget-border-valid`：widget border 值必須是整數，且各邊加總不超過(寬度 or 高度)
8. `alpha-range`：alpha 欄位在 0~255 之間
9. `opacity-range`：opacity 欄位在 0.0~1.0 之間

### Family 限制規則
10. `dialog-max-cta`：`dialog-card` family 中 button-group 成員數 ≤ 2
11. `rail-list-min-items`：`rail-list` family 的 `railItems` 至少 1 個
12. `detail-split-tab-count`：`detail-split` family 的 `tabs` 在 2~6 之間

### Content Contract 規則（依賴 UI-2-0080）
13. `content-contract-requiredFields`：`requiredFields` 中的每個欄位都在 screen spec 中有值
14. `bind-path-declared`：layout 中出現的 bind path 必須在 `contentRequirements.requiredFields` 中宣告
15. `no-dynamic-bind`：不允許使用 `"bind": "dynamic"`（已被 BattleBindData 完全取代）

### Skin 規則
16. `nine-slice-border-not-zero`：`border` 欄位不為 `[0,0,0,0]`（九宮格用途）
17. `skin-slot-references-exist`：layout 中引用的 `skinSlot` 必須存在於 skin manifest 中
