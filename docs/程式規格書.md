# 程式規格書

## 1. 文件目的

本文件只定義新手開場的資料結構、控制器責任、流程銜接與實作邏輯，不重複描述畫面細節。

目標是讓程式與資料結構可直接落地，並與 UI 規格和劇本文案分離管理。

## 2. 系統模組責任

### 2.1 OpeningFlowController

- 控制開場各階段順序
- 管理信件、漫畫、轉場、轉蛋、教學的串接
- 控制每一幕的進出條件

### 2.2 GachaManager

- 控制無限重置十連抽
- 控制初始名將與紅顏抽取池
- 控制保底與確認邏輯

### 2.3 CountdownManager

- 管理紅顏與支援角色的趕路倒數
- 管理主畫面常駐提示
- 管理加速與完成狀態

### 2.4 UI_BloodlineTree

- 呈現祖先矩陣
- 顯示因子繼承與血統關聯
- 支援子嗣預測顯示

### 2.5 AttributePanel

- 呈現名將屬性與因子
- 顯示主因子、次級因子與解鎖狀態
- 支援六角圖或等效圖表

## 3. 靜態資料結構

### 3.1 HeroBase

用途：存放六位初始名將與後續可擴充武將。

建議欄位：

- id
- name
- title
- originRegion
- coreFactorTag
- rarity
- element
- initialRole
- introLine
- portraitKey
- battleVoiceKey

### 3.2 SupportCardBase

用途：存放紅顏、文官、支援型角色。

建議欄位：

- id
- name
- category
- rarity
- routeType
- arrivalTime
- rushPrice
- effectTags
- previewLine
- portraitKey

### 3.3 InheritLogic

用途：存放血統、因子與祖先繼承規則。

建議欄位：

- fatherId
- motherId
- ancestorCount
- allowedRegions
- factorSlots
- mutationRules
- previewWeights
- inheritCaps

### 3.4 OpeningFlowConfig

用途：控制新手開場每個階段的播放順序。

建議欄位：

- phaseId
- phaseName
- durationHint
- entryCondition
- exitCondition
- requiredUi
- requiredEffects
- requiredRewards

### 3.5 GachaConfig

用途：控制名將與紅顏抽取邏輯。

建議欄位：

- poolId
- poolType
- resetEnabled
- guaranteeRarity
- pickCount
- candidateIds
- firstPickOnly
- confirmRequired

### 3.6 CountdownConfig

用途：控制趕路與倒數顯示。

建議欄位：

- targetId
- startTimestamp
- finishTimestamp
- rushPrice
- displayLocation
- notifyEveryLogin
- canAccelerate

## 4. 程式流程規則

- 開場流程應採事件驅動，但每個事件都要可回溯
- 抽卡流程應支援重置與鎖定
- 因子與血統流程應以資料驅動，不硬編成 UI 文本
- 倒數流程應支援離線時間累積
- 所有商業入口都應從同一套配置表讀取

## 5. 驗證重點

- 新手流程是否能在未登入資料情況下先顯示預設內容
- 抽卡重置後是否會正確回到初始狀態
- 因子預覽是否能正確對應角色資料
- 倒數完成後是否能正確切換為已到達狀態
- 商業化入口是否與流程狀態一致
