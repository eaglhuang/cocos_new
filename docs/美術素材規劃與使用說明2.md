# Cyberpunk 專案美術素材規劃與使用說明

> 參考專案：`cocos-example-cyberpunk-master`（Cocos Creator 官方 3D 賽博龐克射擊 Demo）
> 對標目標：三國 2.5D 策略 Demo（本專案）
> 管理規範：遵循 **VFX Core — 特效積木庫 (Effect Building Blocks)** 管理方式
> 最後更新：2026-03-19

---

## 0. 匯入管理原則

本文件所有從 Cyberpunk 專案搬入的素材，**必須遵循本專案已建立的「VFX Core 特效積木庫」管理規範**：

1. **拆解為最小可共用單位（積木）** — 不搬入完整 Prefab，而是將貼圖、Shader、Mesh 拆開歸檔
2. **依視覺功能分類** — 放入 `vfx_core/textures/{分類}/`（glow / fire / smoke / impact / trails 等）
3. **統一命名** — `tex_{分類}_{描述}.png`（如 `tex_smoke_cyberpunk_vent.png`）
4. **登錄到 Registry** — 在 `vfx-block-registry.ts` 的 `VFX_BLOCK_REGISTRY` 中新增 `VfxBlockDef`
5. **宣告使用關係** — 在 `vfx-usage-table.ts` 的 `VFX_EFFECT_TABLE` 中登錄積木組合
6. **壓縮規格遵循 `@SIZE`** — 新建子資料夾使用 `@256` / `@128` / `@no-compress` 命名

> **擴充 SOP 完整步驟** 請見 `assets/bundles/vfx_core/README.md` 第「擴充 SOP」章節。

## 1. 專案概述

`cocos-example-cyberpunk-master` 是 Cocos 官方提供的高品質 3D 第三人稱射擊遊戲範例。它展示了 Cocos Creator 3.x 引擎在 **3D 渲染、物理碰撞、AI 尋路、IK 動畫、動態音效混合** 等方面的最佳實踐。專案採用 **Core / Logic 雙層架構**，核心框架（Core）可複用，業務邏輯（Logic）為遊戲特化。

### 技術亮點

| 特性 | 說明 |
|------|------|
| **Actor-Component 框架** | ActorBase → Actor → 各 Component（Move、Equipment、Sensor），分層清晰 |
| **Data-Driven Action 系統** | JSON 定義行為序列，運行時映射到 class 靜態方法 |
| **A* 導航系統** | 基於 Point-Node 的尋路，含隨機巡邏與最短路徑 |
| **Steering Behaviors** | Seek/Velocity 轉向行為，數學化的 AI 移動方案 |
| **IK 瞄準系統** | Aim IK + Bone Weight，運行時程序化骨骼旋轉 |
| **動態音軌混合** | MusicComposeState / MusicComposeRandom，多音軌 lerp 漸變 |
| **Ghost 殘影特效** | Mesh 拷貝 + Vertex Shader 位移，打擊感視覺增強 |
| **全域事件匯流排** | Msg 類：靜態 on/off/emit，全域解耦 |
| **物件池** | ResPool：自增長陣列池，pop/push + recycle 批次回收 |

### 3.2 ActorBase 基類（Core 層 — 生命週期管理）

`ActorBase` 是所有遊戲實體的基礎類別，管理：
- **生命週期**：`initView()` → `onUpdate()` → `onDestroy()`
- **動畫狀態**：`_anim`（SkeletalAnimation）、`_animationGraph`（AnimationGraph）
- **Action 驅動**：`do(name)` 觸發 JSON 定義的行為序列
- **特效掛載**：`onFx(data)` / `setFx(data)` 管理身上的 VFX
- **數據容器**：`_data` 字典存放所有運行時數據（HP、速度、狀態旗標）

### 3.3 Actor 子類（Logic 層 — 業務邏輯）

`Actor extends ActorBase`，新增：
- **Component 組合**：`ActorMove`（物理移動）、`ActorEquipment`（武器系統）、`ActorBag`（背包）
- **IActorInput 實作**：統一輸入介面（onMove、onFire、onJump、onAim）
- **體力機制**：跑步消耗、跳躍消耗、自然恢復
- **LateUpdate 同步**：將物理速度同步到 AnimationGraph 參數

### 3.4 Steering Behaviors（AI 移動）

```typescript
// SteeringBehaviors.Seek() — 經典 Steering 公式
Seek(targetPos: Vec3) {
    copy(this.desiredVelocity, targetPos);
    this.desiredVelocity.subtract(this.node.worldPosition);
    return this.desiredVelocity.subtract(this.moveVelocity);
}
// 回傳「轉向力」= 期望速度 - 當前速度
```



### 3.5 IK 瞄準系統（程序化骨骼動畫）

```typescript
// AimIK — 元件設定
@property({ type: Node }) rootNode: Node;   // 根骨骼
@property({ type: Node }) aimNode: Node;    // 瞄準端骨骼
@property({ type: [Node] }) boneNodes: [];  // 參與旋轉的骨骼鏈
@property({ type: [CCFloat] }) boneWeights: []; // 每根骨骼的影響權重

// IKSolverAim — 運行時解算
update()  → 計算 clampedIKPosition → 迭代 _solve()
_solve()  → 逐骨骼 _rotateToTarget()
_rotateToTarget() → Quat.rotationTo() 計算最小旋轉 → 權重插值
```

**核心演算法**：
1. 計算目標方向 = `targetPosition - aimNode.worldPosition`
2. 計算軸方向 = `axis × aimNode.worldRotation`
3. `Quat.rotationTo(axisDir, targetDir)` → 最小旋轉四元數
4. 依權重 `Quat.lerp(IDENTITY, rotation, weight)` 混合
5. 逐骨骼由根到末端遞迴施加旋轉

### 3.6 動態音軌混合系統

```typescript
// MusicComposeState — 多音軌 lerp 混合
start() {
    // 所有音軌同時播放，volume 初始為 0
    clips.forEach(clip => { addAudioSource(clip); play(); });
}
update(dt) {
    // 每幀 lerp 趨近目標音量
    curs[i] = math.lerp(curs[i], selects[i], dt * smooth);
    audios[i].volume = curs[i] * Sound.volume;
}

// MusicComposeRandom — 隨機音軌切換
randomPlay() {
    // 依機率開關各音軌，保證至少 1 軌播放
    clips.forEach((_, i) => {
        selects[i] = Math.random() < probability[i] ? 1 : 0;
    });
}
```

### 3.7 Data-Driven Action 系統

```typescript
// JSON 定義（如 actor_player.json）
{
    "attack": {
        "start": [
            { "time": 0, "delay": 0, "name": "on_anig", "data": {"key":"bool_attack","value":true} },
            { "time": 0, "delay": 0, "name": "on_sfx", "data": "sfx_sword_swing" },
            { "time": 0.3, "delay": 0, "name": "on_inst_fx", "data": {"res":"fx_slash","bone":"weapon_end"} }
        ],
        "end": [
            { "time": 0, "delay": 0, "name": "on_anig", "data": {"key":"bool_attack","value":false} }
        ]
    }
}
// UtilAction.do("on_sfx", "sfx_sword_swing") → Sound.on("sfx_sword_swing")
```

**設計模式**：靜態方法註冊表 — 字串 name 映射到 `UtilAction` 的同名靜態方法，實現 **設計師友善的行為配置**。

### 3.8 NavSystem 導航系統

- **資料結構**：Point-Node 圖（每個 Node 有 x/y/z/id/radius + links 鄰接表）
- **尋路演算法**：標準 A*（openTable / closeTable + g/h/f 成本函數）
- **巡邏邏輯**：`randomPaths()` 在鄰接圖上隨機遊走，`randomFirePath()` 在單一 Node 區域內隨機產生戰鬥點
- **最近點查詢**：`findNearest()` 暴力搜尋距離最小的導航點

### 3.9 VFX 特效系統

| 元件 | 功能 | 技術原理 |
|------|------|----------|
| `fx` (core) | 全域特效管理 | 從 ResPool pop prefab → 設定位置 → 播放 → 回收 |
| `FxGhost` | 網格殘影 | 複製 SkinnedMeshRenderer 的 Mesh → 獨立 MeshRenderer 渲染 |
| `FxGhostVertex` | 頂點位移殘影 | 計算移動方向 → 傳入 Shader uniform `direction` → GPU 端頂點偏移 |
| `FxBulletTracer` | 子彈軌跡 | 射線起點/終點 → Mesh 拉伸 → 自動淡出 |
| `FxLightFlash` | 閃光燈 | 動態開關 Light 元件 + 隨機強度 |
| `FxRayLine` | 雷射線 | Line 幾何體 + Emissive 材質 |

## 4. 可匯入素材 — 依 VFX Core 積木庫分類規劃

> **原則**：只匯入風格中性、可共用的貼圖與 Shader。3D 模型、科幻音樂、角色動畫因風格/題材不符，**不搬入**。

### 4.1 ❌ 不搬入的素材（風格不符）

| 類別 | 來源路徑 | 排除原因 |
|------|----------|----------|
| 角色/武器/場景 3D 模型 | `res/model/` | 賽博龐克題材 vs 三國古風 |
| 骨骼動畫 AnimationClip | `animations/` | 綁定特定模型拓撲 |
| BGM 音樂軌 | `res/sound/` | 科幻電子風不符三國題材 |
| 血跡/彈孔貼圖 | `res/effect/blood/`, `bulletHole/` | 現代槍戰元素 |
| 子彈/槍口粒子 | `res/effect/bullet*/`, `muzzle*/` | 現代槍械元素 |

## 6. 總結

### 美術素材價值

Cyberpunk 專案的美術素材大部分因風格差異無法直接搬用（3D 模型、科幻音樂、角色動畫），但其中 **18 個風格中性的積木素材**（煙霧、發光、衝擊、火焰貼圖 + Shader + Mesh）可依照 VFX Core 積木庫規範拆解後匯入。匯入後需完整執行 Registry 登錄與 Usage 宣告流程，確保新積木納入統一管理體系。

### 代碼架構價值（更高）

真正最有價值的是 **代碼架構與設計模式**：
1. **Steering Behaviors** → 讓三國 Demo 的單位移動更流暢自然
2. **MusicComposeState** → 讓戰鬥音樂根據局勢動態變化
3. **Data-Driven Action** → 讓武將技能可以由 JSON 配置，設計師可自行調整
4. **NavSystem A*** → 為棋盤尋路提供更智能的路徑計算
5. **Ghost 殘影** → 為武將衝鋒和大招提供視覺衝擊力

### 管理規範整合

所有搬入素材的管理方式，已完全對齊本專案既有的治理體系：
- **目錄結構** → `vfx_core/textures/{分類}/`
- **命名規範** → `tex_{分類}_{描述}.png`
- **程式登錄** → `vfx-block-registry.ts` + `vfx-usage-table.ts`
- **文件治理** → `vfx_core/README.md` + `vfx_skills/_MANIFEST.md`
