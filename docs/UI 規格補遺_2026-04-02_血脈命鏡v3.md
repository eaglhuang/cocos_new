# UI 規格補遺 2026-04-02 血脈命鏡 v3

> 對應 proof：`突破頁 v3`
> 任務來源：`UI-2-0054`

## 版型結論

`血脈命鏡 v3` 正式收斂為高階展示版型，不與 `General Detail` 混用。

固定區塊如下：

1. `左主角區`
2. `右英靈區`
3. `中央命鏡區`
4. `中央定心者區`
5. `底部 5 格故事帶`
6. `頂部名稱 / 星級 / 稀有度帶`
7. `右下 CTA 或 tips 區`

## 版面權重

- 左主角：`34%~38%`
- 右英靈：`30%~34%`
- 中央命鏡與定心者：`16%~20%`
- 底部故事帶：`18%~22%`

設計重點是 `雙面人物 + 中央命鏡`，不是大量屬性表。

## 元件規格

### A. 左主角區

- 使用近全身角色圖。
- 輪廓需完整，避免被故事帶切腳。
- 可附少量陣營旗、戰場空氣或群眾輪廓，不可太搶戲。

### B. 右英靈區

- 使用同角色血脈 / 英靈版本圖。
- 建議與左圖共用臉型、髮型與甲片邏輯。
- 必須支援：
  - `normal-spirit`
  - `awakened-spirit`
  - `locked-spirit`

### C. 中央命鏡區

- 主圖層：門框或裂隙主體
- 次圖層：淡霧、薄光、命紋符號
- 避免過度高亮與高彩 portal 感

### D. 定心者區

- 小型人物剪影或坐姿像
- aura 為可替換色層
- 適合獨立做成可換色 prefab

### E. 故事帶

- 固定 `5 cell`
- 建議統一比例，不做高低錯位
- 每格支援：
  - 插圖
  - 標題
  - stage index

## 前端拆分難度

### 容易

- 版面切區
- 故事 cell 列表
- CTA / tips / 名稱列
- 未持有標記與稀有度標籤

### 中等

- 左右人物與中央門之間的層次遮罩
- 命鏡淡霧與 aura 疊色
- 英靈圖發光與半透明控制

### 較難但可延後

- 門內動態裂紋
- 故事帶慢速滑動
- 命鏡內部粒子循環

## 建議實作策略

第一階段先做：

1. 靜態版型
2. 可換角色圖
3. 可換 5 格故事圖
4. 可換 aura 色
5. `Loading` / `Awakening` 兩個 screen variant

第二階段才加：

1. 命鏡呼吸光
2. 定心者微發光
3. 英靈淡閃

## 與 AI UI codegen 的關係

這個版型適合先產出：

1. `UI 結構線稿圖`
2. `slot 標註圖`
3. `layout JSON 骨架`
4. `skin token 對照`

原因是它的骨架很清楚：

- 左圖 slot
- 右圖 slot
- 中央門 slot
- 下方故事 cell slot
- 上方名字與 rarity slot

非常適合先用 AI 生成線框，再導回 Cocos UI 節點結構。
