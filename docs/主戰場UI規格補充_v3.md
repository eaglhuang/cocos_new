## Zone 3：虎符卡片列（v3 改造）

### 新增：兵種符號（v3-3）
在卡片 overlay 上新增一個兵種類型符號徽章：

| 子節點 | 規格 |
|---|---|
| **UnitTypeBadge** | Widget: `bottom: 4px, left: 4px`，32×32px 圓形 |
| 背景 | `rgba(0, 0, 0, 0.7)` 圓形 |
| 符號 | 兵種 icon（騎🐴、步⚔、弓🏹、盾🛡、槍🔱），字號 16px |
| 顏色 | 各兵種專屬色：騎`#D4AF37`、步`#3A8FD9`、弓`#9b6dff`、盾`#888`、槍`#2ecc71` |

### 卡片尺寸限制（v3-3）

| 平台 | 規則 |
|---|---|
| **手機**（高度 ≤ 720px 或行動裝置） | 4 卡片等分可用高度（`top:88px → bottom:0`），`flex: 1` 填滿 |
| **Web / 平板**（高度 > 720px） | 單卡最大高度 **240px**；超過時卡片固定 240px，底部留白 |
| **最小高度** | 單卡最低 **140px**，低於此值則啟動 ScrollView 滾動 |

#### 計算公式
```
可用高度 = screenHeight - 88(TopBar) - 8(topPad) - 8(bottomPad)
cardHeight = min(240, (可用高度 - 3×8(間距)) / 4)
if cardHeight < 140: 啟用 ScrollView
```

### 完整卡片子節點（v3 彙整）

```
TigerTallyCard
├── ArtBackground        ← 兵種美術圖（填滿）
├── RarityBorder         ← 品質外框色
├── StatsOverlay         ← top: ⚔攻擊 / ❤生命
├── UnitTypeBadge ← NEW  ← bottom-left: 兵種符號圓徽章
├── UnitName             ← bottom-center: 兵種名稱
├── CostBadge            ← top-right: 糧草消耗
└── DisabledMask         ← 冷卻/不足時灰度遮罩
```

### 精確規格

| 區域 | 屬性 | 值 |
|---|---|---|
| **ControlBar** | Widget | `top: 88px, right: 0` |
| | 尺寸 | 寬 10%（≈192px），高 40px |
| | 背景 | `rgba(8, 8, 14, 0.92)` |
| | 佈局 | 水平排列，居中，間距 8px |
| | 按鈕 | Auto（32×32）、x2（32×32）、⚙（32×32） |
| **BattleLog** | Widget | 緊接 ControlBar 下方 |
| | 寬度 | 10% |
| | 高度 | **展開**：約 38%H（≈410px）；**收合**：0px（隱藏） |
| | 背景 | `rgba(8, 8, 14, 0.88)` |
| | 內容 | Header + ScrollView（同原版） |
| **CollapseBtn** | 位置 | 日誌區底部 |
| | 尺寸 | 全寬 × 28px |
| | 功能 | 點擊收合日誌 → 日誌區高度歸零，按鈕變為小圖示「📋」固定在控制列下方 |
| | 展開 | 再次點擊「📋」→ 日誌展開至 38%H |

### 收合邏輯
```
展開狀態：ControlBar(40px) + BattleLog(~410px) + CollapseBtn(28px) = ~478px
收合狀態：ControlBar(40px) + 收合icon按鈕(32×32px)
過渡動畫：Tween 0.3s ease-out
```

## Zone 7：操作按鈕（v3 改造 — 奧義為大圓）

### 佈局（v3 修訂）

```
        [計謀]
           
  [結束回合]     [單挑]
           ╔═══════════╗
           ║   奧義     ║  ← 大圓 120px + SP 填充環
           ║  (SP 環)   ║
           ╚═══════════╝
```

### 大按鈕：奧義（UltimateBtn）— 含 SP 填充環

| 屬性 | 值 |
|---|---|
| **尺寸** | **120×120px** 圓形（從 80px 升級為大圓） |
| **位置** | 區域右下角（最易觸及，`bottom: 24px, right: 24px`） |
| **外環**（SP 填充環） | 圓環寬 **8px**，底色 `rgba(100,100,120,0.3)` |
| **填充色** | `#3A8FD9`（藍），順時針從 12 點方向填充，隨 SP% 增加 |
| **滿 SP 效果** | 環色變 `#7ec8f7` + 呼吸脈衝 1.5s + 水墨飛濺外框 + 文字「發動」 |
| **未滿 SP** | 文字「奧義」+ SP 百分比（如「72%」），環色暗淡 |
| **內圈** | 水墨風徽章底圖，深色 `rgba(15,15,25,0.9)` |
| **背景** | 漸層 `#1a2a6a` → `#0a0f3a`，金色邊框 2px `#D4AF37` |
| **觸控熱區** | 130×130px |
| **點擊行為** | SP 滿時：彈出奧義選擇小窗（v3-6）；SP 未滿：無反應或 Toast「SP 不足」 |

### 奧義選擇小窗（v3-6 新增）

| 屬性 | 值 |
|---|---|
| **觸發** | 點擊奧義大按鈕（SP 已滿時） |
| **位置** | 從奧義按鈕上方向上展開（bottom anchor = 奧義按鈕 top） |
| **尺寸** | 寬 220px × 高度自適應（奧義數 × 56px + padding） |
| **背景** | `rgba(10, 12, 25, 0.95)` 圓角 8px，金色邊框 1px |
| **項目** | 每項 200×48px，包含：icon（32×32）+ 技能名 + SP 消耗 |
| **範例** | 「🔥 烈火連環 (80 SP)」「⚡ 震吼 (100 SP)」 |
| **選擇** | 點擊一項 → 觸發該奧義 → 小窗關閉 → SP 扣除 |
| **取消** | 點擊小窗外區域 → 關閉不發動 |
| **動畫** | 由下向上 slide + fade-in 0.2s |

## DP 全面移除清單（v3-4）

> 所有「DP (Deploy Point)」概念全面改為「糧草」。

### 用語對照

| 舊用語 | 新用語 |
|---|---|
| DP / Deploy Point | 糧草 (Food) |
| DP: 1500 | 糧草: 8500 / 12000 |
| DpLabel | FoodLabel |
| dpLabel (id) | foodLabel |
| DP 系統 | 糧草系統 |
| 部署消耗 DP | 消耗糧草 |

### 需修改的檔案

| 檔案 | 改動 |
|---|---|
| `battle-hud-main.json` | `DpLabel` → `FoodLabel`，`dpLabel` → `foodLabel` |
| `主戰場UI規格書.md` §2.1 | DP 改為糧草、加入攜帶上限概念 |
| `keep.md` | 確認無 DP 殘留用語 |
| `demo_playbook.md` | 若有 DP 提及需確認 |
| 所有未來 JSON / TS | 禁止使用 `dp` 作為 id 或變數名 |

### 糧草機制說明
- 每場出戰有**糧草攜帶上限**（由武將/關卡決定）
- 回合開始時不自動增加（與傳統 DP 每回合增長不同）
- 部署兵種消耗糧草，糧草不足則無法部署（或觸發 Downgrade）
- 顯示格式：「🌾 糧草 {目前} / {上限}」
