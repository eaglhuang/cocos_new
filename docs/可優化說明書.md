## 2. 優化項目一：音效系統（AudioSystem）

### 為什麼需要

三國 Demo **目前完全沒有音效系統**。`ServiceLoader` 中註冊了 9 個系統，但沒有任何一個負責音效播放。而舊架構的 `AudioMgr` 已經是一個成熟且經過實戰驗證的音效管理器，包含：

- 背景音樂（BGM）播放/暫停
- 單次音效播放（`playOneShot`）
- 循環音效播放/停止
- **50ms 防重複播放**（避免同一音效在短時間內被多次觸發造成音頻爆炸）

### 舊架構原始碼分析

```typescript
// assets/Scripts/Mgr/AudioMgr.ts — 關鍵防重複機制
public play(audio: string, volume = 1) {
    const last = this.audioTime.get(audio);
    const now = Date.now();
    if (!last || (now - last) > 50) {       // 50ms 內不重複播放
        this.audioTime.set(audio, now);
        const clip = ResMgr.ins.getClip(audio);
        this._audioComp.playOneShot(clip, volume);
    }
}
```

### 如何以最佳方式實作

**步驟 1**：建立 `assets/scripts/core/systems/AudioSystem.ts`

```typescript
import { AudioClip, AudioSource, Node } from "cc";
import { ResourceManager } from "./ResourceManager";

export class AudioSystem {
    private bgmSource: AudioSource | null = null;
    private sfxSource: AudioSource | null = null;
    private loopSource: AudioSource | null = null;
    private lastPlayTime = new Map<string, number>();
    private resourceManager: ResourceManager | null = null;
    private clipCache = new Map<string, AudioClip>();

    /** 由 ServiceLoader.initialize() 呼叫 */
    public setup(resourceManager: ResourceManager, hostNode: Node): void {
        this.resourceManager = resourceManager;
        // AudioSource 必須掛載在活躍的 Node 上
        this.bgmSource = hostNode.addComponent(AudioSource);
        this.bgmSource.loop = true;
        this.sfxSource = hostNode.addComponent(AudioSource);
        this.loopSource = hostNode.addComponent(AudioSource);
        this.loopSource.loop = true;
    }

    /** 播放單次音效，50ms 內同名音效不重複 */
    public playSfx(name: string, volume = 1): void {
        const now = Date.now();
        const last = this.lastPlayTime.get(name) ?? 0;
        if (now - last < 50) return;
        this.lastPlayTime.set(name, now);

        const clip = this.clipCache.get(name);
        if (clip && this.sfxSource) {
            this.sfxSource.playOneShot(clip, volume);
        }
    }

    /** 註冊 AudioClip 到快取 */
    public registerClip(name: string, clip: AudioClip): void {
        this.clipCache.set(name, clip);
    }

    public playBgm(clip: AudioClip): void {
        if (!this.bgmSource) return;
        this.bgmSource.clip = clip;
        this.bgmSource.play();
    }

    public stopBgm(): void { this.bgmSource?.stop(); }
    public pauseBgm(): void { this.bgmSource?.pause(); }
    public resumeBgm(): void { this.bgmSource?.play(); }
}
```

**步驟 2**：在 `ServiceLoader` 中註冊


### 與舊架構的關鍵差異

| 面向 | 舊架構 `AudioMgr` | 新架構 `AudioSystem` |
|------|-------------------|---------------------|
| 實例化 | `AudioMgr.ins`（Singleton） | `services().audio`（DI 容器） |

### 如何以最佳方式實作

**策略**：不直接搬移 `EffectCtrl`（它是 Component，需要掛在 Prefab 上），而是將其邏輯抽象為 `EffectSystem` 的方法。

**擴充 `EffectSystem`**：

```typescript
import { Node, Vec3, ParticleSystem, Animation } from "cc";
import { PoolSystem } from "./PoolSystem";

export class EffectSystem {
    private poolSystem: PoolSystem | null = null;

    public setup(poolSystem: PoolSystem): void {
        this.poolSystem = poolSystem;
    }

    /**
     * 播放指定特效 Prefab，自動回收
     * 移植自 PoolMgr.playSkill()，但整合到 DI 架構中
     */
    public playEffect(key: string, position: Vec3, duration: number, parent?: Node): Node | null {
        const node = this.poolSystem?.acquire(key);
        if (!node) return null;

        if (parent) node.parent = parent;
        node.setWorldPosition(position);

        // 播放所有粒子系統和動畫
        this.playAllParticles(node);
        this.playAllAnimations(node);

        // 設定自動回收（若 duration > 0）
        if (duration > 0) {
            setTimeout(() => {
                this.stopAllParticles(node);
                this.stopAllAnimations(node);
                this.poolSystem?.release(key, node);
            }, duration * 1000);
        }

        return node;
    }

    /** 播放節點下所有 ParticleSystem */
    private playAllParticles(node: Node): void {
        const particles = node.getComponentsInChildren(ParticleSystem);
        particles.forEach(p => p.play());
    }

    /** 停止並清除節點下所有 ParticleSystem */
    private stopAllParticles(node: Node): void {
        const particles = node.getComponentsInChildren(ParticleSystem);
        particles.forEach(p => { p.stop(); p.clear(); });
    }

    /** 播放節點下所有 Animation */
    private playAllAnimations(node: Node): void {
        const anims = node.getComponentsInChildren(Animation);
        anims.forEach(a => a.play());
    }

    /** 停止節點下所有 Animation */
    private stopAllAnimations(node: Node): void {
        const anims = node.getComponentsInChildren(Animation);
        anims.forEach(a => a.stop());
    }

    public recycleEffect(key: string, node: Node): void {
        if (!this.poolSystem) { node.destroy(); return; }
        this.stopAllParticles(node);
        this.stopAllAnimations(node);
        this.poolSystem.release(key, node);
    }
}
```

### 使用方式

```typescript
// 在戰鬥場景中播放一個 1.5 秒的爆炸特效
services().effect.playEffect("Boom", targetPosition, 1.5, battleStage);

// 播放一個持續性特效（如 Buff 光環），手動管理回收
const aura = services().effect.playEffect("FireAura", unitPosition, 0);
// ... 稍後手動回收
services().effect.recycleEffect("FireAura", aura);
```

### 與舊架構的映射關係

```
舊：PoolMgr.ins.playSkill("Boom", true, pos, 0.25)
新：services().effect.playEffect("Boom", pos, 0.25)

舊：node.getComponent(EffectCtrl).play()
新：services().effect.playAllParticles(node)  // 內部方法自動處理

舊：PoolMgr.ins.putNode(node)
新：services().effect.recycleEffect("Boom", node)
```

## 4. 優化項目三：傷害飄字系統（BMFont 高效能方案）

### 為什麼需要

目前 `UnitRenderer` 中的 `spawnFloatText()` 使用 `Label` 元件動態建立文字節點：

```typescript
// 現狀：UnitRenderer.spawnFloatText() — 每次傷害都 new Node + addComponent(Label)
const floatNode = new Node(nodeName);
const label = floatNode.addComponent(Label);
label.fontSize = 28;
label.string = text;
```

這有兩個效能問題：
1. **每幀動態建立/銷毀節點**：GC 壓力大
2. **Label 元件渲染成本高**：每個 Label 會產生一次 draw call

舊架構使用 **BMFont + 物件池 + 預製體** 的方案：

```typescript
// assets/Scripts/Ctrl/DmgAnimCtrl.ts — 高效能飄字
init(dmg, font: Font) {
    this.text.font = font;                           // 切換 BMFont 字體
    this.text.fontSize = 56 + 16 * Math.random();   // 隨機大小增加視覺多樣性
    this.text.string = "" + dmg;
    this.node.setPosition(-40 + 80 * Math.random(), 0);  // 隨機水平偏移
    this.anim.play();                                // 播放預設動畫
    this.scheduleOnce(this.recycleSelf, 0.9);        // 0.9 秒後回收
}

recycleSelf() {
    this.anim.stop();
    this.text.string = null;
    PoolMgr.ins.putNode(this.node);  // 回收到物件池，不銷毀
}
```

### 優勢比較

| 面向 | 現行方案（Label + destroy） | BMFont + 物件池方案 |
|------|---------------------------|-------------------|
| Draw Call | 每個飄字 1 次 | 同 BMFont 批次合併 |
| GC 壓力 | 每次 new + destroy | 物件池複用，零 GC |
| 視覺效果 | 純文字 | 可包含圖示（暴擊★、閃避 Miss 圖片） |
| 動畫控制 | Tween 程式碼控制 | Animation 預設動畫，美術可調整 |
| 記憶體 | 每次配置 | 預熱固定數量，記憶體穩定 |

### 如何以最佳方式實作

**步驟 1**：遷移 BMFont 素材

```
res/dmgFont/bmfont/pu.fnt + pu_0.png   → resources/fonts/dmg_normal.fnt
res/dmgFont/bmfont/bao.fnt + bao_0.png → resources/fonts/dmg_crit.fnt
res/dmgFont/bmfont/miss.fnt + miss_0.png → resources/fonts/dmg_miss.fnt
```

**步驟 2**：建立 DmgText Prefab

製作一個包含 `Label`（使用 BMFont）+ `Animation`（飄字動畫）的 Prefab，掛載一個簡化版的 `DmgAnimCtrl` 腳本。

**步驟 3**：在 `EffectSystem` 中實作 `showDamageText()`

```typescript
public showDamageText(position: Vec3, value: number, isCrit = false): void {
    const key = "DmgText";
    const node = this.poolSystem?.acquire(key);
    if (!node) return;

    // 放置到 UI 層並定位
    node.parent = this.uiRoot;
    node.setWorldPosition(position);

    // 設定文字與字體
    const label = node.getComponentInChildren(Label);
    if (label) {
        label.font = isCrit ? this.critFont : this.normalFont;
        label.fontSize = 56 + Math.floor(16 * Math.random());
        label.string = `${value}`;
    }

    // 播放動畫 + 自動回收
    const anim = node.getComponent(Animation);
    anim?.play();

    setTimeout(() => {
        anim?.stop();
        this.poolSystem?.release(key, node);
    }, 900);
}
```

## 5. 優化項目四：GPU Instanced Attribute 視覺回饋

### 為什麼需要

舊架構的 `MeshCtrl` 使用 GPU Instanced Attribute 實現兩個核心視覺效果：

1. **受擊變色**（Rim Color）：透過 `a_rimColor` 瞬間改變角色邊緣光顏色
2. **溶解出入場**（Dissolve）：透過 `a_disRange` 控制溶解進度

```typescript
// assets/Scripts/CombatCtrl/MeshCtrl.ts
set disRange(v: number) {
    this._range = v;
    this.mesh.setInstancedAttribute('a_disRange', [v]);  // GPU 端直接設值
}

set colorType(type: number) {
    this.mesh.setInstancedAttribute('a_rimColor', hurtColor[type]);  // 即時變色
}
```

目前三國 Demo 的 `UnitRenderer` 使用的是 `tween` 縮放動畫來表達受擊（`scale 1.15 → 1.0`），視覺衝擊力不足。

### 如何以最佳方式實作

**注意**：此優化需要自定義 Shader，適合三國 Demo 從方塊升級為 3D 模型時一起實施。

**步驟 1**：建立自定義 Shader（支援 Instanced Attribute）

```glsl
// assets/resources/effects/unit-feedback.effect
// 在 vertex shader 中宣告 instanced attributes
in vec4 a_rimColor;    // 邊緣光顏色 (r, g, b, intensity)
in float a_disRange;   // 溶解範圍 (-1 ~ 0)

// 在 fragment shader 中混合
vec3 finalColor = baseColor.rgb;
float rim = 1.0 - dot(normal, viewDir);
finalColor += a_rimColor.rgb * a_rimColor.a * pow(rim, 2.0);
```

**步驟 2**：在 `UnitRenderer` 中整合 MeshCtrl 邏輯

```typescript
// UnitRenderer 新增方法
public flashUnit(unitId: string, colorType: number): void {
    const view = this.unitViews.get(unitId);
    if (!view) return;

    const mr = view.bodyNode.getComponent(MeshRenderer);
    if (!mr) return;

    // 設定受擊變色
    const color = HURT_COLORS[colorType];
    mr.setInstancedAttribute('a_rimColor', color);

    // 0.2 秒後恢復
    setTimeout(() => {
        mr.setInstancedAttribute('a_rimColor', [0, 0, 0, 0]);
    }, 200);
}

public dissolveUnit(unitId: string, duration: number, direction: 'in' | 'out'): void {
    const view = this.unitViews.get(unitId);
    if (!view) return;

    const mr = view.bodyNode.getComponent(MeshRenderer);
    if (!mr) return;

    const start = direction === 'in' ? -1 : 0;
    const end = direction === 'in' ? 0 : -1;

    // 使用 tween 驅動 instanced attribute
    const ctrl = { range: start };
    tween(ctrl)
        .to(duration, { range: end })
        .call(() => mr.setInstancedAttribute('a_disRange', [ctrl.range]))
        .start();
}
```

### 效能優勢

| 方案 | Draw Call 影響 | CPU 開銷 | 視覺效果 |
|------|---------------|---------|---------|
| 現行 Scale Tween | 無 | 中（每幀插值） | 簡單縮放 |
| GPU Instanced | **0 額外 Draw Call** | 極低（設一次值） | 邊緣光 + 溶解 |

## 6. 優化項目五：PoolSystem 增強（特效播放 + 自動回收）

### 為什麼需要

目前 `PoolSystem` 只有基礎的 `acquire` / `release` / `register`，缺少舊架構 `PoolMgr` 的便捷方法：

- `playSkill()`：取出節點 → 定位 → 播放特效 → 定時回收，一行搞定
- `preloadSkill()`：預載並預播一次，確保首次使用不卡頓
- `copyNode()`：從現有節點複製（不需要 Prefab）

### 如何以最佳方式實作

```typescript
// 擴充 PoolSystem，新增便捷方法
export class PoolSystem {
    // ... 現有方法保持不變

    /**
     * 預載特效：取出 → 播放 → 短暫後回收
     * 確保粒子系統和材質在首次使用前已被 GPU 上傳
     */
    public warmupEffect(key: string, position: Vec3): void {
        const node = this.acquire(key);
        if (!node) return;
        node.setWorldPosition(position);
        node.active = true;

        // 給 GPU 一幀時間上傳資源
        setTimeout(() => this.release(key, node), 650);
    }

    /**
     * 從已有節點複製（不需要 Prefab 引用）
     * 對應舊架構 PoolMgr.copyNode()
     */
    public acquireFromTemplate(templateNode: Node, parent: Node | null): Node {
        const key = templateNode.name;
        if (!this.pools.has(key)) {
            this.pools.set(key, new NodePool());
        }

        const pool = this.pools.get(key)!;
        const node = pool.size() > 0 ? pool.get()! : instantiate(templateNode);
        if (parent) {
            node.parent = parent;
            node.active = true;
        }
        return node;
    }
}
```

- 3D 模型（`.FBX`）— 三國 Demo 目前使用程式化幾何體
- `monster.json` / `skill.json` — 數值體系不同

## 9. 風險評估與注意事項

### ⚠️ AudioSource 掛載問題

舊架構的 `AudioMgr` 使用 `new AudioSourceComponent()` 直接建構，這在 Cocos Creator 中**不安全**，因為 Component 必須掛載在活躍的 Node 上。新架構必須使用 `hostNode.addComponent(AudioSource)` 方式。

### ⚠️ Singleton → DI 轉換

所有從舊架構搬入的邏輯，必須將 `XXX.ins` 模式改為 `services().xxx` 模式：

```typescript
// ❌ 舊模式（不要搬入）
AudioMgr.ins.play("hurt");
PoolMgr.ins.playSkill("Boom", true, pos, 0.25);

// ✅ 新模式
services().audio.playSfx("hurt");
services().effect.playEffect("Boom", pos, 0.25);
```

### ⚠️ Prefab 腳本引用

搬移 Prefab 時，若 Prefab 上掛載的腳本（如 `EffectCtrl`、`BulletCtrl`）不在新架構中，Prefab 會報錯。建議：
1. 先搬入需要的腳本
2. 再搬入引用該腳本的 Prefab
3. 或者建立新的 Prefab，只搬入視覺資源（貼圖、材質、粒子參數）

### ⚠️ Bundle 命名衝突

舊架構使用 `Bundle1` ~ `Bundle6`，新架構使用語義化命名（`vfx_core`、`vfx_skills`）。搬移時統一使用新命名規範，不要混用數字命名。

### ⚠️ 效能預算

| 項目 | 建議上限 |
|------|---------|
| 同時存在的特效 Prefab | ≤ 15 個 |
| 同時存在的傷害飄字 | ≤ 20 個 |
| 特效貼圖單張尺寸 | ≤ 256×256 |
| 音效同時播放數 | ≤ 8 個 |
| 物件池預熱數量 | 每類 5-10 個 |
