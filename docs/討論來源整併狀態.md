# 討論來源整併狀態

> 目的：將 `docs/討論來源/` 與其子目錄中的歷史提案，整理為「已整併可封存」與「仍待整併」兩批，避免後續重複掃描時遺漏 UI / 視覺 / 鏡頭類中間討論。
> 判定原則：若正式規格書、UI 文件、技術文件或 `keep.md` 已吸收主要結論，則列為「已整併可封存」；若僅部分吸收、尚未回寫或仍缺正式承接檔，則列為「仍待整併」。
> 衝突規則：所有舊討論若與現行正式規格衝突，一律以現行正式規格為準。
