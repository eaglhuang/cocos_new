# 美術素材規劃與使用說明

> 本文件完整整理本專案（3DEffectController）中所有美術素材的分類、存放位置、命名規範、製作流程與程式引用關係。
> **最後更新**: 2026-03-18

---

## 1. 專案概述

本專案為一款基於 **Cocos Creator** 的三國題材 **2.5D 策略對推遊戲** Demo，採用 MVC + Service 混合架構。核心玩法為 5 路 × 8 格棋盤上的遭遇戰，包含兵種部署、互剋鏈、武將技能、Buff 系統等機制。美術素材涵蓋角色序列幀、場景背景、UI 圖標、VFX 特效貼圖、Shader Effect 以及影片素材。

---

## 3. 素材分類詳細說明

### 3.1 角色序列幀（Sprites）

| 項目 | 說明 |
|------|------|
| **存放位置** | `assets/resources/sprites/{角色名}/{動作}/` |
| **命名規則** | `{角色名}_{動作}_{兩位數序號}.png`，例如 `zhangfei_idle_01.png` |
| **現有素材** | 張飛待機動畫：15 幀（`zhangfei_idle_01` ~ `zhangfei_idle_15`） |
| **來源格式** | 原始圖集為 3×5 格的合併 PNG（含綠幕背景） |
| **前處理** | 透過 `tools/sprite-pipeline/process-spritesheet.js` 自動拆幀、去綠幕 |
| **程式引用** | `SpriteAutoPreview.ts` 載入 `resources/sprites/{name}/{action}` 目錄下的 SpriteFrame 進行自動播放 |
| **FPS** | 預設 8 fps（由 spec.json 定義），可依角色調整 |

#### 拆幀參數規格（spec.json）
- `gridCols` / `gridRows`：圖集的列數與行數
- `greenKey.hard` / `greenKey.soft`：綠幕去背的硬邊界與軟邊界參數
- `alphaThreshold`：透明度門檻
- `alignPadding`：對齊邊距

#### 擴充方式
1. 將新的角色圖集放入 `tools/sprite-pipeline/input/`
2. 建立對應的 `.spec.json` 描述拆幀參數
3. 執行 `node process-spritesheet.js` 自動產出序列幀
4. 將輸出複製到 `assets/resources/sprites/{角色名}/{動作}/`

### 3.2 場景背景（Textures - Backgrounds）

| 項目 | 說明 |
|------|------|
| **存放位置** | `assets/resources/textures/` |
| **現有素材** | `bg_normal_day.png`（白天平原）、`bg_normal_night.png`（夜晚平原） |
| **設定檔** | `assets/resources/data/scene-backgrounds.json` |
| **程式引用** | `SceneBackground.ts` 透過 `ResourceManager` 載入，使用獨立的 `BGCamera(priority=-1)` + `BACKGROUND_LAYER` 顯示 |
| **渲染層級** | 背景使用獨立相機渲染，`Main Camera` 不清除背景色，避免覆蓋 |

#### 背景對照表（scene-backgrounds.json）
```json
{
  "backgrounds": [
    { "id": "bg_normal_day",   "path": "textures/bg_normal_day",   "timeOfDay": "day",   "label": "白天平原" },
    { "id": "bg_normal_night", "path": "textures/bg_normal_night", "timeOfDay": "night", "label": "夜晚平原" }
  ]
}
```

#### 擴充方式
1. 新增背景圖放入 `assets/resources/textures/`
2. 在 `scene-backgrounds.json` 加一筆記錄
3. 在 `encounters.json` 的關卡設定中引用新背景 ID

### 3.4 VFX 特效貼圖（Bundles - vfx_core）

| 項目 | 說明 |
|------|------|
| **存放位置** | `assets/bundles/vfx_core/textures/` |
| **打包方式** | 獨立 Asset Bundle，建置時自動打包 |
| **圖集** | 目錄內含 `auto-atlas.pac`，Cocos 建置時會將小圖合併為大圖集（1024 或 2048） |

#### 三層分類目錄

| 子目錄 | 用途 | 建議尺寸 | 命名規則 |
|--------|------|----------|----------|
| `icons/` | 特效中心的主圖示（劍、盾牌、愛心等） | 128×128 | `tex_icon_{效果名}.png` |
| `rings/` | 地板上的法陣、圓環、底座 | 128×128 ~ 256×256 | `tex_ring_{效果名}.png` |
| `shapes/` | 點綴圖形（箭頭、星星、十字等） | 64×64 ~ 128×128 | `tex_shape_{類型}_{效果名}.png` |

#### 現有素材清單

| 檔案 | 分類 | 用途 |
|------|------|------|
| `tex_icon_addatk.png` | icons | 攻擊增益中心圖示 |
| `tex_icon_addlife.png` | icons | 生命增益中心圖示 |
| `tex_ring_addatk.png` | rings | 攻擊增益法陣底座 |
| `tex_ring_addlife.png` | rings | 生命增益法陣底座 |
| `tex_shape_arrow_addatk.png` | shapes | 攻擊增益向上箭頭 |
| `tex_shape_arrow_addlife.png` | shapes | 生命增益向上箭頭 |

#### 程式引用
- `BuffGainEffectPool.ts` 使用單一 Quad + Tween 動畫播放 Buff 特效
- `UnitRenderer.playBuffEffect()` 依數值正負分派到 4 組 Pool：`AtkGain / AtkLoss / HpGain / HpLoss`
- 材質使用自訂 `vfx-buff-quad.effect`，依圖層選用不同 technique（transparent / additive / outer-glow）

#### 擴充方式
1. 依照命名規範新增 PNG 到對應子目錄
2. 無需修改程式碼，建置時自動進入圖集
3. 若新增全新 Buff 類型，需在 `BuffGainEffectPool` 中註冊新的 Pool 配置

### 3.6 Shader Effect 檔案

#### 3.6.1 board-jelly.effect（棋盤玻璃感地板）

| 項目 | 說明 |
|------|------|
| **存放位置** | `assets/resources/effects/board-jelly.effect` |
| **用途** | 為棋盤地板格子提供半透明玻璃質感 |
| **程式引用** | `BoardRenderer.ts` 建立棋盤格時套用 |
| **關鍵 Uniform** | `edgeColor`（邊緣色）、`fillColor`（填充色）、`highlightColor`（高亮色）、`jellyParams`（果凍參數） |
| **渲染狀態** | `depthTest: true`、`depthWrite: false`、`cullMode: none`、Alpha Blend |

#### 3.6.2 vfx-buff-quad.effect（Buff 特效 Shader）

| 項目 | 說明 |
|------|------|
| **存放位置** | `assets/resources/effects/vfx-buff-quad.effect` |
| **用途** | 統一 Quad 類 VFX 的材質行為 |
| **程式引用** | `BuffGainEffectPool.ts` 初始化材質時使用 |

**三種 Technique：**

| Technique | Blend 模式 | 用途 | Alpha |
|-----------|-----------|------|-------|
| `transparent` | `src_alpha + one_minus_src_alpha` | 主符號、法陣、箭頭本體 | 1.0 |
| `additive` | `src_alpha + one` | 火花粒子、亮點能量 | 0.45 |
| `outer-glow` | `src_alpha + one` | 外光柔光層 | 0.32 |

**統一 Uniform：**
- `mainTexture`：VFX 貼圖本體
- `mainColor`：色相與整體 alpha
- `uvTransform`：UV 平移/縮放（目前固定 `[1,1,0,0]`）
- `effectParams`：`[alphaCutoff, alphaSoftness, colorBoost, alphaBoost]`

## 4. 美術素材前處理工具鏈

### 4.1 工具位置
`tools/sprite-pipeline/`

### 4.2 三大處理腳本

| 腳本 | 用途 | 輸入 | 輸出 |
|------|------|------|------|
| `process-spritesheet.js` | 角色圖集拆幀 + 去綠幕 | `input/{圖集}.png` + `{圖集}.spec.json` | `output/{角色}/{動作}/` 序列幀 |
| `process-effect-textures.js` | 特效貼圖黑底轉透明 | 設定檔中的 source 路徑 | `_alpha.png` 版本 |
| `process-effect-layers.js` | 特效圖層分離（環/主體/箭頭） | `_alpha.png` 版本 | `_ring_alpha.png`、`_main_alpha.png`、`_arrow_alpha.png` |

### 4.3 拆幀流程（process-spritesheet）

```
輸入圖集 (3×5 格, 含綠幕)
    ↓ 讀取 spec.json 參數
    ↓ 按 grid 切割為單幀
    ↓ 綠幕去背 (hard + soft key)
    ↓ 透明度修剪 + 對齊
    ↓ 輸出 PNG 序列幀
輸出到 output/{角色}/{動作}/
```

**核心參數（default.config.json）：**
- `minComponentArea: 220`：最小連通區域面積
- `componentPadding: 14`：元件邊距
- `alphaThreshold: 10`：透明度閾值
- `greenKey.hard`：硬去綠（`minGreen: 120, minDelta: 38`）
- `greenKey.soft`：軟去綠（`minGreen: 95, minDelta: 20, alphaScale: 0.35`）

### 4.4 特效貼圖處理流程

```
原始黑底特效圖
    ↓ process-effect-textures.js
    ↓ 亮度分析 + 色度過濾
    ↓ 黑底轉 alpha 通道
    ↓ 輸出 _alpha.png
    ↓
    ↓ process-effect-layers.js
    ↓ 以圓心/半徑分離圖層
    ↓ 輸出 ring / main / arrow 三張圖
最終存入 bundles/vfx_core/textures/
```

**黑底轉透明參數（effect-textures.config.json）：**
- `hardMaxLuma: 10`：硬黑底閾值
- `softMaxLuma: 84`：軟黑底閾值
- `maxChroma: 28`：色度上限

**圖層分離參數（effect-layer-splits.config.json）：**
- `centerX/Y`：圓心座標
- `innerRadius/outerRadius`：環形內外半徑
- `feather`：羽化值
- `focusRadius`：焦點半徑
- `arrowBottomOffset/mainTopOffset`：箭頭與主體的偏移量

## 8. 效能相關規範

### 8.1 圖集優化
- `vfx_core/textures/` 下設有 `auto-atlas.pac`，建置時自動合併小圖
- 目標：3 層 VFX 材質（ring + icon + shape）僅需 **1 個 Draw Call**
- 建議圖集最大尺寸：1024×1024 或 2048×2048

### 8.2 特效貼圖規範
- 符號「清晰度」由 `transparent` technique 負責，不依賴 additive 發亮
- 外光只做輔助輪廓與氣氛，不可把主符號洗白
- 解析度不足時優先升級來源圖至 256×256 或 512×512
- Quad VFX 一律 `depthTest=false`、`depthWrite=false`、`cullMode=none`

### 8.3 記憶體管理
- 透過 `ResourceManager` 的 `addRef()` / `decRef()` 管理資源生命週期
- 特效節點使用 `PoolSystem` 物件池回收重用，避免頻繁 `instantiate` / `destroy`
- 切換場景或離開戰鬥時呼叫 `ResourceManager.releaseAsset()` 釋放資源

## 9. 待擴充素材清單

| 類別 | 需求 | 優先順序 |
|------|------|----------|
| 角色序列幀 | 關羽、呂布、曹操的待機/攻擊/受傷動畫 | 高 |
| 兵種序列幀 | 各兵種的行走/攻擊/死亡動畫 | 高 |
| 技能特效 | 張飛震吼、關羽月牙刀斬、呂布天下無雙、曹操兵不厭詐 | 高 |
| 場景背景 | 森林、山地、城池等不同地形主題背景 | 中 |
| VFX 特效 | 攻擊減益（AtkLoss）、生命減益（HpLoss）的專屬貼圖 | 中 |
| UI 素材 | 武將頭像、SP 能量條美化、按鈕樣式 | 中 |
| 地形貼圖 | 河流、山地、森林、沙漠等地形格子貼圖 | 低 |
| 音效/音樂 | 戰鬥 BGM、技能音效、UI 互動音效 | 低 |

## 10. 美術素材製作建議流程

```
1. 美術提供原始素材
   ├── 角色圖集（含綠幕/黑底背景）
   ├── 特效圖（黑底版本）
   └── UI 元件（透明底 PNG）
       ↓
2. 前處理工具鏈
   ├── process-spritesheet.js → 拆幀 + 去背
   ├── process-effect-textures.js → 黑底轉 alpha
   └── process-effect-layers.js → 圖層分離
       ↓
3. 放入對應目錄
   ├── sprites/ → 角色動畫
   ├── bundles/vfx_core/textures/ → 通用特效
   ├── bundles/vfx_skills/ → 技能專屬特效
   ├── icons/ → UI 圖標
   └── textures/ → 場景貼圖
       ↓
4. 更新設定檔（如需要）
   ├── scene-backgrounds.json
   ├── generals.json
   └── encounters.json
       ↓
5. 在 Cocos Creator 中確認匯入與預覽
```
