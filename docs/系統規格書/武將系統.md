## A. 系統描述

武將是整個遊戲的基礎實體。不論是戰鬥、治理、結緣、培育，都以「武將物件」為核心操作對象。每位武將具備六色屬性、遺傳因子槽、血統家族樹、精力值、壽命等完整的數據結構，並根據職能（戰鬥武將 / 專業教官）走向不同的功能路線。

> **Unity 對照**：武將等同於一個帶有多層 Component 的 Entity，底層由 ScriptableObject 驅動的角色資料。

---

## B. 系統目的

1. **核心載體**：所有遊戲子系統（戰鬥、結緣、培育、治理）都以武將為操作單位
2. **長線投資標的**：讓玩家對武將產生情感連結，從抽卡→培育→退役→英靈牆，形成完整的生命週期敘事
3. **數值基石**：提供統一的資料結構（UID、六色屬性、因子槽、血統矩陣），讓所有子系統可共用同一份資料

## C. 商業套路

| 商業點 | 說明 |
|---|---|
| 轉蛋抽取 | 名將池 SSR 高上限（5 轉 1500+），忠誠度永遠 100 |
| 培育次數 | 每轉生一次解鎖一次培育機會，驅動重複抽卡 |
| 精力回復 | 補元散（快速回復精力）為課長剛需道具 |
| 退役虎符 | SSR 名將退休產出特殊兵種虎符，收藏型付費動力 |
| 英靈牆 | 集齊特定組合觸發全局 Buff，長線收集誘因 |

## F. 公式相關

### 爆發力公式（Explosive Power）
$$EP = 50 + [血脈共鳴] + [異脈加成] + [名將契合] + [天命波動(LUK)]$$

| 加分項 | 觸發條件 | 加成 |
|---|---|---|
| 血脈共鳴 | 14 人中 ≥3 個相同標籤 | +15 |
| 異脈加成 | 14 人完全無重複 ID | +20 |
| 名將契合 | 史實/宿敵配對 | +25 |
| 精力加成 | 父母精力皆 100% | +10 |
| 天命波動 | 父母平均 LUK>80: +5~+15; <40: -5~-10 | 浮動 |

### 精力消耗
- **戰鬥奧義**：`Current_Vitality -= (Ult_Skill_Used × 10)`
- **結緣受孕**：男性 -50；女性 -100（進入待產凍結）
- **季度回補**：出征 +25%；待命 +50%

### 因子覺醒
```
if (Genes[i].Is_Locked == true && Trigger_Condition met):
    Genes[i].Is_Locked = false
    // UI 播放碎裂特效
```

### Server 端
| 欄位 | 型別 | 說明 |
|---|---|---|
| UID | VARCHAR(16) PK | 唯一識別碼 |
| Template_ID | VARCHAR(16) | 模板 ID（同名武將共用） |
| Bloodline_ID | VARCHAR(16) | 血源 ID（近親判定用） |
| Player_ID | VARCHAR(32) FK | 所屬玩家 |
| Role | ENUM('Combat','Mentor') | 職能 |
| Stats_JSON | TEXT | 六色屬性快照 |
| Genes_JSON | TEXT | 因子槽序列化 |

| Lifespan | INT | 天命值（隱藏） |
| Vitality | FLOAT | 精力值 |
| Status | ENUM('Active','Training','Retired','Dead') | 狀態 |

## J. 名詞定義

| 名詞 | 代碼/Key | 定義 |
|---|---|---|
| 武將 | General / Combat | 遊戲中所有可操作的角色實體 |
| 教官 | Mentor / Support | 專職於培育模式的支援卡角色 |
| UID | UID | 武將唯一識別碼（如 G001、C102） |
| Template_ID | Template_ID | 同名武將模板（如 ZY_01） |
| Bloodline_ID | Bloodline_ID | 血源判定 ID（如 ZY_FAMILY） |
| 天命值 | Lifespan | 隱藏的壽命上限值 |
| 精力 | Vitality | 戰鬥與結緣共用的精力資源 (0~100) |
| 因子 | Gene / GeneObject | 遺傳因子 {Type, ID, Level, Is_Locked, Discovery_Level} |
| 六色屬性 | Six-Color Stats | STR/INT/LEA/POL/CHA/LUK |
| 爆發力 | Explosive Power (EP) | 由血統決定的子嗣屬性跳升機率隱藏值 |
| 虎符 | Tiger_Tally | SSR 名將退役時產出的特殊兵種裝備道具，裝備在武將身上後方可指揮 |
| 英靈牆 | Spirit_Wall | 退役名將掛牆提供永久全局 Buff |
| 演武筆記 | Battle_Notes | 武將間接教學產物（戰法記錄交給教官） |
| 開獎 | Unveiling | 子嗣 5 歲時揭曉五維與因子的事件 |
| 職能分流 | Role_Branch | 5 歲時決定戰鬥武將或教官的不可逆選擇 |

## K. 目前 UI / 資料對接共識

1. 武將 runtime 資料入口統一使用 `assets/resources/data/generals.json`
2. UI 與戰鬥共用 `assets/scripts/core/models/GeneralUnit.ts` 內的 `GeneralConfig`
3. `GeneralDetailPanel` 為分頁式詳情面板，未公開欄位以鎖定或折疊方式呈現
4. `LobbyScene` 仍可先吃最小欄位集，但之後新增欄位時不得再回頭建立本地版 `GeneralConfig`
