# 轉職與宿命系統規格書

> **來源文件**：轉職與宿命系統設計.md、傭兵職能分流與傳承機制.md

---

## A. 系統描述

武將在 5 歲開獎時進行第一次「宿命分流」——系統根據因子偏好建議武官或文官路線。玩家可接受或抗命。20 歲時可進行「轉職」，從單一職能轉為雙職能（如武官追加軍師副職）。65 歲退役時，退役路線由職能決定：武官→虎符→戰技教官；文官→竹簡→智識教官。宿命系統在 5 歲定下大方向，轉職系統在 20 歲提供軌道修正。

---

## B. 系統目的

1. **5 歲開獎的決策感**：天命分流=開盲盒後的首個重大選擇
2. **20 歲轉職=軌道修正**：給玩家「培養方向出錯」的補救空間
3. **退役分流=資源化**：不同路徑退役的教官有不同加成方向
4. **因子視覺化**：職能分流結合因子色彩，UI 一目了然

---

## C. 商業套路

| 商業點 | 說明 |
|---|---|
| 轉職令 | 20 歲轉職需消耗轉職令 |
| 抗命風險 | 5 歲抗命→成長率下降（但可能大器晚成） |
| 雙職能強化 | 雙職能武將稀有度更高→驅動精心培育 |

## F. 公式相關

### 5 歲宿命推薦
```
if (Blue_STR + Blue_POW > Blue_INT + Blue_POL):
    Recommend = "武官"
else:
    Recommend = "文官"
```

### 抗命懲罰
```
if (Player_Choice != Recommend):
    Growth_Rate × 0.8  // 成長率降 20%
    // 但若觸發「大器晚成」，25 歲後 Growth_Rate × 1.5
```

### 轉職條件
```
if (General.Age >= 20 && has(轉職令)):
    Add_Sub_Class(target_class)
    // 副職能力 = 主職能力 × 0.6
```

### 退役分流
| 主職能 | 退役產物 | 教官類型 |
|---|---|---|
| 武官 | 虎符 | 戰技教官（藍粉因子加成） |
| 文官 | 竹簡 | 智識教官（白綠因子加成） |
| 雙職能 | 虎符+竹簡 | 全能教官（稀有） |

## I. Data Schema 需求

### 本機端 (Client)
```json
{
  "General_Career": {
    "UID": "G201",
    "Primary_Class": "Warrior",
    "Sub_Class": null,
    "Fate_Recommend": "Warrior",
    "Fate_Defied": false,
    "Growth_Modifier": 1.0,
    "Job_Change_Available": true,
    "Retirement_Product": "Tiger_Tally"
  }
}
```

### Server 端
| 欄位 | 型別 | 說明 |
|---|---|---|
| General_UID | VARCHAR(16) FK | 武將 |
| Primary_Class | ENUM('Warrior','Scholar') | 主職能 |
| Sub_Class | ENUM('Warrior','Scholar',NULL) | 副職能 |
| Fate_Defied | BOOLEAN | 是否抗命 |
| Growth_Modifier | FLOAT | 成長率修正 |

## J. 名詞定義

| 名詞 | 代碼/Key | 定義 |

| 宿命分流 | Fate_Assignment | 5 歲時系統建議的職能方向 |
