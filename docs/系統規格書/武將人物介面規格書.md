# 武將人物介面規格書

> **文件目的**：定義武將人物介面的正式資訊架構、顯示規則、鎖定規則與資料對映，作為 `GeneralDetailPanel`、大廳武將卡、詳情彈窗、血統/戰法/適性延伸畫面的母規格。
> **適用範圍**：武將角色頁、詳情頁、彈窗頁、人物卡、收藏頁、未來培育頁的角色摘要模組。
> **核心原則**：預設顯示已知資訊；未解鎖、未發現或不想揭露的欄位，一律以鎖圖示 `🔒`、`?` 佔位、折疊區塊或灰階卡表現。
> **Unity 對照**：此介面相當於 Unity 的角色詳情面板 + 樣板卡片 + 多頁籤資訊視圖，由 ScriptableObject 資料驅動，View 只負責呈現。

---

## 1. 文件定位與設計目標

### 1.1 定位

武將人物介面是玩家查看「一名武將完整面貌」的主要入口。它不是單純的數值頁，而是結合以下資訊的正式角色介面：

1. 基本身份與生命週期
2. 六色屬性與能力評語
3. 血統、因子、爆發力與覺醒狀態
4. 戰法、教官啟發與習得狀態
5. 兵種、地形、天氣適性
6. 隱藏欄位、未解鎖欄位與折疊欄位

### 1.2 設計目標

1. 讓玩家一眼辨識這名武將「是誰、強在哪、能做什麼、還缺什麼」
2. 兼容正式版的長線養成資料，避免未來擴欄位時大改 UI
3. 保留手機可讀性，避免單頁資訊過載
4. 對接現有 `GeneralUnit` 與 `GeneralDetailPanel`，減少實作轉換成本
5. 讓血統、戰法、適性這三大子系統在同一頁形成完整敘事

### 1.3 非目標

1. 不在這份文件中定義戰鬥數值公式本體
2. 不在這份文件中定義培育流程本體
3. 不在這份文件中定義轉蛋或抽卡 UI
4. 不把所有隱藏資訊直接揭露給玩家

## 3. 介面總覽

### 3.1 建議容器形式

武將人物介面建議採用「頁籤式詳情彈窗」或「抽屜式人物面板」。

#### 方案 A：頁籤式詳情彈窗
- 適合完整展示多系統資訊
- 適合中後期正式版
- 建議放在 PopupLayer

#### 方案 B：人物卡 + 詳情抽屜
- 適合大廳、列表、收藏頁
- 點擊人物卡後右側或底部滑出詳情
- 適合手機橫豎切換

### 3.2 建議頁面結構

```text
GeneralDetailPanel
├─ Header（頭像、名稱、稱號、陣營、職能、狀態）
├─ Summary Strip（HP、SP、年齡、精力、EP 評級）
├─ Tab Bar
│  ├─ 基本資料
│  ├─ 六色屬性
│  ├─ 血統與因子
│  ├─ 戰法與技能
│  ├─ 戰場適性
│  └─ 延伸資訊
└─ Footer Actions（返回、收藏、鎖定、比較、分享）
```

### 3.3 顯示優先級

1. 必顯：名稱、稱號、陣營、職能、狀態、HP、SP、年齡、精力
2. 高優先：六色屬性、EP 評級、血統簡覽、已習得戰法、兵種/地形適性
3. 中優先：因子槽、血脈警告、可習得戰法、戰法等級、天氣適性
4. 低優先：UID、Bloodline_ID、Template_ID、來源標籤、進階劇本標籤

### 3.4 框體分層視覺規格

武將人物介面若要達到正式水墨渲染質感，不應再把每個 panel 視為一張單獨的九宮格框，而應拆成可疊合的框體家族。

Unity 對照：這相當於把一個 Panel Prefab 拆成 `Frame`、`Inner Glow/Bleed`、`Fill` 三個子 Image，再由外層容器統一排版；而不是只放一張 sliced sprite 撐完整個視覺。

#### 分層原則

1. `frame`：只承擔外輪廓、金屬紋飾、角花與主要結構線，避免把大面積灰霧直接烘在最外框。
2. `bleed`：負責邊框往內容區滲入的墨暈、霧邊與暗角，是讓框與背景自然過渡的主因。
3. `fill`：負責內容區的大面積紙感、煙墨、山霧或深色底紋，不應以單色實心底板取代。
4. `accent`：只用於 active tab、稀有度高亮、警示狀態或頁面焦點，不應變成每個 panel 都有的常駐發光層。

#### 套用範圍

1. `PopupBody / RightContentArea` 使用大型框體家族，必須具備 `frame + bleed + fill` 三層。
2. `HeaderPanel`、`SummaryStrip`、`FooterPanel` 使用中型框體家族，至少具備 `frame + fill`，建議補 `bleed`。
3. 頁內卡片如血統摘要、技能卡、適性卡使用 section 家族，不得再直接與 header 共用完全相同的視覺層級。
4. `RightTabBar` 與 tab button 應拆成「tab rail」與「tab plate」兩種家族，避免整條頁籤列和單顆按鈕共享同一張小卡框。

#### 質感目標

1. 讓邊框與背景的過渡發生在 `bleed` 層，而不是靠透明邊緣硬撐。
2. 讓角色立繪後方的大底圖、右側內容框、頁內卡片維持同色系不同密度，而不是各自獨立成一塊塊貼紙。
3. 讓 active 與 idle 的差異主要來自 `accent` 與亮度階層，不是單純換色塊。

## 5. UI 資訊架構

### 5.1 Header 區

Header 是人物介面的第一視覺焦點，必須穩定呈現以下內容：

1. 頭像 / 立繪 / 角色剪影
2. 名稱
3. 稱號
4. 陣營
5. 職能（Combat / Mentor）
6. 狀態（Active / Training / Retired / Dead）
7. 稀有度 / 版本標籤（若有）

### 5.2 Summary Strip 區

Summary Strip 放在 Header 下方，快速顯示核心狀態：

1. HP
2. SP
3. 年齡
4. 精力
5. EP 分級
6. 主技能是否可用

### 5.3 Tab 1：基本資料

基本資料頁顯示不需要展開即可理解的核心資訊。

#### 需顯示欄位
- UID
- Template_ID
- Name
- Title
- Faction
- Gender
- Age
- Role
- Status
- Lifespan
- Vitality_Current / Vitality_Max

#### 建議視覺
- 左側放頭像與狀態章
- 右側放文字欄位與短標籤
- `Lifespan` 建議預設隱藏為「天命值：🔒」或條件顯示

### 5.4 Tab 2：六色屬性

六色屬性頁用於顯示能力骨架與戰鬥定位。

#### 需顯示欄位
- STR
- INT
- LEA
- POL
- CHA
- LUK

#### 視覺規則
- 採列表 + 雷達圖雙呈現
- 主力屬性高亮
- 未主修屬性灰化但不隱藏
- 可附帶一句定位評語，例如「武力型」、「謀略型」、「統率型」

### 5.5 Tab 3：血統與因子

這是本規格最重要的區塊之一，需兼顧輕度與硬核玩家。

#### 預設內容
1. EP 卡片
2. EP_Rating 文案
3. 父母簡覽
4. 因子槽簡圖
5. 近親警告或歸宗標籤

#### 展開內容
1. 14 人血統矩陣
2. 虛擬祖先標記
3. Bloodline_ID
4. 覺醒名號
5. 因子來源軌跡

### 5.6 Tab 4：戰法與技能

#### 需顯示欄位
- 主戰法
- 已習得戰法列表
- 可習得戰法列表
- 未啟發戰法列表
- 戰法等級
- TP 消耗
- 戰法類型
- 技能簡述

#### 三態規則
1. 已習得：正常彩色顯示，可點擊看詳情
2. 可習得但未啟發：半透明、鎖定、顯示教官需求
3. 無法習得：灰階、鎖定、顯示因子不符或條件未滿

### 5.7 Tab 5：戰場適性

#### 需顯示欄位
- 兵種適性：騎 / 步 / 弓 / 機械
- 地形適性：平原 / 山地 / 水域 / 林地 / 沙漠
- 天氣適性：晴 / 雨 / 霧 / 風 / 夜 / 雷

#### 視覺規則
- 採 S / A / B / C / D 星級或級別卡
- S 級顯示金色高亮
- D 級顯示警示色與排斥符號
- 可展開文字說明對應加成

### 5.8 Tab 6：延伸資訊

#### 需顯示欄位
- UID
- Template_ID
- Bloodline_ID
- 來源標籤
- 取得方式
- 成長摘要
- 戰績摘要
- 收藏狀態
- 分享 / 比較 / 鎖定按鈕

## 6. 鎖定與遮蔽規則

### 6.1 鎖圖示規則

以下欄位預設使用鎖圖示 `🔒` 或 `?` 佔位：

1. 未解鎖因子
2. 未啟發戰法
3. 隱藏戰法
4. 未展開的 14 人血統矩陣
5. 天命值（若設定為隱藏）
6. 覺醒名號（尚未達成時）
7. 虛擬祖先內容（若玩家尚未解鎖細節）
8. 未公開的稱號來源

### 6.2 遮蔽層級

| 層級 | 意義 | 顯示方式 |
|---|---|---|
| 0 | 已公開 | 完整顯示 |
| 1 | 條件公開 | 灰階或半透明 |
| 2 | 可點擊解鎖 | `🔒 + ? + 提示` |
| 3 | 完全隱藏 | 僅顯示佔位框 |

### 6.3 不想揭露欄位的處理

若資料本身不希望對玩家直接揭露，則：

1. 以 `🔒` 圖示取代數值
2. 顯示欄位類型標籤，不顯示實值
3. 可附帶「尚未解鎖」或「未知」說明
4. 不在 tooltip 中洩露原始數據

## 8. 互動與展開規則

### 8.1 血統展開

1. 預設顯示父母兩張卡
2. 點擊「查看完整血統」後展開 14 人矩陣
3. 再次點擊可收合
4. 若資料不足，仍保留骨架位與鎖定標籤

### 8.2 戰法點擊

1. 點擊已習得戰法卡可進詳情
2. 若為可習得戰法，點擊顯示啟發來源
3. 若為不可習得戰法，點擊僅顯示鎖定原因

### 8.3 適性點擊

1. 點擊某項適性可看加成說明
2. 長按可顯示地形或天氣對應效果
3. 移動端預設使用 tooltip 底部抽屜，不要依賴 hover

## 9. 版面規格建議

### 9.1 桌面版

- 左：頭像 / 人物關鍵資訊
- 中：Tab 內容
- 右：摘要卡 / 快速操作 / 戰法快捷欄
- 建議比例：左側角色主視覺區約 44%~48%，右側內容卡約 36%~40%，最右頁籤欄維持 96~112 px 固定寬度，避免頁籤文字在 Web 與手機橫向模式下被壓扁。
- 摘要卡（HP / SP / 精力 / EP 等）優先靠右上排列，避免壓在立繪底部，維持角色主視覺完整度。

### 9.2 手機版

- 上：頭像與摘要
- 中：Tab 切換
- 下：內容卡垂直排列
- 血統矩陣與戰法列表一律支援折疊
- 手機橫向模式優先沿用「左視覺、右資訊、右側 tab rail」配置，只縮窄左右留白與內容欄寬；直向模式才退化成上下堆疊。
- 頁籤欄文字不可依賴 hover 或超小字硬塞，應允許兩行標籤並保持按鈕高度不少於 84 px。

### 9.3 尺寸建議

1. 觸控目標不小於 44 × 44 px
2. 關鍵卡片最小高度建議 72 px
3. 血統矩陣卡片最小寬度建議 84 px
4. 戰法卡片需保留圖示、名稱、類型、等級四層資訊
5. 角色頁標題建議使用 42~48 px；頁籤文字 14~16 px；內文資訊 18~20 px；次要說明 16~18 px
6. 主要資訊卡內距建議 18~24 px；Tab 內容區左右留白建議 24~32 px；段落間距建議 12~18 px

## 14. 總結

武將人物介面不是單一角色卡，而是玩家理解一名武將完整生命資料的入口。它應該同時承載：

1. 身份
2. 數值
3. 血統
4. 戰法
5. 適性
6. 隱藏與解鎖狀態

本規格的核心精神是：
- 已知就清楚顯示
- 未知就鎖起來
- 深度資訊可展開
- 正式版欄位先預留完整骨架
