## A. 系統描述

戰法系統管理所有武將可習得與使用的戰鬥技能。戰法分為多種類型（突擊/奇襲/控制/地形/合攻/輔助/統帥），透過培育流程的「支援卡靈感事件 → 結業 TP 確認習得」、「因子覺醒」、「戰法秘笈」等途徑習得。綠色因子提供 TP 折扣，決定頂級戰法的可達性。詳細習得流程見本文件 § K。

> **Unity 對照**：相當於 SkillManager + SkillDatabase 架構。每個 Skill 是 ScriptableObject，定義 TP 消耗、效果、動畫觸發、施放範圍。SkillManager 管理角色已學戰法列表與施放冷卻。

> **重要區分一（格子戰法）**：本文件定義的戰法皆為 **格子戰法（Tile Tactic）**——武將在戰場內消耗 TP 施放的即時技能，僅在指定格子上疊加特效（火攻格、陷阱格、營帳格、水流格等），不改變場景背景。格子戰法透過培育中的「支援卡靈感事件 → 結業 TP 確認」取得；詳見 § K。
>
> **重要區分二（場景戰法）**：另有一類 **場景戰法（Scene Gambit）**——進入戰場**前**決定的特殊場景，載入預繪靜態背景並設定初始格子特效。觸發方式：在治理大廳中，由我方軍師或 **INT ≥ 90**（可設定）武將提出「軍事任務」加入任務欄；玩家**接取任務**後，下一場進入的戰場即自動套用指定場景戰法。**目前唯一觸發來源為「接受軍事任務」**，尚無其他發動途徑。完整規格見 **戰法場景規格書.md** § B。

## B. 系統目的

1. **戰鬥核心操作**：戰法是棋盤戰鬥中玩家主動操作的關鍵行為
2. **培育目標導向**：36 回合培育的最終目標之一就是「點出理想戰法」
3. **因子系統連動**：綠色因子提供 TP 折扣，不同血統產生不同戰法組合
4. **差異化角色**：同名武將因隨機副戰法不同而具備獨特性；子嗣有機會繼承先祖家傳戰法，形成血統獨尊
5. **商業驅動**：頂級戰法需高等教官啟發，驅動教官收集與培育投入

## C. 商業套路

| 商業點 | 說明 |
|---|---|
| 教官稀有性 | 頂級戰法需特定名將教官啟發，驅動收集與轉蛋 |
| TP 不足 | 高級戰法消耗大量 TP，需多次培育累積→重複抽卡解鎖轉生培育次數 |
| 戰法秘笈掉落 | 千人斬/演武產出的秘笈為稀有品，驅動持續戰場投入 |
| 戰法隨機化 | 轉蛋同名武將的副戰法隨機，追求特定組合需多次抽取 |
| 戰法升級 | 重複戰法轉為經驗值升級，驅動長線抽卡 |

## F. 公式相關

### 戰法分類說明

**取得方式分兩大類**：

| 類型 | 標記 | 定義 | 可取得途徑 | 範例 |
|---|---|---|---|---|
| **通用戰法** | `[通]` | 任何武將均可習得 | 靈感事件、戰法秘笈、培育 TP | 火計、落石、騎兵衝锋 |  
| **家傳戰法** | `[家]` | 特定血統獨有，不得離開家族範疇 | 血統繼承（生子嗣、轉蛋同名） | 趙家槍法、諸葛陣法、曹魏鐵騎 |  

> 通用戰法取得方式多元。家傳戰法的 `TacticDef.Bloodline_Required` 非空，血統檢查失敗則完全無法習得（非跳過存檔關閉，而是未出現在靈感選單中）。

### TP 消耗規則

```
// === Phase 1：培育訓練中 — 支援卡靈感事件 ===
for each training_session in cultivation[1..36]:
    if (supportCard.appearsInSession && random() < supportCard.inspiration_rate):
        general.addInspiration({
            tactic_id: skill_id,
            source_card: supportCard.template_id,
            turn: currentTurn,
            confirmed: false
        })
        // 同一次培育同一戰法最多 2 次靈感
        // 重複靈感在結業時提供額外 TP -10%

// === Phase 2：培育結業前 — 玩家 TP 點選 ===
if (child.hasGene(Green, skill_category)):
    final_TP = base_TP × 0.5  // 綠色因子折扣
else:
    final_TP = base_TP

// SR/SSR 需有靈感才能確認；N/R 無需靈感
if (general.hasInspiration(skill_id) || skill.rarity <= "R"):
    canConfirm = (available_TP >= final_TP)
else:
    canConfirm = false  // 未獲靈感，不可點選 SR/SSR

// === Phase 3：培育完成 — 習得執行 ===
if (player.confirms(skill_id) && canConfirm):
    available_TP -= final_TP
    general.learnTactic(skill_id, source="INSPIRATION_COMMIT")
    general.removeInspiration(skill_id)
// 未確認靈感一律清除
general.clearUnconfirmedInspirations()
```

### 戰法升級公式

```
// 重複戰法轉為經驗值
skill_exp += duplicate_skill_count × 100
level_up_threshold = current_level × 500

if (skill_exp >= level_up_threshold):
    skill_level += 1
    skill_exp -= level_up_threshold
    // 效果加成 = base_effect × (1 + level × 0.1)
```

### 戰法隨機化（轉蛋武將）

```
// 定义：轉蛋同名武將，副戰法數量 1~3 個隨機
fixed_skill = template.primary_skill        // 主戰法固定（如趙雲=龍膽）

// 副戰法數量隨機：1~3 個（可透過武將等級度標加權重）
random_count = weightedRandom([1, 2, 3], template.sub_skill_count_weights)

learned_subs = []
for i in range(random_count):
    pool = template.skill_pool.filter(s =>
        !s.is_primary &&
        !learned_subs.includes(s) &&
        (s.bloodline_required == null || general.hasBloodline(s.bloodline_required))
    )
    learned_subs.push(pickRandom(pool, weights))

// 家傳戰法必須符合血統才會進入抽取池
```

### 子嗣先祖戰法繼承

> 生子嗣時，小孩有機會繼承「父、母、祖父母、曾祖父母」等先祖的戰法（包含家傳戰法）。繼承數量與子嗣的 LUK 屬性正相關。

```
// 先祖戰法庫：收集父、母、最多四代先祖的可繼承戰法
ancestor_pool = []
for each ancestor in [father, mother, paternal_grandparents, maternal_grandparents]:
    ancestor_pool.extend(ancestor.Learned_Tactics.filter(t =>
        t.Is_Inheritable  // 只有標記可繼承的戰法才入池
    ))

// LUK 決定繼承數量：1~3 個
if child.LUK >= 80: inherit_count = 3
elif child.LUK >= 50: inherit_count = 2
else:                inherit_count = 1

// 家傳戰法優先，在抽取時權重 ×2
inherited = pickTopWeighted(ancestor_pool, inherit_count, {
    family_tactic_weight: 2.0,
    general_tactic_weight: 1.0,
    // 家傳戰法需確認血統限制
    validate: (tactic) =>
    tactic.bloodline_required == null ||
    child.hasBloodline(tactic.bloodline_required)
})

for tactic in inherited:
    child.learnTactic(tactic.id, source="BLOODLINE_INHERIT")
```

**設定說明**：
| LUK 範圍 | 繼承上限 | 備註 |
|---|---|---|
| 80–100 | 3 個 | 大吉，可能繼承到珍稀家傳戰法 |
| 50–79 | 2 個 | 普通 |
| 0–49 | 1 個 | 薄命，僅繼承 1 個最高權重戰法 |

## J. 名詞定義

| 名詞 | 定義 |
|---|---|
| 戰法 | 武將在棋盤戰鬥中可主動或被動使用的戰鬥技能 |
| TP（Tactics Point） | 戰法點數，培育結束後用於購買戰法的資源 |
| 教官啟發 | 培育模式中教官觸發的對談事件，解鎖高級戰法的購買權限 |
| 綠色因子折扣 | 擁有對應綠色因子時，學習該類戰法的 TP 消耗減半 |
| 戰法秘笈 | 道具型物品，使用後直接習得指定戰法 |
| 主戰法 | 每位武將固定的第一戰法（如趙雲=龍膽），不可替換 |
| 副戰法 | 轉蛋武將的第 2、3 戰法，每次抽取隨機生成 |
| 戰法升級 | 重複獲得同一戰法時，轉為經驗值提升該戰法等級 |
| 戰場覺醒戰法 | 在戰場中達成特定條件（如背刺、絕境）時解鎖的隱藏戰法 |
| 被動戰法 | 不需要消耗 TP 點數、持續生效的戰法（如毒沼免疫、山地防禦） |
| 軍事任務 | 治理大廳中，由軍師或 INT ≥ 90（可設定）武將提出的特殊軍事提案；玩家接取後，進入戰場時自動啟用指定場景戰法（預繪背景＋格子初始特效） |
| 戰法靈感 | 培育訓練中支援卡人物觸發事件所產生的戰法學習資格；須在培育結業時消費 TP 確認，才能真正習得；未確認的靈感在培育結束時清除 |
| 靈感確認 | 培育結業 TP 點選的最終動作；選擇確認某靈感即消耗 TP 並習得該戰法 |
| 通用戰法 | 任何武將均可習得的戰法（Accessibility=UNIVERSAL），可透過靈感事件、戰法秘笈、培育 TP 取得 |
| 家傳戰法 | 特定血統獨有的戰法（Accessibility=BLOODLINE）；專屬家族秘傳，血統不符的武將完全無法習得，僅能透過繼承（轉蛋同名武將或生子嗣）取得 |
| 子嗣繼承戰法 | 生子嗣時，小孩依其 LUK 屬性自父母及兩側祖父母的習得戰法池中抽得 1~3 個戰法（家傳戰法權重加倍） |
| 轉蛋副戰法 | 轉蛋同名武將時，除主戰法固定外，隨機確定 1~3 個副戰法（家傳戰法需符合血統） |

## K. 格子戰法獲得流程（培育連動）

> **Unity 對照**：類似培育模式中「教官羁絆技能書」觸發邏輯（如馬娘的技能習得），結業時的「TP 點選介面」對應 Unity 的 Talent Point Tree UI。

> **依賴系統**：本節內容與「培育系統.md」及「教官系統（支援卡）.md」強耦合。

### K-1. 完整習得流程

| 階段 | 觸發條件 | 結果 |
|---|---|---|
| 訓練觸發 | 支援卡人物在某回合「出現」並「參與訓練」 | 機率觸發靈感事件，靈感清單 +1 筆 |
| 靈感累積 | 36 回合內可觸發多次（同戰法上限 2 次） | Inspiration_Tactics 暫存清單 |
| 結業點選 | 培育第 36 回合結束前彈出 TP 點選介面 | 玩家選擇確認哪些靈感 |
| 習得確認 | 玩家點選並消耗 TP | 戰法加入 Learned_Tactics（Source=INSPIRATION_COMMIT） |
| 清除未選 | 培育完成時執行 | 所有 Confirmed=false 的靈感清除 |

### K-2. 靈感觸發機率

| 支援卡稀有度 | 單回合靈感觸發率 | 可教導戰法品質上限 |
|---|---|---|
| SSR 名將教官 | 60% | SSR 戰法 |
| SR 支援卡 | 40% | SR 戰法 |
| R 支援卡 | 20% | R 戰法 |
| N 支援卡 | 5% | N 戰法 |

- 同一回合同一支援卡**最多觸發 1 次**靈感事件
- 同一培育同一戰法**最多保留 2 筆**靈感（重複靈感在結業時提供 TP 額外 -10% 折扣）
- 綠色因子存在時，對應類型戰法的 TP 消耗額外 ×0.5

### K-3. TP 點選介面規格

| UI 元素 | 說明 |
|---|---|
| 靈感區塊 | 列出所有 Confirmed=false 靈感，顯示戰法名稱、來源支援卡、觸發回合 |
| 通用戰法區塊 | 列出所有 N/R 戰法（無需靈感，可直接選購） |
| TP 餘量顯示 | 即時扣除預覽；綠色因子折扣自動套用 |
| 確認按鈕 | 確認後不可取消，培育完成時執行習得 |

### K-4. 場景戰法觸發來源（彙整對照）

> 場景戰法非由培育習得，此表僅供全局對照用。

| 觸發來源 | 操作者 | 條件 | 目前狀態 |
