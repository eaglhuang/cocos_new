## A. 系統描述與核心分層

### 兩層戰法體系

本專案的戰法系統分為 **兩個完全不同的層級**，必須嚴格區分：

| | 場景戰法（Scene Gambit） | 格子戰法（Tile Tactic） |
|---|---|---|
| **觸發時機** | 戰場外，進入戰鬥前 | 戰場內，回合中施放 |
| **發動者** | 高智力軍師（INT 門檻判定） | 任何武將（消耗 TP） |
| **影響範圍** | 整個戰場背景 + 全局 Tile-State 預設 | 指定格子（1~9 格不等） |
| **背景圖** | 專屬預繪場景（AI 繪製靜態背景） | 不改變背景，僅疊加格子特效 |
| **可動態切換？** | ❌ 不可。場景背景為靜態畫面，進場即固定 | ✅ 可。格子特效逐回合變化 |
| **設計理由** | 動態場景渲染成本過高，改用預繪+格子效果模擬 | 即時特效疊加在固定背景上，技術可行 |

> **Unity 對照**：Scene Gambit ≈ 載入不同 Scene 或切換 Skybox + TileMap preset；Tile Tactic ≈ Runtime 在 Grid 上 Instantiate 特效 Prefab。但我們的 Scene 背景是靜態圖不做動態切換，所以只用預繪方式。

### 設計理由

1. **場景背景是畫死的**：每張戰鬥場景背景由 AI 預先繪製，運行中無法自然切換或動態改變（如從晴天突然變成暴風雨）
2. **格子效果是動態的**：武將在戰場上施放奧義或小戰法時，可以在指定格子上放置特效（如火攻 3 格、落石 2 格、暴風 4 格），這些只是格子上方的特效疊加，不改變背景
3. **兩者結合**：場景戰法決定「進哪張地圖」，格子戰法決定「地圖上哪些格子有即時效果」

## B. 場景戰法（Scene Gambit）— 戰前觸發的特殊戰場

### B-1. 觸發流程

> **唯一觸發來源（目前）**：接受治理大廳的「軍事任務」。尚未規劃其他發動途徑。

```
治理大廳（平時）
└── 軍師 或 INT ≥ 90（可設定）之武將
    → 在「任務欄」提出軍事任務（指定場景類型）
        └── [玩家操作] 接取任務
            ↓
        進入下一場戰鬥時，自動套用指定場景戰法
            ├── 載入預繪靜態背景圖
            └── 初始化格子特效（火攻格、陷阱格、營帳格、水流格…）
```

**軍事任務提出資格**：
- 我方軍師角色（特定職位）**無條件**可提出
- 一般武將需 INT ≥ 90（具體門檻可依場景類型設定）
- 每種場景戰法有對應的 INT 最低門檻（見 B-3）

**任務接取效果**：
- 任務進入「待執行」狀態，僅對下一場指定戰鬥生效
- 一旦進入戰場，場景戰法自動觸發，無需額外判定
- 若玩家放棄或未在有效期內出戰，任務失效（無處罰）

### B-2. 發動條件與判定

| 條件 | 說明 |
|---|---|
| **軍師 INT 門檻** | 發動者 INT ≥ 場景戰法所需最低 INT（見 B-3 各場景定義） |
| **情報充足度** | 需對目標國家的情報值 ≥ 50%（來自治理模式他國AI系統 E-4） |
| **糧草條件** | 發動特殊場景需額外消耗糧草（基礎糧草 × 1.5） |
| **成功率公式** | $P = \min(95\%, 50\% + (INT - threshold) \times 2\%)$ |
| **冷卻** | 同一類場景戰法對同一目標，冷卻 3 回合 |

### B-5. 軍師智謀能量 HUD

> 來源：策略戰場視覺與玩法融合設計.md

特殊場景戰鬥中，軍師 HUD 常駐顯示於棋盤格上方狀態欄：
- **軍師頭像** + 「INT 99, 正在執行: 火燒偷襲」文字標註
- **智謀能量條**：右下角操作區「發動技能」按鈕左側，垂直排列能量條
- 能量由下往上填充，滿時噴發金色粒子，表示可發動場景特殊技能
- 能量積累速度 = $INT \times 0.5$ 每回合
- 能量上限 = 100，滿後可觸發場景專屬全局效果（如火焰蔓延加速、大水第二波、落石加劇）

### B-6. 場景戰法 vs AI 觸發

> 來源：治理模式他國AI系統.md E-4、E-5

NPC 國家的 AI 軍師也可以對玩家發動場景戰法：
- AI 選擇場景戰法的條件：兵力充足 + 邦交負值 + 情報充分 + 該國軍師 INT ≥ 門檻
- AI 傾向依據武將特長選擇場景（周瑜型 → 火攻，關羽型 → 水淹）
- 玩家被捲入場景戰法時，進場前有 1 回合準備期（可調整編隊）

## C. 格子戰法（Tile Tactic）— 戰場內即時施放

### C-0. 與場景戰法的區別

格子戰法是武將在 **戰鬥進行中** 消耗 TP 施放的小型戰法，效果僅限 **指定格子的特效疊加**：
- 不改變場景背景圖
- 不改變全局 Tile-State 佈局
- 僅在指定格子上方播放特效（火焰、落石、暴風、水流等）
- 持續 1~3 回合後消失
- 任何武將都可施放（只要有對應戰法和足夠 TP）

> **Unity 對照**：格子戰法 ≈ 在 Grid Cell 位置 Instantiate 一個 ParticleSystem Prefab + 加上 TileBuff 的 Scriptable Modifier。

## D. 場景戰法與格子戰法的交互

### D-1. 疊加規則

場景戰法設定的全局 Tile-State 是 **底層**，格子戰法的效果是 **疊加層**：

```
最終格子效果 = 場景預設 Tile-State（底層） + 格子戰法特效（疊加層）
```

| 情境 | 底層（場景） | 疊加層（格子戰法） | 最終效果 |
|---|---|---|---|
| 火燒場景 + 火計 | Hazard（火焰） | Hazard（火計特效） | 火焰傷害 ×1.5（疊加加成） |

### D-2. 優先權

當底層與疊加層衝突時：
1. **Force-Move** 優先（水流強制推移不被覆蓋）
2. **Hazard** 疊加（多重傷害來源累計）

### F-3. 適性綜合計算

$$
\begin{aligned}
final\_attack &= base\_attack \times (1 + terrain\_mod) \times (1 + weather\_mod) \times (1 + troop\_mod) \\
final\_defense &= base\_defense \times (1 + terrain\_mod) \times (1 + weather\_mod) \times (1 + troop\_mod) \\
final\_move &= base\_move + terrain\_move\_mod + troop\_move\_mod \\
final\_accuracy &= base\_accuracy + terrain\_acc\_mod + weather\_acc\_mod
\end{aligned}
$$

## I. 混種戰法規格

> 來源：血統理論系統.md K 節。混種戰法本質上是特殊的格子戰法。

### I-1. 混種戰法定義

當子嗣繼承了兩個不同系統的高等因子（例如：紅色因子 Lv.4+ 火系 + 藍色因子 Lv.4+ 水系），在培育期間有機率觸發「混種戰法」的學習資格。混種戰法無法透過教官啟發或秘笈習得，只能由特定血統組合自然產生。

### I-2. 觸發條件

| 條件 | 說明 |
|---|---|
| 至少兩種不同色系因子 ≥ Lv.4 | 六色因子中任意兩種高等 |
| 因子色系對應的戰法分類不同 | 例如：火攻 + 地形、控制 + 突擊 |
| 培育 36 回合內觸發覺醒事件 | 隨機出現，非必然 |

### I-3. 格子效果特點

混種戰法的格子效果為兩種元素的融合特效。例如：
- **火水混合**：蒸氣爆裂（Hazard + Force-Move 複合格子效果）
- **雷風混合**：雷暴旋風（大範圍格子控制 + 持續傷害特效）
- **具體清單待正式規格展開後補充**
