# 培育系統規格書

> **來源文件**：鏡像時空育才系統設計.md、養成系統與教官來源定義.md、子嗣系統規格書：三國傳承.md

---

## A. 系統描述

培育系統（對標《賽馬娘》訓練模式）是 36 回合制的鏡像時空育才機制。玩家將 5 歲以上子嗣送入培育時空，用快照（鏡像分身）進行 36 回合培養。外界時間 = 1 季。培育中可指派教官，觸發因子覺醒、學習戰法、選擇分支。培育完成後數值寫回本體。

---

## B. 系統目的

1. **深度養成迴圈**：36 回合決策密度高，每回合都有選擇
2. **時間壓縮**：外界 1 季=培育 36 回合，允許大量培養
3. **教官價值**：退役武將→教官→培育中發揮價值（課金循環）
4. **因子覺醒**：培育是除戰場外的另一條覺醒路線

## C. 商業套路

| 商業點 | 說明 |
|---|---|
| 入學令 | 每季可培育名額有限，增加入學令=更多養成機會 |
| 教官匹配 | 高星教官（退役名將轉成）帶來 TP 加成 |

| 大器晚成 | 培育中隱隱雷聲→「此子在培育中覺醒了！」 |
| 自動培育券 | 後期解鎖自動培育機制的消耗品，單次使用 |

### 自動培育限制條件

> **設計理念**：培育是核心玩法迴圈，自動培育若無門檻開放會大幅降低遊戲耐玩度。因此採取「後期解鎖 + 品質懲罰 + 商業道具消耗」三重門檻。

| 限制項目 | 內容 |
|---|---|
| **前置條件 1** | 玩家累計手動培育畢業次數 ≥ 30 次 |
| **前置條件 2** | 已達成「培育大師」成就（至少 1 名 S 級畢業生） |
| **前置條件 3** | 玩家世代數 ≥ 5（遊戲後期） |
| **消耗道具** | 每次自動培育需消耗「自動培育券」×1（商城/活動獲取） |
| **品質上限** | 自動培育畢業評級最高僅達 A（無法獲得 S 級） |
| **因子覺醒** | 自動培育中不觸發因子覺醒事件 |
| **戰法學習** | 自動培育僅學習基礎戰法（不可學習進階/稀有戰法） |
| **事件機制** | 不觸發隨機事件（好感、受傷、特殊商店等） |
| **教官加成** | 教官 TP 加成僅套用 50%（自動模式教官效果減半） |

## F. 公式相關

### 每回合訓練效果
```
Stat_Gain = Training_Base × (1 + Mentor_TP/100) × (1 + Event_Bonus)
TP_Gain = 5 + Mentor_TP_Add + Event_TP
```

### 教官 TP 加成
```
Mentor_TP_Add = Mentor_Card.TP_Base × (Breakthrough_Level / Max_Level)
// TP_Base = 退役武將 Total_TP × 0.3
```

### 畢業評級
| 評級 | 條件 |
|---|---|
| S | 五維總和 ≥ 450 |
| A | 五維總和 ≥ 350 |
| B | 五維總和 ≥ 250 |
| C | 五維總和 < 250 |

### 因子覺醒在培育中
```
Per_Round_Awakening_Chance = 0.05 × (1 + Mentor_Relevant_Gene ? 0.5 : 0)
// 教官擁有對應因子→覺醒率 +50%
```

### 自動培育懲罰公式

```
// 自動培育統一懲罰係數
AUTO_PENALTY = 0.7  // 所有訓練效果 ×0.7
AUTO_TP_PENALTY = 0.5  // 教官 TP 加成僅 50%

Auto_Stat_Gain = Training_Base × AUTO_PENALTY × (1 + Mentor_TP × AUTO_TP_PENALTY / 100)
// 無事件加成（Event_Bonus = 0）
// 不觸發覺醒（Per_Round_Awakening_Chance = 0）
// 畢業評級上限 = A（五維總和最高 449）
```

### 戰法槽位（Loadout Capacity）

> **來源**：關卡設計與美術挑戰建議.md

武將可裝備的戰法數量由**智力（INT）**決定，形成「智力=戰術廣度」的策略深度。

```
Loadout_Capacity = max(1, floor(INT / 30))  // 智力每 30 點增加 1 個槽位
// 智力  1~29 → 1 槽
// 智力 30~59 → 2 槽
// 智力 60~89 → 3 槽
// 智力 90~99 → 4 槽（目前上限）
```

| INT 範圍 | 戰法槽位數 | 範例武將 |
|---|---|---|
| 1–29 | 1 槽 | 低智武將（純武力型） |
| 30–59 | 2 槽 | 均衡武將 |
| 60–89 | 3 槽 | 謀士型（如郭嘉） |
| 90–99 | 4 槽（上限） | 頂尖謀士（如諸葛亮） |

> **Unity 對照**：類似裝備格/技能格系統（Equipment Slot / Skill Slot）由角色屬性動態決定容量，而非固定值。

## I. Data Schema 需求

### 本機端 (Client) — 培育會話
```json
{
  "Nurture_Session": {
    "Child_UID": "C201",
    "Snapshot_Stats": {"STR":30,"INT":45,"POL":20,"CHR":35,"LUK":10},
    "Current_Round": 12,
    "Total_Rounds": 36,
    "Mentor_ID": "MENTOR_015",
    "TP_Accumulated": 180,
    "Events_Log": [
      {"Round":5, "Type":"Awakening", "Gene":"GREEN_DRAGON_04"},
      {"Round":8, "Type":"Injury", "Penalty":"Skip_1_round"}
    ],
    "Skills_Learned": ["龍膽", "落雷"]
  }
}
```

### Server 端 — 培育紀錄表
| 欄位 | 型別 | 說明 |
|---|---|---|
| Session_ID | VARCHAR(16) PK | 培育會話 ID |
| Child_UID | VARCHAR(16) FK | 培育目標子嗣 |
| Start_Season | INT | 培育開始季度 |
| End_Season | INT | 培育結束季度 |
| Mentor_ID | VARCHAR(16) FK | 教官 ID |
| Final_Stats_JSON | TEXT | 畢業後五維 |
| Rating | CHAR(1) | 評級 S/A/B/C |
| TP_Total | INT | 累積 TP |

## J. 名詞定義

| 名詞 | 代碼/Key | 定義 |
|---|---|---|
| 培育 | Nurture | 36 回合鏡像時空訓練模式 |
| 鏡像快照 | Snapshot | 進入培育時複製的數值分身 |
| 入學令 | Admission_Token | 培育名額的限量道具 |
| 回合 | Round | 培育中的最小時間單位（共 36 回合） |
| 畢業 | Graduation | 36 回合結束，數值寫回本體 |
| 評級 | Rating | 畢業五維總和對應的 S/A/B/C 等級 |
| 事件 | Event | 培育中隨機觸發的因子覺醒/受傷/好感 |
| 自動培育 | Auto_Nurture | 有條件解鎖的快速培育模式；需手動畢業 ≥30 次 + 世代 ≥5 + S 級成就；每次需消耗自動培育券；品質上限 A、無覺醒、無事件 |
| 自動培育券 | Auto_Nurture_Ticket | 解鎖自動培育後使用的消耗道具，透過商城、活動、成就獲得 |
