# 武將背包（倉庫）系統規格書

> **來源文件**：養成系統與教官來源定義.md、傭兵系統規格書：人才經營進化.md、轉蛋與戰場武將處置系統.md

---

## A. 系統描述

武將背包系統管理每位武將的個人持有物品與全域主城倉庫。武將背包儲存其專屬裝備（武器、名馬、古書）、消耗品配額、以及戰法秘笈；主城倉庫則作為全域共用的物資儲存空間，未裝配到特定武將的物品統一存放於此。

> **Unity 對照**：典型的 Inventory + Equipment 系統。武將背包類似角色的 Equipment Slots；主城倉庫類似全域的 Shared Inventory / Stash。兩者透過「裝備/卸下」操作互轉。

---

## B. 系統目的

1. **個人化裝備管理**：讓每位武將可攜帶專屬裝備，影響戰場數值與戰法效果
2. **全域物資池**：主城倉庫作為統一的存取空間，方便玩家管理大量物品
3. **容量限制驅動**：背包/倉庫容量上限驅動物品取捨決策與擴容付費
4. **傳承載具**：退役武將的裝備可轉移到後代或倉庫，延續投資價值

## C. 商業套路

| 商業點 | 說明 |
|---|---|
| 倉庫擴容 | 初始倉庫 50 格，付費擴容每次 +20 格 |
| 背包欄位解鎖 | 基礎 3 欄位（武器/防具/飾品），付費解鎖第 4~6 欄位（名馬/古書/秘寶） |
| 自動整理 | 高級功能：一鍵整理/自動分類，可付費解鎖永久使用 |
| 裝備強化 | 傳家寶物可強化，消耗材料與黃金 |
| 鎖定保護 | 重要裝備可鎖定防止誤賣/誤分解 |

## F. 公式相關

### 背包欄位定義

| 欄位 | 類型 | 解鎖條件 | 效果 |
|---|---|---|---|
| 武器 | 基礎欄 | 預設解鎖 | 武力/智力加成 |
| 防具 | 基礎欄 | 預設解鎖 | 統帥/生命加成 |
| 飾品 | 基礎欄 | 預設解鎖 | 政治/魅力加成 |
| 名馬 | 進階欄 | 付費/成就解鎖 | 移動力 +1、騎兵適性提升 |
| 古書 | 進階欄 | 付費/成就解鎖 | TP 折扣、戰法學習加速 |
| 秘寶 | 進階欄 | 付費/成就解鎖 | 特殊被動效果（如暴擊率+） |

### 倉庫容量

```
base_capacity = 50
expanded_capacity = base_capacity + (expansion_count × 20)
max_capacity = 200  // 上限
```

### 裝備數值加成

```
final_stat = base_stat + Σ(equipped_item[i].stat_bonus)
```

### 裝備品質與來源

| 品質 | 來源 | 基礎加成範圍 |
|---|---|---|
| SSR | 虎符兵種專屬/限時活動 | +50~+100 |

> **虎符裝備**：虎符在背包中作為特殊裝備類管理。裝備到武將後從背包移至武將裝備欄；卸下後回到背包。虎符不可丟棄或販賣。
| SR | 傳家寶物/高級任務 | +20~+49 |
| R | 戰場掉落/商城 | +5~+19 |
| N | 基礎配發/低級掉落 | +1~+4 |

## I. Data Schema 需求

### 本機端 (Client)

```json
{
  "GeneralEquipment": {
    "UID": "string (武將 UID)",
    "Slots": {
      "Weapon": "string | null (Item_ID)",
      "Armor": "string | null",
      "Accessory": "string | null",
      "Horse": "string | null",
      "Book": "string | null",
      "Relic": "string | null"
    },
    "Unlocked_Slots": ["Weapon", "Armor", "Accessory"]
  }
}
```

```json
{
  "Warehouse": {
    "Player_ID": "string",
    "Capacity": "number (當前最大格數)",
    "Items": [
      {
        "Slot_Index": "number",
        "Item_ID": "string",
        "Count": "number",
        "Is_Locked": "boolean",
        "Acquired_At": "string"
      }
    ]
  }
}
```

### Server 端

| 欄位 | 型別 | 說明 |
|---|---|---|
| equip_id | VARCHAR(32) PK | 裝備記錄 ID |
| player_id | VARCHAR(32) FK | 玩家 |
| general_uid | VARCHAR(32) FK NULL | 裝備於哪位武將（NULL=在倉庫） |
| item_id | VARCHAR(32) FK | 道具/裝備 ID |
| slot_type | ENUM | WEAPON/ARMOR/ACCESSORY/HORSE/BOOK/RELIC/WAREHOUSE |
| count | INT | 數量 |
| is_locked | BOOLEAN | 是否鎖定 |
| acquired_at | DATETIME | 取得時間 |

## J. 名詞定義

| 名詞 | 定義 |
|---|---|
| 武將背包 | 每位武將可攜帶的個人裝備欄位，直接影響戰場數值 |
| 主城倉庫 | 全域共用的物品儲存空間，未裝配的物品存放於此 |
| 裝備欄位 | 武將背包中的特定位置（武器/防具/飾品/名馬/古書/秘寶），每欄只能放對應類型的裝備 |
| 傳家寶物 | 從俘虜/退役/限時活動取得的稀有裝備，品質 SR 以上 |
| 分解 | 拆解不需要的裝備回收材料 |
| 倉庫擴容 | 付費增加主城倉庫的最大格數 |
| 裝備傳承 | 武將退役時裝備自動回到倉庫，可由後代繼續使用 |
| 物品鎖定 | 標記重要物品防止誤操作（販賣/分解） |
