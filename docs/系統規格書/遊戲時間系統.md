# 遊戲時間系統規格書

> **來源文件**：遊戲時間流逝與壽命機制.md、三國遊戲模式串接與循環.md

---

## A. 系統描述

遊戲採「季度制」時間推進。每季為最小時間單位，4 季=1 年。三大模式（治理→作戰→培育）以季為單位循環。治理模式中選擇內政行動消耗 1 季；作戰模式可能跨季；培育 36 回合壓縮=外界 1 季。武將每季自動老化，到達壽命上限則退役/死亡。

---

## B. 系統目的

1. **節奏控制**：季度制提供清晰的節奏脈搏
2. **資源約束**：每季行動有限，形成策略取捨
3. **時間壓力**：名將會老死，玩家必須持續繁殖新血
4. **模式切換**：季度是模式切換的節點

## F. 公式相關

### 時間換算
```
1 Season = 1 Game Quarter（遊戲最小單位）
4 Seasons = 1 Game Year
1 Nurture Session = 1 Season = 36 Rounds
1 Battle Campaign = 1~3 Seasons
```

### 武將老化
```
General.Age += 0.25  // 每季
if (General.Age >= General.Max_Lifespan):
    if (random() < Death_Chance):
        General.Status = "Dead"
    else:
        General.Status = "Retired"
```

## K. 四季特色設計（換季機制補充）

> **來源**：遊戲時間與壽命機制改良.md

### 四季特色

| 季節 | 代碼 | 主要特色 | 玩家影響 |
|---|---|---|---|
| 春 | SPRING | 稅收結算、隨機事件（瘟疫/豐收）、宜農業 | 資源結算，留意突發事件 |
| 夏 | SUMMER | 建設效率 ×2、訓練體力消耗較高 | 優先推進建築和科技 |
| 秋 | AUTUMN | 糧草收入高峰、最佳出征時機 | 集中發動攻勢 |
| 冬 | WINTER | 行軍速度 -1、結緣佳期（受孕率加成 +10%） | 適合在後方培育與結緣 |

### 時代更迭（Era_Reset）

每 **60 個遊戲月**（5 個遊戲年）觸發一次全局地圖大重組，防止遊戲後期因一家獨大陷入死局。

```
Era_Reset_Trigger = Total_Game_Months % 60 == 0

保留項目：
  - 子嗣數值、已解鎖科技
  - 血統因子、虎符與英靈卡
  - 玩家個人成就

重置項目：
  - 領土分配與城池歸屬
  - 外交關係
  - AI 勢力規模
```

> **Unity 對照**：類似 Roguelike 遊戲的「大循環重置」機制。保留角色養成（Meta Progress）而重置地圖狀態，確保每個時代週期都有新鮮感與競爭平衡。
