> **來源文件**：傭兵養成與留存系統設計.md

## A. 系統描述

留存系統設計以「情感連結」為核心，透過 5 階段插畫進度、傭兵名牌社交、告別信機制、退役回流等設計，讓玩家對武將產生依附感。同時透過「三國日報」動態敘事引擎，基於玩家前 24 小時真實行為 Log 自動生成個人化事件（復仇、代勞、離線社交），創造「每次上線都有驚喜」的防流失體驗。核心理念：「讓玩家捨不得離開，不是因為沉沒成本，而是因為每個武將都有故事。」

> **Unity 對照**：三國日報類似 Game Event Bus + Roguelike 事件生成器，基於玩家 Log 的「插槽式敘事」設計可對比 Unity 的 ScriptableObject Event System。

---

## B. 系統目的

1. **DAU 留存**：6 日結義倒數→每日登入看傭兵
2. **情感綁定**：插畫進度→「我跟這武將有關係了」
3. **社交留存**：傭兵名牌→加好友→互幫互助
4. **長線留存**：世代傳承→「我的曾祖趙雲的因子在第五代顯現了」

### 三國日報事件引擎

> **設計理念**：每次上線時，系統基於玩家前 24h 的真實 Log（慘敗、缺糧、抽卡失敗、勝利、升星等）自動拼接 Roguelike 風格事件，產生個人化敘事。

```typescript
// === 三國日報 —— Log 採樣與事件生成 ===

// Log 類型（前 24h 採集）
type PlayerLogType =
    | 'BATTLE_WIN' | 'BATTLE_LOSE' | 'RESOURCE_LOW'
    | 'GACHA_FAIL'  | 'GACHA_SUCCESS' | 'GENERAL_RETIRE'
    | 'GENERAL_BORN' | 'TALLY_UPGRADE' | 'DUEL_WIN' | 'DUEL_LOSE'
    | 'LOYALTY_DROP' | 'IDLE_LONG';  // 24h 未登入

// 事件類型
type DailyNewsType =
    | 'COMFORT'     // 安慰（慘敗後武將主動安慰）
    | 'REVENGE'     // 復仇（昨日被搶糧，今日武將幫你反搶）
    | 'AUTO_TASK'   // 代勞驚喜（武將自動幫完成每日任務）
    | 'OFFLINE_SOCIAL'  // 離線社交（關羽在路邊遇到他玩家的張飛並切磋）
    | 'TRAINING'    // 自主特訓（武將離線自行練功，屬性小幅提升）
    | 'GOSSIP'      // 八卦（武將之間的趣味互動消息）
    | 'SHOPPING';   // 團購（武將去市場淘寶，帶回小物品）

// 插槽式敘事模板：[角色] + [情緒] + [行為] + [結果]
interface NarrativeSlot {
    character: string;     // 武將名字
    emotion: string;       // 憤怒/開心/擔憂/得意/淡定
    action: string;        // 行為描述
    result: string;        // 結果描述（含獎勵）
}

// 事件觸發機率（基於 Log 類型）
const NEWS_TRIGGER_TABLE: Record<PlayerLogType, DailyNewsType[]> = {
    BATTLE_LOSE:    ['COMFORT', 'REVENGE'],
    RESOURCE_LOW:   ['REVENGE', 'SHOPPING'],
    GACHA_FAIL:     ['COMFORT', 'GOSSIP'],
    GACHA_SUCCESS:  ['GOSSIP'],
    BATTLE_WIN:     ['TRAINING', 'OFFLINE_SOCIAL'],
    GENERAL_RETIRE: ['COMFORT'],
    GENERAL_BORN:   ['GOSSIP'],
    TALLY_UPGRADE:  ['TRAINING'],
    DUEL_WIN:       ['OFFLINE_SOCIAL'],
    DUEL_LOSE:      ['COMFORT', 'REVENGE'],
    LOYALTY_DROP:   ['COMFORT'],
    IDLE_LONG:      ['AUTO_TASK', 'SHOPPING'],
};

// 每日事件產生數量
const DAILY_NEWS_COUNT = { min: 1, max: 3 };  // 每次登入生成 1~3 條日報

// 代勞機制：武將自動幫完成簡單日常任務
function generateAutoTask(general: General): NarrativeSlot {
    return {
        character: general.name,
        emotion: '得意',
        action: `自動幫你完成了「${pickSimpleTask()}」`,
        result: `獲得 ${pickSmallReward()}！`,
    };
}

// 復仇機制：基於昨日損失自動反搶部分資源
function generateRevenge(general: General, lostResource: string): NarrativeSlot {
    const recover = Math.floor(Math.random() * 0.15 * 100 + 5);  // 5%~20%
    return {
        character: general.name,
        emotion: '憤怒',
        action: `帶隊反搶昨日失去的${lostResource}`,
        result: `取回了 ${recover}% 的${lostResource}！`,
    };
}
```

## K. 7天玩家體驗歷程設計

> **來源**：遊戲機制推演與玩家歷程.md

新玩家入坑後的前 7 天是最關鍵的留存窗口。設計針對「無課玩家」與「課長」兩種典型路徑，確保雙軌都有清晰的體驗節奏與驚喜時刻。

| 天數 | 無課玩家體驗 | 課長體驗 |
|---|---|---|
| Day 1 | 新手引導完整流程 → 首次治理模式體驗 → 免費轉蛋 10 連 | 首次大額轉蛋 → 獲得 SSR 名將 → 開始深度培育規劃 |
| Day 2 | 完成首個 36 回合培育 → 子嗣誕生開獎 → 首個戰鬥關卡 | 多輪轉蛋組建精英隊伍 → 探索血統矩陣策略 |
| Day 3 | 領地擴張戰役 → 首次俘虜處置抉擇 → 傭兵池刷新 | 購買補元散 → 快速量產高品質子嗣 |
| Day 4 | 天才子嗣出現（Is_Daqi 觸發）→ 名士占卜驚喜感 | 因子覺醒事件 → 確認血統路線投入深度養成 |
| Day 5 | 遭遇克制戰法 → 需要策略應對 → 研究適性系統 | 完成首個英靈虎符轉化 → 體驗英靈支援槽加成 |
| Day 6 | 參加首次名將挑戰賽培育期 → 了解賽季機制 | 投入賽季針對性培育 → 季節虎符策略佈局 |
| Day 7 | 確立百年世家長線目標 → 世代傳承的宏圖清晰 | 規劃血統樹最優路線 → 解鎖第一個覺醒名號 |

### 關鍵留存觸發點

| 時刻 | 觸發內容 | 留存機制 |
|---|---|---|
| Day 1 晚 | 三國日報首次生成 | 每日上線習慣養成 |
| Day 2 早 | 子嗣誕生天象異變 | 情感注入（「是我培育的！」） |
| Day 4 | Is_Daqi 名士批語 | 懸念感（「這孩子以後會很強嗎？」） |
| Day 7 | 血統樹形成三代圖 | 沉沒成本 + 長線目標確立 |
