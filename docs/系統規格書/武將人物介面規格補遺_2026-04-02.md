# 武將人物介面規格補遺 2026-04-02

> 對應 proof：`日常人物頁 v3`
> 主卡：`UI-2-0054`

## 補遺目的

本補遺用來正式收斂 `日常人物頁 v3` 的版型、元件拆分與可行性評估，作為現行武將人物介面規格的追加規格。

## 母型結論

`日常人物頁 v3` 正式採用以下骨架：

1. 左側：角色本人近全身立繪
2. 右側：主資訊卡
3. 下方：橫向 6 格連續故事帶

這一版的正式定位是：

- 人物本人仍是主角
- 血脈區是深層人格區
- 故事帶是產品特色內容層

## 視覺重點

1. 血脈資訊不可再只是一般補充欄位，需形成獨立但整合的右側閱讀主題。
2. 祖獸符印 / 命紋徽記要避免直白獸首 icon，應逐步收斂為可 family 化的徽記系統。
3. 故事帶固定採 `6 格` 作為桌機 / 高級版首選配置。

## 六格故事節奏

1. 早年 / 訓練
2. 入世 / 進入歷史
3. 功績 / 名場面
4. 命運開始介入
5. 血脈真相 / 祖因接觸
6. 延續 / 承擔命運

## UI 元件拆分建議

### 主容器

1. `gd-root-frame`
2. `gd-left-portrait-carrier`
3. `gd-right-info-card`
4. `gd-story-strip`

### 資訊模組

1. `gd-nameplate`
2. `gd-attribute-grid`
3. `gd-role-card`
4. `gd-trait-card`
5. `gd-bloodline-card`
6. `gd-bloodline-crest`

### 故事模組

1. `story-strip-rail`
2. `story-cell-01 ~ story-cell-06`
3. `story-caption`（可選）

## 可行性評估

### 易於落地

1. 左人物 / 右資訊 / 下故事帶的版型，與現有 JSON UI pipeline 高度相容。
2. 右側資訊卡本質仍是多區塊卡片堆疊，容易拆 layout。
3. 六格故事帶主要是固定 slot 與縮圖容器，不依賴複雜動畫。
4. 命紋徽記可先以靜態 sprite / badge 落地。

### 中度成本

1. 角色身上的血脈裝飾語言，需要與角色生圖流程一起持續收斂。
2. 血脈區的 background motif / crest family 需要後續補齊整套視覺資產。

### 暫不建議混入日常頁

1. 長駐血脈虛影
2. 大量發光動效
3. 自動捲動式故事跑馬燈

上述內容應留在 `血脈命鏡過場 / 覺醒頁`。
