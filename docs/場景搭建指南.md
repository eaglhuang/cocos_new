# 🚀 戰鬥場景快速搭建指南

## 方案選擇

### ⭐ 推薦：編輯器擴展生成（持久化節點）

使用 `battle-scene-builder` 編輯器擴展，在編輯模式下直接建立節點，停止遊戲後節點不會消失。

> **Unity 對照**：這相當於 Unity 中寫 `Editor Script` 搭配 `MenuItem` 來自動建立 Hierarchy 結構，而非在 Play Mode 中建立。

#### 步驟（3 分鐘）

1. **確認擴展已編譯**
   ```bash
   cd extensions/battle-scene-builder
   cmd /c "npx tsc"
   ```
   確認 `dist/main.js` 和 `dist/scene-script.js` 存在。

2. **在 Cocos Creator 編輯器中開啟場景**
   - 雙擊打開 `assets/demo.scene`

3. **執行擴展選單**
   - 點擊選單：`開發者 → 戰鬥場景生成器 → 生成戰鬥場景`
   - 控制台會顯示：
     ```
     [BattleSceneBuilder] 開始生成戰鬥場景...
       ✓ BattleScene
       ✓ HUD
       ✓ Panel
       ✓ Popup
     [BattleSceneBuilder] ✅ 戰鬥場景生成完成！
     ```

4. **保存場景**
   - `Ctrl+S` 保存場景，節點會持久保留

### 備選：SceneAutoBuilder（現已支援編輯模式生成）

使用 `SceneAutoBuilder.ts` 掛在 `Canvas` 上後，**可直接在編輯模式生成節點並保留在場景中**。

> **注意**：如果你是在 Play Mode 裡才讓它執行，節點仍然會在停止後消失，這和 Unity 的 Play Mode 完全一樣。要持久保留，請在編輯模式生成，或使用上方的編輯器擴展方案。

#### 步驟（5 分鐘）

1. **打開 Cocos Creator 編輯器**
   - 雙擊打開 `assets/demo.scene`

2. **掛載自動生成腳本**
   - 在層級管理器中選中 `Canvas` 節點
   - Inspector 面板 → `Add Component` → `Custom Script` → 搜索 `SceneAutoBuilder`
   - 確認 `Auto Generate` 屬性為**勾選**狀態

3. **在編輯模式等待自動生成**
   - 勾選後會直接生成節點，不需要按播放
   - 生成完成後 `Auto Generate` 會自動取消勾選

4. **保存場景**（`Ctrl+S`）

5. **移除自動生成腳本**（可選）

## 📦 手動綁定元件與引用

生成節點後，**必須手動添加元件並綁定引用**。

> ⚠️ **重要順序**：必須先完成 **B → C → D**（替子節點掛元件），最後再回來做 **A**（綁定引用）。
>
> 原因：BattleScene 的屬性宣告為 `@property(BattleHUD)` 等元件類型（相當於 Unity Inspector 中只接受含特定 Component 的 GameObject），若目標節點還沒掛上對應元件，拖入時會被拒絕。

---

### B. HUD 節點

1. 選中 `Canvas/HUD` 節點
2. Inspector → `Add Component` → `Custom Script` → **`BattleHUD`**
3. 綁定所有子節點：

| 屬性名稱 | 拖入的節點 |
|---------|----------|
| `turnLabel` | `HUD/TurnLabel` |
| `dpLabel` | `HUD/DpLabel` |
| `playerSpBar` | `HUD/PlayerSpBar` |
| `playerSpLabel` | `HUD/PlayerSpLabel` |
| `playerFortressBar` | `HUD/PlayerFortressBar` |
| `playerFortressLabel` | `HUD/PlayerFortressLabel` |
| `enemyFortressBar` | `HUD/EnemyFortressBar` |
| `enemyFortressLabel` | `HUD/EnemyFortressLabel` |
| `statusLabel` | `HUD/StatusLabel` |

---

### C. Panel 節點

1. 選中 `Canvas/Panel` 節點
2. Inspector → `Add Component` → `Custom Script` → **`DeployPanel`**
3. 綁定按鈕：

| 屬性名稱 | 拖入的節點 |
|---------|----------|
| `btnCavalry` | `Panel/BtnCavalry` |
| `btnInfantry` | `Panel/BtnInfantry` |
| `btnShield` | `Panel/BtnShield` |
| `btnArcher` | `Panel/BtnArcher` |
| `btnSkill` | `Panel/BtnSkill` |
| `btnEndTurn` | `Panel/BtnEndTurn` |
| `laneButtons` | 展開數組，`Size = 5`，依序拖入：<br>`Panel/LaneButton1`<br>`Panel/LaneButton2`<br>`Panel/LaneButton3`<br>`Panel/LaneButton4`<br>`Panel/LaneButton5` |
| `selectionLabel` | `Panel/SelectionLabel` |

---

### D. Popup 節點

1. 選中 `Canvas/Popup` 節點
2. Inspector → `Add Component` → `Custom Script` → **`ResultPopup`**
3. 綁定：

| 屬性名稱 | 拖入的節點 |
|---------|----------|
| `titleLabel` | `Popup/TitleLabel` |
| `descLabel` | `Popup/DescLabel` |
| `btnReplay` | `Popup/BtnReplay` |

4. **重要**：確認 `Popup` 節點的 `Active` 屬性為**未勾選**（預設隱藏）

---

### A. BattleScene 節點（最後做）

> ✅ 完成 B、C、D 後，HUD / Panel / Popup 節點都已掛載對應元件，現在才能拖入。

1. 選中 `Canvas/BattleScene` 節點
2. Inspector → `Add Component` → `Custom Script` → **`BattleScene`**
3. 綁定四個引用：

| 屬性名稱 | 拖入的節點 | 說明 |
|---------|----------|------|
| **Hud** | `HUD` 節點 | 節點已有 BattleHUD 元件，可拖入 |
| **Deploy Panel** | `Panel` 節點 | 節點已有 DeployPanel 元件，可拖入 |
| **Result Popup** | `Popup` 節點 | 節點已有 ResultPopup 元件，可拖入 |
| **Grid Debug Label** | Canvas 下的任一 Label（可選）| 顯示文字棋盤，Debug 用 |

4. 保存場景（`Ctrl+S`）

---

## 🎮 運行測試

1. **保存場景**（`Ctrl+S`）
2. **播放遊戲**（`Ctrl+P`）
3. **預期行為**：
   - 左上角顯示「第 1 回合」、「糧草: 30」
   - 底部顯示 4 個兵種按鈕 + 5 個路線按鈕 + 技能/結束按鈕
   - 點擊「結束回合」觸發戰鬥流程
   - 控制台應該會輸出類似：
     ```
     [BattleScene] 開始生成戰鬥場景...
     [BattleController] 玩家武將：張飛
     [BattleController] 敵方武將：呂布
     [EventSystem] UnitMoved: ...
     [EventSystem] UnitDamaged: ...
     ```

4. **調試工具**：
   - 如果綁定了 `gridDebugLabel`，會顯示文字化的 5×8 棋盤狀態
   - 綁定方式：在 `Canvas` 下建立一個 `Label` 節點（或現有任何 Label），然後把該節點拖到 `BattleScene` 元件的 `gridDebugLabel` 欄位。
   - 可在控制台查看詳細的戰鬥日誌

## 🔧 常見問題

### 1. 運行時報錯「Cannot read property 'xxx' of null」
- **原因**：有屬性沒有綁定到節點
- **解決**：檢查 Inspector 中每個元件的屬性，確保沒有顯示 `None` 或紅色感嘆號

### 2. 點擊按鈕沒反應
- **原因**：`DeployPanel` 的 `ctrl` 引用可能為空
- **解決**：確認 `BattleScene` 元件已正確添加並執行了 `start()` 方法

### 3. 控制台顯示「找不到元件」
- **原因**：元件腳本名稱與檔案名不一致
- **解決**：檢查 `@ccclass("BattleHUD")` 等裝飾器的名稱是否正確

### 4. 場景運行後沒有顯示任何 UI
- **原因**：Canvas 可能沒有正確設置 Camera 或 RenderMode
- **解決**：
  1. 選中 `Canvas` 節點
  2. 確認 `Canvas` 元件的 `RenderMode` 設為 `OVERLAY`
  3. 如果是 `INTERSPERSE`，需要綁定 `Camera` 節點

### 5. 節點位置全部重疊在中心
- **原因**：UI 節點的 `UITransform` 可能未正確初始化
- **解決**：重新運行 `SceneAutoBuilder` 腳本（勾選 `clearExisting` 清除舊節點）

## 🧪 下一步：完整測試流程

1. **部署兵種**
   - 點擊「騎兵」按鈕（選擇兵種）
   - 點擊「路 1」按鈕（選擇路線並立即部署）
   - 糧草會即時扣減，格子 `(0, 0)` 出現單位（文字棋盤顯示 `P`）

2. **結束回合**
   - 點擊「結束回合」按鈕
   - 控制台會輸出敵方部署、自動移動、戰鬥結算、特殊行動的日誌

3. **發動技能**
   - 當 SP 條滿 100 時，點擊「發動技能」
   - 應該會看到「⚡ 震吼！敵方全體暈眩 1 回合！」
   - 敵方單位下回合不會移動和攻擊

4. **勝利條件**
   - 擊敗敵方武將（HP 歸零）
   - 或摧毀敵方堡壘（工兵到達前線）
   - 或敵方回合數超過 50

## 📚 進階：自定義遭遇戰

編輯 `assets/resources/data/encounters.json`，可以創建不同的戰鬥場景：

```json
{
  "id": "custom-battle",
  "name": "自訂戰鬥",
  "playerGeneral": "zhang-fei",
  "enemyGeneral": "cao-cao",
  "terrain": [
    ["plain", "plain", "river", "plain", "plain", "plain", "plain", "plain"],
    ["plain", "mountain", "plain", "plain", "plain", "plain", "plain", "plain"],
    ...
  ],
  "initialEnemyUnits": [
    { "type": "infantry", "lane": 2, "depth": 7 }
  ]
}
```
